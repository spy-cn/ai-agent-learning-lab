# 03 - 层次聚类

## 核心思想

> 不需要预先指定 K，而是构建一棵**聚类树（树状图）**，在不同层级切断就能得到不同数量的簇。

```
聚类树（树状图 / Dendrogram）：

        ┌─────────┐
        │         │           ← 在这里切断 → 2个簇
    ┌───┤     ┌───┤
    │   │     │   │           ← 在这里切断 → 3个簇
  ┌─┴─┐ │   ┌─┴─┐ │
  A   B C   D   E F           ← 叶节点：每个样本
```

---

## 两种策略

### 1. 凝聚法（Agglomerative / 自底向上）⭐ sklearn 支持

```
开始：每个样本自成一簇
  A  B  C  D  E  F

第1步：合并最近的 A、B
  [AB]  C  D  E  F

第2步：合并 D、E
  [AB]  C  [DE]  F

第3步：合并 [DE] 和 F
  [AB]  C  [DEF]

...直到所有样本合并成一个大簇
```

### 2. 分裂法（Divisive / 自顶向下）

```
开始：所有样本一个大簇
  [ABCDEF]

第1步：分成两群
  [AB] [CDEF]

...继续细分
```

> sklearn 只实现了凝聚法（`AgglomerativeClustering`）。

---

## 关键：链接方式（linkage）

合并两个簇时，如何定义"簇间距离"？

| linkage | 簇间距离定义 | 特点 |
|---------|------------|------|
| `'ward'` | 合并后方差增加最小 | 紧凑球形簇（默认） |
| `'complete'` | 两簇间最远点距离 | 紧凑，对异常值敏感 |
| `'average'` | 两簇间所有点对的平均距离 | 折中 |
| `'single'` | 两簇间最近点距离 | 能发现细长簇，但易链式合并 |

```
single linkage:           complete linkage:
  簇A    簇B                簇A    簇B
  ● ●    ● ●                ● ●    ● ●
  dist = 最近两点距离        dist = 最远两点距离
       ↑                          ↑
```

---

## sklearn 实现

```python
from sklearn.cluster import AgglomerativeClustering
from sklearn.datasets import make_blobs
import matplotlib.pyplot as plt

X, y = make_blobs(n_samples=200, centers=4, random_state=42)

# 层次聚类
cluster = AgglomerativeClustering(
    n_clusters=4,      # 指定最终簇数
    linkage='ward'     # 链接方式
)
labels = cluster.fit_predict(X)

plt.figure(figsize=(8, 6))
plt.scatter(X[:, 0], X[:, 1], c=labels, cmap='viridis', s=50)
plt.title('层次聚类 (ward linkage)')
plt.colorbar(label='簇')
plt.show()
```

## 树状图可视化（SciPy）

sklearn 本身不画树状图，用 SciPy：

```python
from scipy.cluster.hierarchy import dendrogram, linkage
import matplotlib.pyplot as plt

# 计算层次聚类
linked = linkage(X, method='ward')

# 画树状图
plt.figure(figsize=(12, 6))
dendrogram(linked,
           orientation='top',
           distance_sort='descending',
           show_leaf_counts=True,
           truncate_mode='lastp',  # 只显示最后p个簇
           p=12)                    # 显示12个叶节点
plt.title('层次聚类树状图')
plt.xlabel('样本索引 / 簇')
plt.ylabel('距离')
plt.axhline(y=15, color='red', linestyle='--', label='切断线 → 4个簇')
plt.legend()
plt.show()
```

**如何读树状图**：
- 横轴：样本（或合并后的簇）
- 纵轴：合并时的距离（越高说明合并的两个簇越不相似）
- 纵线越高 → 两簇差异越大
- 在某高度画水平线切割，经过的竖线数 = 簇数

## 不同 linkage 效果对比

```python
from sklearn.datasets import make_moons
from sklearn.cluster import AgglomerativeClustering

# 月牙形数据
X, y = make_moons(n_samples=200, noise=0.05, random_state=42)

fig, axes = plt.subplots(1, 4, figsize=(16, 4))

for ax, link in zip(axes, ['ward', 'complete', 'average', 'single']):
    cluster = AgglomerativeClustering(n_clusters=2, linkage=link)
    labels = cluster.fit_predict(X)
    ax.scatter(X[:, 0], X[:, 1], c=labels, cmap='coolwarm', s=40)
    ax.set_title(f'linkage={link}')

plt.suptitle('不同 linkage 在月牙数据上的效果')
plt.tight_layout()
plt.show()
```

> 💡 `single` linkage 能分开月牙形，但其他三种不行。

## 层次聚类的优缺点

### 优点
- ✅ **不需要预设 K**（通过树状图可视化选择）
- ✅ 能产生**层次结构**（大类中有小类）
- ✅ 树状图提供丰富的信息
- ✅ 结果**确定性**（不像KMeans有随机性）

### 缺点
- ❌ **计算复杂度高** O(n²) ~ O(n³)
- ❌ **不可逆**：一旦合并无法撤销
- ❌ 对噪声和异常值敏感
- ❌ 不适合大数据（> 几万样本）

---

## 添加距离/connectivity 约束

```python
from sklearn.cluster import AgglomerativeClustering
from sklearn.neighbors import kneighbors_graph

# 添加连通性约束 → 能处理复杂形状
connectivity = kneighbors_graph(X, n_neighbors=10, include_self=False)

cluster = AgglomerativeClustering(
    n_clusters=4,
    linkage='ward',
    connectivity=connectivity  # 只允许相邻样本合并
)
labels = cluster.fit_predict(X)
```

## 三种聚类算法对比

| 特性 | KMeans | DBSCAN | 层次聚类 |
|------|--------|--------|---------|
| 需要指定 K | ✅ | ❌ | ✅（或用树状图） |
| 簇形状 | 球形 | 任意 | 取决于 linkage |
| 噪声处理 | 差 | 好 | 中 |
| 大数据 | ⚡快 | 中 | 慢 |
| 可解释性 | 低 | 中 | 高（树状图） |
| 确定性 | 否（随机初始化） | 是 | 是 |

---

## 小结

| 要点 | 内容 |
|------|------|
| 策略 | 凝聚法（自底向上合并） |
| 链接方式 | ward（默认）、complete、average、single |
| 选择簇数 | 从树状图切断 |
| 树状图 | 用 SciPy 的 `dendrogram` |
| 优点 | 层次结构、可视化 |
| 缺点 | 慢、不可逆 |
