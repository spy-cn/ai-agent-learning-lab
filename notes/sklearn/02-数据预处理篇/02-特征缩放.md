# 02 - 特征缩放

## 为什么需要特征缩放？

假设有两个特征：**年龄**（0-100）和**收入**（0-1000000）。

```
样本A:  年龄=30,  收入=50000
样本B:  年龄=35,  收入=55000
```

计算欧氏距离时，收入维度完全碾压年龄维度：

```
距离 = √((35-30)² + (55000-50000)²)
     = √(25 + 25000000)
     ≈ 5000         ← 几乎全由收入决定
```

**基于距离的算法**（KNN、SVM、KMeans）会被大量级特征主导。特征缩放让所有特征在同一尺度上"公平竞争"。

---

## 哪些算法需要缩放？哪些不需要？

| 算法 | 对缩放敏感？ | 原因 |
|------|------------|------|
| KNN | ✅ 强烈需要 | 基于距离 |
| SVM | ✅ 强烈需要 | 基于间隔 |
| KMeans | ✅ 强烈需要 | 基于距离 |
| 线性回归 | ⚠️ 影响不大 | 基于系数，但收敛更慢 |
| 逻辑回归 | ⚠️ 影响不大 | 梯度下降收敛更慢 |
| 神经网络 | ✅ 强烈需要 | 梯度下降 |
| **决策树** | ❌ 不需要 | 基于阈值分裂，与尺度无关 |
| **随机森林** | ❌ 不需要 | 基于树 |
| **朴素贝叶斯** | ❌ 不需要 | 基于概率 |

> 💡 **经验法则**：基于距离/梯度的算法需要缩放；基于树的算法不需要。

---

## 主要缩放方法

### 1. StandardScaler（标准化）⭐ 最常用

将数据转换为**均值为0、标准差为1**的分布。

```
z = (x - μ) / σ
```

```python
from sklearn.preprocessing import StandardScaler
import numpy as np

X = np.array([[1, 1000],
              [2, 3000],
              [3, 5000],
              [4, 7000]])

scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

print("原始数据均值:", X.mean(axis=0))      # [2.5, 4000]
print("缩放后均值:", X_scaled.mean(axis=0))  # [0., 0.]  ≈0
print("缩放后标准差:", X_scaled.std(axis=0))  # [1., 1.]  =1
```

**特点**：
- ✅ 保留分布形状，不压缩到固定范围
- ✅ 适合大多数算法
- ✅ 对异常值有一定鲁棒性（不如 RobustScaler）
- ❌ 不保证值在 [0, 1] 范围内

### 2. MinMaxScaler（归一化）

将数据压缩到 **[0, 1]** 范围。

```
x_scaled = (x - x_min) / (x_max - x_min)
```

```python
from sklearn.preprocessing import MinMaxScaler

scaler = MinMaxScaler()
X_scaled = scaler.fit_transform(X)

print("缩放后最小值:", X_scaled.min(axis=0))  # [0., 0.]
print("缩放后最大值:", X_scaled.max(axis=0))  # [1., 1.]
```

**特点**：
- ✅ 保证所有值在 [0, 1]
- ✅ 适合图像像素值、需要固定范围的场景
- ❌ **对异常值非常敏感**——一个极端值会把其他值压缩到很小的范围

### 3. RobustScaler（鲁棒缩放）

用中位数和 IQR 代替均值和标准差。

```
x_scaled = (x - median) / IQR
```

```python
from sklearn.preprocessing import RobustScaler

# 带异常值的数据
X = np.array([[1], [2], [3], [4], [5], [100]])  # 100 是异常值

standard = StandardScaler().fit_transform(X)
minmax   = MinMaxScaler().fit_transform(X)
robust   = RobustScaler().fit_transform(X)

print("StandardScaler:", standard.flatten())
print("MinMaxScaler:  ", minmax.flatten())
print("RobustScaler:  ", robust.flatten())
```

**特点**：
- ✅ 异常值不会影响缩放参数
- ✅ 适合含大量异常值的数据
- ❌ 不保证均值/范围为特定值

### 4. MaxAbsScaler

缩放到 [-1, 1]，除以最大绝对值。适合**稀疏数据**（大部分为0的数据）。

```python
from sklearn.preprocessing import MaxAbsScaler

X = np.array([[ 1., -10.,  2.],
              [ 2.,   4.,   0.],
              [ 0.,   6.,  -1.]])

scaler = MaxAbsScaler()
X_scaled = scaler.fit_transform(X)
print(X_scaled)
```

### 5. Normalizer（样本缩放）

注意！前面都是**按特征缩放**（每列），Normalizer 是**按样本缩放**（每行），让每个样本的向量长度为1。

```python
from sklearn.preprocessing import Normalizer

X = np.array([[3, 4],
              [1, 2],
              [0, 5]])

normalizer = Normalizer()
X_norm = normalizer.fit_transform(X)
print(X_norm)
# [[0.6, 0.8],    ← 3²+4²=25, √25=5, [3/5, 4/5]
#  [0.447, 0.894],
#  [0., 1.]]
```

适用于**文本分类**（TF-IDF 向量）等场景。

## 方法对比总览

| 缩放器 | 公式 | 结果范围 | 异常值鲁棒 | 适用场景 |
|--------|------|---------|-----------|----------|
| StandardScaler | `(x-μ)/σ` | 无固定范围 | 中等 | 默认选择 |
| MinMaxScaler | `(x-min)/(max-min)` | [0, 1] | 差 | 需要固定范围 |
| RobustScaler | `(x-median)/IQR` | 无固定范围 | 好 | 有异常值 |
| MaxAbsScaler | `x/max|x|` | [-1, 1] | 差 | 稀疏数据 |
| Normalizer | `x/‖x‖` | 单位向量 | - | 文本/样本归一化 |

## ⚠️ 最常见的错误：数据泄露

```python
# ❌ 错误做法！
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)  # OK
X_test_scaled  = scaler.fit_transform(X_test)   # 错！用了测试集信息

# ✅ 正确做法
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)  # 只用训练集 fit
X_test_scaled  = scaler.transform(X_test)       # 用训练集的参数 transform
```

**为什么？** `fit_transform(X_test)` 会用测试集的均值/标准差来缩放测试集，这相当于**让模型间接看到了测试集的分布信息**，导致评估结果虚高。

**铁律**：
- `fit` 只在训练集上做
- `transform` 在训练集和测试集上都做（用训练集的参数）

## 在 Pipeline 中使用（推荐）

```python
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.neighbors import KNeighborsClassifier
from sklearn.model_selection import train_test_split

# 创建 Pipeline：先缩放，再建模
pipe = Pipeline([
    ('scaler', StandardScaler()),
    ('knn', KNeighborsClassifier(n_neighbors=5))
])

# 使用时和普通模型完全一样
pipe.fit(X_train, y_train)
print(f"准确率: {pipe.score(X_test, y_test):.4f}")
# Pipeline 自动保证：测试集用训练集的缩放参数
```

> 💡 **永远用 Pipeline**，从机制上避免数据泄露。详见 [05-数据预处理Pipeline](./05-数据预处理Pipeline.md)

## 实战对比：缩放 vs 不缩放

```python
from sklearn.datasets import load_wine
from sklearn.neighbors import KNeighborsClassifier
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler

wine = load_wine()
X_train, X_test, y_train, y_test = train_test_split(
    wine.data, wine.target, test_size=0.3, random_state=42)

# 不缩放
knn = KNeighborsClassifier(n_neighbors=5)
knn.fit(X_train, y_train)
acc_no_scale = knn.score(X_test, y_test)

# 缩放
scaler = StandardScaler()
X_train_s = scaler.fit_transform(X_train)
X_test_s  = scaler.transform(X_test)
knn.fit(X_train_s, y_train)
acc_scaled = knn.score(X_test_s, y_test)

print(f"不缩放: {acc_no_scale:.4f}")  # ~0.70
print(f"缩放后: {acc_scaled:.4f}")    # ~0.96  ← 提升巨大！
```

Wine 数据集各特征量级差异巨大（酒精含量 vs 脯氨酸含量），不缩放时 KNN 完全被大量级特征主导。

## 小结

| 要点 | 说明 |
|------|------|
| 基于距离/梯度的算法需要缩放 | KNN、SVM、KMeans、神经网络 |
| 基于树的算法不需要缩放 | 决策树、随机森林 |
| 默认选 StandardScaler | 适合大多数场景 |
| 有异常值选 RobustScaler | 用中位数/IQR |
| **永远只在训练集 fit** | 避免】数据泄露 |
| **用 Pipeline** | 从机制上防止出错 |
