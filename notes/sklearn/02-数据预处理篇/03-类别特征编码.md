# 03 - 类别特征编码

## 问题：机器学习模型只认数字

```python
# 原始数据
df = pd.DataFrame({
    '城市': ['北京', '上海', '广州', '北京', '深圳'],
    '颜色': ['红', '蓝', '绿', '红', '蓝'],
    '学历': ['高中', '本科', '硕士', '博士', '本科']
})

# 模型报错！无法处理字符串
# model.fit(df[['城市', '颜色', '学历']], y)  ← ValueError
```

必须将文本类别转换为数值。

---

## 类别特征的两种类型

选择编码方式前，必须区分类别类型：

### 1. 名义类别（Nominal）—— 无顺序关系

类别之间**没有大小/先后关系**。

```
城市: 北京 / 上海 / 广州      ← 北京 ≠ 上海 × 2
颜色: 红 / 蓝 / 绿           ← 红 ≠ 蓝 × 2
性别: 男 / 女                ← 男 ≠ 女 × 2
```

> 用 **OneHot 独热编码**。

### 2. 有序类别（Ordinal）—— 有顺序关系

类别之间**有天然的顺序/等级**。

```
学历: 高中 < 本科 < 硕士 < 博士
评分: 差 < 中 < 良 < 优
尺码: S < M < L < XL
```

> 用 **OrdinalEncoder / LabelEncoder**，映射为 0, 1, 2, 3...

---

## 编码方法详解

### 1. LabelEncoder（标签编码）

将类别映射为整数。**只能用于目标变量 y**，不能用于特征矩阵 X。

```python
from sklearn.preprocessing import LabelEncoder

le = LabelEncoder()
y = ['猫', '狗', '鸟', '猫', '狗']
y_encoded = le.fit_transform(y)

print(y_encoded)  # [1, 2, 0, 1, 2]  (映射: 猫→1, 狗→2, 鸟→0)
print(le.classes_)  # ['鸟', '狗', '猫']  (按字母排序后编号)
```

```python
# 逆转换
original = le.inverse_transform([0, 1, 2])
print(original)  # ['鸟', '猫', '狗']
```

> ⚠️ **不要用 LabelEncoder 编码特征 X！** 如果把"北京=0, 上海=1, 广州=2"喂给线性模型，模型会认为 北京 < 上海 < 广州，这是无意义的。

### 2. OrdinalEncoder（有序编码）

类似 LabelEncoder，但**用于特征矩阵 X**，可同时处理多列。

```python
from sklearn.preprocessing import OrdinalEncoder

# 有序类别
data = [['高中'], ['本科'], ['硕士'], ['博士'], ['本科']]

oe = OrdinalEncoder(categories=[['高中', '本科', '硕士', '博士']])
encoded = oe.fit_transform(data)

print(encoded)
# [[0.]    高中
#  [1.]    本科
#  [2.]    硕士
#  [3.]    博士
#  [1.]]   本科
```

> 💡 通过 `categories` 参数指定顺序，确保编码符合逻辑。

### 3. OneHotEncoder（独热编码）⭐ 处理名义类别

为每个类别创建一个二进制列。

```python
from sklearn.preprocessing import OneHotEncoder

data = [['北京'], ['上海'], ['广州'], ['北京'], ['深圳']]

encoder = OneHotEncoder(sparse_output=False)  # 返回密集矩阵
encoded = encoder.fit_transform(data)

print(encoded)
# [[1. 0. 0. 0.]   北京
#  [0. 1. 0. 0.]   上海
#  [0. 0. 1. 0.]   广州
#  [1. 0. 0. 0.]   北京
#  [0. 0. 0. 1.]]  深圳

print(encoder.categories_)  # [array(['上海', '北京', '广州', '深圳'])]
```

#### 处理多列

```python
data = [['北京', '红'],
        ['上海', '蓝'],
        ['广州', '绿']]

encoder = OneHotEncoder(sparse_output=False)
encoded = encoder.fit_transform(data)
print(encoded.shape)  # (3, 7) ← 4城市 + 3颜色 = 7列

# 查看特征名
print(encoder.get_feature_names_out())
# ['x0_上海' 'x0_北京' 'x0_广州' 'x0_深圳' 'x1_红' 'x1_绿' 'x1_蓝']
```

#### 处理未知类别

```python
# 测试集出现了训练集没有的新类别怎么办？
encoder = OneHotEncoder(handle_unknown='ignore', sparse_output=False)
encoder.fit([['北京'], ['上海']])  # 只见过北京、上海

# 预处理新数据
test = [['广州']]  # 没见过的城市
encoded = encoder.transform(test)
print(encoded)  # [[0. 0.]]  ← 全0，而不是报错
```

#### drop 参数（避免共线性）

```python
# 线性模型中，独热编码后如果保留所有类别列，
# 会导致多重共线性（虚拟变量陷阱）
encoder = OneHotEncoder(drop='first', sparse_output=False)
# 丢掉第一个类别列，用 N-1 列表示 N 个类别
```

## 方法选择决策树

```
类别特征
   │
   ├── 有顺序关系？（学历、评分、尺码）
   │       └── YES → OrdinalEncoder
   │
   └── 无顺序关系？（城市、颜色、性别）
           │
           ├── 类别很少？（< 15个）
           │       └── YES → OneHotEncoder
           │
           └── 类别很多？（> 15个）
                   │
                   ├── 基于树的模型
                   │       └── LabelEncoder 或 OrdinalEncoder
                   │
                   └── 线性/SVM/KNN
                           └── 考虑 TargetEncoder 或频率编码
```

## pandas 的编码方法（对比）

```python
import pandas as pd

df = pd.DataFrame({'城市': ['北京', '上海', '广州', '北京']})

# pandas 独热编码
df_onehot = pd.get_dummies(df, columns=['城市'])
print(df_onehot)
#    城市_上海  城市_北京  城市_广州
# 0    False    True    False
# 1    True     False   False
# ...

# pandas 有序编码
size_map = {'S': 0, 'M': 1, 'L': 2, 'XL': 3}
df['尺码'] = df['尺码'].map(size_map)
```

| 对比项 | sklearn OneHotEncoder | pandas get_dummies |
|--------|----------------------|-------------------|
| 训练/测试一致性 | ✅ `fit` / `transform` | ❌ 测试集可能少列 |
| 处理新类别 | ✅ `handle_unknown` | ❌ 需手动对齐 |
| 集成 Pipeline | ✅ 完美支持 | ❌ 不支持 |
| 使用便捷度 | 代码稍多 | 一行搞定 |

> 💡 **推荐 sklearn 版本**，因为可以放进 Pipeline，自动处理训练/测试集一致性。

## 实战：综合编码示例

```python
import pandas as pd
from sklearn.preprocessing import OneHotEncoder, OrdinalEncoder
from sklearn.compose import ColumnTransformer

# 构造数据
df = pd.DataFrame({
    '城市':   ['北京', '上海', '广州', '北京', '深圳'],
    '学历':   ['本科', '硕士', '高中', '博士', '本科'],
    '年龄':   [25, 30, 22, 45, 28],
    '收入':   [8000, 12000, 5000, 20000, 9000]
})

# 定义列
nominal_cols = ['城市']                        # 名义类别
ordinal_cols = ['学历']                        # 有序类别
numeric_cols = ['年龄', '收入']                 # 数值

# 有序类别的顺序
education_order = ['高中', '本科', '硕士', '博士']

# 用 ColumnTransformer 同时处理（详见 Pipeline 章节）
preprocessor = ColumnTransformer([
    ('num', 'passthrough', numeric_cols),                                    # 数值列不变
    ('ord', OrdinalEncoder(categories=[education_order]), ordinal_cols),     # 有序编码
    ('nom', OneHotEncoder(drop='first', sparse_output=False), nominal_cols), # 独热编码
])

result = preprocessor.fit_transform(df)
print(f"处理后形状: {result.shape}")
print(result)
```

## 高频编码（Target/Frequency Encoding）

当类别**非常多**时（如邮编、用户ID），独热编码会产生大量列。此时可用频率编码：

```python
# 用类别出现的频率代替
freq = df['城市'].value_counts(normalize=True)
df['城市_freq'] = df['城市'].map(freq)
```

> sklearn 1.3+ 提供了 `TargetEncoder`（还在实验阶段）：
```python
from sklearn.preprocessing import TargetEncoder  # 需要 sklearn >= 1.3
```

---

## 小结

| 场景 | 推荐方法 |
|------|----------|
| 编码目标变量 y | `LabelEncoder` |
| 有序类别特征 | `OrdinalEncoder` |
| 名义类别（类别少） | `OneHotEncoder` |
| 名义类别（类别多） | 频率编码 / TargetEncoder |
| 线性模型 + 独热编码 | 加 `drop='first'` 避免共线性 |
| 放进 Pipeline | 用 `ColumnTransformer` 组合 |
