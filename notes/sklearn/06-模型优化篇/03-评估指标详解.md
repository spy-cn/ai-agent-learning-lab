# 03 - 评估指标详解

## 为什么准确率不够用？

```python
# 一个癌症检测模型：
#   99% 的人是健康的
#   1% 的人有癌症

# 模型：所有人都预测为"健康"
#   准确率 = 99%  ← 看起来很高！

# 但这个模型完全没用——它一个癌症都没检测出来。
```

不同任务需要不同的评估指标。

---

## 混淆矩阵（基础）

### 二分类

```
                    预测
                 正例    负例
              ┌────────┬────────┐
        正例  │  TP    │  FN    │   TP=True Positive (真阳性)
真 实   负例  │  FP    │  TN    │   FN=False Negative (假阴性)
              └────────┴────────┘   FP=False Positive (假阳性)
                                  TN=True Negative (真阴性)
```

```python
from sklearn.metrics import confusion_matrix, ConfusionMatrixDisplay
import matplotlib.pyplot as plt

y_true = [0, 0, 0, 1, 1, 1, 1, 1]
y_pred = [0, 1, 0, 1, 1, 1, 0, 1]

cm = confusion_matrix(y_true, y_pred)
print(cm)
# [[2 1]    ← TN=2, FP=1
#  [1 4]]   ← FN=1, TP=4

# 可视化
disp = ConfusionMatrixDisplay(cm, display_labels=['健康', '癌症'])
disp.plot(cmap='Blues')
plt.title('混淆矩阵')
plt.show()
```

### 多分类

```python
cm = confusion_matrix(y_true_multiclass, y_pred_multiclass)
# 3×3 矩阵，行是真实类别，列是预测类别
```

## 分类指标

### 1. 准确率（Accuracy）

$$\text{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN}$$

```python
from sklearn.metrics import accuracy_score
accuracy_score(y_true, y_pred)
```

- 适合：类别均衡
- 不适合：类别不平衡

### 2. 精确率（Precision）

> 预测为正的样本中，真正是正的比例。

$$\text{Precision} = \frac{TP}{TP + FP}$$

```python
from sklearn.metrics import precision_score
precision_score(y_true, y_pred)
```

**关注精确率的场景**：垃圾邮件分类
- FP（误判为垃圾邮件）代价高 → 不希望把正常邮件判为垃圾邮件

### 3. 召回率（Recall / Sensitivity）

> 真实的正例中，被找出来的比例。

$$\text{Recall} = \frac{TP}{TP + FN}$$

```python
from sklearn.metrics import recall_score
recall_score(y_true, y_pred)
```

**关注召回率的场景**：癌症检测
- FN（漏诊癌症）代价高 → 宁可误判也不要漏掉

### 4. F1分数

> 精确率和召回率的**调和平均**。

$$F1 = 2 \times \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}}$$

```python
from sklearn.metrics import f1_score
f1_score(y_true, y_pred)
```

**为什么用调和平均？**
- 算术平均：Precision=1.0, Recall=0.0 → 平均=0.5（看起来还行）
- 调和平均：Precision=1.0, Recall=0.0 → F1=0.0（惩罚偏科）

### 指标关系总览

```
                    预测正    预测负
                   ┌────────┬────────┐
             正例  │  TP    │  FN    │
                    │        │        │
             负例  │  FP    │  TN    │
                   └────────┴────────┘

精确率 = TP / (TP + FP)     ← 横向看（预测为正中真的正）
召回率 = TP / (TP + FN)     ← 纵向看（真实正中被找出）
F1     = 两者调和平均         ← 平衡指标
```

## classification_report（一键报告）

```python
from sklearn.metrics import classification_report

print(classification_report(y_true, y_pred, target_names=['健康', '癌症']))
```

输出：
```
              precision    recall  f1-score   support

        健康       0.67      0.67      0.67         3
        癌症       0.80      0.80      0.80         5

    accuracy                           0.75         8
   macro avg     0.73      0.73      0.73         8
weighted avg     0.75      0.75      0.75         8
```

| 行 | 含义 |
|----|------|
| 各类别 | 该类的 precision/recall/f1 |
| accuracy | 整体准确率 |
| macro avg | 各类指标的简单平均（每类等权重） |
| weighted avg | 按样本数加权的平均 |

## ROC 曲线与 AUC

### ROC 曲线

> 不同阈值下，**召回率（TPR）** vs **假正率（FPR）** 的曲线。

```
TPR (召回率)
 1 │            ┌────────  ← 好模型（贴近左上角）
   │         ╱
   │       ╱
   │     ╱
   │   ╱        ← 随机猜测（对角线）
 0 │──╱─────────
   0           1
       FPR (假正率)
```

```python
from sklearn.metrics import roc_curve, auc, RocurveDisplay

# 需要概率而不是硬预测
y_prob = model.predict_proba(X_test)[:, 1]

fpr, tpr, thresholds = roc_curve(y_test, y_prob)
roc_auc = auc(fpr, tpr)

plt.figure(figsize=(6, 6))
plt.plot(fpr, tpr, 'b-', linewidth=2, label=f'ROC (AUC={roc_auc:.3f})')
plt.plot([0, 1], [0, 1], 'r--', label='随机')
plt.xlabel('假正率 (FPR)')
plt.ylabel('召回率 (TPR)')
plt.title('ROC 曲线')
plt.legend()
plt.grid(True)
plt.show()
```

### AUC

> ROC 曲线下面积。范围 [0, 1]。

| AUC | 含义 |
|-----|------|
| 1.0 | 完美 |
| 0.9 | 优秀 |
| 0.7 | 还行 |
| 0.5 | 随机猜测 |
| <0.5 | 比随机还差 |

```python
from sklearn.metrics import roc_auc_score
auc_score = roc_auc_score(y_test, y_prob)
```

> 💡 AUC 的优势：**对类别不平衡不敏感**，且综合考虑了所有阈值。

---

## PR 曲线（适合不平衡数据）

当正例非常稀少时（如欺诈检测），PR 曲线比 ROC 更有信息量。

```python
from sklearn.metrics import precision_recall_curve, average_precision_score

precision, recall, thresholds = precision_recall_curve(y_test, y_prob)
ap = average_precision_score(y_test, y_prob)

plt.plot(recall, precision, 'g-')
plt.xlabel('召回率')
plt.ylabel('精确率')
plt.title(f'PR 曲线 (AP={ap:.3f})')
plt.grid(True)
plt.show()
```

## 回归指标

```python
from sklearn.metrics import (
    mean_squared_error,      # MSE
    mean_absolute_error,     # MAE
    r2_score,                # R²
    mean_squared_log_error,  # MSLE
    mean_absolute_percentage_error  # MAPE
)
```

| 指标 | 公式 | 特点 |
|------|------|------|
| **MSE** | $\frac{1}{n}\sum(y-\hat{y})^2$ | 对大误差敏感（平方惩罚） |
| **RMSE** | $\sqrt{\text{MSE}}$ | 与y同单位，直观 |
| **MAE** | $\frac{1}{n}\sum|y-\hat{y}|$ | 对异常值鲁棒 |
| **R²** | $1 - \frac{SS_{res}}{SS_{tot}}$ | [0,1]，解释方差比例 |
| **MAPE** | $\frac{1}{n}\sum|\frac{y-\hat{y}}{y}|$ | 百分比误差 |

```python
mse  = mean_squared_error(y_test, y_pred)
rmse = np.sqrt(mse)
mae  = mean_absolute_error(y_test, y_pred)
r2   = r2_score(y_test, y_pred)
mape = mean_absolute_percentage_error(y_test, y_pred)

print(f"MSE:  {mse:.4f}")
print(f"RMSE: {rmse:.4f}")
print(f"MAE:  {mae:.4f}")
print(f"R²:   {r2:.4f}")
print(f"MAPE: {mape:.2%}")
```

## 多分类指标

```python
# 多分类的 precision/recall/f1 需要指定 average 参数
from sklearn.metrics import precision_score

precision_score(y_true, y_pred, average='macro')      # 各��简单平均
precision_score(y_true, y_pred, average='weighted')    # 按样本数加权
precision_score(y_true, y_pred, average='micro')       # 全局计算（=准确率）
precision_score(y_true, y_pred, average=None)          # 返回每类的值
```

| average | 含义 | 适用 |
|---------|------|------|
| `'macro'` | 各类等权重 | 关注少数类 |
| `'weighted'` | 按样本数加权 | 类别不平衡 |
| `'micro'` | 全局计算 | 等价于准确率 |

## 指标选择决策树

```
什么任务？
│
├── 回归
│     ├── 一般 → R², RMSE
│     ├── 有异常值 → MAE
│     └── 关注百分比 → MAPE
│
├── 二分类
│     ├── 类别均衡 → accuracy
│     ├── 类别不平衡 → F1, AUC
│     ├── 关注假阳性（垃圾邮件）→ precision
│     ├── 关注假阴性（癌症）→ recall
│     └── 需要阈值分析 → ROC/PR曲线
│
└── 多分类
      ├── 各类同等重要 → macro F1
      └── 按样本量加权 → weighted F1
```

## scoring 参数（用于 GridSearchCV）

```python
from sklearn.model_selection import cross_val_score

# 默认用模型自带的 score（准确率/R²）
cross_val_score(model, X, y, cv=5)

# 指定其他指标
cross_val_score(model, X, y, cv=5, scoring='f1')
cross_val_score(model, X, y, cv=5, scoring='roc_auc')
cross_val_score(model, X, y, cv=5, scoring='neg_mean_squared_error')

# 查看所有可用的 scoring
from sklearn.metrics import get_scorer_names
print(get_scorer_names())
```

> ⚠️ sklearn 的 scoring 中，回归指标是**负值**（如 `neg_mean_squared_error`），因为 sklearn 遵循"越大越好"的统一约定。

---

## 小结

| 指标 | 适用 | 关注 |
|------|------|------|
| Accuracy | 均衡分类 | 整体正确率 |
| Precision | FP代价高 | 预测的纯度 |
| Recall | FN代价高 | 找全率 |
| F1 | 平衡 | P和R的调和 |
| AUC | 不平衡 | 全阈值综合 |
| R² | 回归 | 解释方差 |
| RMSE | 回归 | 误差大小 |
| MAE | 回归有异常值 | 误差大小 |
