# 02 - 超参数调优

## 参数 vs 超参数

| 类型 | 说明 | 谁决定 | 示例 |
|------|------|--------|------|
| **参数** | 模型从数据中学到的 | 算法自己 | 回归系数 w、树的结构 |
| **超参数** | 训练前手动设定的 | 你 | K值、学习率、max_depth |

```python
# 超参数 → 你设定
model = RandomForestClassifier(n_estimators=100, max_depth=10)

# 参数 → 模型学习
model.fit(X_train, y_train)
print(model.feature_importances_)  # 这是学到的参数
```

---

## 三种调参策略

### 1. GridSearchCV（网格搜索）⭐ 最常用

> 尝试所有参数组合，用交叉验证选最好的。

```python
from sklearn.model_selection import GridSearchCV
from sklearn.ensemble import RandomForestClassifier

param_grid = {
    'n_estimators': [50, 100, 200],
    'max_depth': [5, 10, None],
    'max_features': ['sqrt', 'log2']
}

grid = GridSearchCV(
    RandomForestClassifier(random_state=42),
    param_grid,
    cv=5,
    scoring='accuracy',
    n_jobs=-1,          # 并行
    verbose=1            # 打印进度
)

grid.fit(X_train, y_train)

print(f"最佳参数: {grid.best_params_}")
print(f"最佳准确率: {grid.best_score_:.4f}")

# 最佳模型已经在 grid.best_estimator_ 中
print(f"测试集准确率: {grid.best_estimator_.score(X_test, y_test):.4f}")
```

#### 网格搜索的可视化

```
参数组合（3 × 3 × 2 = 18种）：

         max_features=sqrt    max_features=log2
       ┌─────────────────────┬─────────────────────┐
depth=5│ [50,5,sqrt]         │ [50,5,log2]         │
       │ [100,5,sqrt]        │ [100,5,log2]        │
       │ [200,5,sqrt]        │ [200,5,log2]        │
       ├─────────────────────┼─────────────────────┤
depth=10│ [50,10,sqrt]       │ [50,10,log2]        │
       │ [100,10,sqrt]       │ [100,10,log2]       │
       │ [200,10,sqrt]       │ [200,10,log2]       │
       ├─────────────────────┼─────────────────────┤
depth=N│ [50,N,sqrt]        │ [50,N,log2]         │
       │ [100,N,sqrt]       │ [100,N,log2]        │
       │ [200,N,sqrt]       │ [200,N,log2]        │
       └─────────────────────┴─────────────────────┘

每种组合 × 5折CV = 18 × 5 = 90次训练
```

### 2. RandomizedSearchCV（随机搜索）

> 不尝试所有组合，而是**随机采样**指定数量的组合。

```python
from sklearn.model_selection import RandomizedSearchCV
from scipy.stats import randint, uniform

# 用分布代替固定列表
param_dist = {
    'n_estimators': randint(50, 500),       # 50-500随机整数
    'max_depth': [5, 10, 20, None],
    'max_features': uniform(0.5, 0.5),      # 0.5-1.0随机浮点
    'min_samples_leaf': randint(1, 20)
}

random_search = RandomizedSearchCV(
    RandomForestClassifier(random_state=42),
    param_dist,
    n_iter=50,          # 只尝试50种随机组合
    cv=5,
    scoring='accuracy',
    n_jobs=-1,
    random_state=42
)

random_search.fit(X_train, y_train)
print(f"最佳参数: {random_search.best_params_}")
print(f"最佳准确率: {random_search.best_score_:.4f}")
```

#### 为什么随机搜索有效？

```
网格搜索：              随机搜索：
  参数A: 5个值             参数A: 随机
  参数B: 5个值             参数B: 随机
  参数C: 5个值             参数C: 随机
  总共 125 次              总共 25 次
  
  → 在不重要的参数上       → 在每个参数上都有
    浪费了大量尝试          充分的探索
```

> 💡 当参数空间很大时，**随机搜索通常比网格搜索更高效**。

### 3. HalvingSearchCV（减半搜索）⭐ sklearn 新功能

> 先用少量数据评估所有候选，淘汰一半，再用更多数据评估剩下的...

```python
from sklearn.model_selection import HalvingGridSearchCV

halving = HalvingGridSearchCV(
    RandomForestClassifier(random_state=42),
    param_grid,
    cv=5,
    factor=2,            # 每轮保留 1/factor 的候选
    n_jobs=-1
)
halving.fit(X_train, y_train)
```

---

## Grid vs Random vs Halving 对比

| 方法 | 尝试组合数 | 效率 | 适合 |
|------|-----------|------|------|
| GridSearch | 全部组合 | 慢但全面 | 参数少（<3个，每个<5值） |
| Randomized | 你指定 | 快 | 参数多、连续参数 |
| Halving | 自适应 | 最快 | 大量候选、大数据 |

## 搜索空间设计技巧

### 1. 从粗到细

```python
# 第1轮：粗搜索（大范围）
param_grid_1 = {
    'C': [0.001, 0.01, 0.1, 1, 10, 100, 1000],
    'gamma': [0.001, 0.01, 0.1, 1, 10, 100]
}

# 第2轮：细搜索（在小范围细化）
# 假设第1轮最佳 C=1, gamma=0.1
param_grid_2 = {
    'C': [0.5, 1, 2, 5],
    'gamma': [0.05, 0.1, 0.2, 0.5]
}
```

### 2. 对数尺度搜索

```python
# 对于学习率、正则化等参数，在对数空间搜索更合理
import numpy as np

param_grid = {
    'C': np.logspace(-3, 3, 7),      # [0.001, 0.01, ..., 1000]
    'gamma': np.logspace(-4, 2, 7)
}
```

### 3. 条件参数空间

```python
# 不同模型有不同的参数
param_grid = [
    {
        'kernel': ['linear'],
        'C': [0.1, 1, 10]
    },
    {
        'kernel': ['rbf'],
        'C': [0.1, 1, 10],
        'gamma': ['scale', 'auto', 0.1, 1]
    }
]
grid = GridSearchCV(SVC(), param_grid, cv=5)
```

## 评估调参结果

### 查看所有结果

```python
import pandas as pd

cv_results = pd.DataFrame(grid.cv_results_)
# 查看排名前5的参数组合
top5 = cv_results.nsmallest(5, 'rank_test_score')[
    ['params', 'mean_test_score', 'std_test_score', 'rank_test_score']
]
print(top5)
```

### 可视化参数影响

```python
import matplotlib.pyplot as plt

# 单参数影响
results = pd.DataFrame(grid.cv_results_)
for param in ['param_n_estimators', 'param_max_depth']:
    plt.figure(figsize=(8, 4))
    grouped = results.groupby(param)['mean_test_score'].mean()
    plt.plot(grouped.index.astype(str), grouped.values, 'bo-')
    plt.xlabel(param)
    plt.ylabel('平均准确率')
    plt.title(f'{param} 的影响')
    plt.grid(True)
    plt.show()
```

## ⚠️ 调参注意事项

### 1. 数据泄露

```python
# ❌ 错误：先对全部数据预处理，再调参
X_scaled = StandardScaler().fit_transform(X)  # 全部数据！
grid.fit(X_scaled, y)

# ✅ 正确：Pipeline + GridSearch
pipe = Pipeline([('scaler', StandardScaler()), ('model', SVC())])
grid.fit(pipe, X, y)  # CV内部自动正确处理
```

### 2. 过度调参

```python
# ❌ 调了几百种组合，在验证集上找到了"最佳"
# 但这只是在验证集上"过拟合"
```

> 💡 保持参数空间合理，不要过度搜索。最终用**独立的测试集**评估。

### 3. 不要在测试集上调参

```
训练集 → 训练模型
验证集 → 调超参数（GridSearchCV 内部用 CV 实现）
测试集 → 最终评估（只看一次！）
```

---

## 小结

| 方法 | 适用 | 建议 |
|------|------|------|
| GridSearchCV | 参数少 | 全面但慢 |
| RandomizedSearchCV | 参数多 | 高效推荐 |
| HalvingSearchCV | 大数据 | 最快 |

**调参流程**：
1. 先用默认参数跑个基线
2. 从粗到细搜索
3. 用 Pipeline 防止泄露
4. 最终用独立测试集验证
