# 01 - 交叉验证

## 为什么需要交叉验证？

### train_test_split 的问题

```python
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)
model.fit(X_train, y_train)
model.score(X_test, y_test)  # 比如 0.85
```

**问题**：这个 0.85 可靠吗？

```
如果换一种划分方式呢？

划分方式1：  [████████░░] → 准确率 0.85
划分方式2：  [███░░█████] → 准确率 0.92
划分方式3：  [█████░░███] → 准确率 0.78

哪个才是模型的真实水平？
```

**单次划分的评估方差很大**，尤其是数据量小时。

---

## K折交叉验证（K-Fold CV）

### 核心思想

> 把数据分成 K 份，轮流用其中 1 份做测试、其余 K-1 份做训练，重复 K 次，取平均。

```
5折交叉验证：

第1折：  [测试]  [训练] [训练] [训练] [训练]  → 准确率1
第2折：  [训练]  [测试] [训练] [训练] [训练]  → 准确率2
第3折：  [训练]  [训练] [测试] [训练] [训练]  → 准确率3
第4折：  [训练]  [训练] [训练] [测试] [训练]  → 准确率4
第5折：  [训练]  [训练] [训练] [训练] [测试]  → 准确率5

最终准确率 = mean(准确率1...5) ± std(准确率1...5)
```

---

## sklearn 实现

### cross_val_score（最常用）

```python
from sklearn.model_selection import cross_val_score
from sklearn.ensemble import RandomForestClassifier
from sklearn.datasets import load_iris

X, y = load_iris(return_X_y=True)
model = RandomForestClassifier(random_state=42)

# 5折交叉验证
scores = cross_val_score(model, X, y, cv=5)

print(f"各折准确率: {scores}")
print(f"平均: {scores.mean():.4f} ± {scores.std():.4f}")
```

### cross_validate（更详细）

```python
from sklearn.model_selection import cross_validate

results = cross_validate(
    model, X, y,
    cv=5,
    scoring=['accuracy', 'precision_macro', 'recall_macro', 'f1_macro'],
    return_train_score=True
)

print(f"训练集准确率: {results['train_accuracy'].mean():.4f}")
print(f"验证集准确率: {results['test_accuracy'].mean():.4f}")
print(f"验证集F1:     {results['test_f1_macro'].mean():.4f}")
```

## 不同的 K-Fold 策略

### 1. KFold（普通）

```python
from sklearn.model_selection import KFold

kf = KFold(n_splits=5, shuffle=True, random_state=42)
scores = cross_val_score(model, X, y, cv=kf)
```

### 2. StratifiedKFold（分层）⭐ 分类推荐

> 保持每折中**各类别的比例一致**。

```python
from sklearn.model_selection import StratifiedKFold

skf = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
scores = cross_val_score(model, X, y, cv=skf)
```

```
普通 KFold：           StratifiedKFold：
  折1: [A,B,A,B,A]       折1: [A,A,A,B,B]   ← 各折类别比例一致
  折2: [A,A,A,A,A]       折2: [A,A,A,B,B]
  折3: [B,B,B,B,B]       折3: [A,A,A,B,B]
  ↑ 类别不均衡！          ↑ 每折都有A和B
```

> 💡 `cross_val_score` 在分类任务中**默认使用 StratifiedKFold**。

### 3. GroupKFold（分组）

> 确保同一组的数据**不同时出现在训练集和测试集**。

```python
from sklearn.model_selection import GroupKFold

# 比如：同一个患者的数据不能同时出现在训练集和测试集
groups = [0, 0, 0, 1, 1, 1, 2, 2, 2, 3, 3, 3]  # 患者ID
gkf = GroupKFold(n_splits=4)
scores = cross_val_score(model, X, y, cv=gkf, groups=groups)
```

**应用场景**：
- 医疗：同一患者的多次检查
- 时间序列：避免未来数据泄露到训练集
- 推荐系统：同一用户的多条记录

### 4. TimeSeriesSplit（时间序列）

```python
from sklearn.model_selection import TimeSeriesSplit

tscv = TimeSeriesSplit(n_splits=5)
# 训练数据始终在测试数据之前（不能"用未来预测过去"）

# 折1: 训练[1]       测试[2]
# 折2: 训练[1,2]     测试[3]
# 折3: 训练[1,2,3]   测试[4]
```

## 选择 K 值

| K 值 | 训练集占比 | 特点 |
|------|----------|------|
| 5 | 80% | 常用，平衡 |
| 10 | 90% | 更精确，但慢 |
| 3 | 67% | 快速验证 |
| LOO（留一法） | N-1/N | 极慢，几乎不用 |

> 💡 **最常用：5折或10折**。

## 交叉验证的黄金法则

### ⚠️ 永远在 Pipeline 外面做 CV

```python
# ❌ 错误！数据泄露
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)        # 用了全部数据的统计量
scores = cross_val_score(model, X_scaled, y, cv=5)  # CV时已泄露

# ✅ 正确！把预处理放进 Pipeline
pipe = Pipeline([
    ('scaler', StandardScaler()),
    ('model', RandomForestClassifier())
])
scores = cross_val_score(pipe, X, y, cv=5)  # CV内部自动正确处理
```

**为什么？** 如果先对全部数据做标准化，再划分训练/测试，测试集的统计信息已经泄露到训练中。

> Pipeline 在每次 CV 迭代中**只用训练折 fit，再 transform 测试折**，彻底避免泄露。

## 用 CV 比较模型

```python
from sklearn.linear_model import LogisticRegression
from sklearn.svm import SVC
from sklearn.ensemble import RandomForestClassifier, GradientBoostingClassifier
from sklearn.model_selection import cross_val_score
from sklearn.datasets import load_wine

X, y = load_wine(return_X_y=True)

models = {
    '逻辑回归': LogisticRegression(max_iter=5000),
    'SVM':      SVC(),
    '随机森林':  RandomForestClassifier(random_state=42),
    'GBDT':     GradientBoostingClassifier(random_state=42),
}

print(f"{'模型':<10} {'平均准确率':<15} {'标准差':<10}")
print("-" * 40)
for name, model in models.items():
    scores = cross_val_score(model, X, y, cv=5, scoring='accuracy')
    print(f"{name:<10} {scores.mean():.4f}        ±{scores.std():.4f}")
```

## cross_val_predict

```python
from sklearn.model_selection import cross_val_predict

# 获取交叉验证的预测结果（每个样本都被"测试"过一次）
y_pred_cv = cross_val_predict(model, X, y, cv=5)

# 可以用来画混淆矩阵等
from sklearn.metrics import confusion_matrix
print(confusion_matrix(y, y_pred_cv))
```

## 可视化 CV 策略

```python
from sklearn.model_selection import KFold, StratifiedKFold
import matplotlib.pyplot as plt
import numpy as np

def plot_cv(cv, X, y, title):
    fig, ax = plt.subplots(figsize=(10, 3))
    for i, (train_idx, test_idx) in enumerate(cv.split(X, y)):
        ax.scatter(train_idx, [i]*len(train_idx), c='blue', marker='s', s=20, label='训练' if i==0 else '')
        ax.scatter(test_idx, [i]*len(test_idx), c='red', marker='o', s=20, label='测试' if i==0 else '')
    ax.set_title(title)
    ax.set_xlabel('样本索引')
    ax.set_ylabel('CV迭代')
    ax.legend()
    ax.set_yticks(range(i+1))
    plt.show()

X_demo = np.arange(30).reshape(-1, 1)
y_demo = np.array([0]*15 + [1]*15)

plot_cv(KFold(5), X_demo, y_demo, 'KFold（普通）')
plot_cv(StratifiedKFold(5, shuffle=True, random_state=42), X_demo, y_demo, 'StratifiedKFold（分层）')
```

## 小结

| 要点 | 内容 |
|------|------|
| 为什么用CV | 单次划分不可靠 |
| 最常用 | 5折分层交叉验证 |
| 分类默认 | StratifiedKFold |
| 分组数据 | GroupKFold |
| 时间序列 | TimeSeriesSplit |
| 黄金法则 | **预处理必须在 Pipeline 内** |
