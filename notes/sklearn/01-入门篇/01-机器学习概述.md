# 01 - 机器学习概述

## 什么是机器学习？

Tom Mitchell 的经典定义：

> 如果一个程序在任务 T 上的性能 P 随着经验 E 的增加而提升，我们就说它从经验 E 中学习。

通俗讲，机器学习不是通过人工编写规则来解决问题，而是让计算机**从数据中自动学习规律**，然后利用学到的规律进行预测或决策。

### 与传统编程的对比

```
传统编程：  数据 + 规则（人写） ──► 答案
机器学习：  数据 + 答案 ──► 规则（机器学）
```

举例：识别垃圾邮件
- 传统做法：人工编写规则——"如果包含'中奖'、'免费'、发件人在黑名单……则判定为垃圾邮件"
- 机器学习：给算法几千封已标记的邮件，让它自己学习区分垃圾邮件的特征

## 三大学习范式

### 1. 监督学习（Supervised Learning）

数据有**标签**（label），模型学习从输入到输出的映射。

| 子类型 | 说明 | 示例 |
|--------|------|------|
| **分类** | 预测离散类别 | 垃圾邮件检测（是/否）、图像识别（猫/狗/鸟） |
| **回归** | 预测连续数值 | 房价预测、温度预测 |

```
输入特征 X ──► [ 模型 ] ──► 预测标签 ŷ
  (email)         ↑              (垃圾?)
                  │
            训练时：已知真实标签 y
```

### 2. 无监督学习（Unsupervised Learning）

数据**没有标签**，模型自己发现数据中的结构。

| 子类型 | 说明 | 示例 |
|--------|------|------|
| **聚类** | 将相似样本分组 | 客户分群、新闻话题发现 |
| **降维** | 压缩特征空间 | 数据可视化、去噪 |
| **异常检测** | 找出异常样本 | 信用卡欺诈检测 |

### 3. 强化学习（Reinforcement Learning）

智能体通过与环境交互、试错来学习最优策略。

> ⚠️ scikit-learn **不包含**强化学习算法。如需学习，请了解 OpenAI Gym、Stable Baselines3 等。

## 核心概念一览

### 特征（Feature）与标签（Label）

```
样本：  [面积=120㎡, 位置=朝阳, 楼层=8]  →  标签：价格=850万
         └──────── 特征 X ────────┘       └ 标签 y ┘
```

- **特征矩阵 X**：形状 `(n_samples, n_features)`，每行一个样本，每列一个特征
- **标签向量 y**：形状 `(n_samples,)`，每个样本对应一个目标值

### 训练集与测试集

```
完整数据
├── 训练集（70%-80%）──► 训练模型（fit）
└── 测试集（20%-30%）──► 评估模型（score）
```

**为什么要分？** 模型在训练集上表现好不代表它真的学到了规律——可能只是"死记硬背"（过拟合）。用没见过的测试集才能反映真实泛化能力。

### 过拟合与欠拟合

| 状态 | 表现 | 比喻 |
|------|------|------|
| **欠拟合** | 训练集和测试集都差 | 考试前没复习，啥都不会 |
| **刚好的拟合** | 训练集好、测试集也好 | 学懂了，能举一反三 |
| **过拟合** | 训练集极好、测试集差 | 把答案背下来了，换个题就不会 |

```
误差
 │
高│  ╲  欠拟合区域
  │   ╲
  │    ╲________________
  │     训练误差         ╲  过拟合区域
低│                      ╲
  │                       ╲
  └──────────────────────────► 模型复杂度
```

## 机器学习项目的一般流程

```
1. 定义问题          "要预测什么？分类还是回归？"
      │
2. 收集数据          数据决定了模型的上限
      │
3. 数据探索与清洗    了解数据分布，处理缺失值/异常值
      │
4. 特征工程          把原始数据变成模型能用的特征
      │
5. 选择模型          根据问题类型和数据特点选算法
      │
6. 训练与调参        fit → 评估 → 调参 → 再评估
      │
7. 评估与比较        用测试集/交叉验证比较多个模型
      │
8. 部署与监控       上线后持续监控模型表现
```

> 💡 **黄金法则**：永远不要用测试集来调参。调参要用单独的验证集或交叉验证。

## sklearn 在机器学习流程中的位置

scikit-learn 覆盖了流程中的第 3-7 步：

```
数据收集 ──► [数据预处理] ──► [特征工程] ──► [模型训练] ──► [模型评估] ──► 部署
              ↑               ↑              ↑              ↑
            sklearn         sklearn        sklearn        sklearn
          (preprocessing)  (feature_*)    (各算法)       (metrics)
```

## 小结

| 概念 | 一句话理解 |
|------|-----------|
| 监督学习 | 有标准答案，照着学 |
| 无监督学习 | 没有答案，自己找规律 |
| 训练集 | 练习题 |
| 测试集 | 考试题 |
| 特征 | 样本的属性（面积、位置…） |
| 标签 | 要预测的目标（价格） |
| 过拟合 | 背答案，不会举一反三 |
