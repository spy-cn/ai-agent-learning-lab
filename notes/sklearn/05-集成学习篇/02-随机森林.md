# 02 - 随机森林

## 核心思想

> 训练**很多棵决策树**，让它们投票表决。

```
                    输入特征
                       │
         ┌─────────┬───┴───┬─────────┐
         │         │       │         │
    随机子集1  随机子集2  随机子集3  ... 随机子集N
         │         │       │         │
      决策树1   决策树2  决策树3    决策树N
         │         │       │         │
         ▼         ▼       ▼         ▼
       预测1     预测2   预测3      预测N
         │         │       │         │
         └─────────┴───┬───┴─────────┘
                       │
                  投票 / 平均
                       │
                    最终预测
```

---

## 随机森林 = Bagging + 决策树 + 特征随机

### 相比普通 Bagging 的"额外随机"

普通 Bagging 只在**数据**层面随机（bootstrap 抽样）。

随机森林在两个层面引入随机性：
1. **样本随机**：每棵树用 bootstrap 抽样的数据子集
2. **特征随机**：每次分裂时，只从**随机选的一部分特征**中选最优分裂

```
普通决策树分裂：     考虑所有特征，选最优分裂
随机森林的树分裂：   随机选 √n_features 个特征，从中选最优分裂
```

**为什么这样更好？**
- 避免所有树都选同一个强特征 → 树之间更多样化 → 集成效果更好

## sklearn 实现

### 分类

```python
from sklearn.ensemble import RandomForestClassifier
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import train_test_split

data = load_breast_cancer()
X_train, X_test, y_train, y_test = train_test_split(
    data.data, data.target, test_size=0.3, random_state=42)

# 训练
rf = RandomForestClassifier(
    n_estimators=100,    # 树的数量
    max_depth=None,      # 每棵树的最大深度
    random_state=42
)
rf.fit(X_train, y_train)

print(f"训练集准确率: {rf.score(X_train, y_train):.4f}")
print(f"测试集准确率: {rf.score(X_test, y_test):.4f}")
```

### 回归

```python
from sklearn.ensemble import RandomForestRegressor

rf_reg = RandomForestRegressor(n_estimators=100, random_state=42)
rf_reg.fit(X_train, y_train)
print(f"R²: {rf_reg.score(X_test, y_test):.4f}")
```

## 关键超参数

```python
RandomForestClassifier(
    n_estimators=100,       # 树的数量（最重要）
    max_depth=None,         # 每棵树的最大深度
    min_samples_split=2,    # 分裂最小样本数
    min_samples_leaf=1,     # 叶节点最小样本数
    max_features='sqrt',    # 分裂时考虑的特征数
    bootstrap=True,         # 是否 bootstrap 抽样
    oob_score=False,        # 是否用 OOB 评估
    n_jobs=-1,              # 并行（-1=用所有CPU核心）
    random_state=42
)
```

### 参数详解

#### n_estimators（树的数量）

```python
# 树越多，准确率越高（但边际递减）
# 但训练时间也线性增加
for n in [10, 50, 100, 200, 500]:
    rf = RandomForestClassifier(n_estimators=n, random_state=42, n_jobs=-1)
    rf.fit(X_train, y_train)
    print(f"n={n}: {rf.score(X_test, y_test):.4f}")
```

> 💡 经验：**100-200 棵树**通常足够。更多收益递减。

#### max_features（分裂时的特征随机性）

| 值 | 分类默认 | 回归默认 | 含义 |
|----|---------|---------|------|
| `'sqrt'` | ✅ | | √n_features |
| `'log2'` | | | log₂(n_features) |
| `None` | | ✅ | 所有特征 |
| `0.5` | | | 50%的特征 |
| 整数 | | | 指定数量 |

> 💡 减小 `max_features` → 树更多样化 → 可能提升集成效果

#### oob_score（袋外评估）

```python
rf = RandomForestClassifier(oob_score=True, random_state=42)
rf.fit(X_train, y_train)
print(f"OOB 准确率: {rf.oob_score_:.4f}")
```

**什么是 OOB？**
- Bootstrap 抽样中，每棵树大约有 **37%** 的样本没被选中（Out-Of-Bag）
- 这些 OOB 样本可以用来评估这棵树 → 免费的验证集！
- 不需要额外划分验证集就能评估模型

## 特征重要性

随机森林是最常用的特征重要性评估工具：

```python
import numpy as np
import matplotlib.pyplot as plt

importances = rf.feature_importances_
std = np.std([tree.feature_importances_ for tree in rf.estimators_], axis=0)

# 排序
indices = np.argsort(importances)[::-1]

# 可视化 Top 10
plt.figure(figsize=(10, 6))
plt.bar(range(10), importances[indices[:10]],
        yerr=std[indices[:10]], align='center')
plt.xticks(range(10),
           [data.feature_names[i] for i in indices[:10]],
           rotation=45, ha='right')
plt.xlabel('特征')
plt.ylabel('重要性')
plt.title('随机森林特征重要性 Top 10')
plt.tight_layout()
plt.show()
```

### 特征重要性的两种计算方式

```python
# 1. MDI（平均不纯度减少）- 默认，基于训练集
importances_mdi = rf.feature_importances_

# 2. 排列重要性 - 基于测试集，更可靠
from sklearn.inspection import permutation_importance
result = permutation_importance(rf, X_test, y_test, n_repeats=10, random_state=42)
importances_perm = result.importances_mean
```

> 💡 排列重要性更可靠，因为它直接衡量"打乱某特征后准确率下降多少"。

## 随机森林 vs 单棵决策树

```python
from sklearn.tree import DecisionTreeClassifier

# 单棵决策树（无限制 → 过拟合）
tree = DecisionTreeClassifier(random_state=42)
tree.fit(X_train, y_train)
print(f"单棵树 - 训练: {tree.score(X_train, y_train):.4f}")  # 1.0000
print(f"单棵树 - 测试: {tree.score(X_test, y_test):.4f}")    # 0.9415

# 随机森林（100棵树）
rf = RandomForestClassifier(n_estimators=100, random_state=42)
rf.fit(X_train, y_train)
print(f"随机森林 - 训练: {rf.score(X_train, y_train):.4f}")  # 1.0000
print(f"随机森林 - 测试: {rf.score(X_test, y_test):.4f}")    # 0.9708  ← 提升！
```

## 优缺点

### 优点
- ✅ **开箱即用**，默认参数就不错
- ✅ 不需要特征缩放
- ✅ 能评估特征重要性
- ✅ 不容易过拟合（比单棵树好得多）
- ✅ 提供 OOB 评估
- ✅ **可并行**训练（`n_jobs=-1`）

### 缺点
- ❌ 比单棵树慢（N棵树）
- ❌ 模型大（内存占用）
- ❌ 可解释性比单棵树差（虽然有特征重要性）
- ❌ **无法外推**（回归的预测值在训练集范围内）
- ❌ 对稀疏高维数据不如线性模型

---

## 调参建议

```python
from sklearn.model_selection import GridSearchCV

param_grid = {
    'n_estimators': [100, 200],
    'max_depth': [10, 20, None],
    'max_features': ['sqrt', 'log2'],
    'min_samples_leaf': [1, 5],
}

grid = GridSearchCV(RandomForestClassifier(random_state=42),
                    param_grid, cv=5, n_jobs=-1)
grid.fit(X_train, y_train)
print(f"最佳参数: {grid.best_params_}")
```

**调参顺序建议**：
1. 先固定 `n_estimators=100`
2. 调 `max_depth` 和 `min_samples_leaf`（控制过拟合）
3. 调 `max_features`
4. 最后增大 `n_estimators`

|------|------|
| 核心思想 | 多棵决策树 + 两个层面随机 + 投票 |

| OOB 评估 | 不用划分验证集 |
| 优点 | 强大、开箱即用、可并行 |
| 缺点 | 慢、模型大、无法外推 |
