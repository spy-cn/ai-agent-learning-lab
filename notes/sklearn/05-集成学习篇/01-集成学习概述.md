# 01 - 集成学习概述

## 核心思想

> **三个臭皮匠，顶个诸葛亮。** 把多个"弱模型"组合成一个"强模型"。

```
单个模型：            集成模型：
准确率 80%            模型A ──┐
                      模型B ──┤── 投票/加权 → 准确率 88%
                      模型C ──┘
```

### 为什么集成有效？

```
假设有3个模型，每个独立地以 80% 的准确率做判断：

多数表决正确的概率 = 至少2个模型同时正确的概率
= C(3,2)×0.8²×0.2 + C(3,3)×0.8³
= 3×0.128 + 0.512
= 0.384 + 0.512 = 0.896  ← 接近90%！
```

**前提条件**：各模型要有**多样性**（犯不同的错误）。如果三个模型完全一样，集成没有意义。

---

## 三大集成策略

### 1. Bagging（装袋法）

> 多次有放回抽样，训练多个模型，**投票/平均**。

```
原始数据
    │
    ├── bootstrap 抽样1 ──► 模型1 ──┐
    ├── bootstrap 抽样2 ──► 模型2 ──┤
    ├── bootstrap 抽样3 ──► 模型3 ──┤── 投票 / 平均
    └── ...                        ──┘
```

代表：**随机森林**

**特点**：
- 模型**并行**训练（互相独立）
- 主要降低**方差**（适合高方差的基模型，如决策树）
- 各模型用不同的数据子集

### 2. Boosting（提升法）

> 串行训练，每个新模型**修正前一个的错误**。

```
原始数据 ──► 模型1 ──► 关注模型1的错误 ──► 模型2 ──► 关注错误 ──► 模型3 ──► ...
                                                                     │
                                                           最终 = 加权组合
```

代表：**AdaBoost、GBDT**

**特点**：
- 模型**串行**训练（后一个依赖前一个）
- 主要降低**偏差**（适合欠拟合的基模型）
- 逐步聚焦于难样本

### 3. Stacking（堆叠法）

> 用多个不同类型的模型预测，再用一个"元模型"组合它们的预测。

```
原始数据
    │
    ├── 模型A(决策树) ──┐
    ├── 模型B(SVM)    ──┤── 预测结果作为新特征 ──► 元模型（如逻辑回归）──► 最终预测
    └── 模型C(KNN)    ──┘
```

**特点**：
- 使用**不同类型**的基模型（保证多样性）
- 元模型学习如何最好地组合基模型

## 偏差-方差分解

理解集成的关键在于**偏差-方差权衡**。

```
总误差 = 偏差² + 方差 + 不可约误差

偏差（Bias）：     模型预测的平均值与真实值的差距
                  → 高偏差 = 欠拟合（模型太简单）

方差（Variance）： 模型对不同训练集的预测变化程度
                  → 高方差 = 过拟合（模型太敏感）
```

### 打靶比喻

```
        低方差              高方差
        (稳定)             (不稳定)

低偏差  ●●●  ← 好！       • •• ← 过拟合
(准确)  ●●●                • •

高偏差  ○○○  ← 欠拟合    ◌ ◌◌ ← 最差
(偏离)  ○○○               ◌ ◌
```

### 集成如何帮助？

| 集成方法 | 目标 | 适合的基模型 |
|----------|------|------------|
| Bagging | 降低**方差** | 高方差模型（深决策树） |
| Boosting | 降低**偏差** | 高偏差模型（浅决策树） |
| Stacking | 两者兼顾 | 多样化的模型组合 |

## sklearn 中的集成模块

```python
from sklearn.ensemble import (
    # Bagging 系列
    BaggingClassifier,
    BaggingRegressor,
    RandomForestClassifier,     # ← 重点
    RandomForestRegressor,

    # Boosting 系列
    AdaBoostClassifier,         # ← 重点
    AdaBoostRegressor,
    GradientBoostingClassifier, # ← 重点
    GradientBoostingRegressor,
    HistGradientBoostingClassifier,  # 快速版

    # Voting / Stacking
    VotingClassifier,           # ← 重点
    VotingRegressor,
    StackingClassifier,         # ← 重点
    StackingRegressor,
)
```

## 集成学习的效果对比

```python
from sklearn.datasets import make_classification
from sklearn.model_selection import cross_val_score
from sklearn.tree import DecisionTreeClassifier
from sklearn.ensemble import (
    RandomForestClassifier, AdaBoostClassifier, GradientBoostingClassifier,
    VotingClassifier, StackingClassifier
)
from sklearn.svm import SVC
from sklearn.linear_model import LogisticRegression

X, y = make_classification(n_samples=1000, n_features=20,
                           n_informative=15, random_state=42)

models = {
    '决策树（单模型）': DecisionTreeClassifier(random_state=42),
    '随机森林（Bagging）': RandomForestClassifier(random_state=42),
    'AdaBoost': AdaBoostClassifier(random_state=42),
    'GBDT': GradientBoostingClassifier(random_state=42),
}

print(f"{'模型':<20} {'准确率（5折CV）':<15}")
print("-" * 40)
for name, model in models.items():
    scores = cross_val_score(model, X, y, cv=5)
    print(f"{name:<20} {scores.mean():.4f} ± {scores.std():.4f}")
```

预期输出：
```
模型                 准确率（5折CV）
----------------------------------------
决策树（单模型）      0.8820 ± 0.0120
随机森林（Bagging）   0.9360 ± 0.0080   ← 提升
AdaBoost             0.9100 ± 0.0150   ← 提升
GBDT                 0.9380 ± 0.0100   ← 提升最大
```

## 集成学习的代价

| 方面 | 单模型 | 集成模型 |
|------|--------|---------|
| 准确率 | 基准 | 通常更高 |
| 训练时间 | 快 | 慢 N倍 |
| 预测时间 | 快 | 慢 N倍 |
| 内存 | 小 | 大 |
| 可解释性 | 高 | 降低 |
| 调参难度 | 低 | 高 |

> 💡 集成是用**计算资源**换**准确率**。在准确率至关重要的场景（如比赛、生产）中值得。

---

## 何时用集成？

```
是否需要集成？
    │
    ├── 数据小，追求简单可解释？
    │     └── 单模型即可
    │
    ├── 基线模型效果不够好？
    │     └── 试试随机森林（最简单的集成）
    │
    ├── 需要挤最后一丝准确率？
    │     └── GBDT / Stacking
    │
    └── 参加比赛 / 生产环境？
          └── 必须用集成！
```
