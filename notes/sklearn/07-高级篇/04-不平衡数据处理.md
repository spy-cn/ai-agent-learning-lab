# 04 - 不平衡数据处理

## 什么是不平衡数据？

```
信用卡欺诈：   99.9% 正常 vs 0.1% 欺诈
疾病检测：     95% 健康 vs 5% 患病
故障预测：     99% 正常 vs 1% 故障
```

### 问题

```python
# 一个全部预测"正常"的模型
model.predict = lambda X: np.zeros(len(X))  # 全部预测为0（正常）

# 准确率 = 99.9%  ← 看起来很好
# 但欺诈样本一个都没抓到 → 模型毫无价值
```

---

## 解决方案

### 策略一：class_weight（调整类别权重）

> 让模型更重视少数类。

```python
from sklearn.ensemble import RandomForestClassifier

# 方式1：自动平衡
model = RandomForestClassifier(class_weight='balanced', random_state=42)

# 方式2：手动指定
model = RandomForestClassifier(class_weight={0: 1, 1: 10}, random_state=42)
# 少数类（1）的权重是多数类的10倍
```

```python
# 其他支持的模型
LogisticRegression(class_weight='balanced')
SVC(class_weight='balanced')
DecisionTreeClassifier(class_weight='balanced')
```

### 策略二：重采样

#### 欠采样（减少多数类）

```python
from imblearn.under_sampling import RandomUnderSampler

rus = RandomUnderSampler(random_state=42)
X_resampled, y_resampled = rus.fit_resample(X, y)
# 多数类被随机删除，直到与少数类数量相等
```

#### 过采样（增加少数类）

```python
from imblearn.over_sampling import RandomOverSampler, SMOTE

# 简单复制
ros = RandomOverSampler(random_state=42)
X_res, y_res = ros.fit_resample(X, y)

# SMOTE（合成新样本）
smote = SMOTE(random_state=42)
X_res, y_res = smote.fit_resample(X, y)
```

> ⚠️ `imblearn` 需要单独安装：`pip install imbalanced-learn`

### 策略三：调整决策阈值

```python
# 默认阈值 0.5，可以降低以增加召回率
model = LogisticRegression(random_state=42)
model.fit(X_train, y_train)

y_prob = model.predict_proba(X_test)[:, 1]

# 用 0.3 作为阈值（更倾向于预测为正例）
y_pred = (y_prob > 0.3).astype(int)

from sklearn.metrics import classification_report
print(classification_report(y_test, y_pred))
```

### 策略四：使用合适的评估指标

```python
# ❌ 不要用 accuracy
# ✅ 用 precision, recall, F1, AUC
from sklearn.metrics import (
    f1_score, precision_score, recall_score,
    roc_auc_score, average_precision_score
)

print(f"F1:        {f1_score(y_test, y_pred):.4f}")
print(f"AUC:       {roc_auc_score(y_test, y_prob):.4f}")
print(f"PR-AUC:    {average_precision_score(y_test, y_prob):.4f}")
```

---

## 策略对比

| 方法 | 优点 | 缺点 | 适用 |
|------|------|------|------|
| class_weight | 简单 | 依赖模型 | 首选 |
| 欠采样 | 快 | 丢失信息 | 多数类数据量大 |
| 过采样 | 保留信息 | 可能过拟合 | 数据少 |
| SMOTE | 效果好 | 计算慢 | 数据少 |
| 调阈值 | 灵活 | 需要概率输出 | 微调 |

## 实战建议

```python
# 综合策略
from sklearn.pipeline import Pipeline
from sklearn.ensemble import GradientBoostingClassifier
from sklearn.metrics import classification_report
from imblearn.pipeline import Pipeline as ImbPipeline
from imblearn.combine import SMOTETomek

# 用 imblearn 的 Pipeline（支持重采样）
pipe = ImbPipeline([
    ('smote', SMOTETomek(random_state=42)),       # 重采样
    ('model', GradientBoostingClassifier(
        random_state=42,
        # 不需要 class_weight（GBDT 不支持）
    ))
])

pipe.fit(X_train, y_train)
y_pred = pipe.predict(X_test)
print(classification_report(y_test, y_pred))
```

> 💡 `imblearn.pipeline.Pipeline` 与 sklearn 的 Pipeline 用法相同，但中间可以插入重采样步骤。

---

## 小结

| 策略 | 方法 |
|------|------|
| 调权重 | `class_weight='balanced'` |
| 欠采样 | `RandomUnderSampler` |
| 过采样 | `SMOTE` |
| 调阈值 | 降低阈值增加召回 |
| 评估 | F1、AUC、PR-AUC |
