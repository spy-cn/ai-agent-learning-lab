# 分词技术

分词是中文 NLP 的第一步，也是影响下游任务效果的关键环节。

## 为什么分词很重要？

```
英文: 天然空格分词
  "I love natural language processing"
  → ["I", "love", "natural", "language", "processing"]

中文: 没有空格，需要自动分词
  "我爱自然语言处理"
  → ? "我 / 爱 / 自然 / 语言 / 处理"
  → ? "我 / 爱 / 自然语言 / 处理"  
  → ? "我 / 爱 / 自然语言处理"
  → ? "我爱 / 自然语言处理"

分词错误传播:
  错误分词 → 错误词性 → 错误句法 → 错误语义 → 错误理解
```

## 中文分词的挑战

### 歧义问题

```python
# 经典歧义示例

ambiguity_cases = [
    {
        'text': '结婚的和尚未结婚的',
        'option1': '结婚 / 的 / 和 / 尚未 / 结婚 / 的',    # 正确
        'option2': '结婚 / 的 / 和尚 / 未 / 结婚 / 的',    # 错误(但合法)
    },
    {
        'text': '研究生命的起源',
        'option1': '研究 / 生命 / 的 / 起源',              # 正确
        'option2': '研究生 / 命 / 的 / 起源',              # 错误
    },
    {
        'text': '武汉市长江大桥',
        'option1': '武汉市 / 长江大桥',                    # 武汉市的长江大桥
        'option2': '武汉 / 市长 / 大桥',                   # 武汉市长叫大桥?
    },
    {
        'text': '他说的确实在理',
        'option1': '他 / 说 / 的 / 确实 / 在理',          # 正确
        'option2': '他 / 说 / 的确 / 实在 / 理',          # 错误
    },
    {
        'text': '这份报告我写不好',
        'option1': '这 / 份 / 报告 / 我 / 写 / 不好',     # 我写不好
        'option2': '这 / 份 / 报告 / 我 / 写不 / 好',     # 错误
    },
]

for case in ambiguity_cases:
    print(f"原文: {case['text']}")
    print(f"  解析1: {case['option1']}")
    print(f"  解析2: {case['option2']}")
    print()
```

### 未登录词问题

```
未登录词 (OOV - Out of Vocabulary):

词典中有的词 (登录词):
  自然语言、计算机、手机...

新词/专有名词 (未登录词):
  人名: 张伟、李华
  地名: 蜀山区、海淀区
  机构: 字节跳动、OpenAI
  新词: yyds、栓Q、绝绝子
  专业术语: 大语言模型、RAG

解决思路:
  1. 词典 + 规则 (覆盖登录词)
  2. 统计模型 (处理未登录词)
  3. 字符级模型 (无词表限制)
```

## 1. Jieba 分词

```python
import jieba

# === 三种分词模式 ===

text = "自然语言处理是人工智能的重要分支"

# 1. 精确模式 (默认): 最精确切分
words_precise = jieba.lcut(text)
print(f"精确模式: {' / '.join(words_precise)}")
# 自然语言 / 处理 / 是 / 人工智能 / 的 / 重要 / 分支

# 2. 全模式: 所有可能的切分
words_full = jieba.lcut(text, cut_all=True)
print(f"全模式: {' / '.join(words_full)}")
# 自然 / 自然语言 / 语言 / 处理 / 是 / 人工 / 人工智能 / 智能 / 的 / 重要 / 分支

# 3. 搜索引擎模式: 在精确基础上再切分短词
words_search = jieba.lcut_for_search(text)
print(f"搜索引擎模式: {' / '.join(words_search)}")
# 自然 / 语言 / 自然语言 / 处理 / 是 / 人工 / 智能 / 人工智能 / 的 / 重要 / 分支
```

### 关键词提取

```python
# === TF-IDF 关键词提取 ===
from jieba import analyse

text = """
自然语言处理是人工智能领域中的重要方向。
它研究能实现人与计算机之间用自然语言进行有效通信的理论和方法。
自然语言处理是一门融语言学、计算机科学、数学于一体的科学。
深度学习技术的突破极大地推动了自然语言处理的发展。
预训练语言模型如BERT和GPT在各种NLP任务中取得了优异的效果。
"""

# TF-IDF 关键词
keywords_tfidf = analyse.extract_tags(text, topK=5, withWeight=True)
print("TF-IDF 关键词:")
for word, weight in keywords_tfidf:
    print(f"  {word}: {weight:.4f}")

# TextRank 关键词
keywords_textrank = analyse.textrank(text, topK=5, withWeight=True)
print("\nTextRank 关键词:")
for word, weight in keywords_textrank:
    print(f"  {word}: {weight:.4f}")
```

## 4. Subword 分词 (深度学习时代)

### 为什么需要 Subword？

```
传统分词的问题:
  1. 词表太大 (几十万词)
  2. 未登录词无法处理
  3. 形态丰富的语言效率低 (德语/芬兰语)

Subword 方案:
  介于字符级和词级之间
  常见词保持完整 → playing
  罕见词拆分为子词 → un + happi + ly
  完全未知的拆为字符 → xyz → x + y + z

优势:
  ✓ 词表大小可控 (通常 30K~50K)
  ✓ 无未登录词问题
  ✓ 捕获词缀信息
```

### BPE (Byte Pair Encoding)

```
BPE 算法流程:

初始: 所有词拆为字符
  low: l o w </w>
  lower: l o w e r </w>
  newest: n e w e s t </w>
  widest: w i d e s t </w>

第1次合并 (lo 出现2次):
  low: lo w </w>
  lower: lo w e r </w>

第2次合并 (low 出现2次):
  low: low </w>
  lower: low e r </w>

第3次合并 (er 出现2次):
  lower: low er </w>
  newest: n e w er </w>

...继续合并直到达到目标词表大小
```

```python
# HuggingFace Tokenizers 实现 BPE
from tokenizers import Tokenizer
from tokenizers.models import BPE
from tokenizers.trainers import BpeTrainer
from tokenizers.pre_tokenizers import Whitespace

# 创建 BPE 分词器
tokenizer = Tokenizer(BPE(unk_token="<unk>"))
tokenizer.pre_tokenizer = Whitespace()
trainer = BpeTrainer(
    vocab_size=5000,
    special_tokens=["<unk>", "<pad>", "<bos>", "<eos>"]
)

# 训练
corpus = [
    "自然语言处理是人工智能的重要分支",
    "深度学习推动了自然语言处理的发展",
    "预训练模型取得了突破性进展",
    # ... 大量文本
] * 100

tokenizer.train_from_iterator(corpus, trainer)

# 使用
text = "自然语言处理很重要"
encoding = tokenizer.encode(text)
print(f"原文: {text}")
print(f"Token: {encoding.tokens}")
print(f"IDs: {encoding.ids}")
```

## 5. 分词评估

```python
def evaluate_segmentation(gold, predicted):
    """评估分词质量
    
    Args:
        gold: 正确分词列表 [['我', '爱', 'NLP'], ...]
        predicted: 预测分词列表
    """
    correct = 0
    total_gold = 0
    total_pred = 0
    
    for g_words, p_words in zip(gold, predicted):
        total_gold += len(g_words)
        total_pred += len(p_words)
        
        # 转换为位置区间
        g_set = set()
        pos = 0
        for word in g_words:
            g_set.add((pos, pos + len(word)))
            pos += len(word)
        
        p_set = set()
        pos = 0
        for word in p_words:
            p_set.add((pos, pos + len(word)))
            pos += len(word)
        
        correct += len(g_set & p_set)
    
    precision = correct / total_pred if total_pred > 0 else 0
    recall = correct / total_gold if total_gold > 0 else 0
    f1 = 2 * precision * recall / (precision + recall) if (precision + recall) > 0 else 0
    
    return {
        'precision': precision,
        'recall': recall,
        'f1': f1,
    }

# 测试
gold = [
    ['我', '爱', '自然语言', '处理'],
    ['深度', '学习', '很', '重要'],
]
predicted = [
    ['我', '爱', '自然', '语言', '处理'],   # 自然语言 切错了
    ['深度', '学习', '很', '重要'],          # 完全正确
]

result = evaluate_segmentation(gold, predicted)
print(f"分词评估:")
print(f"  精确率(P): {result['precision']:.4f}")
print(f"  召回率(R): {result['recall']:.4f}")
print(f"  F1值:     {result['f1']:.4f}")
```
