# 位置编码

Transformer 的核心组件 Self-Attention 有一个致命问题：**它完全没有位置感知能力**。打乱输入序列的顺序，Self-Attention 的输出除了顺序外完全不变。但语言中词序至关重要——"狗咬人"和"人咬狗"含义截然不同。**位置编码（Positional Encoding）** 就是用来给模型注入位置信息的机制，是 Transformer 不可或缺的组件。从原始的正弦余弦编码，到 GPT 的可学习编码，再到现代大模型广泛使用的 RoPE 旋转位置编码，位置编码技术不断演进。

---

## 一、为什么需要位置编码

### 1.1 Self-Attention 的位置盲点

```
Self-Attention 是排列不变的（Permutation Invariant）:

  输入 A: "猫 追 老鼠"
  输入 B: "老鼠 追 猫"

  ┌─────────────────────────────────────────────┐
  │  Self-Attention 把输入看作"集合"而非"序列"  │
  │                                             │
  │  打乱顺序 → 每个词的输出内容完全不变        │
  │  （只是输出位置跟着移动）                    │
  │                                             │
  │  这对语言理解是灾难性的!                    │
  │  "我 打 你" ≠ "你 打 我"                    │
  └─────────────────────────────────────────────┘

  对比 RNN:
    RNN 天然有序列顺序 —— h₁ 先算，h₂ 后算，顺序隐含在计算图中

  对比 CNN:
    CNN 也有局部位置信息 —— 不同 kernel 位置对应不同输入位置

  Transformer 必须显式注入位置信息!
```

### 1.2 位置编码的思路

```
解决方案: 给每个位置加上一个"位置标签"

  原始嵌入:  Embed(token)         ← 只有词义信息
  加位置后:  Embed(token) + PE(pos)  ← 词义 + 位置信息

  ┌────────────────────────────────────────────┐
  │                                            │
  │  位置  Token   Token Embedding   PE        │
  │  ────  ─────   ──────────────   ───       │
  │   0    "我"    [0.2, -0.1, ...] [0.0, 1.0]│
  │   1    "爱"    [0.5, 0.3, ...]  [0.8, 0.6]│
  │   2    "NLP"   [0.1, 0.9, ...]  [0.9, -0.4│
  │                                            │
  │  最终输入 = Token Embedding + PE            │
  │                                            │
  └────────────────────────────────────────────┘
```

## 二、正弦余弦编码（原始 Transformer）

### 2.1 公式

```
Sinusoidal Positional Encoding (原论文方案):

  PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
  PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

  其中:
    pos    = 位置索引 (0, 1, 2, ...)
    i      = 维度索引 (0, 1, ..., d_model/2-1)
    d_model = 嵌入维度 (如 512)

  直觉:
    - 偶数维用 sin，奇数维用 cos
    - 不同维度对应不同频率的正弦波
    - 高频维度编码局部位置，低频维度编码全局位置
```

### 2.2 为什么用正弦余弦

```
正弦余弦编码的优势:

  ① 不同频率形成"多重时钟"
  ─────────────────────────────────
  维度 0 (高频): sin(pos / 1)      ← 每个位置都不同
  维度 100 (中频): sin(pos / 100)  ← 相邻位置接近
  维度 500 (低频): sin(pos / 10000)← 远距离才变化

  ② 相对位置可线性表达
  ─────────────────────────────────
  利用三角恒等式:
    sin(pos + k) = sin(pos)cos(k) + cos(pos)sin(k)
    cos(pos + k) = cos(pos)cos(k) - sin(pos)sin(k)

  这意味着 PE(pos+k) 可以表示为 PE(pos) 的线性组合
  模型容易学会"相对位置"关系

  ③ 可外推到更长序列
  ─────────────────────────────────
  正弦函数对任意位置都有定义
  理论上可处理训练中未见过的更长序列
```

### 2.3 可视化

```
正弦余弦位置编码热力图 (d_model=64):

位置 →  0    1    2    3    ...   63
  ┌────┬────┬────┬────┬────┬────┬────┐
0 │ 0.0│ 0.8│ 0.9│ 0.5│... │0.0 │0.0 │  ← 高频(快速变化)
1 │ 0.0│ 0.6│ 1.0│ 0.9│... │0.0 │0.0 │
2 │ 0.0│ 0.3│ 0.6│ 0.8│... │0.1 │0.0 │
  │ ...                                │
30│ 0.0│ 0.0│ 0.0│ 0.1│... │0.5 │0.5 │  ← 低频(缓慢变化)
31│ 1.0│ 1.0│ 1.0│ 0.9│... │0.8 │0.9 │
  └────┴────┴────┴────┴────┴────┴────┘

  上方高频维度像"秒针"，每个位置都不同
  下方低频维度像"时针"，远距离才变化
  → 类似时钟的多尺度位置编码
```

## 三、可学习位置编码（GPT 使用）

### 3.1 方案

```
Learned Positional Embedding:

  与词嵌入类似，为每个位置学习一个向量:

    PE = Embedding(max_len, d_model)

  示例 (max_len=512, d_model=768):
    位置 0 → 查表得到 [0.12, -0.34, ...] (768维)
    位置 1 → 查表得到 [0.56, 0.78, ...]
    ...

  特点:
    + 简单直接，模型自己学最优位置表示
    + 在训练长度内表现往往优于正弦编码
    - 无法外推到训练时未见过的更长序列
    - max_len 固定，限制了模型处理长文本的能力
```

### 3.2 GPT 的选择

```
GPT 系列使用可学习位置编码:

  GPT-2:  max_len=1024
  GPT-3:  max_len=2048

  问题: 如果输入超过 max_len 怎么办?
  → GPT 无法处理超过训练长度的序列
  → 这是可学习编码的局限性

  对比: 原始 Transformer 的正弦编码理论上可外推
        但实际外推效果也有限
```

## 四、相对位置编码

### 4.1 动机

```
绝对 vs 相对位置:

  绝对位置编码:
    "词A在位置3，词B在位置5"
    关注的是每个词的绝对位置

  相对位置编码:
    "词A和词B相差2个位置"
    关注的是词与词之间的相对距离

  直觉:
    "我喜欢吃苹果" 和 "昨天我喜欢吃苹果"
    "我"和"吃"的关系（主谓）不变，只是绝对位置变了
    相对位置编码让模型更容易学会这种关系
```

### 4.2 方案

```
相对位置编码（Relative Position Encoding）:

  在计算 Attention 时，加入相对位置偏置:

  原始 Attention:
    score(i,j) = Q_i · K_j

  相对位置 Attention:
    score(i,j) = Q_i · K_j + b_{rel(i-j)}

  其中 b_{rel(i-j)} 是关于相对距离 i-j 的可学习偏置

  ┌────────────────────────────────────┐
  │  常见的相对距离桶（bucket）映射:   │
  │                                    │
  │  相对距离 d → bucket(d)            │
  │  d=0       → bucket 0              │
  │  d=±1      → bucket 1, 2           │
  │  d=±2      → bucket 3, 4           │
  │  d=±(4~8)  → bucket 5-12           │
  │  d=±(8~16) → bucket 13-28          │
  │  ...                               │
  │                                    │
  │  对数分桶，近距离精细，远距离粗略  │
  └────────────────────────────────────┘

  代表模型: T5, Transformer-XL, BART
```

## 五、RoPE 旋转位置编码

### 5.1 核心思想

```
RoPE (Rotary Position Embedding):
  用旋转矩阵编码位置 —— 当前大模型的主流选择

  核心思想:
    把 Q, K 向量看作二维平面上的点
    根据位置 pos 对其进行旋转

  ┌────────────────────────────────────────────┐
  │                                            │
  │  位置 pos 的旋转:                          │
  │                                            │
  │   (Q_2i,   Q_2i+1)                        │
  │      ↗                                    │
  │     /  旋转角度 θ = pos / 10000^(2i/d)     │
  │    /                                      │
  │   ↙                                       │
  │                                            │
  │  RoPE(q, pos) = q · R(pos)                │
  │                                            │
  │  其中 R(pos) 是分块旋转矩阵:               │
  │                                            │
  │     ┌ cos(θ)  -sin(θ)  0       0    ┐    │
  │     │ sin(θ)   cos(θ)  0       0    │    │
  │     │   0        0    cos(θ') sin(θ')│   │
  │     │   0        0   -sin(θ') cos(θ')│   │
  │     └                               ┘    │
  │                                            │
  │  θ' = pos / 10000^(2(i+1)/d)             │
  │                                            │
  └────────────────────────────────────────────┘
```

### 5.2 RoPE 的优势

```
RoPE 为什么成为主流:

  ① 编码相对位置
  ─────────────────────────────────
    Q_m · K_n = |q|·|k|·cos(mθ - nθ)
                        ↑ 只依赖相对距离 m-n

  ② 可外推
  ─────────────────────────────────
    旋转矩阵对任意位置都有定义
    配合 NTK-aware scaling 可外推到很长序列

  ③ 不引入额外参数
  ─────────────────────────────────
    不像可学习编码需要 Embedding 表
    RoPE 只用数学公式，零参数

  ④ 高效实现
  ─────────────────────────────────
    可用逐元素乘法实现，计算开销极小

  使用 RoPE 的模型: Llama, Mistral, Qwen, ChatGLM, ...
```

### 5.3 RoPE 实现

```
RoPE 的高效实现（逐元素操作）:

  给定向量 q = [q₀, q₁, q₂, q₃, ...]

  步骤1: 把向量分成两半
    q_front = [q₀, q₁, ...]
    q_back  = [q_{d/2}, q_{d/2+1}, ...]

  步骤2: 旋转
    q'_front = q_front · cos(θ) - q_back · sin(θ)
    q'_back  = q_back  · cos(θ) + q_front · sin(θ)

  步骤3: 拼接
    q' = concat(q'_front, q'_back)

  这种实现避免了完整的矩阵乘法，非常高效
```

## 六、ALiBi（Attention with Linear Biases）

### 6.1 方案

```
ALiBi (Attention with Linear Biases Position Embedding):

  不在嵌入层加位置信息，而是在 Attention 分数上加线性偏置:

  修改后的 Attention:
    score(i,j) = Q_i · K_j + (-m · |i - j|)

  其中 m 是每个头的斜率（固定，几何递减）

  ┌──────────────────────────────────────────────┐
  │  ALiBi 的注意力偏置矩阵（示意图）:          │
  │                                              │
  │         j=0  j=1  j=2  j=3  j=4             │
  │  i=0  [ 0   -m   -2m  -3m  -4m]             │
  │  i=1  [ 0    0   -m   -2m  -3m]             │
  │  i=2  [ 0    0    0   -m   -2m]             │
  │  i=3  [ 0    0    0    0   -m ]             │
  │  i=4  [ 0    0    0    0    0 ]             │
  │                                              │
  │  距离越远，惩罚越大（线性递减）              │
  │  softmax 后远处注意力权重自然变小            │
  └──────────────────────────────────────────────┘

  不同头使用不同斜率 m:
    头1: m = 1/2^0 = 1.0       ← 强距离惩罚
    头2: m = 1/2^1 = 0.5
    头3: m = 1/2^2 = 0.25
    ...
    头8: m = 1/2^7 ≈ 0.008     ← 弱距离惩罚
```

### 6.2 ALiBi 的优势

```
ALiBi 特点:

  ① 极强的外推能力
  ─────────────────────────────────
    训练时 max_len=512，推理时可处理 2000+ 长度
    性能几乎不下降

  ② 零额外参数
  ─────────────────────────────────
    斜率 m 是固定的（几何序列），无需学习

  ③ 计算高效
  ─────────────────────────────────
    只在 attention scores 上加偏置

  代表模型: BLOOM, MPT
```

## 七、代码实现

### 7.1 正弦余弦位置编码

```python
import torch
import torch.nn as nn
import math
import matplotlib.pyplot as plt


class SinusoidalPositionalEncoding(nn.Module):
    """原始 Transformer 的正弦余弦位置编码"""

    def __init__(self, d_model=512, max_len=5000):
        super().__init__()

        # 创建位置编码矩阵
        pe = torch.zeros(max_len, d_model)
        position = torch.arange(0, max_len).unsqueeze(1).float()

        # 计算频率项: 1 / 10000^(2i/d_model)
        div_term = torch.exp(
            torch.arange(0, d_model, 2).float() *
            -(math.log(10000.0) / d_model)
        )

        pe[:, 0::2] = torch.sin(position * div_term)  # 偶数维 sin
        pe[:, 1::2] = torch.cos(position * div_term)  # 奇数维 cos

        pe = pe.unsqueeze(0)  # (1, max_len, d_model)
        self.register_buffer('pe', pe)

    def forward(self, x):
        """x: (batch, seq_len, d_model)"""
        return x + self.pe[:, :x.size(1)]


# 测试
if __name__ == "__main__":
    pe = SinusoidalPositionalEncoding(d_model=64, max_len=100)
    x = torch.randn(2, 10, 64)
    output = pe(x)
    print(f"输入形状: {x.shape}")
    print(f"输出形状: {output.shape}")

    # 可视化位置编码
    pe_matrix = pe.pe[0, :20, :16].numpy()
    print(f"位置编码矩阵 (20位置 x 16维):")
    print(f"形状: {pe_matrix.shape}")
    print(f"位置0的前8维: {pe_matrix[0, :8]}")
    print(f"位置1的前8维: {pe_matrix[1, :8]}")
```

```python
class LearnedPositionalEncoding(nn.Module):
    """GPT 使用的可学习位置编码"""

    def __init__(self, d_model=768, max_len=1024):
        super().__init__()
        self.pos_embedding = nn.Embedding(max_len, d_model)

    def forward(self, x):
        """x: (batch, seq_len, d_model)"""
        batch_size, seq_len, _ = x.size()
        positions = torch.arange(seq_len, device=x.device)
        return x + self.pos_embedding(positions).unsqueeze(0)
```

### 7.3 RoPE 旋转位置编码

```python
class RotaryPositionEmbedding(nn.Module):
    """RoPE 旋转位置编码（Llama/Mistral 使用）"""

    def __init__(self, dim=64, max_len=2048, base=10000):
        super().__init__()

        # 计算频率
        inv_freq = 1.0 / (base ** (torch.arange(0, dim, 2).float() / dim))
        self.register_buffer('inv_freq', inv_freq)

        # 预计算 cos/sin
        t = torch.arange(max_len).float()
        freqs = torch.outer(t, inv_freq)  # (max_len, dim/2)
        emb = torch.cat([freqs, freqs], dim=-1)  # (max_len, dim)

        self.register_buffer('cos_cached', emb.cos())
        self.register_buffer('sin_cached', emb.sin())

    def rotate_half(self, x):
        """将后半部分取负并翻转"""
        x1 = x[..., :x.size(-1) // 2]
        x2 = x[..., x.size(-1) // 2:]
        return torch.cat([-x2, x1], dim=-1)

    def forward(self, x, seq_len=None):
        """
        x: (batch, heads, seq_len, dim)
        """
        if seq_len is None:
            seq_len = x.size(-2)

        cos = self.cos_cached[:seq_len].unsqueeze(0).unsqueeze(0)
        sin = self.sin_cached[:seq_len].unsqueeze(0).unsqueeze(0)

        # 旋转: x * cos + rotate_half(x) * sin
        return x * cos + self.rotate_half(x) * sin


def apply_rope_to_qk(q, k, rope):
    """对 Q 和 K 应用 RoPE"""
    q_rotated = rope(q)
    k_rotated = rope(k)
    return q_rotated, k_rotated


# 测试
if __name__ == "__main__":
    dim = 64
    num_heads = 8
    seq_len = 32
    batch_size = 2

    rope = RotaryPositionEmbedding(dim=dim // num_heads, max_len=2048)

    # 模拟多头注意力的 Q, K
    q = torch.randn(batch_size, num_heads, seq_len, dim // num_heads)
    k = torch.randn(batch_size, num_heads, seq_len, dim // num_heads)

    q_rot, k_rot = apply_rope_to_qk(q, k, rope)
    print(f"RoPE 测试:")
    print(f"  Q 原始: {q.shape}, 旋转后: {q_rot.shape}")
    print(f"  K 原始: {k.shape}, 旋转后: {k_rot.shape}")
    print(f"  cos 缓存形状: {rope.cos_cached.shape}")
```

### 7.4 ALiBi 实现

```python
def get_alibi_slopes(num_heads):
    """获取 ALiBi 的斜率（几何序列）"""
    def get_slopes_power_of_2(n):
        start = 2 ** (-(2 ** -(math.log2(n) - 3)))
        return [start * start ** (i + 1) for i in range(n)]

    if math.log2(num_heads).is_integer():
        return get_slopes_power_of_2(num_heads)
    else:
        # 非二次幂头数的处理
        closest_power_of_2 = 2 ** round(math.log2(num_heads))
        return (
            get_slopes_power_of_2(closest_power_of_2)[:num_heads]
        )


class ALiBiAttention(nn.Module):
    """带 ALiBi 偏置的注意力"""

    def __init__(self, d_model=512, num_heads=8, max_len=2048):
        super().__init__()
        self.num_heads = num_heads
        self.d_k = d_model // num_heads

        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)

        # 预计算 ALiBi 偏置矩阵
        slopes = torch.tensor(get_alibi_slopes(num_heads))
        position = torch.arange(max_len).unsqueeze(1) - torch.arange(max_len).unsqueeze(0)
        bias = slopes.unsqueeze(1).unsqueeze(1) * position  # (heads, 1, max_len, max_len)
        self.register_buffer('bias', bias)

    def forward(self, x):
        batch_size, seq_len, _ = x.size()

        Q = self.W_q(x).view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        K = self.W_k(x).view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        V = self.W_v(x).view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)

        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(self.d_k)

        # 加入 ALiBi 偏置
        scores = scores + self.bias[:, :, :seq_len, :seq_len]

        attn = torch.softmax(scores, dim=-1)
        output = torch.matmul(attn, V)
        return output


# 测试
if __name__ == "__main__":
    alibi_attn = ALiBiAttention(d_model=512, num_heads=8, max_len=512)
    x = torch.randn(2, 20, 512)
    output = alibi_attn(x)
    print(f"ALiBi Attention:")
    print(f"  输入: {x.shape}")
    print(f"  输出: {output.shape}")
    slopes = get_alibi_slopes(8)
    print(f"  8个头的斜率: {[f'{s:.4f}' for s in slopes]}")
```

```
│ RoPE         │    0        │   强     │ 显式     │   高     │ Llama      │
│ ALiBi        │    0        │   极强   │ 显式     │   高     │ BLOOM      │
└──────────────┴─────────────┴──────────┴──────────┴──────────┴────────────┘

外推性 = 处理比训练时更长的序列的能力
```

---

## 小结

| 要点 | 内容 |
|------|------|
| 为什么需要位置编码 | Self-Attention 是排列不变的，缺乏位置信息 |
| 正弦余弦编码 | 原始 Transformer 方案，多频率，零参数，可外推 |
| 可学习编码 | GPT 使用，简单有效，但无法外推 |
| 相对位置编码 | T5/BART 使用，关注词间相对距离，外推性中等 |
| RoPE | 用旋转编码位置，显式相对位置，外推性强，Llama 使用 |
| ALiBi | 在 attention 分数加线性偏置，外推性极强，BLOOM 使用 |
| 多频率 | 不同维度对应不同频率，类似时钟的多尺度编码 |
| 外推性 | RoPE 和 ALiBi 外推能力强，是现代大模型主流 |
| 选择建议 | 短文本可用可学习；长文本/大模型推荐 RoPE 或 ALiBi |
