# T5 与编码器-解码器架构

前面我们学了 BERT（Encoder-only）和 GPT（Decoder-only），它们各有所长：BERT 擅长理解，GPT 擅长生成。那么有没有一种架构能**兼顾理解和生成**？**T5（Text-to-Text Transfer Transformer）** 给出了优雅的答案——把**所有 NLP 任务统一为"文本输入 → 文本输出"**的范式。T5 使用完整的 **Encoder-Decoder 架构**，与 BART 一起成为这一流派的代表。本篇将深入探讨 T5、BART 以及三大 Transformer 架构的全面对比。

---

## 一、T5 核心思想

### 1.1 Text-to-Text 范式

```
T5 的核心创新: 统一所有任务为 text-to-text

  ┌──────────────────────────────────────────────────┐
  │                                                  │
  │  输入文本 → [T5 模型] → 输出文本                  │
  │                                                  │
  │  无论什么任务，都是: 文本 → 文本                  │
  │                                                  │
  │  翻译:                                           │
  │    "translate English to German: Thank you"       │
  │    → "Danke"                                     │
  │                                                  │
  │  摘要:                                           ││
  │    "summarize: [长文章...]"                       │
  │    → "文章摘要..."                                │
  │                                                  │
  │  分类:                                           │
  │    "sentiment: This movie is great"               │
  │    → "positive"                                  │
  │                                                  │
  │  问答:                                           │
  │    "question: ... context: ..."                   │
  │    → "答案"                                      │
  │                                                  │
  │  回归:                                           │
  │    "stsb sentence1: ... sentence2: ..."           │
  │    → "3.8"  (相似度分数)                          │
  │                                                  │
  └──────────────────────────────────────────────────┘
```

### 1.2 为什么 Text-to-Text 是革命性的

```
传统方式: 每个任务需要不同的模型和输出头

  ┌──────────────────────────────────────────────┐
  │                                              │
  │  分类任务 → BERT + 分类头 (输出类别)         │
  │  问答任务 → BERT + 起止位置预测头            │
  │  翻译任务 → Encoder-Decoder + 词表 softmax   │
  │  摘要任务 → 另一个模型...                    │
  │                                              │
  │  问题: 每个任务需要定制，无法统一             │
  │                                              │
  └──────────────────────────────────────────────┘

T5 方式: 一个模型处理所有任务

  ┌──────────────────────────────────────────────┐
  │                                              │
  │  所有任务 → 同一个 T5 模型 → 文本输出        │
  │                                              │
  │  优势:                                       │
  │  + 统一的输入输出格式                         │
  │  + 用同一套损失函数 (语言模型 loss)           │
  │  + 预训练和微调用相同范式                     │
  │  + 多任务学习更简单                           │
  │  + 零样本/少样本能力                          │
  │                                              │
  └──────────────────────────────────────────────┘
```

## 二、T5 架构

### 2.1 Encoder-Decoder 结构

```
T5 架构 (完整的 Encoder-Decoder):

   ┌─────────────────────────┐    ┌──────────────────────────────┐
   │        ENCODER          │    │          DECODER             │
   │                         │    │                              │
   │  输入: "translate: ..." │    │  输出: "Danke"               │
   │       ↓                 │    │       ↓                     │
   │  [Encoder × N]          │    │  [Decoder × N]              │
   │                         │    │                              │
   │  双向 Self-Attention    │───→│  Cross Attention            │
   │  (完整理解输入)          │    │  + Masked Self-Attention    │
   │       ↓                 │    │  (生成输出)                  │
   │  编码器输出             │    │       ↓                     │
   │                         │    │  Linear + Softmax           │
   └─────────────────────────┘    └──────────────────────────────┘

  对比:
    BERT:   只有 Encoder (双向注意力)
    GPT:    只有 Decoder (因果注意力)
    T5:     Encoder + Decoder 都有
```

### 2.2 T5 的特殊设计

```
T5 vs 原始 Transformer 的区别:

  ┌──────────────────────────────────────────────────┐
  │                                                  │
  │  T5 的特殊之处:                                  │
  │                                                  │
  │  ① 简化的 LayerNorm                              │
  │  ──────────────────────────────────────────────  │
  │  去掉均值中心化，只用标准差归一化                  │
  │                                                  │
  │  ② 相对位置编码                                   │
  │  ──────────────────────────────────────────────  │
  │  T5 使用 Bucketed 相对位置编码                    │
  │  在 Attention 中加入相对位置偏置                   │
  │                                                  │
  │  ③ Encoder 不使用因果掩码                         │
  │  ──────────────────────────────────────────────  │
  │  Encoder 可以完整看到输入 (双向)                  │
  │  Decoder 使用因果掩码 (自回归生成)                │
  │                                                  │
  │  ④ 无偏置的 FFN                                   │
  │  ──────────────────────────────────────────────  │
  │  前馈网络不使用偏置项                             │
  │                                                  │
  └──────────────────────────────────────────────────┘
```

### 2.3 T5 任务前缀设计

```
T5 的任务前缀 (Task Prefix):

  通过输入文本的前缀来区分不同任务:

  ┌──────────────────────────────────────────────────┐
  │                                                  │
  │  任务       输入前缀              输出示例       │
  │  ─────     ──────────────────    ──────────     │
  │  翻译      "translate X to Y:"   翻译结果        │
  │  摘要      "summarize:"          摘要文本        │
  │  分类      "sentiment:"          positive/neg    │
  │  问答      "question: context:"  答案            │
  │  NER       "ner:"                实体标签        │
  │  回归      "stsb:"               分数            │
  │  完成      (无前缀)              续写文本        │
  │                                                  │
  │  前缀让模型知道"这个文本要做什么任务"             │
  │  → 多任务学习的统一接口                          │
  │                                                  │
  └──────────────────────────────────────────────────┘
```

## 三、BART

### 3.1 BART 架构

```
BART (Bidirectional and Auto-Regressive Transformers):

  本质: 一个标准的 seq2seq Transformer (Encoder-Decoder)

  ┌──────────────────────────────────────────────────┐
  │                                                  │
  │  BART 的预训练: 去噪自编码器 (Denoising Autoenc) │
  │                                                  │
  │  原始文本: "The cat sat on the mat"              │
  │       ↓ 噪声函数                                 │
  │  损坏文本: "The [MASK] sat [MASK] the mat"       │
  │       ↓ Encoder                                  │
  │  编码表示: 理解损坏文本的语义                    │
  │       ↓ Decoder (自回归)                         │
  │  重建文本: "The cat sat on the mat"              │
  │                                                  │
  │  目标: 让模型学会从损坏文本恢复原始文本           │
  │  → 学到深层语言理解 + 生成能力                   │
  │                                                  │
  └──────────────────────────────────────────────────┘
```

### 3.2 BART 的噪声类型

```
BART 使用的多种噪声策略:

  ┌──────────────────────────────────────────────────┐
  │                                                  │
  │  ① Token Masking (Token 掩码)                    │
  │  ──────────────────────────────────────────────  │
  │  原始: "The cat sat on the mat"                  │
  │  损坏: "The [MASK] sat on the mat"               │
  │  (类似 BERT 的 MLM)                              │
  │                                                  │
  │  ② Token Deletion (Token 删除)                   │
  │  ──────────────────────────────────────────────  │
  │  原始: "The cat sat on the mat"                  │
  │  损坏: "The sat the mat"  (删掉了 "cat", "on")   │
  │  (模型需推断缺失的位置)                           │
  │                                                  │
  │  ③ Text Infilling (文本填充)                     │
  │  ──────────────────────────────────────────────  │
  │  原始: "The cat sat on the mat"                  │
  │  损坏: "The [MASK] on the mat"                   │
  │  (可能遮挡多个词，用单个 [MASK] 替换)            │
  │  (模型需预测遮挡了多少词)                         │
  │                                                  │
  │  ④ Sentence Permutation (句子乱序)               │
  │  ──────────────────────────────────────────────  │
  │  原始: "A. B. C. D."                             │
  │  损坏: "C. A. D. B."                             │
  │  (打乱文档中句子的顺序)                           │
  │                                                  │
  │  ⑤ Document Rotation (文档旋转)                  │
  │  ──────────────────────────────────────────────  │
  │  原始: "A B C D"                                 │
  │  损坏: "C D A B"                                 │
  │  (从随机位置开始旋转)                             │
  │                                                  │
  │  这些噪声让 BART 同时学会:                       │
  │  - 语言理解 (识别缺失/乱序)                      │
  │  - 语言生成 (重建原始文本)                        │
  │                                                  │
  └──────────────────────────────────────────────────┘
```

## 四、Encoder-Decoder 优劣势

### 4.1 优势

```
Encoder-Decoder 架构的优势:

  ┌──────────────────────────────────────────────────┐
  │                                                  │
  │  ① Encoder 的双向理解                            │
  │  ──────────────────────────────────────────────  │
  │  Encoder 可以完整看到整个输入                     │
  │  理解能力 > Decoder-only 的单向理解               │
  │                                                  │
  │  ② Decoder 的自回归生成                          │
  │  ──────────────────────────────────────────────  │
  │  Decoder 逐步生成，可控性好                       │
  │  可以用 beam search 等解码策略                    │
  │                                                  │
  │  ③ 适合 Seq2Seq 任务                             │
  │  ──────────────────────────────────────────────  │
  │  翻译、摘要等"输入→输出不同长度"的任务            │
  │  Encoder 理解长输入，Decoder 生成短输出            │
  │                                                  │
  │  ④ Cross Attention 提供显式对齐                   │
  │  ──────────────────────────────────────────────  │
  │  Decoder 生成时可以"查看"Encoder 的特定位置       │
  │  提供了可解释的源-目标对齐                        │
  │                                                  │
  └──────────────────────────────────────────────────┘
```

### 4.2 劣势

```
Encoder-Decoder 架构的劣势:

  ┌──────────────────────────────────────────────────┐
  │                                                  │
  │  ① 参数效率较低                                  │
  │  ──────────────────────────────────────────────  │
  │  有两套 Transformer 堆叠 (Encoder + Decoder)     │
  │  同参数量下，单层深度不如 Decoder-only            │
  │                                                  │
  │  ② 推理速度较慢                                  │
  │  ──────────────────────────────────────────────  │
  │  生成时需要同时计算 Encoder 和 Decoder            │
  │  KV Cache 利用率不如 Decoder-only 高              │
  │                                                  │
  │  ③ 预训练目标不够通用                             │
  │  ──────────────────────────────────────────────  │
  │  BART/T5 需要设计噪声/掩码策略                   │
  │  GPT 的"预测下一个词"更简单直接                   │
  │                                                  │
  │  ④ 超大模型规模下 Scaling 效果                    │
  │  ──────────────────────────────────────────────  │
  │  实践表明: 超大模型(>100B)时                     │
  │  Decoder-only 的 Scaling 效果更好                 │
  │  (这也是 GPT-3 之后主流转向 Decoder-only)        │
  │                                                  │
  └──────────────────────────────────────────────────┘
```

## 六、HuggingFace 使用 T5

### 6.1 基础使用

```python
from transformers import T5Tokenizer, T5ForConditionalGeneration
import torch

# 加载 T5 模型
model_name = "t5-small"  # 可选: t5-small, t5-base, t5-large, t5-3b, t5-11b
tokenizer = T5Tokenizer.from_pretrained(model_name)
model = T5ForConditionalGeneration.from_pretrained(model_name)

print(f"模型参数量: {sum(p.numel() for p in model.parameters()):,}")
print(f"词表大小: {tokenizer.vocab_size}")

# T5 的多任务能力
def t5_predict(input_text, max_length=50):
    """通用 T5 预测"""
    input_ids = tokenizer.encode(input_text, return_tensors="pt")

    with torch.no_grad():
        outputs = model.generate(
            input_ids,
            max_length=max_length,
            num_beams=4,
            early_stopping=True
        )

    return tokenizer.decode(outputs[0], skip_special_tokens=True)


# 测试各种任务
print("\n=== 翻译任务 ===")
print(t5_predict("translate English to German: Hello, how are you?"))

print("\n=== 摘要任务 ===")
long_text = (
    "summarize: Artificial intelligence (AI) is intelligence demonstrated by machines, "
    "unlike the natural intelligence displayed by humans and animals. "
    "Leading AI textbooks define the field as the study of intelligent agents: "
    "any device that perceives its environment and takes actions that maximize "
    "its chance of successfully achieving its goals."
)
print(t5_predict(long_text))

print("\n=== 问答任务 ===")
qa_input = "question: What is the capital of France? context: France is a country in Western Europe. Its capital is Paris."
print(t5_predict(qa_input))

print("\n=== 情感分类 ===")
print(t5_predict("sentiment: This movie is absolutely wonderful!"))

print("\n=== 句子相似度 ===")
stsb_input = "stsb sentence1: The cat is on the mat. sentence2: A cat is sitting on a rug."
print(t5_predict(stsb_input))
```

### 6.2 翻译任务微调示例

```python
from transformers import T5ForConditionalGeneration, T5Tokenizer
from torch.utils.data import Dataset, DataLoader
import torch
import torch.optim as optim


class TranslationDataset(Dataset):
    """翻译数据集"""
    def __init__(self, src_texts, tgt_texts, tokenizer, max_len=128):
        self.src_texts = src_texts
        self.tgt_texts = tgt_texts
        self.tokenizer = tokenizer
        self.max_len = max_len

    def __len__(self):
        return len(self.src_texts)

    def __getitem__(self, idx):
        src = f"translate English to Chinese: {self.src_texts[idx]}"
        tgt = self.tgt_texts[idx]

        src_encoding = self.tokenizer(
            src, truncation=True, padding='max_length',
            max_length=self.max_len, return_tensors="pt"
        )
        tgt_encoding = self.tokenizer(
            tgt, truncation=True, padding='max_length',
            max_length=self.max_len, return_tensors="pt"
        )

        return {
            'input_ids': src_encoding['input_ids'].flatten(),
            'attention_mask': src_encoding['attention_mask'].flatten(),
            'labels': tgt_encoding['input_ids'].flatten()
        }


# 模拟数据
en_texts = ["Hello", "Thank you", "Good morning", "I love you"]
zh_texts = ["你好", "谢谢", "早上好", "我爱你"]

# 准备训练
tokenizer = T5Tokenizer.from_pretrained("t5-small")
model = T5ForConditionalGeneration.from_pretrained("t5-small")
dataset = TranslationDataset(en_texts, zh_texts, tokenizer)
dataloader = DataLoader(dataset, batch_size=2, shuffle=True)

optimizer = optim.AdamW(model.parameters(), lr=3e-5)

# 训练循环 (简化版)
print("开始训练 T5 翻译模型...")
for epoch in range(3):
    model.train()
    total_loss = 0
    for batch in dataloader:
        optimizer.zero_grad()
        outputs = model(
            input_ids=batch['input_ids'],
            attention_mask=batch['attention_mask'],
            labels=batch['labels']
        )
        loss = outputs.loss
        loss.backward()
        optimizer.step()
        total_loss += loss.item()
    print(f"Epoch {epoch+1}: Loss = {total_loss/len(dataloader):.4f}")

# 测试翻译
model.eval()
test_input = tokenizer("translate English to Chinese: Hello", return_tensors="pt")
with torch.no_grad():
    output = model.generate(**test_input, max_length=20)
print(f"翻译结果: {tokenizer.decode(output[0], skip_special_tokens=True)}")
```

### 6.3 使用 BART

```python
from transformers import BartTokenizer, BartForConditionalGeneration

model_name = "facebook/bart-base"

inputs = tokenizer(text, return_tensors="pt", max_length=512, truncation=True)

        early_stopping=True
    )

print(f"BART 参数量: {sum(p.numel() for p in model.parameters()):,}")

print(f"摘要: {summary}")

### 9.1 三大架构终极对比

```
┌──────────────────┬───────────────┬───────────────┬───────────────┐
│                  │ Encoder-Only  │ Decoder-Only  │ Enc-Dec       │
│                  │ (BERT 系)     │ (GPT 系)      │ (T5/BART 系)  │
├──────────────────┼───────────────┼───────────────┼───────────────┤

├──────────────────┼───────────────┼───────────────┼───────────────┤
│ 架构组成         │ Encoder × N   │ Decoder × N   │ Encoder + Dec │
├──────────────────┼───────────────┼───────────────┼───────────────┤
│ 注意力           │ 双向          │ 单向(因果)    │ Enc双向+Dec单向│
├──────────────────┼───────────────┼───────────────┼───────────────┤
│ 预训练目标       │ 掩码还原      │ 下一词预测    │ 去噪重建      │
├──────────────────┼───────────────┼───────────────┼───────────────┤

│ 推理速度         │ 快(一次性)    │ 中(逐token)   │ 中(逐token)   │
├──────────────────┼───────────────┼───────────────┼───────────────┤
│ 训练复杂度       │ 中            │ 低            │ 高            │
├──────────────────┼───────────────┼───────────────┼───────────────┤
│ 少样本能力       │ 不适用        │ 强            │ 中等          │
├──────────────────┼───────────────┼───────────────┼───────────────┤
│ 超大规模趋势     │ 减少          │ 主流          │ 较少          │
├──────────────────┼───────────────┼───────────────┼───────────────┤
│ 部署难度         │ 低            │ 中            │ 中            │
└──────────────────┴───────────────┴───────────────┴───────────────┘

### 9.2 历史演进

```
Transformer 架构的发展历程:

  2017  Transformer (原始 Encoder-Decoder)
   │
  2018  BERT (Encoder-Only) ──→ 理解任务 SOTA
   │    GPT-1 (Decoder-Only) ──→ 效果一般
   │
  2019  GPT-2 (Decoder-Only) ──→ Zero-shot 初现
   │    T5 (Encoder-Decoder) ──→ 统一 text-to-text
   │    BART (Encoder-Decoder) ──→ 去噪自编码
   │    RoBERTa, ALBERT (Encoder-Only 改进)
   │
  2020  GPT-3 (Decoder-Only) ──→ 涌现能力, Few-shot
   │    → Decoder-Only 开始成为主流
   │
  2022+ Llama, Mistral, Qwen (全部 Decoder-Only)
   │    → 超大模型时代几乎都用 Decoder-Only
   │
  现在  Encoder-Decoder 在中小规模/特定任务仍有价值
        Encoder-Only 在理解任务依然高效
        Decoder-Only 是通用大模型的主流选择
```

## 小结

| 要点 | 内容 |
|------|------|
| T5 全称 | Text-to-Text Transfer Transformer |
| T5 核心创新 | 统一所有 NLP 任务为"文本→文本"范式 |
| T5 架构 | 完整的 Encoder-Decoder |
| T5 位置编码 | Bucketed 相对位置编码 |
| T5 任务前缀 | 用文本前缀区分任务（如 "translate:"） |
| BART | 去噪自编码器，多种噪声策略 |
| BART 噪声 | Token Masking/Deletion/Infilling + 句子乱序 |
| Enc-Dec 优势 | 双向理解 + 自回归生成，适合 Seq2Seq |
| Enc-Dec 劣势 | 参数效率低，超大模型 Scaling 不如 Decoder-Only |
| 三大架构 | Encoder-Only（BERT）、Decoder-Only（GPT）、Enc-Dec（T5） |
| BERT 最佳 | 分类、NER、抽取式问答 |
| GPT 最佳 | 生成、对话、创意写作、通用大模型 |
| T5/BART 最佳 | 翻译、摘要等 Seq2Seq 任务 |
| 当前趋势 | 超大模型以 Decoder-Only 为主流（Llama/Mistral/Qwen） |
| 选择原则 | 根据任务类型、规模、效率需求选择架构 |
