# 注意力机制

前面介绍的 RNN 和 CNN 各有所长，但也都有限制：RNN 难以并行且远距离信息会衰减，CNN 只关注局部窗口。**注意力机制（Attention）** 提供了一种全新思路——让模型在处理每个位置时，**动态地"关注"序列中最相关的其他位置**，不管它们距离多远。这一机制是 Transformer 和现代大模型的基石，可以说是现代深度学习 NLP 最重要的创新。

---

## 一、Attention 的动机

### 1.1 传统序列模型的问题

```
RNN 的瓶颈：逐步传递，远距离信息衰减

  句子: "我 [10个词] 昨天 去 [10个词] 北京"

  RNN 处理"北京"时，需要经过 20+ 个时间步才能回忆起"我"
  ┌───┐   ┌───┐       ┌───┐   ┌───┐       ┌───┐
  │我 │ → │.. │ → ... → │昨天│ → │.. │ → ... → │北京│
  └───┘   └───┘       └───┘   └───┘       └───┘
    └────────── 梯度/信息逐步衰减 ──────────┘

CNN 的瓶颈：只看局部窗口

  句子: "我 [10个词] 昨天 去 [10个词] 北京"

  CNN 处理"北京"时，受限于 kernel_size，看不到远处的"我"
  ┌─────────┐
  │ kernel=3│ 只能看到相邻3个词
  └─────────┘

注意力的突破：任意两词直接交互，无视距���

  "我" ←──attention──→ "北京"   距离再远也能直接关注
  ┌─────────────────────────────────┐
  │  我    ...    昨天   ...   北京  │
  │   ●───────────────────────●    │  ← 直接连边
  │           互相看到              │
  └─────────────────────────────────┘
```

### 1.2 直觉：人类阅读的注意力

```
人类读句子时的注意力分配（示例）:

  句子: "苹果 公司 昨天 发布 了 新款 iPhone"

  读到"iPhone"时，人脑会回溯关注:
    苹果(主语) ●━━━━━━━━━━━━━━━━┓  ← 关注度高（语义相关）
    公司      ●━━━━━━━━━━━━━━━━┫
    昨天      ●                    ← 中等关注（时间）
    发布      ●━━━━━━━━━━━━━━━━┫  ← 关注度高（动作）
    了         ●                    ← 低（功能词）
    新款      ●━━━━━━━━━━━━━━━━┛  ← 关注度高（属性）
    iPhone    ← 当前词

  注意力 = 对每个词分配一个权重（0~1），权重之和为1
  模型自动学习"应该关注哪些词"
```

## 二、注意力公式

### 2.1 核心公式

```
缩放点积注意力（Scaled Dot-Product Attention）:

  ┌────────────────────────────────────────────────────┐
  │  Attention(Q, K, V) = softmax(Q · K^T / √d_k) · V │
  └────────────────────────────────────────────────────┘

  其中:
  - Q (Query):  查询向量，"我想找什么"   形状 (seq_len_q, d_k)
  - K (Key):    键向量，"我有什么特征"   形状 (seq_len_k, d_k)
  - V (Value):  值向量，"我提供什么内容" 形状 (seq_len_k, d_v)

直觉类比（图书馆检索）:
  - Query: 你的搜索词 "深度学习入门"
  - Key:   每本书的标题/标签
  - Value: 每本书的内容

  1. 用 Query 和每本书的 Key 比较 → 得到相关度分数
  2. softmax 把分数归一化为权重
  3. 用权重对每本书的 Value 加权求和 → 综合答案
```

### 2.2 分步计算图解

```
步骤1: Q 和 K 计算相似度（点积）
  Q · K^T  →  得到打分矩阵

        K₁   K₂   K₃
  Q₁  [ s₁₁  s₁₂  s₁₃ ]     s_ij = Q_i · K_j
  Q₂  [ s₂₁  s₂₂  s₂₃ ]     （query i 对 key j 的关注分数）

步骤2: 缩放（除以 √d_k）
  s_ij / √d_k
  作用: 防止点积过大导致 softmax 梯度消失
  原理: d_k 越大，点积值方差越大；除以 √d_k 把方差拉回 1

步骤3: softmax 归一化（按行）
        K₁    K₂    K₃
  Q₁  [0.7   0.2   0.1 ]    每行和为1，表示权重分配
  Q₂  [0.1   0.8   0.1 ]

步骤4: 权重矩阵乘以 V，加权求和
  输出_i = Σ_j (weight_ij × V_j)

       ┌                     ┐   ┌    ┐
       │ 0.7  0.2  0.1       │   │ V₁ │
       │ 0.1  0.8  0.1       │ × │ V₂ │   =   输出 (seq_q, d_v)
       └                     ┘   │ V₃ │
                                   └    ┘

最终输出形状: (seq_len_q, d_v)
每个输出位置都是对所有 V 的加权组合
```

### 2.3 为什么除以 √d_k

```
假设 Q 和 K 的元素是均值0、方差1的独立随机变量:

  点积 q · k = Σ_{i=1}^{d_k} q_i × k_i

  均值 = 0
  方差 = d_k   （d_k 个方差为1的项求和）

  → d_k 越大，点积值的波动越大

  当 d_k = 64 时，点积标准差 = √64 = 8
  softmax(8) 和 softmax(-8) 之间的差异会被放大到接近 one-hot
  → 梯度几乎为 0，训练停滞

  除以 √d_k 后，方差回到 1，softmax 输出更平滑，梯度健康

  ┌──────────────────────────────────┐
  │  √d_k 的本质: 把点积的方差归一化  │
  └──────────────────────────────────┘
```

## 三、Self-Attention（自注意力）

### 3.1 什么是 Self-Attention

```
Self-Attention: Q、K、V 都来自同一个输入序列

  输入序列: X = [x₁, x₂, x₃, x₄]  （例如4个词的嵌入）

  通过三个线性变换得到 Q、K、V:
    Q = X · W_Q    （每个词变成"查询"）
    K = X · W_K    （每个词变成"键"）
    V = X · W_V    （每个词变成"值"）

  然后套用 Attention(Q, K, V)

  效果: 每个词都去"查看"序列中所有词（包括自己），重新表示自己

示例: 处理"我 爱 自然 语言 处理"

  处理"语言"时:
    Q_语言 与所有词的 K 计算分数:
      K_我:    0.05  (低相关)
      K_爱:    0.05  (低相关)
      K_自然:  0.30  (高相关，"自然语言"是固定搭配)
      K_语言:  0.40  (自身)
      K_处理:  0.20  (中相关，"语言处理"相关)

    V 的加权组合 → "语言" 的新表示融合了"自然""处理"的信息
```

### 3.2 Self-Attention 的完整计算

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class SelfAttention(nn.Module):
    def __init__(self, embed_dim):
        super().__init__()
        self.embed_dim = embed_dim
        # 三个线性层生成 Q、K、V
        self.W_q = nn.Linear(embed_dim, embed_dim)
        self.W_k = nn.Linear(embed_dim, embed_dim)
        self.W_v = nn.Linear(embed_dim, embed_dim)

    def forward(self, x):
        # x: (batch, seq_len, embed_dim)
        Q = self.W_q(x)  # (batch, seq_len, embed_dim)
        K = self.W_k(x)
        V = self.W_v(x)

        # 计算注意力分数: Q · K^T / √d_k
        d_k = self.embed_dim
        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(d_k)
        # scores: (batch, seq_len, seq_len)

        # softmax 归一化（最后一个维度）
        attn_weights = F.softmax(scores, dim=-1)

        # 加权求和
        output = torch.matmul(attn_weights, V)
        # output: (batch, seq_len, embed_dim)
        return output, attn_weights

# 测试
attn = SelfAttention(embed_dim=8)
x = torch.randn(2, 5, 8)  # batch=2, seq_len=5, embed_dim=8
output, weights = attn(x)
print(f"输入: {x.shape}")
print(f"输出: {output.shape}")        # (2, 5, 8) ← 同形状
print(f"注意力权重: {weights.shape}")  # (2, 5, 5) ← 每个词对其他词的关注度
print(f"\n第1个样本的注意力矩阵:")
print(weights[0])
print(f"\n每行和（应=1）: {weights[0].sum(dim=-1)}")
```

## 四、Bahdanau Attention vs Luong Attention

### 4.1 两种经典注意力

```
Attention 最早用于机器翻译（编码器-解码器架构）

架构回顾:
  Encoder: x₁, x₂, ..., x_n → 隐状态 h₁, h₂, ..., h_n
  Decoder: 逐步生成 y₁, y₂, ..., y_m
  问题: 传统方法只用 Encoder 最后一个隐状态作为"上下文"
        → 早期信息易丢失

Attention 的解决方案:
  每生成一个 y_t，都动态地从所有 Encoder 隐状态中"挑选"相关信息

┌─────────────────────────────────────────────────────────────┐
│  Bahdanau Attention (加性, 2014)                            │
│  ────────────────────────────────                           │
│  score(y_t, h_i) = v^T · tanh(W_1·h_i + W_2·s_t)          │
│  - 用一个小神经网络（MLP）计算相关性                        │
│  - Query = Decoder 当前隐状态 s_t                           │
│  - Key,Value = Encoder 隐状态 h_i                           │
│  - 可学习参数更多，表达力强                                  │
├─────────────────────────────────────────────────────────────┤
│  Luong Attention (乘性, 2015)                               │
│  ────────────────────────────────                           │
│  score(y_t, h_i) = s_t^T · W · h_i   (general)             │
│                 或 s_t^T · h_i        (dot)                 │
│  - 直接用点积或线性变换计算相关性                            │
│  - 计算更简单、更快                                          │
│  - Transformer 的 scaled dot-product 即源自此              │
└─────────────────────────────────────────────────────────────┘

  s_t: Decoder 隐状态
  h_i: Encoder 隐状态
  W, W_1, W_2, v: 可学习参数
```

### 4.2 对比表

```
┌──────────────┬──────────────────────┬──────────────────────┐
│    维度      │   Bahdanau (加性)    │    Luong (乘性)      │
├──────────────┼──────────────────────┼──────────────────────┤
│ 提出时间     │ 2014                 │ 2015                 │
│ 打分函数     │ MLP（tanh+线性）     │ 点积或双线性          │
│ 计算复杂度   │ 较高                 │ 较低                 │
│ 参数量       │ 多                   │ 少                   │
│ 适用场景     │ Query和Key维度不同时 │ Query和Key维度相同时 │
│ 对齐位置     │ 在上一时刻计算对齐    │ 在当前时刻计算对齐    │
│ Transformer  │ ×                    │ ✓（演化）            │
└──────────────┴──────────────────────┴──────────────────────┘
```

### 4.3 PyTorch 实现 Bahdanau Attention

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class BahdanauAttention(nn.Module):
    """加性注意力（用于 Encoder-Decoder 架构）"""
    def __init__(self, hidden_dim):
        super().__init__()
        self.W1 = nn.Linear(hidden_dim, hidden_dim, bias=False)  # for encoder
        self.W2 = nn.Linear(hidden_dim, hidden_dim, bias=False)  # for decoder
        self.v = nn.Linear(hidden_dim, 1, bias=False)

    def forward(self, query, keys, values, mask=None):
        # query: (batch, hidden_dim) ← Decoder 当前隐状态
        # keys: (batch, seq_len, hidden_dim) ← Encoder 全部隐状态
        # values: 同 keys（这里 key=value）
        # mask: (batch, seq_len) 可选，1=有效，0=padding

        # 扩展 query 以便广播: (batch, 1, hidden) → (batch, seq_len, hidden)
        query_expanded = self.W2(query).unsqueeze(1)         # (B, 1, H)
        keys_projected = self.W1(keys)                       # (B, L, H)
        # 打分
        scores = self.v(torch.tanh(query_expanded + keys_projected)).squeeze(-1)
        # scores: (batch, seq_len)

        # 应用 mask（把 padding 位置的分数设为 -inf）
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))

        # softmax 归一化
        attn_weights = F.softmax(scores, dim=-1)              # (batch, seq_len)

        # 加权求和
        context = torch.bmm(attn_weights.unsqueeze(1), values).squeeze(1)
        # context: (batch, hidden_dim)
        return context, attn_weights

# 测试
attn = BahdanauAttention(hidden_dim=16)
query = torch.randn(2, 16)           # batch=2
keys = torch.randn(2, 5, 16)         # seq_len=5
context, weights = attn(query, keys, keys)
print(f"Bahdanau Attention:")
print(f"  context: {context.shape}")    # (2, 16)
print(f"  weights: {weights.shape}")    # (2, 5)
print(f"  权重和: {weights.sum(dim=-1)}")  # 应为1
```

## 五、PyTorch 实现完整 Attention

### 5.1 带注意力的文本分类模型

将 Self-Attention 用于文本分类：让模型动态关注句子中最重要的词。

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class AttentionClassifier(nn.Module):
    """带 Self-Attention 的文本分类模型"""
    def __init__(self, vocab_size, embed_dim, hidden_dim, num_classes, dropout=0.3):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        # Q、K、V 投影
        self.W_q = nn.Linear(embed_dim, hidden_dim)
        self.W_k = nn.Linear(embed_dim, hidden_dim)
        self.W_v = nn.Linear(embed_dim, hidden_dim)
        self.scale = math.sqrt(hidden_dim)
        # 分类层: 把 attention 输出映射到类别
        self.fc = nn.Linear(hidden_dim, num_classes)
        self.dropout = nn.Dropout(dropout)

    def forward(self, x):
        # x: (batch, seq_len)
        mask = (x != 0).unsqueeze(1)  # (batch, 1, seq_len) 用于屏蔽 padding
        emb = self.embedding(x)       # (batch, seq_len, embed_dim)

        Q = self.W_q(emb)             # (batch, seq_len, hidden)
        K = self.W_k(emb)
        V = self.W_v(emb)

        # 注意力分数
        scores = torch.matmul(Q, K.transpose(-2, -1)) / self.scale
        # scores: (batch, seq_len, seq_len)

        # mask: 把 padding 位置的分数设为 -inf
        scores = scores.masked_fill(mask == 0, float('-inf'))

        attn_weights = F.softmax(scores, dim=-1)
        # 加权求和
        attended = torch.matmul(attn_weights, V)  # (batch, seq_len, hidden)

        # 池化: 对所有位置取平均（带 mask）
        valid_mask = (x != 0).float().unsqueeze(-1)  # (batch, seq_len, 1)
        pooled = (attended * valid_mask).sum(dim=1) / valid_mask.sum(dim=1).clamp(min=1)

        pooled = self.dropout(pooled)
        return self.fc(pooled), attn_weights

# 测试
model = AttentionClassifier(vocab_size=1000, embed_dim=32, hidden_dim=32, num_classes=2)
x = torch.randint(1, 1000, (4, 10))
logits, weights = model(x)
print(f"输入: {x.shape}")
print(f"logits: {logits.shape}")    # (4, 2)
print(f"注意力权重: {weights.shape}")  # (4, 10, 10)
```

### 5.2 完整训练流程

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math
from torch.utils.data import Dataset, DataLoader
from collections import Counter
import numpy as np

torch.manual_seed(42)
np.random.seed(42)

# 数据
texts = [
    "这部 电影 真的 太 好看 了", "剧情 紧凑 引人入胜 推荐",
    "演员 演技 精彩 出色", "画面 精美 值得 一看",
    "这部 电影 真的 太 烂 了", "剧情 烂 演技 差",
    "浪费 时间 无聊 透顶", "画面 粗糙 不 推荐",
] * 15
labels = [1, 1, 1, 1, 0, 0, 0, 0] * 15

counter = Counter()
for t in texts: counter.update(t.split())
vocab = {'<pad>': 0, '<unk>': 1}
for w, _ in counter.most_common(): vocab[w] = len(vocab)

max_len = 10
def text_to_seq(text):
    idx = [vocab.get(w, 1) for w in text.split()][:max_len]
    return idx + [0] * (max_len - len(idx))

class DS(Dataset):
    def __init__(self, texts, labels):
        self.x = torch.tensor([text_to_seq(t) for t in texts])
        self.y = torch.tensor(labels)
    def __len__(self): return len(self.y)
    def __getitem__(self, i): return self.x[i], self.y[i]

loader = DataLoader(DS(texts, labels), batch_size=16, shuffle=True)

# 训练
model = AttentionClassifier(len(vocab), embed_dim=32, hidden_dim=32, num_classes=2)
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=0.005)

print("=== Attention 文本分类训练 ===")
for epoch in range(25):
    total_loss, correct, total = 0, 0, 0
    for bx, by in loader:
        optimizer.zero_grad()
        logits, _ = model(bx)
        loss = criterion(logits, by)
        loss.backward()
        optimizer.step()
        total_loss += loss.item() * len(by)
        correct += (logits.argmax(1) == by).sum().item()
        total += len(by)
    if epoch % 5 == 0:
        print(f"  Epoch {epoch:3d} | Loss: {total_loss/total:.4f} | Acc: {correct/total:.4f}")

# 推理 + 查看注意力
model.eval()
test = "这部 电影 好看 推荐"
x = torch.tensor([text_to_seq(test)])
with torch.no_grad():
    logits, weights = model(x)
    pred = logits.argmax(-1).item()
    print(f"\n'{test}' → {'正面' if pred == 1 else '负面'}")
    print("\n注意力权重矩阵（每个词对其他词的关注）:")
    tokens = test.split()
    w = weights[0][:len(tokens), :len(tokens)]
    for i, ti in enumerate(tokens):
        vals = ' '.join(f'{w[i][j]:.2f}' for j in range(len(tokens)))
        print(f"  {ti}: {vals}")
```

## 六、注意力的优势

### 6.1 长距离依赖

```
对比 RNN 和 Attention 处理远距离依赖:

  句子: "The animal didn't cross the street because it was too tired"

  关注 "it" 指代什么？

  RNN: 从"animal"到"it"相隔多个词，信息逐步衰减
       ┌───┐  ┌───┐  ┌───┐  ┌───┐  ┌───┐  ┌───┐  ┌───┐
       │ani│→ │...│→ │...│→ │...│→ │...│→ │...│→ │it │
       └───┘  └───┘  └───┘  └───┘  └───┘  └───┘  └───┘
         └──── 路径长度 O(n) ────────┘

  Attention: "it" 可以直接关注到 "animal"，路径长度 O(1)
       ┌─────┐  ┌───┐  ┌───┐  ┌───┐  ┌───┐  ┌───┐  ┌───┐
       │ ani │  │...│  │...│  │...│  │...│  │...│  │ it │
       └──┬──┘  └───┘  └───┘  └───┘  └───┘  └───┘  └─┬─┘
          └────────────── 直接连接 ─────────────────┘
                          路径长度 O(1)
```

### 6.2 并行化

```
RNN: 必须按顺序计算（t 依赖 t-1）
  t=1 → t=2 → t=3 → t=4 → ...   串行
  
  时间复杂度: O(n) 无法并行
  GPU 利用率低

Attention: 所有位置同时计算
  t=1 ┐
  t=2 ├→ 矩阵乘法，一次完成
  t=3 ├
  t=4 ┘
  
  时间复杂度: O(1)（一次矩阵运算）
  GPU 利用率高

  → 这是 Transformer 能高效训练超大模型的关键
```

### 6.3 可解释性

```
Attention 权重提供了一种"可解释性":

  输入: "这部电影真的太烂了"
  
  分类: 负面 😞

  模型关注的词（注意力权重）:
    这  ▓░░░░░░  0.05
    部  ▓░░░░░░  0.05
    电影 ▓▓░░░░░  0.10
    真的 ▓░░░░░░  0.05
    太  ▓▓░░░░░  0.10
    烂  ▓▓▓▓▓▓▓  0.60  ← 模型最关注这个字
    了  ▓░░░░░░  0.05

  → 通过权重可以知道模型"依据什么做决策"
```

## 七、从 Attention 到 Transformer 的过渡

### 7.1 Self-Attention 的局限

```
Self-Attention 虽好，但单独使用有几个问题:

  问题1: 缺少位置信息
  ────────────────────
  Self-Attention 对输入顺序不敏感
  "我 爱 你" 和 "你 爱 我" 在纯 Self-Attention 下计算结果相同
  → 需要加入位置编码（Positional Encoding）

  问题2: 单一注意力头可能不够
  ────────────────────────────
  一个词和其他词的关系是多方面的:
    "苹果" 既是"公司名"，又是"水果"
  → 需要多头注意力（Multi-Head Attention），从不同角度关注

  问题3: 只有注意力，缺少非线性变换
  ─────────────────────────────
  需要前馈网络（Feed-Forward Network）做进一步特征变换

  问题4: 深层网络训练不稳定
  ─────────────────────────
  需要残差连接（Residual Connection）和层归一化（Layer Normalization）
```

### 7.2 Transformer 的组成

```
Transformer = Multi-Head Attention + Positional Encoding
              + Feed-Forward Network + Residual + LayerNorm

  ┌─────────────────────────────────────┐
  │        Transformer Block            │
  │                                     │
  │   输入 + 位置编码                    │
  │      ↓                              │
  │   ┌──────────────┐                  │
  │   │ Multi-Head   │ ← 多头注意力     │
  │   │ Attention    │                  │
  │   └──────┬───────┘                  │
  │          + ← 残差连接                │
  │        LayerNorm                    │
  │          ↓                          │
  │   ┌──────────────┐                  │
  │   │ Feed-Forward │ ← 两层全连接     │
  │   │ Network      │                  │
  │   └──────┬───────┘                  │
  │          + ← 残差连接                │
  │        LayerNorm                    │
  │          ↓                          │
  │        输出                          │
  └─────────────────────────────────────┘

  这些组件的组合就是 Transformer，是 BERT、GPT 等大模型的基础。
  下一篇我们将详细拆解 Transformer 的每个组件。
```

### 7.3 Attention 家族演进

```
Attention 发展时间线:

  2014  Bahdanau Attention  用于 RNN 编解码器，解决长序列翻译
   │
  2015  Luong Attention     简化为乘性，计算更快
   │
  2017  Scaled Dot-Product  Transformer 核心部件
   │    Attention
   │
  2017  Multi-Head          多角度并行关注
   │    Attention
   │
  2018+ BERT / GPT 系列     基于（多头）注意力构建大模型
   │
  现在 Flash Attention      IO 感知优化，训练更快更省显存
```

## 小结

| 要点 | 内容 |
|------|------|
| Attention 动机 | 让模型动态关注序列中不同位置，无视距离 |
| 核心公式 | `Attention(Q,K,V) = softmax(QK^T/√d_k)V` |
| Q/K/V 含义 | Query 查询、Key 键、Value 值，类比图书馆检索 |
| 缩放因子 √d_k | 防止点积过大导致 softmax 梯度消失 |
| Self-Attention | Q、K、V 来自同一序列，词与词互相关注 |
| Bahdanau | 加性注意力，MLP 打分，参数多 |
| Luong | 乘性注意力，点积打分，参数少 |
| 优势1 | 长距离依赖：任意两词直接交互，路径 O(1) |
| 优势2 | 并行化：矩阵运算，GPU 友好 |
| 优势3 | 可解释性：权重可视化模型关注点 |
| 局限 | 无位置感知、单头视角有限 → 需要多头+位置编码 |
| 向 Transformer | Self-Attention + 多头 + 位置编码 + FFN + 残差 + LayerNorm |
