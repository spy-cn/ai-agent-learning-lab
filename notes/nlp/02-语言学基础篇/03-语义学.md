# 语义学

语义学研究语言的意义——从词义到句义再到篇章意义。

## 语义学层次

```
语义研究的层次:

┌─────────────────────────────────────────────┐
│  词义 (Lexical Semantics)                   │
│  → 每个词是什么意思                         │
│  → 同义、反义、多义                         │
│  → WordNet, 词嵌入                          │
├─────────────────────────────────────────────┤
│  组合语义 (Compositional Semantics)         │
│  → 词如何组合成句子的意义                   │
│  → "the cat" + "chased" + "the mouse"      │
│  → 语义角色标注                             │
├─────────────────────────────────────────────┤
│  篇章语义 (Discourse Semantics)             │
│  → 句子如何构成篇章的意义                   │
│  → 指代消解、蕴含关系                       │
└─────────────────────────────────────────────┘
```

## 1. 词义 (Lexical Semantics)

### 词汇关系

```
词与词之间的语义关系:

同义 (Synonymy):     意义相同或相似
  happy ≈ glad ≈ joyful
  快乐 ≈ 高兴 ≈ 愉快

反义 (Antonymy):     意义相反
  hot ↔ cold
  大 ↔ 小

上下位 (Hypernymy):  包含关系 (is-a)
  animal ⊃ dog ⊃ poodle
  动物 ⊃ 狗 ⊃ 贵宾犬
  
  dog 是 animal 的下位词 (hyponym)
  animal 是 dog 的上位词 (hypernym)

整体-部分 (Meronymy): 部分关系 (has-a)
  car HAS-A wheel
  手 HAS-A 手指

多义 (Polysemy):     一个词多种相关意义
  bank = 银行? 河岸?
  head = 头? 领导? 

同音异义 (Homonymy): 同音不同义
  bat = 蝙蝠? 球棒?
  bank = 银行? 河岸?
```

### WordNet

```python
import nltk
from nltk.corpus import wordnet as wn

nltk.download('wordnet', quiet=True)
nltk.download('omw-1.4', quiet=True)

# === 同义词集 (Synsets) ===
dog_synsets = wn.synsets('dog')
print(f"'dog' 的同义词集:")
for synset in dog_synsets:
    print(f"  {synset.name()}: {synset.definition()}")

# 输出:
# dog.n.01: a member of the genus Canis...
# frump.n.01:...
# ...

# === 同义词 ===
happy = wn.synset('happy.a.01')
lemmas = [l.name() for l in happy.lemmas()]
print(f"\n'happy' 的同义词: {lemmas}")

# === 反义词 ===
happy_lemma = wn.lemma('happy.a.01.happy')
antonym = happy_lemma.antonyms()[0]
print(f"'happy' 的反义词: {antonym.name()}")

# === 上下位关系 ===
dog = wn.synset('dog.n.01')
# 上位词
hypernyms = dog.hypernyms()
print(f"\n'dog' 的上位词: {[h.name() for h in hypernyms]}")
# [Synset('canine.n.02')]

# 上溯到根
path = dog.hypernym_path()
print(f"分类路径: {[s.name() for s in path]}")
# entity.n.01 → physical_entity.n.01 → ... → mammal.n.01 → 
# carnivore.n.01 → canine.n.02 → dog.n.01

# === 相似度计算 ===
dog = wn.synset('dog.n.01')
cat = wn.synset('cat.n.01')
car = wn.synset('car.n.01')

print(f"\n相似度:")
print(f"  dog-cat:  {dog.path_similarity(cat):.4f}")
print(f"  dog-car:  {dog.path_similarity(car):.4f}")
```

### 词义消歧 (WSD)

```python
# 同一个词在不同上下文有不同含义

def word_sense_disambiguation(word, context_sentence):
    """基于 Lesk 算法的词义消歧"""
    from nltk.wsd import lesk
    
    # 分词
    context = context_sentence.lower().split()
    
    # Lesk 算法: 选择与上下文词重叠最多的词义
    sense = lesk(context, word)
    
    if sense:
        return {
            'word': word,
            'sense': sense.name(),
            'definition': sense.definition(),
            'examples': sense.examples(),
        }
    return None

# bank 的两种含义
result1 = word_sense_disambiguation('bank', 
    "I deposited money in the bank")
print(f"金融语境: {result1['sense']} - {result1['definition']}")

result2 = word_sense_disambiguation('bank', 
    "The boat reached the river bank")
print(f"河流语境: {result2['sense']} - {result2['definition']}")
```

## 2. 组合语义

### 语义角色标注 (SRL)

```
语义角色标注: 谁对谁做了什么

句子: "John gave Mary a book at school"

语义角色:
  Predicate (谓语): gave
  Agent  (施事):    John    — 主动行为者
  Theme  (受事):    a book  — 动作对象
  Goal   (目标):    Mary    — 接收者
  Location (地点):  school  — 发生地点

         gave (谓语)
        / | | \
     John Mary book school
     施事 目标 受事 地点
```

```python
# 使用 spaCy + Allennlp 或 HuggingFace 做语义角色标注

# 简化版: 用 spaCy 做基础角色分析
import spacy

nlp = spacy.load('en_core_web_sm')

def analyze_semantic_roles(text):
    """简化的语义角色分析"""
    doc = nlp(text)
    
    roles = {}
    
    for token in doc:
        if token.dep_ == 'ROOT':
            roles['predicate'] = token.lemma_  # 谓语
            
            # 分析依存关系确定角色
            for child in token.children:
                if child.dep_ == 'nsubj':
                    roles['agent'] = child.text     # 施事
                elif child.dep_ in ('dobj', 'obj'):
                    roles['theme'] = child.text      # 受事
                elif child.dep_ == 'dative':
                    roles['recipient'] = child.text  # 接收者
                elif child.dep_ in ('prep', 'agent'):
                    roles['instrument'] = child.text # 工具
                elif child.dep_ == 'advmod':
                    roles['manner'] = child.text     # 方式
    
    return roles

text = "The chef cooked a delicious meal quickly"
roles = analyze_semantic_roles(text)
print(f"句子: {text}")
print(f"语义角色: {roles}")
# {'predicate': 'cook', 'agent': 'chef', 'theme': 'meal', 'manner': 'quickly'}
```

## 3. 语义关系

### 文本蕴含 (Textual Entailment)

```
文本蕴含: 判断前提(P)是否蕴含假设(H)

关系类型:
  ✓ 蕴含 (Entailment):    P为真→H必为真
  ✗ 矛盾 (Contradiction): P为真→H必为假  
  ○ 中立 (Neutral):       无法确定

示例:
  蕴含:
    P: "A man is playing guitar on stage"
    H: "Someone is performing music"
    → P 蕴含 H ✓

  矛盾:
    P: "A man is playing guitar on stage"  
    H: "The stage is empty"
    → P 矛盾 H ✗

  中立:
    P: "A man is playing guitar on stage"
    H: "The man is famous"
    → 无法判断 ○
```

```python
# 使用 HuggingFace 做文本蕴含
from transformers import pipeline

# 零样本分类可以用于蕴含判断
classifier = pipeline("zero-shot-classification", 
                      model="facebook/bart-large-mnli")

# 方法: 将假设作为候选标签
premise = "A person is training his horse for a competition."
hypothesis = "A person is at a horse show."

result = classifier(premise, [hypothesis], multi_label=True)
print(f"前提: {premise}")
print(f"假设: {hypothesis}")
print(f"蕴含概率: {result['scores'][0]:.4f}")
```

## 4. 语义表示

### 词嵌入: 分布式语义

```
分布式假设 (Distributional Hypothesis):
"You shall know a word by the company it keeps"
                                    — J.R. Firth

词的意义由其使用上下文决定:

统计共现:
         cat    dog    car    book
mouse    845    210     5     12
pet      520    680     2     10
drive     15     20    980      5
read       8     12      3    750

→ cat 和 dog 有相似的上下文分布
→ 它们的词向量空间距离相近

可视化 (2D投影):
    
    pet •  cat
         • dog
        
                   • car
                   
    • book
    read

→ 语义相近的词聚集在一起
```

### 命题逻辑表示

```python
# 将自然语言转换为逻辑表示

"""
自然语言 → 逻辑表达式

"所有猫都是动物"    → ∀x. Cat(x) → Animal(x)
"有些猫是黑色的"    → ∃x. Cat(x) ∧ Black(x)
"John喜欢猫"        → Likes(John, Cat)
"没有猫是狗"        → ¬∃x. Cat(x) ∧ Dog(x)

三段论推理:
  P1: 所有猫都是动物          ∀x. Cat(x) → Animal(x)
  P2: Tom是猫                 Cat(Tom)
  C:  Tom是动物               Animal(Tom)
"""

# 简化的逻辑推理示例
class LogicInference:
    """简单的逻辑推理引擎"""
    
    def __init__(self):
        self.facts = set()
        self.rules = []
    
    def add_fact(self, predicate, entity):
        """添加事实"""
        self.facts.add((predicate, entity))
    
    def add_rule(self, condition_pred, result_pred):
        """添加规则: 如果X是condition_pred, 则X是result_pred"""
        self.rules.append((condition_pred, result_pred))
    
    def infer(self):
        """前向推理"""
        new_facts = True
        while new_facts:
            new_facts = False
            for cond, result in self.rules:
                for pred, entity in list(self.facts):
                    if pred == cond and (result, entity) not in self.facts:
                        self.facts.add((result, entity))
                        new_facts = True
    
    def query(self, predicate, entity=None):
        """查询"""
        if entity:
            return (predicate, entity) in self.facts
        return [e for p, e in self.facts if p == predicate]

# 使用
engine = LogicInference()

# 事实
engine.add_fact('Cat', 'Tom')
engine.add_fact('Animal', 'Jerry')  # Jerry是动物

# 规则: 所有猫是动物, 所有动物会死
engine.add_rule('Cat', 'Animal')
engine.add_rule('Animal', 'Mortal')

# 推理
engine.infer()

# 查询
print(f"Tom是猫: {engine.query('Cat', 'Tom')}")
print(f"Tom是动物: {engine.query('Animal', 'Tom')}")  # 推理得出
print(f"Tom会死: {engine.query('Mortal', 'Tom')}")     # 推理得出
print(f"所有动物: {engine.query('Animal')}")
```

## 5. 中文语义特点

```
中文语义的独特之处:

1. 量词系统 (语义分类)
   一本 书    (装订物)
   一张 纸    (扁平物)
   一条 河    (细长物)
   一头 牛    (大动物)
   一只 鸟    (小动物)
   一棵 树    (植物)
   → 量词反映了物体的语义类别

2. 动补结构
   洗 干净    (wash + clean = wash clean)
   走 过去    (walk + over = walk over)
   打 破      (hit + broken = break)
   → 动作+结果的语义组合

3. 主题评论结构
   这本书，我觉得很难
   主题: 这本书
   评论: 我觉得很难
   → 主题-评论是中文重要语义结构

4. 成语的语义
   画蛇添足 (画蛇+添足→多此一举)
   守株待兔 (守株+待兔→不劳而获)
   → 成语意义不是字面意义
```

## 小结

| 概念 | 定义 | NLP应用 |
|------|------|---------|
| **词义** | 词的含义 | 词嵌入、WordNet |
| **同义词** | 意义相似 | 搜索扩展 |
| **多义性** | 一词多义 | 词义消歧(WSD) |
| **语义角色** | 谁对谁做什么 | 信息抽取 |
| **文本蕴含** | P→H的逻辑关系 | 推理问答 |
| **语义相似度** | 句义接近程度 | 语义搜索 |
| **分布式假设** | 上下文决定词义 | Word2Vec/BERT |

> 语义学是 NLP 的核心——从"看到文字"到"理解意义"是智能的本质。现代深度学习通过向量表示隐式地捕捉语义。
