# 问答系统：让机器理解并回答问题

问答系统（Question Answering, QA）是 NLP 中最具挑战性也最贴近"智能"定义的任务之一：给定一个问题，系统需要给出准确、简洁的答案。从搜索引擎返回相关网页，到智能助手直接回答事实类问题，再到企业知识库的精准问答，QA 技术正在重塑信息获取的方式。本章将系统介绍问答系统的分类、核心算法和 RAG 架构。

## 一、问答系统分类

```
┌──────────────────────────────────────────────────────────────┐

├──────────────────────────────────────────────────────────────┤
│                                                              │
│  按知识来源分类：                                              │
│                                                              │
│  ┌─────────────────┐  ┌─────────────────┐                    │
│  │  MRC 阅读理解    │  │  KBQA 知识库问答 │                    │
│  │  从文档抽取答案  │  │  查询结构化知识  │                    │
│  │  SQuAD, CoNLP   │  │  Wikidata, DBpedia│                   │
│  └─────────────────┘  └─────────────────┘                    │
│                                                              │
│  ┌─────────────────┐  ┌─────────────────┐                    │
```

| 类型 | 输入 | 知识源 | 输出 | 典型数据集 |
|------|------|-------|------|----------|
| MRC 阅读 | 问题 + 文档 | 文档 | 文档片段 | SQuAD、CoQA |

| 开放域 QA | 问题 | 全网/百科 | 答案文本 | Natural Questions、TrivialQA |

## 二、抽取式 QA

抽取式问答是 MRC 阅读理解的主流形式：给定一个问题和一段文档，模型预测答案在文档中的**起始位置**和**结束位置**。

```
抽取式 QA 工作流程

问题：百度的创始人是谁？
文档：李彦宏是百度的创始人，他于 2000 年创立了百度公司。

Step 1：BERT 编码 [CLS] 问题 [SEP] 文档 [SEP]
┌──────────────────────────────────────────────────────────┐
│ [CLS] 百度 的 创始人 是 谁 [SEP] 李彦宏 是 百度 的创始人 ...│
│  h0   h1 h2 h3  h4 h5 h6 h7  h8  h9 h10 h11 h12 h13 ... │
└──────────────────────────────────────────────────────────┘

Step 2：起始位置预测（每个 token 是答案起点的概率）
         ... 李彦宏  ... 
             ▲
         0.85 ← 最高概率

Step 3：结束位置预测
         ... 李彦宏  ... 
             ▲    ▲
            0.85 0.90 ← 最高概率

Step 4：抽取答案片段 = "李彦宏"
```

### BERT 抽取式 QA 完整代码

```python
"""
BERT 抽取式问答完整示例
模型：BertForQuestionAnswering
"""
import torch
from transformers import (
    AutoTokenizer,
    AutoModelForQuestionAnswering,
)

model_name = "uer/roberta-base-chinese-extractive-qa"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForQuestionAnswering.from_pretrained(model_name)

def answer_question(question, context):
    """从 context 中抽取问题的答案"""
    inputs = tokenizer(
        question,
        context,
        truncation="only_second",   # 只截断文档，不截断问题
        max_length=384,
        stride=128,                 # 滑动窗口处理长文档
        return_overflowing_tokens=True,
        return_offsets_mapping=True,
        padding="max_length",
    )

    # 移除无用的 batch 维度
    input_ids = inputs["input_ids"]
    attention_mask = inputs["attention_mask"]

    # 记录哪些 token 属于文档（用于还原字符位置）
    sequence_ids = inputs.sequence_ids(0)
    context_start = 0
    while sequence_ids[context_start] != 1:
        context_start += 1
    context_end = len(sequence_ids) - 1
    while sequence_ids[context_end] != 1:
        context_end -= 1

    model.eval()
    with torch.no_grad():
        outputs = model(
            input_ids=torch.tensor(input_ids),
            attention_mask=torch.tensor(attention_mask),
        )

    start_logits = outputs.start_logits[0]
    end_logits = outputs.end_logits[0]

    # 找到起始和结束位置（限制在文档范围内）
    start_idx = start_logits[context_start:context_end+1].argmax() + context_start
    end_idx = end_logits[start_idx:context_end+1].argmax() + start_idx

    # 还原到原文的字符位置
    offsets = inputs["offset_mapping"][0]
    start_char = offsets[start_idx][0]
    end_char = offsets[end_idx][1]

    return context[start_char:end_char]

# 测试
context = (
    "李彦宏是百度的创始人兼董事长，他于 1968 年出生于山西省阳泉市。"
    "2000 年 1 月，李彦宏在中关村创立了百度公司，最初专注于搜索引擎技术。"
    "经过二十多年的发展，百度已经成为中国最大的互联网公司之一。"
)
questions = [
    "百度的创始人是谁？",
    "李彦宏哪一年出生？",
    "百度公司什么时候成立的？",
]
for q in questions:
    ans = answer_question(q, context)
    print(f"Q: {q}\nA: {ans}\n")

# 输出示例：
# Q: 百度的创始人是谁？
# A: 李彦宏
# Q: 李彦宏哪一年出生？
# A: 1968 年
# Q: 百度公司什么时候成立的？
# A: 2000 年 1 月
```

## 三、生成式 QA

生成式问答不限于从文档复制，而是生成自然的语言答案。它适合"是否类""原因类"等需要推理或概括的问题。

```python
"""
生成式 QA：使用 seq2seq 模型生成答案
"""
from transformers import pipeline

qa_generator = pipeline(
    "text2text-generation",
    model="google/flan-t5-base",
)
question = "Why is the sky blue?"
context = (
    "The sky appears blue because air molecules scatter "
    "shorter wavelengths of light more than longer wavelengths. "
    "This phenomenon is called Rayleigh scattering."
)
prompt = f"question: {question} context: {context}"
result = qa_generator(prompt, max_length=50)
print(result[0]["generated_text"])
# 输出：Because air molecules scatter shorter wavelengths of light.
```

### 抽取式 vs 生成式 对比

| 维度 | 抽取式 QA | 生成式 QA |
|------|---------|---------|
| 答案形式 | 文档原文片段 | 自然语言生成 |
| 答案边界 | 明确起止位置 | 无明确边界 |
| 可控性 | 高（来自文档） | 较低（可能产生幻觉） |
| 训练目标 | 起止位置回归 | 序列生成 |
| 适用场景 | 事实类、可定位 | 推理类、概括类 |
| 典型模型 | BERT、RoBERTa | T5、BART、GPT |

## 四、评估指标

```python
"""
SQuAD 风格评估指标：Exact Match (EM) 和 F1
"""
import re
import string
from collections import Counter

def normalize_answer(s):
    """标准化答案：小写、去标点、去冠词、去多余空格"""
    s = s.lower()
    s = re.sub(r"\b(a|an|the)\b", " ", s)
    s = "".join(ch for ch in s if ch not in string.punctuation)
    s = " ".join(s.split())
    return s

def exact_match(pred, truth):
    return int(normalize_answer(pred) == normalize_answer(truth))

def f1_score(pred, truth):
    pred_tokens = normalize_answer(pred).split()
    truth_tokens = normalize_answer(truth).split()
    common = Counter(pred_tokens) & Counter(truth_tokens)
    num_same = sum(common.values())
    if num_same == 0:
        return 0
    precision = num_same / len(pred_tokens)
    recall = num_same / len(truth_tokens)
    return 2 * precision * recall / (precision + recall)

pred = "the Transformer architecture"
truth = "Transformer architecture"

print(f"EM: {exact_match(pred, truth)}")  # 0（因为大小写和冠词差异）
print(f"F1: {f1_score(pred, truth):.4f}")  # 经过标准化后约 0.80

# 实际项目中使用 datasets 库：
# from datasets import load_metric
# squad_metric = load_metric("squad")
```

```
EM vs F1 图示

真实答案：Barack Obama
预测答案：Barack Hussein Obama

EM = 0  （字符串不完全一致）
F1 = 2 * (2/3) * (2/2) / ((2/3) + (2/2)) = 0.80
       ↑           ↑
   精确率 2/3    召回率 2/2
```

| 指标 | 含义 | 范围 |
|------|------|------|
| EM | 答案完全一致的比例 | 0–100% |
| F1 | token 级别的 F1 分数 | 0–100% |
| 共识 | 多个答案的稳定性 | – |

## 五、RAG 检索增强生成

**Retrieval-Augmented Generation (RAG)** 把检索和生成结合起来：先从大规模知识库中检索相关文档，再让大模型基于检索到的文档生成答案。这是解决大模型"知识过时"和"幻觉"问题的关键技术。

```
RAG 架构：检索 + 生成

┌─────────────────────────────────────────────────────────────┐
│                         RAG 完整流程                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌──────────┐    ┌──────────┐    ┌──────────────────────┐   │
│  │ 用户问题 │───►│ Query    │───►│  向量数据库           │   │
│  │          │    │ Encoder  │    │  (Chroma, Faiss, ...) │   │
│  └──────────┘    └──────────┘    └──────────┬───────────┘   │
│                                              │ 检索 Top-K    │
│                                              ▼              │
│                                    ┌──────────────────┐    │
│                                    │ 相关文档片段       │    │
│                                    │ doc1, doc2, ...   │    │
│                                    └────────┬─────────┘    │
│                                             │              │
│                                             ▼              │
│                  ┌───────────────────────────────────┐     │
│                  │  Prompt 构建                        │     │
│                  │  context: doc1 + doc2 + ...        │     │
│                  │  question: 用户问题                  │     │
│                  └──────────────┬────────────────────┘     │
│                                 │                          │
│                                 ▼                          │
│                  ┌───────────────────────────────┐         │
│                  │   LLM 生成器 (GPT, Llama, ...) │         │
│                  └──────────────┬────────────────┘         │
│                                 │                          │
│                                 ▼                          │
│                         ┌───────────────┐                  │
│                         │   最终答案     │                  │
│                         └───────────────┘                  │
└─────────────────────────────────────────────────────────────┘
```

```python
"""
完整的 RAG 问答系统示例
依赖：pip install langchain langchain-community chromadb sentence-transformers
"""
from langchain_community.document_loaders import TextLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter
from langchain_community.embeddings import HuggingFaceEmbeddings
from langchain_community.vectorstores import Chroma
from langchain_community.llms import HuggingFacePipeline
from langchain.chains import RetrievalQA

# ─── 1. 加载并切分文档 ─────────────────────────────────────
loader = TextLoader("knowledge.txt")
docs = loader.load()
splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,
    chunk_overlap=50,
)
chunks = splitter.split_documents(docs)

# ─── 2. 构建向量数据库 ─────────────────────────────────────
embeddings = HuggingFaceEmbeddings(
    model_name="sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2"
)
vector_db = Chroma.from_documents(chunks, embeddings, persist_directory="./chroma_db")

# ─── 3. 构建检索器 ──────────────────��──────────────────
retriever = vector_db.as_retriever(
    search_type="similarity",
    search_kwargs={"k": 3},   # 检索 Top-3 相关文档
)

# ─── 4. 构建 RAG 链 ─────────────────────────────────────
llm = HuggingFacePipeline.from_model_id(
    model_id="Qwen/Qwen2.5-0.5B-Instruct",
    task="text2text-generation",
    pipeline_kwargs={"max_new_tokens": 200},
)

qa_chain = RetrievalQA.from_chain_type(
    llm=llm,
    chain_type="stuff",         # 把所有检索文档"塞"进一个 prompt
    retriever=retriever,
    return_source_documents=True,
)

# ─── 5. 提问 ─────────────────────────────────────
question = "公司年假政策是怎样的？"
result = qa_chain({"query": question})
print(f"问题：{question}")
print(f"答案：{result['result']}")
print("\n参考来源：")
for i, doc in enumerate(result["source_documents"]):
    print(f"  [{i+1}] {doc.page_content[:80]}...")
```

## 七、KBQA 知识库问答

KBQA 面向结构化知识图谱：把自然语言问题翻译成 SPARQL / Cypher 等查询语句，从知识库中取回答案。

```python
"""
KBQA 示例：基于模板匹配
"""
import re

knowledge = {
    ("百度", "创始人"): "李彦宏",
    ("百度", "成立时间"): "2000年",
    ("阿里巴巴", "创始人"): "马云",
    ("腾讯", "创始人"): "马化腾",
}

def kbqa(question):
    # 简化版：正则模板匹配
    patterns = [
        (r"(.+?)的创始人", "创始人"),
        (r"(.+?)什么时候成立", "成立时间"),
    ]
    for pattern, relation in patterns:
        match = re.match(pattern, question)
        if match:
            entity = match.group(1)
            return knowledge.get((entity, relation), "未知")
    return "抱歉，我无法回答这个问题。"

print(kbqa("百度的创始人是谁？"))        # 李彦宏
print(kbqa("阿里巴巴的创始人是谁？"))    # 马云
```

更先进的 KBQA 系统会使用语义解析（Text-to-SPARQL）或基于大模型的函数调用。

## 八、常见挑战

| 挑战 | 描述 | 应对策略 |
|------|------|---------|
| 答案不可定位 | 推理类问题无明确片段 | 使用生成式 QA |
| 文档过长 | 超出模型上下文窗口 | 滑动窗口 / 段落检索 |
| 多跳推理 | 答案需要跨多个文档推理 | Multi-hop QA（HotpotQA）|
| 幻觉 | 生成式 QA 编造答案 | RAG + 引用追溯 |
| 检索质量差 | 检索器召回不足 | 混合检索（向量 + 关键词）|
| 领域知识缺失 | 通用模型不懂业务 | 领域微调 + 领域知识库 |

## 小结

| 要点 | 内容 |
|------|------|
| 任务分类 | MRC、KBQA、FAQ、开放域 QA |
| 答案形式 | 抽取式（复制片段）、生成式（自然语言）|
| 抽取式 QA | BERT 预测起止位置（`BertForQuestionAnswering`）|
| 生成式 QA | seq2seq 模型生成自然答案 |
| RAG 架构 | 检索（向量数据库）+ LLM 生成 |
| 评估指标 | Exact Match、F1 |
| 典型数据集 | SQuAD、CoQA、Natural Questions、HotpotQA |
| 实践建议 | 检索质量决定 RAG 上限、引用追溯防幻觉 |
