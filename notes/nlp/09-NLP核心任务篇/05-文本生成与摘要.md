# 文本生成与摘要：让机器学会写作

文本生成（Text Generation）是 NLP 中最具创造力的任务：给定一段输入（可以是空、提示词、原文），模型自动生成流畅、连贯、符合要求的自然语言文本。从自动摘要、故事续写，到对话生成、代码生成，文本生成技术正以前所未有的方式重塑内容生产。本章将系统讲解生成策略、摘要方法、评估指标和工程实践。

## 二、生成策略

解码策略决定了模型在每个时间步如何从概率分布中选择下一个 token。这是文本生成的核心环节，直接决定生成质量。

### 1. Greedy Search

每一步选择概率最高的 token，简单但容易陷入局部最优。

```
Greedy Search 图解

词表概率（简化）：
            我     你    他    其他
Step 1:   0.60▶  0.25  0.10  0.05
            │
            ▼ 选中 我

            爱    是    想    其他
Step 2:   0.50▶  0.30  0.15  0.05
            │
            ▼ 选中 爱

最终输出：我爱 ...

问题：可能错过全局更优的"你 好"
```

### 2. Beam Search

保留 top-k 候选序列，最终选择累计概率最高的。

```
Beam Search 图解（beam_width=2）

Step 1 候选：{我(0.60), 你(0.25)}

Step 2 扩展：
   我→爱 (0.60×0.50=0.30) ★
   我→是 (0.60×0.30=0.18)
   你→好 (0.25×0.80=0.20) ★
   你→是 (0.25×0.10=0.03)
   保留 top-2: {我爱(0.30), 你好(0.20)}

Step 3 继续扩展 ... 最终选累计概率最高的

优点：全局更优
缺点：生成文本偏向常见表达，缺乏多样性
```

### 3. Top-k Sampling

从概率最高的 k 个 token 中按概率随机采样。

```
Top-k Sampling 图解（k=3）

原始分布：
   我(0.40) 你(0.30) 他(0.15) 她(0.10) 它(0.03) ...

Top-k=3 后保留：我(0.40) 你(0.30) 他(0.15)
重新归一化：    我(0.47) 你(0.35) 他(0.18)
按概率采样 ───► 可能选 你（更有创造力）

优点：增加多样性
缺点：k 固定，遇到尖锐分布仍可能出错
```

### 4. Top-p Sampling（Nucleus Sampling）

动态选择累计概率达到 p 的最小 token 集合。

```
Top-p Sampling 图解（p=0.7）

情况 A：分布尖锐
   我(0.60) 你(0.20) 他(0.10) ...
   累计：0.60 → 0.80 ✓ 已达 p=0.7
   采样集：{我, 你}

情况 B：分布平坦
   我(0.25) 你(0.20) 他(0.18) 她(0.15) 它(0.12) ...
   累计：0.25 → 0.45 → 0.63 → 0.78 ✓
   采样集：{我, 你, 他, 她}

优点：根据分布形状动态调整候选数
缺点：极端情况下候选过大
```

### 5. Temperature

温度参数控制概率分布的"陡峭程度"，调节生成的"保守度"。

```
Temperature 公式：p_i' = softmax(logit_i / T)

T → 0：分布变得尖锐，趋近 Greedy
T = 1：分布不变
T → ∞：分布变得平坦，趋近均匀随机

图解：
                T=0.5      T=1.0      T=2.0
               (保守)     (原分布)    (奔放)
               ▲          ▲           ▲
   我      ┌──┐ 0.8   ┌──┐ 0.5    ┌──┐ 0.3
   你      ┌──┐ 0.15  ┌──┐ 0.3    ┌──┐ 0.28
   他      ┌──┐ 0.04  ┌──┐ 0.15   ┌──┐ 0.22
   她      └──┘ 0.01  └──┘ 0.05   └──┘ 0.20
```

### 代码实现

```python
"""
HuggingFace 文本生成：对比不同策略
"""
from transformers import AutoTokenizer, AutoModelForCausalLM
import torch

model_name = "uer/gpt2-chinese-cluecorpussmall"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForCausalLM.from_pretrained(model_name)

prompt = "人工智能的未来"
inputs = tokenizer(prompt, return_tensors="pt")

# 1. Greedy Search
print("=== Greedy Search ===")
greedy_output = model.generate(
    **inputs,
    max_new_tokens=50,
    do_sample=False,           # 贪心
    no_repeat_ngram_size=2,    # 禁止 2-gram 重复
)
print(tokenizer.decode(greedy_output[0], skip_special_tokens=True))

# 2. Beam Search
print("\n=== Beam Search ===")
beam_output = model.generate(
    **inputs,
    max_new_tokens=50,
    num_beams=5,               # Beam 宽度
    no_repeat_ngram_size=2,
    early_stopping=True,
)
print(tokenizer.decode(beam_output[0], skip_special_tokens=True))

# 3. Top-k Sampling
print("\n=== Top-k Sampling ===")
topk_output = model.generate(
    **inputs,
    max_new_tokens=50,
    do_sample=True,
    top_k=50,                  # Top-k 采样
    no_repeat_ngram_size=2,
)
print(tokenizer.decode(topk_output[0], skip_special_tokens=True))

# 4. Top-p Sampling（推荐）
print("\n=== Top-p Sampling ===")
topp_output = model.generate(
    **inputs,
    max_new_tokens=50,
    do_sample=True,
    top_p=0.92,                # Top-p 采样
    temperature=0.8,           # 温度
    no_repeat_ngram_size=2,
)
print(tokenizer.decode(topp_output[0], skip_special_tokens=True))
```

**策略选择指南**：

| 场景 | 推荐策略 | 理由 |
|------|---------|------|
| 摘要、翻译 | Beam Search | 要求准确、流畅 |
| 故事、对话 | Top-p + Temperature | 需要多样性 |
| 代码生成 | Greedy / 低 Temperature | 要求确定性 |
| 诗歌、创意 | Top-p + 高 Temperature | 鼓励新颖 |

## 三、文本摘要

文本摘要分为两大流派：**抽取式**和**生成式**。

```
抽取式 vs 生成式摘要

原文：
"李彦宏于 2000 年在中关村创立了百度公司。最初，百度专注于搜索引擎技术。
 经过二十多年的发展，百度已经成为中国最大的互联网公司之一，
 业务涵盖搜索、云计算、人工智能等多个领域。"

抽取式摘要：从原文挑选关键句
┌──────────────────────────────────────────────┐
│  句子评分：                                   │
│  S1: 0.85  ★ 李彦宏创立百度                    │
│  S2: 0.42    最初专注搜索                      │
│  S3: 0.78  ★ 百度成为最大互联网公司            │
└──────────────────────────────────────────────┘
摘要：李彦宏创立百度。百度成为最大互联网公司。

生成式摘要：用自然语言重新组织
┌──────────────────────────────────────────────┐
│  BART/T5 理解全文后生成：                       │
│  "百度由李彦宏于 2000 年创立，已发展为中国       │
│   最大的互联网公司之一。"                       │
└──────────────────────────────────────────────┘
```

### 抽取式摘要：BERTSUM

```python
"""
简化版抽取式摘要：基于句子评分
"""
import torch
from transformers import AutoTokenizer, AutoModel
from sklearn.metrics.pairwise import cosine_similarity
import numpy as np

model_name = "bert-base-chinese"
tokenizer = AutoTokenizer.from_pretrained(model_name)
bert_model = AutoModel.from_pretrained(model_name)

def extractive_summary(text, num_sentences=3):
    """基于 BERT 句子嵌入 + TextRank 思路的抽取式摘要"""
    # 切句
    import re
    sentences = re.split(r"[。！？.!?]", text)
    sentences = [s.strip() for s in sentences if len(s.strip()) > 5]

    if len(sentences) <= num_sentences:
        return text

    # 编码每个句子
    embeddings = []
    for sent in sentences:
        inputs = tokenizer(sent, return_tensors="pt", truncation=True, max_length=128)
        with torch.no_grad():
            output = bert_model(**inputs)
        # 取 [CLS] 向量作为句子表示
        embeddings.append(output.last_hidden_state[0, 0, :].numpy())
    embeddings = np.array(embeddings)

    # 计算句子相似度矩阵
    sim_matrix = cosine_similarity(embeddings)

    # TextRank：相似度矩阵幂迭代
    scores = np.ones(len(sentences)) / len(sentences)
    for _ in range(50):
        scores = sim_matrix @ scores
        scores = scores / scores.sum()

    # 选 top-k 句子（按原文顺序）
    top_indices = np.argsort(scores)[-num_sentences:]
    top_indices.sort()
    summary = "。".join([sentences[i] for i in top_indices]) + "。"
    return summary

# 测试
text = """
人工智能是计算机科学的一个分支，致力于研究、开发用于模拟、延伸和扩展人的智能的理论、方法、技术及应用系统。
人工智能的历史可以追溯到 1956 年的达特茅斯会议。近年来，深度学习的突破让 AI 迎来了新一轮发展浪潮。
神经网络、自然语言处理、计算机视觉等领域取得了显著进展。大语言模型如 GPT、BERT 等深刻改变了人们的生活方式。
未来，人工智能将在医疗、教育、交通等领域发挥更大的作用。
"""
print(extractive_summary(text, num_sentences=2))
```


**抽取式 vs 生成式 对比**：

| 维度 | 抽取式 | 生成式 |
|------|-------|-------|
| 答案形式 | 原文句子 | 重写的新句子 |
| 可读性 | 一般（句子拼接生硬）| 高（自然流畅）|
| 准确性 | 高（来自原文）| 较低（可能产生幻觉）|
| 计算成本 | 低 | 高 |
| 模型 | BERTSUM、TextRank | BART、T5、PEGASUS |
| 适用场景 | 信息抽取、严格事实 | 新闻摘要、会议纪要 |

## 四、ROUGE 评估指标

ROUGE（Recall-Oriented Understudy for Gisting Evaluation）是摘要任务的标准评估指标，核心思想是衡量生成摘要覆盖了参考摘要中多少 n-gram。

```
ROUGE 计算示例

生成摘要：我 爱 自然 语言 处理
参考摘要：我 热爱 自然 语言 处理

ROUGE-1（unigram 召回率）：
  匹配：我、自然、语言、处理 = 4 个
  参考总数：5 个
  ROUGE-1 Recall = 4/5 = 0.80

ROUGE-2（bigram 召回率）：
  生成 bigram：(我爱, 爱自然, 自然语言, 语言处理)
  参考 bigram：(我热爱, 热爱自然, 自然语言, 语言处理)
  匹配：(自然语言, 语言处理) = 2 个
  ROUGE-2 Recall = 2/4 = 0.50

ROUGE-L（最长公共子序列 LCS）：
  生成：我 _ 自然 语言 处理
  参考：我 热爱 自然 语言 处理
  LCS = [我, 自然, 语言, 处理]，长度 4
  ROUGE-L = F1 based on LCS
```

```python
"""
ROUGE 计算示例
依赖：pip install rouge-chinese
"""
from rouge_chinese import Rouge

generated = "我 爱 自然 语言 处理"
reference = "我 热爱 自然 语言 处理"

# 注意：rouge-chinese 需要先用 jieba 分词
import jieba
gen_tokens = " ".join(jieba.cut(generated))
ref_tokens = " ".join(jieba.cut(reference))

rouge = Rouge()
scores = rouge.get_scores(gen_tokens, ref_tokens)[0]
print(f"ROUGE-1: P={scores['rouge-1']['p']:.4f} R={scores['rouge-1']['r']:.4f} F={scores['rouge-1']['f']:.4f}")
print(f"ROUGE-2: P={scores['rouge-2']['p']:.4f} R={scores['rouge-2']['r']:.4f} F={scores['rouge-2']['f']:.4f}")
print(f"ROUGE-L: P={scores['rouge-l']['p']:.4f} R={scores['rouge-l']['r']:.4f} F={scores['rouge-l']['f']:.4f}")
```

**ROUGE vs BLEU 对比**：

| 指标 | 方向 | 关注点 | 典型任务 |
|------|------|-------|---------|
| BLEU | Precision 导向 | 生成中有多少在参考里 | 机器翻译 |
| ROUGE | Recall 导向 | 参考中有多少被生成覆盖 | 文本摘要 |

## 六、HuggingFace 文本生成完整示例

```python
"""
完整的文本生成 Pipeline：支持多种任务
"""
from transformers import AutoTokenizer, AutoModelForCausalLM, TextGenerationPipeline
import torch

model_name = "Qwen/Qwen2.5-0.5B-Instruct"   # 示例模型，可替换
tokenizer = AutoTokenizer.from_pretrained(model_name, trust_remote_code=True)
model = AutoModelForCausalLM.from_pretrained(
    model_name,
    trust_remote_code=True,
    torch_dtype=torch.float16 if torch.cuda.is_available() else torch.float32,
)

# 构建对话格式
def chat(prompt, max_new_tokens=200):
    messages = [{"role": "user", "content": prompt}]
    text = tokenizer.apply_chat_template(
        messages, tokenize=False, add_generation_prompt=True
    )
    inputs = tokenizer(text, return_tensors="pt")
    outputs = model.generate(
        **inputs,
        max_new_tokens=max_new_tokens,
        do_sample=True,
        top_p=0.8,
        temperature=0.7,
        repetition_penalty=1.1,
    )
    response = outputs[0][inputs["input_ids"].shape[1]:]
    return tokenizer.decode(response, skip_special_tokens=True)

print(chat("用三句话介绍一下自然语言处理的发展历史。"))
```

## 七、常见挑战与对策

| 挑战 | 表现 | 对策 |
|------|------|------|
| 重复生成 | "我我爱爱自自然然..." | `repetition_penalty`、`no_repeat_ngram_size` |
| 事实错误 | 编造不存在的事实 | RAG + 引用、事实核查 |
| 长度失控 | 过长或过短 | `max_length`、`length_penalty` |
| 风格漂移 | 不符合任务风格 | Prompt 工程、指令微调 |
| 安全性 | 生成有害内容 | 安全过滤器、RLHF 对齐 |
| 多语言混杂 | 中英文夹杂 | 语言约束、专用模型 |

## 小结

| 要点 | 内容 |
|------|------|
| 任务家族 | 摘要、续写、翻译、对话、代码生成 |
| 解码策略 | Greedy、Beam Search、Top-k、Top-p、Temperature |
| 抽取式摘要 | BERTSUM、TextRank：从原文选关键句 |
| 生成式摘要 | BART、T5、PEGASUS：重新组织语言 |
| 评估指标 | ROUGE（召回导向）、BLEU（精度导向）|
| 质量维度 | 字面匹配 + 语言质量 + 主观感受 + 安全性 |
| 实践建议 | 推荐组合：Top-p(0.9) + Temperature(0.7) + repetition_penalty |
