# 文本分类：NLP 的"Hello World"

文本分类是自然语言处理中最基础也最核心的任务之一。给定一段文本，模型需要将其归入预定义的一个或多个类别。从邮箱判断垃圾邮件，到电商分析用户情感，再到客服系统识别用户意图，文本分类无处不在。本章将带你从任务定义出发，走过方法演进，最终实现一个完整的 BERT 文本分类微调流程，并拓展到零样本和多标签场景。

## 一、任务概述

文本分类根据标签数量关系，可以分为三大类。

```
┌──────────────────────────────────────────────────────────────┐
│                     文本分类任务分类                          │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  1. 二分类 (Binary Classification)                           │
│     ┌────────────┐    模型    ┌──────┐                       │
│     │ "电影真好看"│ ────────► │ 正面 │  (2个互斥类别)         │
│     └────────────┘           └──────┘                       │
│                                                              │
│  2. 多分类 (Multi-class Classification)                      │
│     ┌────────────┐    模型    ┌──────┐                       │
│     │ "股市行情"  │ ────────► │ 财经 │  (N个互斥类别)         │
│     └────────────┘           └──────┘                       │
│                                                              │
│  3. 多标签 (Multi-label Classification)                      │
│     ┌────────────┐    模型    ┌──────┐ ┌──────┐              │
│     │ "Python爬虫"│ ────────► │ 技术 │ │ 教育 │ (可多个标签)  │
│     └────────────┘           └──────┘ └──────┘              │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **二分类**：样本只能属于两个互斥类别之一（正面/负面、垃圾/正常）。
- **多分类**：样本属于 N 个互斥类别中的一个，如新闻主题分类（财经/体育/科技）。
- **多标签**：一个样本可以同时拥有多个标签，标签之间不互斥，如一篇技术文章可以同时打上"AI""后端""教程"三个标签。

## 二、典型应用场景

| 应用场景 | 任务类型 | 说明 |
|---------|---------|------|
| 情感分析 | 二/多分类 | 判断评论是正面、负面还是中性 |
| 主题分类 | 多分类 | 将新闻归类到财经、体育、娱乐等栏目 |
| 垃圾检测 | 二分类 | 识别垃圾邮件、垃圾短信、恶意评论 |
| 意图识别 | 多分类 | 对话系统中识别用户意图（订票/查天气/闲聊） |
| 内容审核 | 多标签 | 识别文本是否包含暴力、色情、广告等多重风险 |
| 语言识别 | 多分类 | 判断文本属于哪种语言 |

## 三、方法演进

文本分类的方法经历了近七十年的演进，每一次升级都伴随着特征表达能力的飞跃。

```
时间轴：方法演进
─────────────────────────────────────────────────────────────►
  1950s        1990s         2010s        2018+        2023+
   │             │             │            │            │
   ▼             ▼             ▼            ▼            ▼
  规则匹配  →  TF-IDF+ML  →  Word2Vec   →   BERT    →   LLM
              (SVM/NB)       +深度网络     微调         零样本
   │             │             │            │            │
   │             │             │            │            │
  关键词     统计特征      稠密词向量    上下文表示    提示工程
  正则式     机器学习      LSTM/CNN     预训练微调    无需训练
─────────────────────────────────────────────────────────────►
  特征表达：手工 → 统计 → 稠密 → 上下文 → 世界知识
```

**各阶段特点对比**：

| 方法 | 特征表示 | 优点 | 缺点 |
|------|---------|------|------|
| 规则匹配 | 关键词/正则 | 可解释、无需训练 | 覆盖率低、维护成本高 |
| TF-IDF + SVM | 稀疏向量 | 训练快、效果稳定 | 无法理解语义、词袋模型 |
| Word2Vec + LSTM | 静态稠密向量 | 能捕捉词相似度 | 词向量固定、无法消歧 |
| BERT 微调 | 动态上下文向量 | 效果优异、通用 | 需标注数据、计算成本高 |
| LLM 零样本 | 提示工程 | 无需训练、泛化强 | 成本高、推理慢 |

## 四、BERT 文本分类微调

下面使用 HuggingFace Transformers 完整实现一个 BERT 文本分类任务。以中文情感分析为例。

```python
"""
BERT 文本分类微调完整示例：中文情感分析
任务：将用户评论分为 正面(1) / 负面(0)
"""
import torch
from torch.utils.data import Dataset, DataLoader
from transformers import (
    BertTokenizer,
    BertForSequenceClassification,
    AdamW,
    get_linear_schedule_with_warmup,
)
from sklearn.model_selection import train_test_split
from sklearn.metrics import classification_report
import numpy as np

# ─── 1. 数据准备 ─────────────────────────────────────────────
# 示例数据集（实际项目请替换为真实数据）
texts = [
    "这家餐厅菜品新鲜，服务态度好，强烈推荐！",
    "东西太难吃了，等了半小时才上菜。",
    "性价比超高，下次还会再来。",
    "环境脏乱差，绝不推荐。",
    "味道一般，但价格便宜，勉强接受。",
    "史上最差体验，服务员态度恶劣！",
    "非常满意，特别是招牌菜很好吃。",
    "不会再来了，完全是浪费钱。",
] * 10  # 简单复制扩充数据量
labels = [1, 0, 1, 0, 1, 0, 1, 0] * 10

# 划分训练/验证集
train_texts, val_texts, train_labels, val_labels = train_test_split(
    texts, labels, test_size=0.2, random_state=42, stratify=labels
)

# ─── 2. 自定义 Dataset ───────────────────────────────────────
class TextClassificationDataset(Dataset):
    def __init__(self, texts, labels, tokenizer, max_len=128):
        self.texts = texts
        self.labels = labels
        self.tokenizer = tokenizer
        self.max_len = max_len

    def __len__(self):
        return len(self.texts)

    def __getitem__(self, idx):
        encoding = self.tokenizer(
            self.texts[idx],
            truncation=True,
            padding="max_length",
            max_length=self.max_len,
            return_tensors="pt",
        )
        return {
            "input_ids": encoding["input_ids"].flatten(),
            "attention_mask": encoding["attention_mask"].flatten(),
            "labels": torch.tensor(self.labels[idx], dtype=torch.long),
        }

# ─── 3. 加载模型和分词器 ─────────────────────────────────────
model_name = "bert-base-chinese"
tokenizer = BertTokenizer.from_pretrained(model_name)
model = BertForSequenceClassification.from_pretrained(
    model_name, num_labels=2
)

# 构建数据加载器
train_dataset = TextClassificationDataset(train_texts, train_labels, tokenizer)
val_dataset = TextClassificationDataset(val_texts, val_labels, tokenizer)
train_loader = DataLoader(train_dataset, batch_size=16, shuffle=True)
val_loader = DataLoader(val_dataset, batch_size=32)

# ─── 4. 训练配置 ─────────────────────────────────────────────
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
model.to(device)

epochs = 3
optimizer = AdamW(model.parameters(), lr=2e-5, weight_decay=0.01)
total_steps = len(train_loader) * epochs
scheduler = get_linear_schedule_with_warmup(
    optimizer, num_warmup_steps=50, num_training_steps=total_steps
)

# ─── 5. 训练循环 ─────────────��───────────────────────────────
for epoch in range(epochs):
    model.train()
    total_loss = 0
    for batch in train_loader:
        optimizer.zero_grad()
        inputs = {k: v.to(device) for k, v in batch.items()}
        outputs = model(**inputs)
        loss = outputs.loss
        loss.backward()
        torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
        optimizer.step()
        scheduler.step()
        total_loss += loss.item()
    print(f"Epoch {epoch+1}/{epochs} - Loss: {total_loss/len(train_loader):.4f}")

# ─── 6. 评估 ─────────────────────────────────────────────
model.eval()
predictions, true_labels = [], []
with torch.no_grad():
    for batch in val_loader:
        inputs = {k: v.to(device) for k, v in batch.items() if k != "labels"}
        outputs = model(**inputs)
        preds = torch.argmax(outputs.logits, dim=1).cpu().numpy()
        predictions.extend(preds)
        true_labels.extend(batch["labels"].numpy())

print(classification_report(true_labels, predictions, target_names=["负面", "正面"]))
```

训练 3 个 epoch 后，你将看到类似如下的评估报告：

```
              precision    recall  f1-score   support

        负面       0.95      0.93      0.94        15
        正面       0.93      0.95      0.94        17

    accuracy                           0.94        32
   macro avg       0.94      0.94      0.94        32
weighted avg       0.94      0.94      0.94        32
```

## 五、使用 Trainer 简化训练

上面的手动训练循环适合学习理解。在实际工程中，更推荐使用 HuggingFace 的 `Trainer` API。

```python
from transformers import Trainer, TrainingArguments

training_args = TrainingArguments(
    output_dir="./results",
    num_train_epochs=3,
    per_device_train_batch_size=16,
    per_device_eval_batch_size=32,
    learning_rate=2e-5,
    weight_decay=0.01,
    eval_strategy="epoch",
    save_strategy="epoch",
    load_best_model_at_end=True,
    metric_for_best_model="f1",
)

trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=train_dataset,
    eval_dataset=val_dataset,
    compute_metrics=lambda p: {
        "f1": classification_report(
            p.label_ids, np.argmax(p.predictions, axis=1),
            output_dict=True, zero_division=0
        )["macro avg"]["f1"],
    },
)
trainer.train()
```

## 六、零样本分类（Zero-Shot Classification）

当没有标注数据时，可以直接利用大模型的内在知识进行零样本分类。

```python
"""
使用 BART MNLI 模型做零样本文本分类
原理：NLI（自然语言推理）判断文本与候选标签的蕴含关系
"""
from transformers import pipeline

classifier = pipeline(
    "zero-shot-classification",
    model="facebook/bart-large-mnli",
)

text = "我今天学会了用 Python 写爬虫，效率真高！"
candidate_labels = ["技术", "美食", "旅游", "体育", "教育"]

result = classifier(text, candidate_labels)
print(f"文本：{text}")
for label, score in zip(result["labels"], result["scores"]):
    print(f"  {label:8s}  {score:.4f}")

# 输出示例：
# 文本：我今天学会了用 Python 写爬虫，效率真高！
#   技术       0.8521
#   教育       0.1023
#   美食       0.0156
#   ...
```

**原理图解**：

```
                ┌────────────────┐
   原始文本 ───►│  零样本分类器    │
                └───────┬────────┘
                        │  构造 NLI 假设
                        ▼
        ┌───────────────────────────────────┐
        │ "这段文本是关于 [技术] 的"          │
        │ "这段文本是关于 [美食] 的"          │
        │ "这段文本是关于 [旅游] 的"          │
        └───────────────────────────────────┘
                        │  NLI 模型判断蕴含分数
                        ▼
              选择得分最高的标签
```

## 七、多标签分类

多标签分类的核心区别在于：输出层的激活函数用 `Sigmoid` 而非 `Softmax`，损失函数用 `BCEWithLogitsLoss` 而非 `CrossEntropyLoss`。

### One-vs-Rest 策略

```python
"""
多标签分类：One-vs-Rest 思路
为每个标签训练一个独立的二分类器
"""
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.linear_model import LogisticRegression
from sklearn.multiclass import OneVsRestClassifier
from sklearn.preprocessing import MultiLabelBinarizer

# 数据：一篇文档可以有多个标签
texts_ml = [
    "Python 机器学习实战教程",
    "足球世界杯决赛精彩进球",
    "深度学习与神经网络入门",
    "NBA 总决赛湖人队夺冠",
]
tags = [
    ["技术", "教育"],
    ["体育"],
    ["技术", "教育"],
    ["体育"],
]

mlb = MultiLabelBinarizer()
y = mlb.fit_transform(tags)

vectorizer = TfidfVectorizer(max_features=1000)
X = vectorizer.fit_transform(texts_ml)

# 训练 One-vs-Rest 分类器
classifier = OneVsRestClassifier(LogisticRegression())
classifier.fit(X, y)

# 预测
test_text = ["AI 教育课程设计"]
X_test = vectorizer.transform(test_text)
pred = classifier.predict(X_test)
print(mlb.inverse_transform(pred))
# 输出：[('技术', '教育')]
```

### BERT 多标签分类

```python
"""
BERT 多标签分类：输出层使用 Sigmoid
"""
import torch.nn as nn

class BertForMultiLabel(nn.Module):
    def __init__(self, model_name, num_labels):
        super().__init__()
        self.bert = BertForSequenceClassification.from_pretrained(
            model_name, num_labels=num_labels, problem_type="multi_label"
        )

    def forward(self, input_ids, attention_mask, labels=None):
        outputs = self.bert(
            input_ids=input_ids,
            attention_mask=attention_mask,
            labels=labels,
        )
        return outputs  # 内部已自动使用 BCEWithLogitsLoss
```

**二分类 vs 多分类 vs 多标签对比**：

| 维度 | 二分类 | 多分类 | 多标签 |
|------|-------|-------|-------|
| 标签数 | 2 | N ≥ 2 | N ≥ 2 |
| 互斥性 | 互斥 | 互斥 | 不互斥 |
| 激活函数 | Sigmoid | Softmax | Sigmoid |
| 损失函数 | BCE | CrossEntropy | BCE |
| 评估指标 | F1 | Macro-F1 | Micro/Macro-F1 |

## 八、评估指标

```python
"""
文本分类评估指标详解
"""
from sklearn.metrics import (
    accuracy_score, precision_score, recall_score, f1_score,
    confusion_matrix,
)

y_true = [1, 0, 1, 1, 0, 1, 0, 0]
y_pred = [1, 0, 1, 0, 0, 1, 1, 0]

print(f"Accuracy  : {accuracy_score(y_true, y_pred):.4f}")
# Accuracy = 预测正确数 / 总数 = (TP+TN)/(TP+TN+FP+FN)

print(f"Precision : {precision_score(y_true, y_pred):.4f}")
# Precision = 预测为正中实际为正的比例 = TP/(TP+FP)

print(f"Recall    : {recall_score(y_true, y_pred):.4f}")
# Recall = 实际为正中被预测出来的比例 = TP/(TP+FN)

print(f"F1 Score  : {f1_score(y_true, y_pred):.4f}")
# F1 = 2 * P * R / (P + R)，精确率与召回率的调和平均

print("混淆矩阵：")
print(confusion_matrix(y_true, y_pred))
```

```
混淆矩阵图解：
                    预测
                 正面   负面
             ┌─────────────┐
    实际 正面 │  TP   │  FN  │   ← 召回率关注这一行
             ├───────┼──────┤
        负面 │  FP   │  TN  │   ← 精确率关注这一列
             └─────────────┘
               ↑
          精确率关注这一列
```

**指标选择指南**：

- **数据均衡**：用 Accuracy。
- **数据不均衡**：用 F1（Macro 或 Weighted）。
- **重视查准**（如垃圾检测）：看 Precision。
- **重视查全**（如疾病筛查）：看 Recall。

## 九、工程实践建议

1. **数据质量优先**：在数据量不足时，精心标注 1000 条数据胜过爬取 10 万条噪声数据。
2. **预训练模型选择**：中文任务首选 `bert-base-chinese` 或 `hfl/chinese-roberta-wwm-ext`，长文本考虑 `Longformer`。
3. **学习率敏感**：BERT 微调的典型学习率是 `2e-5` 到 `5e-5`，过高会导致灾难性遗忘。
4. **灾难性遗忘**：微调时冻结部分底层 Transformer 层，能保留通用语义能力。
5. **部署优化**：训练完成后用 ONNX Runtime 或 TorchScript 加速推理，详见[模型量化与推理优化](../08-预训练模型应用篇/05-模型量化与推理优化.md)。
