# 机器翻译：让语言不再成为障碍

机器翻译（Machine Translation, MT）旨在自动将一种语言（源语言）的文本翻译成另一种语言（目标语言）。从早期的规则匹配，到统计机器翻译的辉煌，再到神经网络带来的革命性提升，机器翻译是 NLP 发展史上最具有标志性、也是商业化最成功的任务之一。本章将系统讲解三种范式的演进，并给出基于 Transformer 的完整代码实践。

## 一、发展简史

```
机器翻译三大范式演进
─────────────────────────────────────────────────────────────►
  1950s-1980s       1990s-2010s         2014+              2017+
     │                  │                  │                  │
     ▼                  ▼                  ▼                  ▼
   RBMT      →       SMT         →      NMT         →     Transformer
  规则机器翻译      统计机器翻译       神经机器翻译        注意力革命
     │                  │                  │                  │
   规则+词典        IBM Model 1-5        Seq2Seq            完全注意力
   双语专家         Phrase-Based         Encoder-Decoder     抛弃 RNN
                    Moses 系统           + Attention         并行训练
     │                  │                  │                  │
   ──────────��──  BLEU 评估诞生  ─────────────────────────────►
─────────────────────────────────────────────────────────────►
  人工翻译质量对比： GE < RBMT < SMT < RNN-NMT < Transformer-NMT ≤ 人类
```

| 范式 | 核心思想 | 优点 | 缺点 |
|------|---------|------|------|
| RBMT（规则） | 人工编写语法+词典 | 解释性强 | 规则爆炸、跨语言不通用 |
| SMT（统计） | 从双语语料学翻译概率 | 自学习、通用 | 局部建模、特征工程繁琐 |
| RNN-NMT | Seq2Seq 端到端学习 | 全局上下文 | 长序列衰减、训练慢 |
| Transformer-NMT | 自注意力全局建模 | 并行训练、长程依赖 | 显存消耗大 |

## 二、Seq2Seq 架构

Seq2Seq（Encoder-Decoder）是神经机器翻译的奠基性架构：编码器把源语言压缩成一个向量表示，解码器基于该表示逐 token 生成目标语言。

```
Seq2Seq 编码器-解码器架构

源语言（中文）：我 爱 自然语言 处理

┌─────────── Encoder (BiLSTM) ───────────┐
│ 我 ─►│ 爱 ─►│ 自然 ─►│ 语言 ─►│ 处理 ─►│
│  h1    h2    h3      h4      h5         │
└─────────────────────────────────────────┘
                          │
                          ▼  上下文向量 c
┌─────────── Decoder (LSTM) ─────────────┐
│   I   love  NLP    <eos>                │
│    ▲    ▲     ▲                         │
│   h1'  h2'  h3'                         │
└─────────────────────────────────────────┘
        │     │     │
        ▼     ▼     ▼
       I    love   NLP

解码过程：每一步根据 (c, 已生成 token) 预测下一 token
```

**信息瓶颈问题**：所有源句子信息被压缩到固定维度的上下文向量 `c`，长句子难以表达完整，导致翻译质量随源句长度增加而下降。这正是注意力机制要解决的核心痛点。

## 三、注意力机制

注意力机制让解码器在生成每个目标 token 时，能"回看"编码器的所有隐藏状态，动态决定关注源句子的哪些位置。

```
注意力机制工作流程

源句：我   爱   自然  语言   处理
      h1   h2   h3    h4     h5

解码时生成 "love"：
   ┌─────────────────────────────────┐
   │ Decoder 隐藏状态 s2 ──┐          │
   └─────────────────────────────────┘
                          │
                          ▼
        对齐分数：score(s2, h_i)
        ┌────────────────────────────┐
        │  h1:0.10  h2:0.65  h3:0.15 │
        │  h4:0.05  h5:0.05          │
        └────────────────────────────┘
                          │ softmax
                          ▼
        注意力权重 α
        ┌────────────────────────────┐
        │  0.10      0.65      0.15  │
        │  0.05      0.05            │
        └────────────────────────────┘
                          │ 加权求和
                          ▼
        上下文向量 c2 = Σ α_i * h_i
                          │
                          ▼
        拼接 [s2 ; c2] → softmax → "love"
```

- **score 函数**：计算 decoder 状态与 encoder 状态的相似度，常见有 `dot`、`general`、`concat` 三种。
- **soft attention**（主流）：所有位置都有非零权重，可端到端训练。
- **hard attention**：只关注一个位置，使用强化学习训练，较少使用。

## 四、Transformer 翻译模型

Transformer 用**多头自注意力**替代 RNN，实现了完全并行化的翻译建模，是当前所有主流翻译模型的基石。

```
Transformer 用于翻译的结构

源句：I love NLP
目标：我爱自然语言

┌─────────────────────────────────────────────────┐
│                  Transformer                     │
│                                                  │
│  ┌──────────── Encoder ────────────┐             │
│  │  源 token embedding             │             │
│  │      + 位置编码                  │             │
│  │      ▼                          │             │
│  │  ┌──────────────────┐           │             │
│  │  │ Multi-Head       │           │             │
│  │  │ Self-Attention   │ × 6层     │             │
│  │  │ + FFN + 残差      │           │             │
│  │  └──────────────────┘           │             │
│  │      ▼                          │             │
│  │  Encoder 输出 H (源语义表示)     │             │
│  └──────────────┬──────────────────┘             │
│                 │  Encoder-Decoder Attention     │
│  ┌───────────── ▼ Decoder ──────────┐             │
│  │  目标 token embedding (shifted)   │             │
│  │      + 位置编码 + Mask            │             │
│  │      ▼                            │             │
│  │  ┌──────────────────┐             │             │
│  │  │ Masked Multi-Head│             │             │
│  │  │ Self-Attention   │             │             │
│  │  └─────────┬────────┘             │             │
│  │            ▼                      │             │
│  │  ┌──────────────────┐             │             │
│  │  │ Encoder-Decoder  │ × 6层       │             │
│  │  │ Attention        │             │             │
│  │  └─────────┬────────┘             │             │
│  │            ▼                      │             │
│  │  ┌──────────────────┐             │             │
│  │  │ FFN + 残差        │             │             │
│  │  └─────────┬────────┘             │             │
│  │            ▼                      │             │
│  │      Linear + Softmax             │             │
│  │            ▼                      │             │
│  │      目标词概率分布                 │             │
│  └────────────────────────────────────┘             │
└─────────────────────────────────────────────────────┘
```

Transformer 的核心公式：

```
Attention(Q, K, V) = softmax(Q K^T / √d_k) V

MultiHead = Concat(head_1, ..., head_h) W^O
其中 head_i = Attention(Q W_i^Q, K W_i^K, V W_i^V)
```

## 五、Beam Search 解码

贪心解码每步选概率最高的 token，容易陷入局部最优；Beam Search 保留 top-k 候选序列，能找到全局更优解。

```
解码策略对比：翻译 "I love NLP" → 中文

【Greedy Search】（k=1）
─────────────────────────────────
步骤1：我 (0.6) > 你 (0.3) > 他 (0.1)   → 选 我
步骤2：爱 (0.5) > 是 (0.3) > 想 (0.2)   → 选 爱
步骤3：NLP(0.4) > 编程(0.3) > AI(0.3)   → 选 NLP
结果：我爱 NLP（局部最优，可能错过更优组合）


【Beam Search】（k=2，保留两个候选）
─────────────────────────────────
步骤1：候选 = {我(0.6), 你(0.3)}

步骤2：扩展每个候选
   我+爱 (0.6×0.5=0.30)  ★
   我+是 (0.6×0.3=0.18)
   你+爱 (0.3×0.5=0.15)
   你+好 (0.3×0.7=0.21)  ★
   保留 top-2: {我爱(0.30), 你好(0.21)}

步骤3：继续扩展 ...
最终：选择累计概率最高的完整序列
```

```python
"""
使用 HuggingFace 进行 Beam Search 翻译
"""
from transformers import AutoTokenizer, AutoModelForSeq2SeqLM

model_name = "Helsinki-NLP/opus-mt-en-zh"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForSeq2SeqLM.from_pretrained(model_name)

def translate(text, num_beams=4, max_length=128):
    inputs = tokenizer(text, return_tensors="pt")
    outputs = model.generate(
        **inputs,
        num_beams=num_beams,        # Beam Search 宽度
        max_length=max_length,
        length_penalty=0.6,         # 长度惩罚，>1 倾向长句
        early_stopping=True,
    )
    return tokenizer.decode(outputs[0], skip_special_tokens=True)

# 测试翻译
sentences = [
    "I love natural language processing.",
    "Machine translation bridges language barriers.",
    "The transformer architecture revolutionized NLP.",
]
for src in sentences:
    tgt = translate(src)
    print(f"EN: {src}")
    print(f"ZH: {tgt}\n")
```

## 六、BLEU 评估指标

BLEU（Bilingual Evaluation Understudy）是机器翻译最经典的自动评估指标，核心思想是衡量机器翻译与人工参考翻译在 n-gram 上的重叠程度。

```
BLEU 计算流程

候选译文（机器）：我   爱   自然  语言   处理
参考译文（人工）：我   热爱 自然  语言   处理

1-gram 匹配：我✓ 爱✗ 自然✓ 语言✓ 处理✓  → 4/5 = 0.80
2-gram 匹配：(我爱)✗ (爱自然)✗ (自然语言)✓ (语言处理)✓ → 2/4 = 0.50
3-gram 匹配：(我爱自然)✗ ... → 1/3 = 0.33
4-gram 匹配：0/2 = 0.00

几何平均精度：BP × exp(Σ w_n × log p_n)

长度惩罚 BP（Brevity Penalty）：
   当候选短于参考时扣分，防止系统为追求高精度只输出短句
   BP = min(1, exp(1 - 参考长度/候选长度))
```

```python
"""
BLEU 计算示例（使用 sacrebleu）
"""
from sacrebleu import corpus_bleu

# 候选译文（系统输出）
candidates = [
    "我 爱 自然 语言 处理",
    "机器 翻译 改变 了 世界",
]

# 参考译文（可以有多个）
references = [
    ["我 热爱 自然 语言 处理", "机器 翻译 改变 了 整个 世界"],
]

bleu = corpus_bleu(candidates, references)
print(f"BLEU score: {bleu.score:.2f}")
print(f"  1-gram precision: {bleu.precisions[0]:.4f}")
print(f"  2-gram precision: {bleu.precisions[1]:.4f}")
print(f"  3-gram precision: {bleu.precisions[2]:.4f}")
print(f"  4-gram precision: {bleu.precisions[3]:.4f}")
print(f"  BP (brevity penalty): {bleu.bp:.4f}")
```

**BLEU 解读**：

| BLEU | 质量级别 | 说明 |
|------|---------|------|
| < 10 | 很差 | 几乎不可用 |
| 10–20 | 难懂 | 大意可猜 |
| 20–30 | 粗糙 | 有明显错误 |
| 30–40 | 可理解 | 流畅度不足 |
| 40–50 | 高质量 | 接近人工翻译 |
| > 50 | 优秀 | 质量优异 |

## 七、HuggingFace 翻译 Pipeline

最快的上手方式是使用 `pipeline`：

```python
from transformers import pipeline

# 英译中
translator_en_zh = pipeline(
    "translation_en_to_zh",
    model="Helsinki-NLP/opus-mt-en-zh",
    max_length=128,
)

result = translator_en_zh("Artificial intelligence is transforming every industry.")
print(result[0]["translation_text"])
# 输出：人工智能正在改变每一个行业。

# 中译英（使用多语言模型 mBART）
translator_multi = pipeline(
    "translation",
    model="facebook/mbart-large-50-many-to-many-mt",
    tokenizer="facebook/mbart-large-50-many-to-many-mt",
    src_lang="zh_CN",
    tgt_lang="en_XX",
)
result = translator_multi("今天天气真好，适合出去散步。")
print(result[0]["translation_text"])
# 输出：The weather is really nice today, suitable for a walk.
```

## 八、中英翻译微调实践

如果有领域数据（如医疗、法律、口语化场景），可以在开源翻译模型基础上微调。

```python
"""
微调翻译模型示例（基于 MarianMT）
"""
from transformers import (
    AutoTokenizer,
    AutoModelForSeq2SeqLM,
    Seq2SeqTrainingArguments,
    Seq2SeqTrainer,
    DataCollatorForSeq2Seq,
)
from datasets import Dataset

model_name = "Helsinki-NLP/opus-mt-en-zh"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForSeq2SeqLM.from_pretrained(model_name)

# 准备数据：实际项目使用 WMT、UM-Corpus 等公开数据集
data = [
    {"en": "Hello world", "zh": "你好世界"},
    {"en": "How are you?", "zh": "你好吗？"},
    {"en": "I am fine, thank you.", "zh": "我很好，谢谢。"},
    {"en": "Natural language processing is fascinating.", "zh": "自然语言处理很迷人。"},
]
dataset = Dataset.from_list(data)

def preprocess(example):
    model_inputs = tokenizer(
        example["en"], max_length=128, truncation=True
    )
    labels = tokenizer(
        text_target=example["zh"], max_length=128, truncation=True
    )
    model_inputs["labels"] = labels["input_ids"]
    return model_inputs

tokenized = dataset.map(preprocess, batched=False)

data_collator = DataCollatorForSeq2Seq(tokenizer, model=model)

training_args = Seq2SeqTrainingArguments(
    output_dir="./mt-finetuned",
    num_train_epochs=3,
    per_device_train_batch_size=8,
    learning_rate=5e-5,
    eval_strategy="no",
    save_strategy="epoch",
    predict_with_generate=True,       # 用生成式评估
    fp16=torch.cuda.is_available(),
)

trainer = Seq2SeqTrainer(
    model=model,
    args=training_args,
    train_dataset=tokenized,
    data_collator=data_collator,
)
trainer.train()
```

## 九、常见挑战

| 挑战 | 描述 | 应对策略 |
|------|------|---------|
| 词表不一致 | 中英 token 切分差异大 | 用 SentencePiece / BBPE 统一词表 |
| 长句翻译 | 长序列 BLEU 急剧下降 | 长句切分、覆盖率惩罚 |
| 领域迁移 | 通用模型在医疗/法律领域掉点 | 领域预训练 + 微调 |
| 低资源语言 | 双语语料稀缺 | 回译、伪数据、多语言迁移 |
| 文化差异 | 直译不合习惯 | 人工后编辑、风格控制 |

## 小结

| 要点 | 内容 |
|------|------|
| 三大范式 | 规则翻译 → 统计翻译（SMT）→ 神经翻译（NMT）|
| 核心架构 | Seq2Seq Encoder-Decoder + Attention |
| Transformer | 多头自注意力，完全并行训练 |
| 解码策略 | Greedy、Beam Search（主流）、Sampling |
| 评估指标 | BLEU（n-gram 精度 + 长度惩罚）|
| 典型模型 | MarianMT、mBART、NLLB、Google Translate |
| HuggingFace | `AutoModelForSeq2SeqLM` + `Seq2SeqTrainer` |
| 实践建议 | 微调用领域数据、Beam 宽度 4–6、关注低资源 |
