# 序列标注：给每个 Token 贴标签

序列标注（Sequence Labeling）是 NLP 中一类 token 级别的任务：输入是一个 token 序列，输出是为每个 token 预测的标签序列。命名实体识别（NER）、词性标注（POS）、语义角色标注（SRL）都属于这一家族。与文本分类不同，序列标注关注的是**细粒度**的标签预测，是信息抽取、知识图谱构建等下游任务的基础。

## 一、核心任务

```
┌──────────────────────────────────────────────────────────────┐
│                     序列标注三大经典任务                       │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  1. NER（命名实体识别）                                       │
│     输入：李彦宏是百度的创始人                                 │
│     输出：[李彦宏]_PER 是 [百度]_ORG 的创始人                  │
│     识别文本中的人名、地名、机构名等实体                        │
│                                                              │
│  2. POS Tagging（词性标注）                                   │
│     输入：我 爱 自然语言 处理                                  │
│     输出：我/Pron 爱/V 自然语言/Noun 处理/Noun                 │
│     为每个词标注词性                                           │
│                                                              │
│  3. Chunking（组块分析）                                      │
│     输入：The big cat sat on the mat                          │
│     输出：[NP The big cat] [VP sat] [PP on] [NP the mat]      │
│     识别短语块（名词短语、动词短语等）                          │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 二、标注体系

序列标注的关键挑战是：**如何把 token 级的标签组合成完整的实体片段**。三种主流标注体系应运而生。

### BIO 标注

- **B-X**：实体 X 的开始（Begin）
- **I-X**：实体 X 的内部（Inside）
- **O**：非实体（Outside）

### BIOES 标注

- **B-X**：实体开始
- **I-X**：实体内部
- **E-X**：实体结束（End）
- **S-X**：单 token 实体（Single）
- **O**：非实体

### BILOU 标注

与 BIOES 等价，只是把 E 叫 L（Last），把 S 叫 U（Unit）。

```
BIO / BIOES / BILOU 标注对比图解

原句：李彦宏 是 百度 的 创始人
实体：李彦宏(PER)     百度(ORG)

Token 序列：李  彦  宏  是  百  度  的  创  始  人

BIO     ：B-PER I-PER I-PER O  B-ORG I-ORG O  O  O  O

BIOES   ：B-PER I-PER E-PER O  B-ORG E-ORG O  O  O  O

BILOU   ：B-PER I-PER L-PER O  B-ORG L-ORG O  O  O  O

实体边界：
         ├─ 李彦宏 ─┤     ├百度┤
         人名实体         机构实体
```

**为什么需要这些标签体系？**

如果只用 PER/ORG/O 三类标签，模型无法区分相邻的两个同类实体。例如"刘强东、章泽天"中两个人名会被错误地合并为一个实体。引入 B/I 边界标签后，模型就能在 `B-PER` 处明确开启一个新实体。

## 三、方法演进

```
方法演进时间轴
─────────────────────────────────────────────────────────────►
  1990s        2000s         2010s        2017+        2019+
   │             │             │            │            │
   ▼             ▼             ▼            ▼            ▼
   HMM     →    CRF      →  BiLSTM-CRF  →  BERT    →   LLM
  隐马尔可夫   条件随机场    双向LSTM+CRF   微调         生成式
   │             │             │            │            │
   │             │             │            │            │
  生成式      判别式        神经网络       预训练       提示抽取
  独立假设    全局特征      自动特征       上下文       统一范式
              全局归优
─────────────────────────────────────────────────────────────►
  特征工程：手工规则 → 全局优化 → 自动学习 → 上下文 → 世界知识
```

| 方法 | 建模思路 | 优点 | 缺点 |
|------|---------|------|------|
| HMM | 建模 `(观测, 状态)` 联合分布 | 理论清晰、可解释 | 假设独立、特征工程弱 |
| CRF | 建模 `P(状态\|观测)` 条件分布 | 全局最优、特征灵活 | 特征仍需手工设计 |
| BiLSTM-CRF | LSTM 自动学特征 + CRF 约束 | 无需手工特征 | 缺少预训练知识 |
| BERT | 预训练表示 + Token 分类头 | 效果最佳 | 计算成本高 |

## 四、BERT NER 完整代码

使用 `BertForTokenClassification` 实现命名实体识别。

```python
"""
BERT 命名实体识别完整示例
任务：识别中文文本中的人名(PER)、地名(LOC)、机构名(ORG)
"""
import torch
from torch.utils.data import Dataset, DataLoader
from transformers import (
    AutoTokenizer,
    AutoModelForTokenClassification,
    AdamW,
)
from seqeval.metrics import classification_report as seq_classification_report
from seqeval.metrics import f1_score

# ─── 1. 标签定义（BIO 体系）────────────────────────────────────
label_list = ["O", "B-PER", "I-PER", "B-ORG", "I-ORG", "B-LOC", "I-LOC"]
label2id = {label: i for i, label in enumerate(label_list)}
id2label = {i: label for i, label in enumerate(label_list)}
num_labels = len(label_list)

# ─── 2. 示例数据（实际项目请使用 PeopleDaily、MSRA、OntoNotes 等数据集）──
# 每个样本是 token 列表 + 对应标签列表（已经分好词）
# 注意：中文 NER 可以按字切分（字级别），也可以按词切分（词级别）
train_data = [
    (["李", "彦", "宏", "是", "百", "度", "的", "创", "始", "人"],
     ["B-PER", "I-PER", "I-PER", "O", "B-ORG", "I-ORG", "O", "O", "O", "O"]),
    (["马", "云", "在", "杭", "州", "创", "立", "了", "阿", "里", "巴", "巴"],
     ["B-PER", "I-PER", "O", "B-LOC", "I-LOC", "O", "O", "O", "B-ORG", "I-ORG", "I-ORG", "I-ORG"]),
    (["腾", "讯", "总", "部", "在", "深", "圳"],
     ["B-ORG", "I-ORG", "O", "O", "O", "B-LOC", "I-LOC"]),
    (["雷", "军", "出", "生", "于", "湖", "北"],
     ["B-PER", "I-PER", "O", "O", "O", "B-LOC", "I-LOC"]),
] * 20  # 扩充数据

# ─── 3. 数据预处理：对齐 subword 与标签 ────────────────────────
class NERDataset(Dataset):
    """
    关键难点：BERT 分词器会将一个字拆成多个 subword，
    需要把标签对齐到 subword 上：
      - 首 subword：保留原标签
      - 后续 subword：标记为 -100（CrossEntropy 忽略）
    """
    def __init__(self, data, tokenizer, max_len=128):
        self.data = data
        self.tokenizer = tokenizer
        self.max_len = max_len

    def __len__(self):
        return len(self.data)

    def __getitem__(self, idx):
        tokens, tags = self.data[idx]
        # 添加 [CLS] 和 [SEP]
        tokens_full = ["[CLS]"] + tokens + ["[SEP]"]
        tags_full = ["O"] + tags + ["O"]

        # 分词并对齐
        input_ids = []
        label_ids = []
        for token, tag in zip(tokens_full, tags_full):
            sub_ids = self.tokenizer.encode(token, add_special_tokens=False)
            input_ids.extend(sub_ids)
            # 首 subword 用原标签，其余用 -100
            label_ids.extend([label2id[tag]] + [-100] * (len(sub_ids) - 1))

        # 截断 / 填充
        input_ids = input_ids[: self.max_len]
        label_ids = label_ids[: self.max_len]
        attention_mask = [1] * len(input_ids)

        padding_len = self.max_len - len(input_ids)
        input_ids += [0] * padding_len
        attention_mask += [0] * padding_len
        label_ids += [-100] * padding_len

        return {
            "input_ids": torch.tensor(input_ids),
            "attention_mask": torch.tensor(attention_mask),
            "labels": torch.tensor(label_ids),
        }

# ─── 4. 加载模型 ─────────────────────────────────────────────
model_name = "bert-base-chinese"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForTokenClassification.from_pretrained(
    model_name,
    num_labels=num_labels,
    id2label=id2label,
    label2id=label2id,
)

dataset = NERDataset(train_data, tokenizer)
loader = DataLoader(dataset, batch_size=8, shuffle=True)

# ─── 5. 训练循环 ─────────────────────────────────────────────
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
model.to(device)
optimizer = AdamW(model.parameters(), lr=3e-5)

epochs = 3
for epoch in range(epochs):
    model.train()
    total_loss = 0
    for batch in loader:
        optimizer.zero_grad()
        inputs = {k: v.to(device) for k, v in batch.items()}
        outputs = model(**inputs)
        loss = outputs.loss
        loss.backward()
        optimizer.step()
        total_loss += loss.item()
    print(f"Epoch {epoch+1}/{epochs} - Loss: {total_loss/len(loader):.4f}")

# ─── 6. 推理函数：将 token 预测还原成实体 ──────────────────────
def extract_entities(text, model, tokenizer, id2label):
    """从文本中抽取命名实体"""
    model.eval()
    # 按字切分（中文 NER 常用字级别）
    tokens = list(text)
    inputs = tokenizer(
        tokens,
        is_split_into_words=True,
        return_tensors="pt",
        truncation=True,
        padding=True,
    ).to(device)

    with torch.no_grad():
        logits = model(**inputs).logits
    preds = logits.argmax(dim=2)[0].cpu().numpy()

    # 还原 token 级别的标签
    word_ids = inputs.word_ids(batch_index=0)
    entities = []
    current_entity = None

    for idx, word_id in enumerate(word_ids):
        if word_id is None:
            continue
        label = id2label[preds[idx]]
        if label.startswith("B-"):
            if current_entity:
                entities.append(current_entity)
            current_entity = {"type": label[2:], "start": word_id, "end": word_id}
        elif label.startswith("I-") and current_entity:
            current_entity["end"] = word_id
        else:  # O
            if current_entity:
                entities.append(current_entity)
                current_entity = None
    if current_entity:
        entities.append(current_entity)

    # 还原成文本片段
    results = []
    for ent in entities:
        text_span = "".join(tokens[ent["start"]: ent["end"] + 1])
        results.append({"entity": text_span, "type": ent["type"]})
    return results

# 测试推理
text = "李彦宏是百度的创始人"
print(extract_entities(text, model, tokenizer, id2label))
# 输出示例：
# [{"entity": "李彦宏", "type": "PER"}, {"entity": "百度", "type": "ORG"}]
```

## 五、实体级评估 vs 标签级评估

NER 评估有两种粒度，区别在于"什么算一次预测正确"。

```
┌──────────────────────────────────────────────────────────────┐
│  标签级评估 (Token-Level)                                     │
│                                                              │
│  预测：李/B-PER 彦/I-PER 宏/B-PER   ← "宏"被错误标为 B        │
│  真实：李/B-PER 彦/I-PER 宏/I-PER                            │
│                                                              │
│  标签级：3 个标签中 2 个正确 → 准确率 67%                     │
│  实体级：整个"李彦宏"实体边界错误 → 不算一个正确实体          │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│  实体级评估 (Entity-Level)                                   │
│                                                              │
│  一个实体只有当 类型、起始位置、结束位置 都正确时，           │
│  才算一次正确预测（严格模式）                                  │
└──────────────────────────────────────────────────────────────┘
```

```python
from seqeval.metrics import classification_report

y_true = [["B-PER", "I-PER", "O", "B-ORG"]]
y_pred = [["B-PER", "I-PER", "O", "B-ORG"]]

print(classification_report(y_true, y_pred))
# seqeval 自动做实体级评估，匹配 (类型, 边界) 一致才算正确
```

| 评估方式 | 粒度 | 用途 | 典型工具 |
|---------|------|------|---------|
| 标签级 | token | 分析模型在 token 层面表现 | `sklearn.metrics` |
| 实体级 | entity | 衡量实际信息抽取效果 | `seqeval`、`conlleval` |

**实践建议**：业务评估以实体级为准（下游应用关心的是完整实体），调试分析参考标签级。

## 六、中文 NER 的特殊挑战

### 1. 分词歧义

中文没有空格分隔，"南京市长江大桥"可以切分为：

- 南京市/长江大桥（地名 + 桥名）
- 南京/市长/江大桥（地名 + 职务 + 人名）

切分方式直接影响实体识别结果。

```
错误分词导致实体丢失：

原文：南京市长江大桥
错误：南京  市长  江大桥        ← 把"大桥"错切进人名
     B-LOC B-PER I-PER

正确：南京市  长江大桥          ← 字级别标注能避免这个问题
     B-LOC   B-LOC
```

**对策**：字级别（按字切分）NER 是主流，能避免分词错误传播。

### 2. 嵌套实体

```
北京大学 学生会 主席
└─ ORG ─┘
        └── 嵌套 ORG ──┘
```

"北京大学"是 ORG，"北京大学学生会"也是 ORG，形成嵌套。扁平 BIO 体系无法表达，需要使用嵌套 NER 专用方法（如 biaffine 模型）。

### 3. 实体边界模糊

```
小米手机 vs 小米 公司
└─ 产品?     └─ ORG ┘
```

"小米"在不同上下文中是不同类型的实体，依赖上下文消歧。BERT 的动态表示天然适合这种场景。

### 4. 低资源与领域迁移

| 场景 | 挑战 | 应对策略 |
|------|------|---------|
| 医疗 NER | 标注稀缺 | 预训练 + 少样本学习 |
| 法律 NER | 长文本 | Longformer / 滑动窗口 |
| 社交媒体 | 非规范表达 | 对抗训练 + 数据增广 |

## 七、使用 Pipeline 快速体验

```python
from transformers import pipeline

# HuggingFace 提供的现成 NER pipeline（英文）
ner_pipe = pipeline("ner", model="dslim/bert-base-NER", aggregation_strategy="simple")
result = ner_pipe("Tim Cook is the CEO of Apple Inc. in California.")
for ent in result:
    print(f"{ent['word']:20s} {ent['entity_group']:8s} {ent['score']:.4f}")

# 输出：
# Tim Cook             PER      0.9987
# Apple Inc.           ORG      0.9965
# California           LOC      0.9991
```
