# 模型微调：让预训练模型适配你的任务

预训练模型虽然强大，但它是在通用语料上训练的。当你面对特定领域（如医疗、法律、金融）时，直接使用的效果往往不够好。模型微调（Fine-tuning）就是用你的领域数据进一步训练模型，让它成为该领域的专家。本章将系统讲解微调的原理与实战。

## 一、微调概念

### 1.1 为什么需要微调

```
┌─────────────────────────────────────────────────────────────┐
│                  预训练 → 微调 范式                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  阶段1：预训练（耗资百万，大厂完成）                          │
│  ┌──────────────────────────────────────────────┐           │
│  │  海量文本 (TB级)  ──→  通用预训练模型          │           │
│  │  Wikipedia + 新闻 + 书籍       掌握语言规律    │           │
│  └──────────────────────┬──────────────���────────┘           │
│                         │                                    │
│                         ▼                                    │
│  阶段2：微调（成本低，你可以做）                              │
│  ┌──────────────────────────────────────────────┐           │
│  │  领域数据 (MB~GB)  ──→  专业模型              │           │
│  │  医疗/法律/金融文本           适配特定任务      │           │
│  └──────────────────────────────────────────────┘           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

预训练 vs 微调对比：

| 维度 | 预训练 | 微调 |
|------|--------|------|
| 数据量 | TB 级 | MB ~ GB 级 |
| 训练时间 | 数周 | 数小时 |
| 计算资源 | 数百 GPU | 1~8 GPU |
| 学习率 | 1e-4 ~ 1e-3 | 1e-5 ~ 5e-5 |
| 目标 | 学习通用语言知识 | 适配特定任务 |

### 1.2 微调的本质

微调是在预训练权重的基础上，用较小的学习率继续训练，使模型适应新任务。

```python
# 微调的核心：从预训练权重初始化，而非随机初始化
from transformers import AutoModelForSequenceClassification

# 方式1：随机初始化（从头训练，不推荐）
# model = MyModel()  # 随机权重

# 方式2：加载预训练权重（微调，推荐）
model = AutoModelForSequenceClassification.from_pretrained(
    "bert-base-chinese",
    num_labels=2  # 你的任务类别数
)

# 只修改分类头（随机初始化），骨干网络保留预训练权重
# Warning 会提示：分类头的权重是新初始化的
```

## 二、全参数微调 vs 参数高效微调（PEFT）

### 2.1 两种策略对比

```
全参数微调                          参数高效微调 (PEFT)
┌─────────────────┐                ┌─────────────────┐
│                 │                │  ❄ 冻结原参数    │
│  更新所有参数    │                │  ┌───────────┐  │
│  ████████████   │                │  │ 预训练模型 │  │  (冻结)
│  ████████████   │                │  │ ██████████│  │
│  ████████████   │                │  └─────┬─────┘  │
│                 │                │        │        │
│  需要大显存      │                │   ┌────┴────┐   │
│  存储完整模型    │                │   │ 新增适配 │   │ (训练)
│                 │                │   │ 参数少   │   │
└─────────────────┘                │   └─────────┘   │
                                   └─────────────────┘
   更新100%参数                       更新0.1%~1%参数
```

| 特性 | 全参数微调 | PEFT (如 LoRA) |
|------|-----------|----------------|
| 更新参数量 | 100% | 0.1% ~ 1% |
| 显存占用 | 高 | 低 |
| 训练速度 | 慢 | 快 |
| 存储成本 | 每个任务一份完整模型 | 仅存增量参数（MB级）|
| 效果 | 基准 | 接近全参（部分场景持平）|
| 适用 | 小/中型模型 | 大模型（7B+）|

```python
# 对比参数量
from transformers import AutoModelForSequenceClassification

model = AutoModelForSequenceClassification.from_pretrained("bert-base-chinese")

total_params = sum(p.numel() for p in model.parameters())
trainable_params = sum(p.numel() for p in model.parameters() if p.requires_grad)

print(f"BERT-base 总参数: {total_params / 1e6:.1f}M")
print(f"可训练参数: {trainable_params / 1e6:.1f}M (100%)")

# 全参数微调：全部 102M 参数都更新
# LoRA 微调：冻结 102M，只训练新增的 ~0.8M 参数
```

### 3.2 完整代码示例

```python
"""
中文情感分析微调完整示例
数据集：ChnSentiCorp（中文评论情感语料）
"""
import numpy as np
from datasets import load_dataset, Dataset
from transformers import (
    AutoTokenizer,
    AutoModelForSequenceClassification,
    TrainingArguments,
    Trainer,
    EvalPrediction
)

# ============================================
# 步骤1：准备数据
# ============================================
# 使用 HuggingFace datasets 加载
dataset = load_dataset("seamew/ChnSentiCorp")

print("数据集信息：")
print(f"  训练集: {len(dataset['train'])} 条")
print(f"  验证集: {len(dataset['validation'])} 条")
print(f"  测试集: {len(dataset['test'])} 条")
print(f"\n样本示例：")
print(f"  文本: {dataset['train'][0]['text'][:50]}...")
print(f"  标签: {dataset['train'][0]['label']}")

# 如果用自己的数据，可以这样：
# my_data = Dataset.from_dict({
#     "text": ["文本1", "文本2", ...],
#     "label": [0, 1, ...]
# })

# ============================================
# 步骤2：加载 Tokenizer 并处理数据
# ============================================
model_name = "hfl/chinese-roberta-wwm-ext"
tokenizer = AutoTokenizer.from_pretrained(model_name)

def preprocess_function(examples):
    """批量 tokenize"""
    return tokenizer(
        examples["text"],
        padding="max_length",
        truncation=True,
        max_length=128
    )

# 映射到整个数据集（batched=True 加速）
tokenized_datasets = dataset.map(
    preprocess_function,
    batched=True,
    remove_columns=["text"]  # 删除原始文本列
)

print(f"\nTokenize 后的样本特征: {list(tokenized_datasets['train'].features.keys())}")

# ============================================
# 步骤3：加载模型
# ============================================
model = AutoModelForSequenceClassification.from_pretrained(
    model_name,
    num_labels=2  # 二分类：正面/负面
)

# ============================================
# 步骤4：配置训练参数
# ============================================
training_args = TrainingArguments(
    output_dir="./results",              # 输出目录
    num_train_epochs=3,                  # 训练轮数
    per_device_train_batch_size=32,      # 训练批大小
    per_device_eval_batch_size=64,       # 评估批大小
    warmup_steps=500,                    # 预热步数
    weight_decay=0.01,                   # 权重衰减
    learning_rate=2e-5,                  # 学习率
    logging_dir="./logs",                # 日志目录
    logging_steps=100,                   # ���志间隔
    evaluation_strategy="epoch",         # 每个 epoch 评估
    save_strategy="epoch",               # 每个 epoch 保存
    load_best_model_at_end=True,         # 最终加载最优模型
    metric_for_best_model="accuracy",    # 选择标准
)

# ============================================
# 步骤5：定义评估指标
# ============================================
def compute_metrics(eval_pred: EvalPrediction):
    """计算准确率、F1 等指标"""
    logits, labels = eval_pred
    predictions = np.argmax(logits, axis=-1)
    
    from sklearn.metrics import accuracy_score, f1_score, precision_score, recall_score
    
    return {
        "accuracy": accuracy_score(labels, predictions),
        "f1": f1_score(labels, predictions, average="binary"),
        "precision": precision_score(labels, predictions),
        "recall": recall_score(labels, predictions),
    }

# ============================================
# 步骤6：创建 Trainer 并训练
# ============================================
trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=tokenized_datasets["train"],
    eval_dataset=tokenized_datasets["validation"],
    tokenizer=tokenizer,
    compute_metrics=compute_metrics,
)

# 开始训练
print("开始训练...")
trainer.train()

# ============================================
# 步骤7：评估
# ============================================
print("\n最终测试集评估：")
test_results = trainer.evaluate(tokenized_datasets["test"])
for key, value in test_results.items():
    print(f"  {key}: {value:.4f}")

# ============================================
# 步骤8：保存模型
# ============================================
trainer.save_model("./sentiment-roberta")
print("\n模型已保存到 ./sentiment-roberta")
```

## 四、Token Classification（NER）微调

命名实体识别（NER）是对每个 token 进行分类，而非对整句分类。

```
文本分类：                    Token 分类（NER）：
┌──────────────┐             ┌──────────────────────────────┐
│              │             │ Token    Label               │
│ "文本" → 标签 │             │ 张三  →  B-PER               │
│              │             │ 在    →  O                   │
│ 一个句子一个  │             │ 北京  →  B-LOC               │
│ 分类结果      │             │ 工作  →  O                   │
└──────────────┘             └──────────────────────────────┘
                              一个 token 一个标签
```

```python
"""
NER 微调示例（使用 MSRA NER 数据集）
"""
import numpy as np
from datasets import load_dataset
from transformers import (
    AutoTokenizer,
    AutoModelForTokenClassification,
    TrainingArguments,
    Trainer,
    DataCollatorForTokenClassification
)
from seqeval.metrics import classification_report

# === 步骤1：加载带 BIO 标注的数据 ===
# 数据格式：每行一个字+标签，空行分隔句子
# 例：
#   张 B-PER
#   三 I-PER
#   在 O
#   北 B-LOC
#   京 I-LOC

label_list = ["O", "B-PER", "I-PER", "B-ORG", "I-ORG", "B-LOC", "I-LOC"]
label2id = {label: i for i, label in enumerate(label_list)}
id2label = {i: label for i, label in enumerate(label_list)}

# === 步骤2：加载模型和 tokenizer ===
model_name = "bert-base-chinese"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForTokenClassification.from_pretrained(
    model_name,
    num_labels=len(label_list),
    id2label=id2label,
    label2id=label2id
)

# === 步骤3：tokenize 并对齐标签 ===
def tokenize_and_align_labels(examples):
    """处理 NER 数据：tokenize 后标签需对齐"""
    tokenized_inputs = tokenizer(
        examples["tokens"],
        truncation=True,
        is_split_into_words=True  # 输入已分词
    )
    
    labels = []
    for i, label in enumerate(examples["ner_tags"]):
        word_ids = tokenized_inputs.word_ids(batch_index=i)
        previous_word_idx = None
        label_ids = []
        for word_idx in word_ids:
            if word_idx is None:
                label_ids.append(-100)  # 特殊 token 不计算 loss
            elif word_idx != previous_word_idx:
                label_ids.append(label[word_idx])
            else:
                label_ids.append(-100)  # 子词只标注第一个
            previous_word_idx = word_idx
        labels.append(label_ids)
    
    tokenized_inputs["labels"] = labels
    return tokenized_inputs

# === 步骤4：训练（与分类类似）===
data_collator = DataCollatorForTokenClassification(tokenizer)

training_args = TrainingArguments(
    output_dir="./ner-results",
    evaluation_strategy="epoch",
    learning_rate=2e-5,
    per_device_train_batch_size=16,
    num_train_epochs=3,
    weight_decay=0.01,
)

def compute_metrics(p):
    predictions, labels = p
    predictions = np.argmax(predictions, axis=2)
    
    # 转换为标签字符串
    true_labels = [
        [label_list[l] for l in label if l != -100]
        for label in labels
    ]
    true_predictions = [
        [label_list[p] for (p, l) in zip(prediction, label) if l != -100]
        for prediction, label in zip(predictions, labels)
    ]
    
    report = classification_report(true_labels, true_predictions, output_dict=True)
    return {
        "precision": report["micro avg"]["precision"],
        "recall": report["micro avg"]["recall"],
        "f1": report["micro avg"]["f1-score"],
    }

trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=tokenized_train,  # 假设已处理
    eval_dataset=tokenized_val,
    tokenizer=tokenizer,
    data_collator=data_collator,
    compute_metrics=compute_metrics,

trainer.train()
```

## 五、HuggingFace Trainer API 详解

### 5.1 Trainer 架构

```
┌─────────────────────────────────────────────────┐
│                    Trainer                      │
├─────────────────────────────────────────────────┤
│                                                 │
│  TrainingArguments    ┌─────────────────────┐  │
│  ┌──────────────┐     │   训练循环           │  │
│  │ output_dir   │     │                     │  │
│  │ epochs       │     │  for epoch:         │  │
│  │ batch_size   │────→│    for batch:       │  │
│  │ lr           │     │      forward()      │  │
│  │ warmup       │     │      loss()         │  │
│  │ weight_decay │     │      backward()     │  │
│  │ fp16         │     │      optimizer.step │ │
│  └──────────────┘     │                     │  │
│                       │  自动处理：          │  │
│  DataCollator         │  - 梯度累积          │  │
│  (动态 padding)       │  - 混合精度          │  │
│                       │  - 多卡训练          │  │
│                       │  - 日志/检查点       │  │
│                       └─────────────────────┘  │
└─────────────────────────────────────────────────┘
```

### 5.2 重要参数详解

```python
from transformers import TrainingArguments

training_args = TrainingArguments(
    # === 基本设置 ===
    output_dir="./output",               # 输出目录
    
    # === 训练超参数 ===
    num_train_epochs=3,                  # 训练轮数（与 max_steps 二选一）
    # max_steps=1000,                    # 最大训练步数（覆盖 epochs）
    per_device_train_batch_size=32,      # 每卡训练 batch size
    per_device_eval_batch_size=64,       # 评估 batch size
    gradient_accumulation_steps=1,       # 梯度累积步数（显存不够时增大）
    learning_rate=2e-5,                  # 学习率
    
    # === 正则化 ===
    weight_decay=0.01,                   # 权重衰减
    max_grad_norm=1.0,                   # 梯度裁剪
    label_smoothing_factor=0.0,          # 标签平滑
    
    # === 学习率调度 ===
    warmup_steps=500,                    # 线性预热步数
    # warmup_ratio=0.1,                  # 或按比例（总步数的10%）
    lr_scheduler_type="linear",          # 调度器类型
    
    # === 混合精度 ===
    fp16=False,                          # 启用半精度（加速训练）
    bf16=False,                          # BFloat16（Ampere+ GPU 支持）
    
    # === 评估与保存 ===
    evaluation_strategy="epoch",         # 评估策略: "no"/"steps"/"epoch"
    save_strategy="epoch",               # 保存策略
    save_total_limit=3,                  # 最多保存几个检查点
    load_best_model_at_end=True,         # 训练结束加载最优
    
    # === 日志 ===
    logging_steps=100,                   # 日志间隔
    logging_dir="./logs",                # TensorBoard 日志目录
    report_to="tensorboard",             # 报告到：tensorboard/wandb/mlflow
    
    # === 分布式 ===
    # 自动检测多卡，无需手动设置
)
```

### 5.3 学习率调度策略

```
学习率
  ↑
  │     ╱╲                  ← warmup 阶段线性增长
  │    ╱  ╲
lr │   ╱    ╲____              ← 达到峰值后衰减
  │  ╱          ╲___
  │ ╱                ╲___
  │╱                      ╲___  ← 最终降到接近 0
  └─────────────────────────────→ 训练步数
   warmup_steps        total_steps
```

### 5.4 自定义训练循环

如果 Trainer 不够灵活，可以手动编写训练循环：

```python
"""
手动训练循环示例
适用于需要完全控制的场景
"""
import torch
from torch.optim import AdamW
from transformers import get_linear_schedule_with_warmup
from torch.utils.data import DataLoader

# === 准备 DataLoader ===
train_loader = DataLoader(
    tokenized_datasets["train"],
    batch_size=32,
    shuffle=True,
    collate_fn=data_collator  # 动态 padding
)

# === 优化器与调度器 ===
optimizer = AdamW(model.parameters(), lr=2e-5, weight_decay=0.01)

num_epochs = 3
total_steps = len(train_loader) * num_epochs
scheduler = get_linear_schedule_with_warmup(
    optimizer,
    num_warmup_steps=int(0.1 * total_steps),
    num_training_steps=total_steps
)

# === 训练循环 ===
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
model.to(device)

for epoch in range(num_epochs):
    model.train()
    total_loss = 0
    
    for step, batch in enumerate(train_loader):
        # 数据搬到 GPU
        batch = {k: v.to(device) for k, v in batch.items()}
        
        # 前向传播
        outputs = model(**batch)
        loss = outputs.loss
        
        # 反向传播
        loss.backward()
        total_loss += loss.item()
        
        # 梯度裁剪
        torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
        
        # 更新参数
        optimizer.step()
        scheduler.step()
        optimizer.zero_grad()
        
        # 日志
        if step % 100 == 0:
            print(f"Epoch {epoch+1}, Step {step}/{len(train_loader)}, "
                  f"Loss: {loss.item():.4f}, LR: {scheduler.get_last_lr()[0]:.2e}")
    
    avg_loss = total_loss / len(train_loader)
    print(f"Epoch {epoch+1} 完成，平均 Loss: {avg_loss:.4f}")
```

## 六、微调常见问题与解决方案

### 6.1 过拟合

```
Loss
  ↑
  │ 训练 Loss ────────────────────
  │                              ╱
  │                            ╱
  │                          ╱
  │验证 Loss       ╱─────╲ ╱
  │              ╱       ╳
  │            ╱       ╱ ↑ 过拟合！验证 loss 上升
  │          ╱       ╱
  │        ╱       ╱
  └─────────────────────────────→ Epoch
```

```python
# 过拟合解决方案
```

### 6.2 显存不足（OOM）

```python
# 显存优化策略
memory_solutions = """
1. 减小 batch_size（最直接的方法）
   per_device_train_batch_size=8  # 或更小

2. 梯度累积（等效大 batch）
   per_device_train_batch_size=8
   gradient_accumulation_steps=4  # 等效 batch_size=32

3. 混合精度训练
   fp16=True  # 显存减半，速度提升

4. 梯度检查点（时间换空间）
   model.gradient_checkpointing_enable()

5. 使用 LoRA（下一章详解）
   只训练 0.1% 的参数

6. AdamW 8-bit 优化器
   pip install bitsandbytes
   optimizer = bnb.optim.AdamW8bit(model.parameters(), lr=2e-5)
"""

# 显存估算公式
def estimate_memory(model_size_billion, batch_size, seq_len):
    """粗略估算显存（GB）"""
    param_memory = model_size_billion * 2  # FP16
    grad_memory = model_size_billion * 2   # 梯度
    optim_memory = model_size_billion * 4  # Adam 状态
    activation = batch_size * seq_len * model_size_billion * 0.01
    
    total = param_memory + grad_memory + optim_memory + activation
    return total

print("显存估算（FP32，batch=32，seq=128）：")
for size, name in [(0.1, "BERT-base"), (0.3, "BERT-large"), (7, "LLaMA-7B")]:
    mem = estimate_memory(size, 32, 128)
    print(f"  {name:12s} ({size}B 参数): ~{mem:.1f} GB")
```

```python
# 常见数据问题
data_issues = """
1. 数据不平衡
   解决：class_weight / 过采样 / 欠采样

2. 标签错误
   解决：人工抽检 / 交叉验证 / 主动学习

3. 数据泄露
   解决：严格区分训练/验证/测试集

4. 领域偏移
   解决：继续预训练（MLM）再微调

5. 文本过长
   解决：truncation=True / 滑动窗口 / Longformer
"""

# 数据不平衡处理示例
from sklearn.utils.class_weight import compute_class_weight
import numpy as np
import torch

# 假设标签分布不均
labels = [0] * 900 + [1] * 100  # 9:1 不平衡
class_weights = compute_class_weight(
    "balanced",
    classes=np.unique(labels),
    y=labels
)
print(f"类别权重: {class_weights}")  # [0.67, 6.0]

# 自定义加权 loss
class WeightedTrainer(Trainer):
    def compute_loss(self, model, inputs, return_outputs=False):
        labels = inputs.pop("labels")
        outputs = model(**inputs)
        weight = torch.tensor(class_weights, device=outputs.logits.device)
        loss_fn = torch.nn.CrossEntropyLoss(weight=weight)
        loss = loss_fn(outputs.logits, labels)
        return (loss, outputs) if return_outputs else loss
```

## 小结

| 要点 | 内容 |
|------|------|
| 微调本质 | 预训练权重 + 领域数据 + 小学习率 → 专业模型 |
| 全参 vs PEFT | 全参效果好但贵；PEFT 更新 0.1% 参数，效果接近 |
| 文本分类 | AutoModelForSequenceClassification + Trainer |
| NER 微调 | AutoModelForTokenClassification + 标签对齐 |
| Trainer API | 封装训练循环，支持混合精度/分布式/日志 |
| 学习率 | BERT 微调推荐 2e-5 ~ 5e-5 + warmup |
| 显存优化 | 小 batch + 梯度累积 + fp16 + 梯度检查点 |
| 过拟合 | early stopping + 数据增强 + 正则化 |
