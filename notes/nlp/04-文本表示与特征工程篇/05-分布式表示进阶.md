# 分布式表示进阶：从静态词向量到动态语义表示

Word2Vec、GloVe、FastText 等方法有一个共同的局限：**每个词只有唯一的向量表示，无论它出现在什么上下文中**。这意味着"bank"在"river bank"（河岸）和"bank account"（银行账户）中拥有完全相同的向量。本章将探索从**句向量**到**动态上下文表示**（ELMo、BERT）的演进路径，理解 NLP 表示方法如何一步步逼近人类语言的复杂性。

---

## 一、为什么静态词向量不够？

### 1.1 一词多义问题

```
┌──────────────────────────────────────────────────────────────┐
│              静态词向量 vs 动态词向量                         │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  考虑 "bank" 这个词的不同含义:                               │
│                                                              │
│  句子A: "I sat by the river bank, watching the water."      │
│         bank = 河岸（地理概念）                              │
│                                                              │
│  句子B: "I deposited money in the bank yesterday."          │
│         bank = 银行（金融机构）                              │
│                                                              │
│  句子C: "The plane will bank to the left."                  │
│         bank = 倾斜（航空动作）                              │
│                                                              │
│  ┌─────────────────────────────────────────────────┐         │
│  │  静态表示 (Word2Vec/GloVe/FastText):            │         │
│  │                                                 │         │
│  │     vec("bank") = [0.3, -0.1, 0.5, ...]        │         │
│  │                                                 │         │
│  │     所有句子用同一个向量！                       │         │
│  │     无法区分不同含义 ✗                          │         │
│  └─────────────────────────────────────────────────┘         │
│                                                              │
│  ┌─────────────────────────────────────────────────┐         │
│  │  动态表示 (ELMo/BERT):                          │         │
│  │                                                 │         │
│  │     句子A: vec("bank") = [0.2, 0.8, -0.3, ...]  │  河岸  │
│  │     句子B: vec("bank") = [-0.5, 0.1, 0.9, ...]  │  银行  │
│  │     句子C: vec("bank") = [0.7, -0.4, 0.2, ...]  │  倾斜  │
│  │                                                 │         │
│  │     根据上下文动态计算向量 ✓                    │         │
│  └─────────────────────────────────────────────────┘         │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 二、Doc2Vec：句向量与文档向量

Doc2Vec（Le & Mikolov, 2014）是 Word2Vec 的自然扩展：**不仅学习词向量，还学习文档（或句子）向量**。

### 2.1 两种架构

```
┌──────────────────────────────────────────────────────────────┐
│                  Doc2Vec 的两种架构                          │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  ════════════ PV-DM (Distributed Memory) ════════════       │
│                                                              │
│         文档向量 D                                           │
│           │                                                  │
│     ┌─────┴──────┐                                           │
│     │  上下文词   │                                           │
│     │  w1, w2    │                                           │
│     │  w3...     │                                           │
│     └─────┬──────┘                                           │
│           │                                                  │
│      ┌────┴────┐                                             │
│      │ 求和/拼接│ → 预测 → 中心词 w                           │
│      └─────────┘                                             │
│                                                              │
│  特点: 文档向量 + 上下文词 → 预测中心词                      │
│  类比: 类似 CBOW，但额外注入文档向量                        │
│                                                              │
│  ════════════ PV-DBOW (Distributed Bag of Words) ════       │
│                                                              │
│         文档向量 D                                           │
│           │                                                  │
│           ▼                                                  │
│      ┌──────────┐                                            │
│      │  预测    │ → 文档中随机采样的词                       │
│      └──────────┘                                            │
│                                                              │
│  特点: 仅用文档向量 → 预测文档中的词                         │
│  类比: 类似 Skip-gram，但以文档为单位                        │
│  速度: 比 PV-DM 快（不用考虑上下文词）                      │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 2.2 Doc2Vec 训练代码

```python
from gensim.models.doc2vec import Doc2Vec, TaggedDocument
import jieba

# ---------- Prepare tagged documents ----------
documents = [
    "自然语言处理是人工智能的重要分支",
    "深度学习改变了自然语言处理领域",
    "机器学习和深度学习是AI核心技术",
    "计算机视觉专注于图像和视频理解",
    "语音识别将语音转换为文本",
    "自然语言处理包括分词词性标注",
    "深度学习推动了语音识别的突破",
    "人工智能正在改变我们的生活方式",
    "机器学习算法包括监督学习和无监督学习",
    "神经网络是深度学习的基础架构"
]

# Doc2Vec requires TaggedDocument format
tagged_docs = [
    TaggedDocument(words=list(jieba.cut(doc)), tags=[f"DOC_{i}"])
    for i, doc in enumerate(documents)
]

print(f"文档数: {len(tagged_docs)}")
print(f"示例: {tagged_docs[0]}")

# ---------- Train Doc2Vec (PV-DM) ----------
model_dm = Doc2Vec(
    documents=tagged_docs,
    vector_size=100,        # Document vector dimension
    window=5,               # Context window
    min_count=1,            # Min word frequency
    workers=4,
    dm=1,                   # 1=PV-DM, 0=PV-DBOW
    epochs=100,
    seed=42
)

print(f"\n=== PV-DM 模型 ===")
print(f"文档向量维度: {model_dm.vector_size}")

# ---------- Train Doc2Vec (PV-DBOW) ----------
model_dbow = Doc2Vec(
    documents=tagged_docs,
    vector_size=100,
    window=5,
    min_count=1,
    workers=4,
    dm=0,                   # PV-DBOW mode
    epochs=100,
    seed=42
)

# ---------- Get document vectors ----------
doc_vectors = {tag: model_dm.dv[tag] for tag in model_dm.dv.index_to_key}
print(f"\n文档向量示例 (DOC_0 前10维): {doc_vectors['DOC_0'][:10].round(3)}")

# ---------- Infer vector for new document ----------
new_doc = list(jieba.cut("深度学习是未来的方向"))
new_vec = model_dm.infer_vector(new_doc)
print(f"\n新文档 '深度学习是未来的方向' 的向量（前10维）:")
print(f"  {new_vec[:10].round(3)}")

# ---------- Find most similar documents ----------
similar = model_dm.dv.most_similar("DOC_0", topn=3)
print(f"\n与 DOC_0 最相似的文档:")
for tag, score in similar:
    idx = int(tag.split("_")[1])
    print(f"  {tag} (score={score:.4f}): {documents[idx]}")
```

### 2.3 PV-DM vs PV-DBOW 对比

| 特性 | PV-DM | PV-DBOW |
|------|-------|---------|
| 输入 | 文档向量 + 上下文词 | 仅文档向量 |
| 预测目标 | 中心词 | 文档中的词 |
| 类比 | 类似 CBOW | 类似 Skip-gram |
| 训练速度 | 较慢 | 较快 |
| 质量 | 通常更好 | 略差 |
| 内存 | 较大 | 较小 |
| 推荐 | 质量优先 | 速度优先 |

## 三、Skip-thought Vectors

Skip-thought（Kiros et al., 2015）将 Word2Vec 的 Skip-gram 思想从**词级别**扩展到**句子级别**。

```
┌──────────────────────────────────────────────────────────────┐
│              Skip-thought Vectors 原理                       │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  Word2Vec Skip-gram:                                         │
│     中心词 → 预测 → 周围词                                   │
│                                                              │
│     [w1]  [w2]  [w3]  [w4]  [w5]                            │
│               ↑                                              │
│            预测周围                                          │
│                                                              │
│  Skip-thought:                                               │
│     中心句 → 预测 → 前一句 + 后一句                         │
│                                                              │
│     [句子A]  [句子B]  [句子C]                                │
│               ↑                                              │
│          用编码器编码                                        │
│          用解码器生成前后句                                  │
│                                                              │
│  架构: Encoder-Decoder (GRU/LSTM)                           │
│                                                              │
│     ┌─────────┐                                             │
│     │ Encoder │ ← 句子B: "I went to the store"              │
│     └────┬────┘                                             │
│          │                                                   │
│     ┌────┴────┐                                             │
│     │ 句向量  │                                             │
│     └────┬────┘                                             │
│      ┌───┴───┐                                              │
│      ▼       ▼                                              │
│  ┌──────┐ ┌──────┐                                          │
│  │Decoder│ │Decoder│                                         │
│  └──┬───┘ └──┬───┘                                          │
│     ▼       ▼                                               │
│  句子A   句子C                                               │
│  "I arrived"  "I bought milk"                               │
│                                                              │
│  训练后，Encoder 的隐藏状态就是句向量                        │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

**特点**：能捕捉句子间的逻辑连贯性，但训练成本高（需要编码器-解码器架构和大量文本对）。

## 四、ELMo：上下文词向量

ELMo（Embeddings from Language Models，Peters et al., 2018）是**动态词向量**的里程碑，它基于**双向 LSTM** 生成依赖上下文的词向量。

### 4.1 核心思想

```
┌──────────────────────────────────────────────────────────────┐
│                    ELMo 核心思想                             │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  传统: 每个词有固定向量                                      │
│     vec("play") = [0.1, -0.3, 0.5, ...]  (永远不变)        │
│                                                              │
│  ELMo: 词向量取决于完整句子上下文                            │
│                                                              │
│     句子A: "Let's play outside"                              │
│        vec("play") = f("play", "Let's ___ outside")         │
│                     = [0.8, -0.2, 0.1, ...]  (玩耍)         │
│                                                              │
│     句子B: "Shakespeare's play is famous"                    │
│        vec("play") = f("play", "Shakespeare's ___ is famous")│
│                     = [-0.3, 0.7, 0.4, ...]  (戏剧)         │
│                                                              │
│  实现: 双向 LSTM 语言模型                                    │
│                                                              │
│  ┌──────────────────────────────────────────────┐           │
│  │              ELMo 双向 LSTM                  │           │
│  │                                              │           │
│  │  前向 LSTM:  → → → → → → → → →               │           │
│  │              w1  w2  w3  w4  w5              │           │
│  │              ← ← ← ← ← ← ← ← ←               │           │
│  │  后向 LSTM:                                  │           │
│  │                                              │           │
│  │  ELMo(w_i) = [前向LSTM(w_i),                 │           │
│  │              后向LSTM(w_i),                  │           │
│  │              词嵌入(w_i)]                    │           │
│  │                                              │           │
│  │  融合多层表示 → 上下文相关的词向量           │           │
│  └──────────────────────────────────────────────┘           │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 4.2 ELMo 的多层表示融合

ELMo 不只使用 LSTM 最后一层，而是**加权融合所有层**：

```
┌──────────────────────────────────────────────────────────┐
│               ELMo 多层表示融合                          │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ELMo 对每个词生成 L+2 层表示（L=LSTM层数）:            │
│                                                          │
│  词 "play" 的多层表示:                                   │
│                                                          │
│  Layer 0: 词嵌入层                                      │
│     [0.2, -0.1, 0.5, ...]  ← 静态词向量               │
│                                                          │
│  Layer 1: 第一层双向 LSTM                               │
│     [0.4, 0.3, -0.2, ...]  ← 句法信息                 │
│                                                          │
│  Layer 2: 第二层双向 LSTM                               │
│     [-0.1, 0.6, 0.3, ...]  ← 语义信息                 │
│                                                          │
│  最终 ELMo 向量 = 加权求和:                             │
│                                                          │
│     ELMo = s_0 · Layer0                                 │
│          + s_1 · Layer1                                 │
│          + s_2 · Layer2                                 │
│                                                          │
│  其中 s_0, s_1, s_2 是可学习权重                        │
│  不同任务会自动学到不同的层权重                         │
│                                                          │
│  发现:                                                   │
│     • 底层（Layer 0）适合: 词性标注、句法分析           │
│     • 中层（Layer 1）适合: 句法任务                    │
│     • 高层（Layer 2）适合: 语义任务、情感分析          │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

## 五、Universal Sentence Encoder

Google 的 Universal Sentence Encoder（USE，Cer et al., 2018）专注于生成高质量的**句子级向量**。

```
┌──────────────────────────────────────────────────────────────┐
│            Universal Sentence Encoder (USE)                  │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  目标: 将任意长度的句子编码为固定维度的向量                  │
│                                                              │
│  两种架构:                                                   │
│                                                              │
│  ① Transformer 版（DAN replaced by Transformer）            │
│     输入: 词嵌入序列                                        │
│     ↓                                                        │
│     Transformer 编码器（多头注意力）                        │
│     ↓                                                        │
│     句向量 (512维)                                          │
│                                                              │
│     优点: 精度高、捕捉长距离依赖                            │
│     缺点: 计算量大                                          │
│                                                              │
│  ② DAN 版 (Deep Averaging Network)                          │
│     输入: 词嵌入序列                                        │
│     ↓                                                        │
│     求平均 (Average)                                       │
│     ↓                                                        │
│     多层前馈网络 (MLP)                                     │
│     ↓                                                        │
│     句向量 (512维)                                          │
│                                                              │
│     优点: 计算快、简单                                      │
│     缺点: 精度略低                                          │
│                                                              │
│  输出: 512维句向量                                          │
│  用途: 句子相似度、聚类、检索、分类                        │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 六、sentence-transformers 库

`sentence-transformers`（Reimers & Gurevych, 2019）是基于 BERT/RoBERTa 的句子嵌入库，提供了更易用的接口和更优质的句向量。

### 6.1 为什么需要 Sentence-BERT？

```
┌──────────────────────────────────────────────────────────────┐
│           原始 BERT vs Sentence-BERT 用于句子相似度          │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  原始 BERT 的问题:                                           │
│                                                              │
│     计算两个句子的相似度，需要:                              │
│     Sentence A + Sentence B → [CLS] → similarity             │
│                                                              │
│     问题: 每对句子都要跑一次 BERT                            │
│     10,000 个句子找相似对 → 需要 C(10000,2) ≈ 5000万次!     │
│     → 计算上不可行                                           │
│                                                              │
│  Sentence-BERT 的改进:                                       │
│                                                              │
│     Sentence A → BERT → Pooling → vec_A                     │
│     Sentence B → BERT → Pooling → vec_B                     │
│                                                              │
│     sim(A, B) = cosine(vec_A, vec_B)                        │
│                                                              │
│     优势:                                                    │
│     ✓ 每个句子只需编码一次                                   │
│     ✓ 10,000个句子只需编码10,000次                          │
│     ✓ 向量可预先计算、存储、索引                            │
│     ✓ 适合大规模检索                                        │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 6.2 使用示例

```python
# pip install sentence-transformers
from sentence_transformers import SentenceTransformer, util
import numpy as np

# ---------- Load pre-trained model ----------
# Models: https://www.sbert.net/docs/pretrained_models.html
# Multilingual: 'paraphrase-multilingual-MiniLM-L12-v2' (supports Chinese)
model = SentenceTransformer('paraphrase-multilingual-MiniLM-L12-v2')

# ---------- Encode sentences ----------
sentences = [
    "自然语言处理是人工智能的重要分支",
    "NLP是AI的核心领域",
    "今天的天气非常不错",
    "深度学习推动了语言理解的发展",
    "人工智能正在改变世界",
    "神经网络需要大量数据训练"
]

# Get sentence embeddings (384-dimensional)
embeddings = model.encode(sentences)

print(f"=== Sentence-Transformer 结果 ===")
print(f"句子数: {len(sentences)}")
print(f"向量维度: {embeddings.shape[1]}")  # 384

# ---------- Compute similarity ----------
similarities = util.cos_sim(embeddings, embeddings)

print("\n句子相似度矩阵:")
print(f"{'':>4}", end="")
for j in range(len(sentences)):
    print(f"  S{j}", end="")
print()

for i in range(len(sentences)):
    print(f"S{i}  ", end="")
    for j in range(len(sentences)):
        print(f"{similarities[i][j]:.2f}", end="  ")
    print(f"  {sentences[i][:15]}...")

# ---------- Semantic search ----------
query = "AI和语言处理"
query_vec = model.encode([query])[0]
scores = util.cos_sim(query_vec, embeddings)[0]

print(f"\n语义搜索: '{query}'")
ranked = sorted(enumerate(scores), key=lambda x: -x[1])
for rank, (idx, score) in enumerate(ranked):
    print(f"  排名{rank+1} (score={score:.4f}): {sentences[idx]}")

# ---------- Clustering ----------
from sklearn.cluster import KMeans
import numpy as np

n_clusters = 3
kmeans = KMeans(n_clusters=n_clusters, random_state=42)
cluster_labels = kmeans.fit_predict(embeddings)

print(f"\n聚类结果 ({n_clusters} 个簇):")
for c in range(n_clusters):
    members = [sentences[i] for i in range(len(sentences)) if cluster_labels[i] == c]
    print(f"  簇{c+1}: {members}")
```

## 小结

| 要点 | 内容 |
|------|------|
| **静态词向量局限** | 无法处理一词多义，"bank"在所有语境下向量相同 |
| **Doc2Vec** | PV-DM（文档向量+上下文→中心词）、PV-DBOW（文档向量→词） |
| **Skip-thought** | 用句子预测前后句，编码器输出即句向量 |
| **ELMo 核心** | 双向LSTM生成上下文相关词向量，加权融合多层表示 |
| **ELMo 优势** | 同一词在不同句子中得到不同向量，解决多义问题 |
| **Universal Sentence Encoder** | 专注句向量，Transformer或DAN架构，512维输出 |
| **Sentence-BERT** | BERT + 池化策略，每个句子独立编码，适合大规模检索 |
| **动态 vs 静态** | 动态表示（ELMo/BERT）能根据上下文调整词向量 |
| **演进方向** | 离散→稠密、静态→动态、词级→句级、单义→多义 |
| **实践建议** | 句子任务用Sentence-BERT，复杂理解用BERT，快速基线用TF-IDF |
| **下一步** | 进入深度学习篇，系统学习 RNN、CNN、Attention 等核心架构 |
