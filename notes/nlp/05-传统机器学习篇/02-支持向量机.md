# 支持向量机

支持向量机（Support Vector Machine, SVM）是传统机器学习中最强大的分类算法之一。在深度学习崛起之前，SVM在文本分类领域长期占据**霸主地位**。即使在今天，在中小规模文本分类任务上，线性SVM依然是极具竞争力的基线模型——它往往能在极少调参的情况下接近甚至超越复杂模型的表现。

---

## 一、SVM核心思想：最大间隔超平面

### 1.1 直觉理解

想象桌面上有两种颜色的球（红球和蓝球），你要放一根棍子把它们分开。棍子有很多种放法，但最优的放法是让棍子**离两边最近的球都尽可能远**——这就是SVM的核心思想。

```
        红球                  蓝球
         ○                     ●
            ○              ●
                ○        ●
    ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─    ← 最大间隔超平面 w·x + b = 0
                ○        ●
            ○              ●
         ○                     ●

    |←── 间隔 (margin) ──→|
    |← 最小距离 →|← 最小距离 →|
```

### 1.2 数学描述

给定训练集 $\{(x_i, y_i)\}$，其中 $y_i \in \{+1, -1\}$，SVM要找到一个超平面 $w \cdot x + b = 0$，使得：

$$\max_{w,b} \frac{2}{\|w\|} \quad \text{s.t.} \quad y_i(w \cdot x_i + b) \geq 1, \quad \forall i$$

等价于：

$$\min_{w,b} \frac{1}{2}\|w\|^2 \quad \text{s.t.} \quad y_i(w \cdot x_i + b) \geq 1$$

### 1.3 支持向量

距离超平面最近的那些数据点（即满足 $y_i(w \cdot x_i + b) = 1$ 的点）称为**支持向量**。它们"支撑"起了决策边界——删除其他非支持向量的点不影响模型，这就是SVM名字的由来。

```
              决策边界
       ┃         │         ┃
   ○   ┃   ○     │   ●     ┃   ●
       ┃         │         ┃
       ┃         │         ┃
   ○   ┃   ○     │         ┃   ●
       ┃         │         ┃
       ↑         ↑         ↑
     SV(○)   超平面       SV(●)
     支持向量            支持向量
```

## 二、线性SVM vs 核SVM

### 2.1 线性不可分问题

很多时候数据不是线性可分的。此时有两种策略：

1. **软间隔（Soft Margin）**：允许一些点分错，引入松弛变量和惩罚参数C
2. **核技巧（Kernel Trick）**：将数据映射到高维空间，使其在高维空间线性可分

### 2.2 核技巧直观理解

```
原始空间（线性不可分）        高维空间（线性可分）
    ●  ○                         ○ ○ ○
  ● ┌───┐ ○                    ┌─────────┐
    │ ○  │                     │  ○○○    │     ← 在3D空间中
  ● │ ●● │ ○                   │ ●●●●●   │        可以用平面分开
    │○○○│                      │         │
  ● └───┘ ○                    └─────────┘
    ● ○                        ● ● ● ● ●

   核函数 K(x,y) = φ(x)·φ(y)  完成了升维映射
```

### 2.3 软间隔的目标函数

$$\min_{w,b,\xi} \frac{1}{2}\|w\|^2 + C \sum_{i=1}^{n} \xi_i$$

其中 $\xi_i \geq 0$ 是松弛变量，允许第 $i$ 个样本分错或落在间隔内；$C$ 是惩罚参数，控制对误分类的容忍度：

- **C大**：对误分类惩罚大 → 容易过拟合（低偏差、高方差）
- **C小**：对误分类容忍高 → 容易欠拟合（高偏差、低方差）

---

## 三、常用核函数

| 核函数 | 公式 | 特点 | 适用场景 |
|--------|------|------|----------|
| 线性核 | $K(x,y) = x \cdot y$ | 速度快，无参数 | 文本分类（高维稀疏） |
| RBF核 | $K(x,y) = e^{-\gamma\|x-y\|^2}$ | 能处理非线性，需调γ | 低维非线性数据 |
| 多项式核 | $K(x,y) = (\gamma x \cdot y + r)^d$ | 灵活但参数多 | 图像、一般分类 |
| Sigmoid核 | $K(x,y) = \tanh(\gamma x \cdot y + r)$ | 类似神经网络 | 特定场景 |

### 3.1 为什么文本分类常用线性核？

文本特征通常是**高维稀疏**的（词表上万维但每篇文档只有几百个词）。在高维空间中，数据往往已经近似线性可分：

```
高维稀疏文本空间（线性可分）
         ●
    ●         ●
       ●  │     ●
 ─ ─ ─ ─ ─ ┼ ─ ─ ─ ─ ─    ← 线性超平面就能分得很好
       ○  │     ○
    ○         ○
         ○
```

此外，RBF核在高维稀疏空间中计算代价巨大且容易过拟合，线性核反而更快更好。

## 四、文本分类中为什么LinearSVC效果好

### 4.1 实证经验

在多项NLP基准测试中，LinearSVC的表现规律如下：

| 模型 | 训练速度 | 预测速度 | 效果 | 参数调优难度 |
|------|----------|----------|------|-------------|
| MultinomialNB | 极快 | 极快 | 良好 | 低 |
| LinearSVC | 快 | 快 | **优秀** | 低 |
| SVC(rbf) | 慢 | 慢 | 一般 | 高 |
| Logistic Regression | 快 | 快 | 优秀 | 低 |
| 深度学习 | 极慢 | 中 | 最优（大数据时） | 高 |

### 4.2 sklearn中LinearSVC vs SVC(kernel='linear')

```python
# 推荐：速度快（基于liblinear）
from sklearn.svm import LinearSVC
clf = LinearSVC(C=1.0)

# 不推荐（大数据集极慢，基于libsvm）
from sklearn.svm import SVC
clf = SVC(kernel='linear', C=1.0)
```

## 五、sklearn LinearSVC / SVC 用法

### 5.1 LinearSVC 示例

```python
from sklearn.svm import LinearSVC
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.pipeline import Pipeline
from sklearn.model_selection import GridSearchCV
from sklearn.metrics import classification_report

# 模拟中文新闻数据
texts = [
    "中国 股市 今日 大幅 上涨 创业板 涨幅",
    "央行 发布 货币政策 利率 调整 公告",
    "基金 重仓 股 暴跌 投资者 损失",
    "华为 发布 新款 手机 芯片 性能",
    "苹果 新品 发布会 显示屏 处理器",
    "谷歌 推出 AI 大模型 搜索",
    "皇马 签约 球星 转会费 合同",
    "世界杯 决赛 球队 晋级 冠军",
    "NBA 季后赛 球员 受伤 比赛",
    "中科院 研究 人员 发现 新 材料",
    "NASA 发射 火箭 太空 探测",
    "量子 计算 突破 实验 结果",
]
labels = ["财经", "财经", "财经", "科技", "科技", "科技",
          "体育", "体育", "体育", "科学", "科学", "科学"]

# Pipeline：TF-IDF + LinearSVC
pipeline = Pipeline([
    ('tfidf', TfidfVectorizer()),
    ('clf', LinearSVC(C=1.0, max_iter=10000)),
])

# 网格搜索调C参数
param_grid = {'clf__C': [0.1, 0.5, 1.0, 2.0, 5.0]}
grid = GridSearchCV(pipeline, param_grid, cv=3, scoring='f1_macro')
grid.fit(texts, labels)

print(f"最佳参数: {grid.best_params_}")
print(f"最佳F1: {grid.best_score_:.4f}")

# 预测新文本
test = ["央行 利率 下调 股市 利好", "新款 手机 拍照 测评"]
for t, pred in zip(test, grid.predict(test)):
    print(f"[{pred}] {t}")
```

### 5.2 SVC 与核函数对比

```python
from sklearn.svm import SVC

# 线性核（适合文本）
svc_linear = SVC(kernel='linear', C=1.0)

# RBF核（需要调γ和C）
svc_rbf = SVC(kernel='rbf', C=1.0, gamma='scale')

# 多项式核
svc_poly = SVC(kernel='poly', degree=3, C=1.0)
```

## 六、完整文本分类Pipeline

下面展示一个完整的、可直接复用的SVM文本分类pipeline：

```python
from sklearn.svm import LinearSVC
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.pipeline import Pipeline
from sklearn.model_selection import train_test_split, cross_val_score
from sklearn.metrics import classification_report, confusion_matrix
import numpy as np

# ===== 1. 数据准备 =====
# （实际应用中替换为真实数据集）
documents = [
    "这家 酒店 服务 很好 房间 干净 推荐",
    "环境 不错 早餐 丰富 位置 方便",
    "酒店 设施 陈旧 卫生 差 失望",
    "隔音 效果 差 服务 一般",
    "手机 屏幕 清晰 拍照 好看 流畅",
    "性价比 很高 电池 续航 满意",
    "手机 卡顿 发热 严重 退货",
    "屏幕 破损 质量 差 假货",
] * 10  # 复制以增加样本量

labels = (["好评", "好评", "差评", "差评",
           "好评", "好评", "差评", "差评"]) * 10

X_train, X_test, y_train, y_test = train_test_split(
    documents, labels, test_size=0.2, random_state=42, stratify=labels
)

# ===== 2. 构建Pipeline =====
svm_clf = Pipeline([
    ('tfidf', TfidfVectorizer(
        max_features=5000,    # 最大特征数
        ngram_range=(1, 2),   # 使用unigram + bigram
        min_df=2,             # 最小��档频率
    )),
    ('clf', LinearSVC(C=1.0, max_iter=10000)),
])

# ===== 3. 交叉验证评估 =====
scores = cross_val_score(svm_clf, X_train, y_train, cv=5, scoring='f1_macro')
print(f"交叉验证 F1: {scores.mean():.4f} ± {scores.std():.4f}")

# ===== 4. 训练与测试 =====
svm_clf.fit(X_train, y_train)
y_pred = svm_clf.predict(X_test)

print("\n分类报告:")
print(classification_report(y_test, y_pred))

# ===== 5. 查看最重要的特征 =====
feature_names = svm_clf.named_steps['tfidf'].get_feature_names_out()
coef = svm_clf.named_steps['clf'].coef_

for i, label in enumerate(svm_clf.named_steps['clf'].classes_):
    top_indices = np.argsort(coef[i])[-5:][::-1]
    print(f"\n[{label}] 最重要特征:")
    for idx in top_indices:
        print(f"  {feature_names[idx]:>10s}  (权重: {coef[i][idx]:.4f})")
```

## 七、SVM vs 朴素贝叶斯对比

| 对比维度 | 朴素贝叶斯 (NB) | 支持向量机 (SVM) |
|----------|-----------------|-------------------|
| 模型类型 | 生成式 | 判别式 |
| 优化目标 | 最大化联合概率 $P(x,y)$ | 最大化分类间隔 |
| 特征独立假设 | 是（朴素假设） | 否 |
| 训练速度 | 极快 | 中等（大数据下较快） |
| 分类效果 | 良好（基线） | **优秀** |
| 对数据量的要求 | 少量即可 | 中等以上更优 |
| 可解释性 | 高（概率可查） | 中（权重向量可查） |
| 概率输出 | 天然输出概率 | 需Platt缩放等技巧 |
| 典型应用 | 垃圾邮件过滤 | 新闻分类、情感分析 |

## 八、小结

| 要点 | 内容 |
|------|------|
| 核心思想 | 寻找最大间隔超平面，使分类边界离两类数据都尽可能远 |
| 支持向量 | 距离超平面最近的点，它们决定了决策边界 |
| 软间隔 | 引入松弛变量 $\xi$ 和惩罚参数C，允许少量误分类 |
| 核函数 | 将数据映射到高维空间使其线性可分 |
| 文本分类首选 | LinearSVC——高维稀疏文本空间中线性核最快最好 |
| C参数 | 大C=严格分类（易过拟合）；小C=容忍错误（易欠拟合） |
| sklearn 类 | `LinearSVC`（推荐）或 `SVC(kernel='linear')` |
| 对比NB | SVM效果通常更优，NB速度更快且天然输出概率 |
