# 条件随机场

条件随机场（Conditional Random Field, CRF）是序列标注任务中**最经典的判别式模型**。它解决了HMM作为生成式模型的诸多局限，能够灵活地融合丰富的、重叠的特征。在深度学习时代之前，CRF是命名实体识别（NER）、词性标注等任务的**state-of-the-art**方法。即便在今天，BiLSTM-CRF、Transformer-CRF等模型仍将CRF作为解码层——CRF的标签转移矩阵依然是序列标注的利器。

---

## 一、判别式模型 vs 生成式模型

### 1.1 核心区别

```
生成式模型（HMM）:               判别式模型（CRF）:
  建模 P(X, Y)                     建模 P(Y | X)
  即"观测和标签的联合分布"           即"给定观测，标签的条件分布"

  P(Y|X) = P(X,Y) / P(X)          直接对 P(Y|X) 建模
         = P(X|Y)·P(Y) / P(X)       不需要建模 P(X)

  需要建模:                        需要建模:
    P(X|Y)  ← 发射概率                P(Y|X)  ← 直接建模
    P(Y)    ← 先验                    不需要 P(X|Y)
```

### 1.2 类比理解

| 类比 | 生成式 | 判别式 |
|------|--------|--------|
| 学画画 | 学习画猫和画狗，再看新图最像哪个 | 直接学习"猫和狗的区别在哪里" |
| 分类 | 先学每类的分布，再比较 | 直接学分类边界 |
| 公式 | $P(Y\|X) = \frac{P(X\|Y)P(Y)}{P(X)}$ | 直接学 $P(Y\|X)$ |
| 典型模型 | 朴素贝叶斯、HMM | 逻辑回归、SVM、CRF |

### 1.3 为什么NLP序列标注偏好判别式？

在序列标注中，观测序列 $X$（句子中的词）包含丰富的特征：词本身、前后缀、大小写、上下文词、词性等。判别式模型**不需要建模 $P(X)$**，可以直接利用这些重叠特征。

## 二、线性链CRF的结构

### 2.1 从一般CRF到线性链CRF

一般CRF可以定义在任意图结构上，但NLP中最常用的是**线性链CRF（Linear-chain CRF）**，其结构与序列的线性顺序对应。

```
一般CRF（无向图）:              线性链CRF:

    Y₁ ── Y₂ ── Y₃                Y₁ ── Y₂ ── Y₃
     │  ╲  ╱  │  ╲                 │     │     │
     │   ╳    │   ╲                │     │     │
     │  ╱  ╲  │    ╲               │     │     │
    X₁    X₂  X₃   X₄             X₁    X₂    X₃

  （任意依赖，复杂）             （只相邻Y依赖，高效）
```

### 2.2 线性链CRF的数学定义

给定观测序列 $X = x_1, ..., x_n$，标签序列 $Y = y_1, ..., y_n$ 的条件概率：

$$P(Y|X) = \frac{1}{Z(X)} \exp \left( \sum_{t=1}^{n} \sum_{k} \lambda_k f_k(y_{t-1}, y_t, X, t) \right)$$

其中：
- $f_k(y_{t-1}, y_t, X, t)$ 是**特征函数**
- $\lambda_k$ 是特征函数对应的权重（模型学习的就是这些权重）
- $Z(X) = \sum_{Y'} \exp(\sum_t \sum_k \lambda_k f_k(y'_{t-1}, y'_t, X, t))$ 是**配分函数**（归一化项）

### 2.3 与逻辑回归的关系

```
逻辑回归 = CRF的"序列版"

逻辑回归:  P(y|x) = exp(w·f(x,y)) / Σ_y' exp(w·f(x,y'))
                                         ↑ 对所有类别归一化

线性链CRF: P(Y|X) = exp(Σ λₖfₖ) / Σ_Y' exp(Σ λₖfₖ)
                                      ↑ 对所有可能的标签序列归一化

         CRF本质上就是对整个标签序列做"全局归一化"的逻辑回归
```

## 三、特征函数设计

特征函数是CRF的核心——它决定了模型能利用哪些信息。每个特征函数接收四个参数：$(y_{t-1}, y_t, X, t)$，即"前一个标签、当前标签、整个观测序列、当前位置"。

### 3.1 两类特征函数

```python
# ========== 转移特征（依赖标签间的关系）==========
def f_transition(prev_tag, curr_tag, X, t):
    """前一个标签 → 当前标签的转移"""
    if prev_tag == "B-PER" and curr_tag == "I-PER":
        return 1    # 人名实体内部 B→I 转移
    return 0

def f_no_O_to_I(prev_tag, curr_tag, X, t):
    """约束：O不能直接到I-*（非法转移）"""
    if prev_tag == "O" and curr_tag.startswith("I-"):
        return 1    # 惩罚这种非法序列
    return 0

# ========== 状态特征（依赖当前标签和观测）==========
def f_is_capital(curr_tag, X, t):
    """当前词首字母大写 → 可能是人名开头"""
    word = X[t]
    if curr_tag == "B-PER" and word[0].isupper():
        return 1
    return 0

def f_context_word(curr_tag, X, t):
    """前一个词是'总统' → 当前词可能是人名"""
    if t > 0 and X[t-1] == "总统" and curr_tag == "B-PER":
        return 1
    return 0

def f_suffix(curr_tag, X, t):
    """以'公司'结尾 → 可能是组织名"""
    word = X[t]
    if curr_tag == "B-ORG" and word.endswith("公司"):
        return 1
    return 0
```

### 3.2 特征模板（实战中常用的套路）

在实际工程中，通常使用**特征模板**自动生成大量特征函数：

```
BOS/EOS            句首/句末标记
wₜ                 当前词
wₜ₋₁, wₜ₊₁        前一个/后一个词
wₜ₋₂, wₜ₊₂        前两个/后两个词
wₜ₋₁+wₜ           前词+当前词（bigram）
wₜ+wₜ₊₁           当前词+后词（bigram）
前缀/后缀          首字、尾字
是否数字/大写/标点  词形特征
```

### 3.3 ASCII图解特征函数如何工作

```
  句子:  总统   访问   了    北京
  位置:   t=0   t=1   t=2   t=3
          │     │     │     │
  ┌───────┼─────┼─────┼─────┤
  │  特征函数表 (在t=1处激活的)        │
  ├───────────────────────────────────┤
  │  f1: wₜ = "访问"                   │
  │  f2: wₜ₋₁ = "总统"                │
  │  f3: wₜ₊₁ = "了"                 │
  │  f4: wₜ₋₁+wₜ = "总统访问"         │
  │  f5: 前缀="讠"                     │
  │  f6: 转移 B-O → B-O               │
  │  ...                              │
  └───────────────────────────────────┘
          │
          ▼
  每个 fₖ 有权重 λₖ，加权求和 → 决定每个标签序列的得分
```

## 四、CRF vs HMM vs Softmax对比

| 对比维度 | HMM | 逐位置Softmax | CRF |
|----------|-----|---------------|-----|
| 模型类型 | 生成式 $P(X,Y)$ | 判别式 $P(Y_t\|X)$ | 判别式 $P(Y\|X)$ |
| 归一化方式 | 局部 | 局部（每个位置独立） | **全局**（整个序列） |
| 能否利用重叠特征 | 否 | 是 | 是 |
| 标签间依赖 | 转移概率 $a_{ij}$ | **不建模** | 转移特征 $\lambda_k$ |
| 非法标签序列 | 可能输出 | **可能输出** | 可以约束禁止 |
| 全局最优序列 | 维特比 | 逐位置argmax | 维特比（线性链） |
| 训练方式 | 计数+平滑 | 梯度下降 | 梯度下降（LBFGS/L-BFGS） |

### 4.1 为什么逐位置Softmax不好？

```
逐位置Softmax:  每个位置独立分类，不管整体

  句子:   张三  是  好  人
  标签:   O    B  I   O    ← B后面可以接I

  但也可能输出:
  句子:   张三  是  好  人
  标签:   O    I  O   I    ← 不合法！O后面直接到I

CRF:  全局归一化，通过转移特征约束:
  特征 f(O→I) = -∞  ← 禁止这种非法转移
  → 保证输出合法的标签序列
```

## 五、sklearn-crfsuite 完整NER代码

### 5.1 安装


### 5.2 完整NER示例

```python
import sklearn_crfsuite
from sklearn_crfsuite import metrics

# ========== 1. 准备数据（已标注的NER语料）==========
# 每个词标注为: 词, 词性, 实体标签(BIO体系)
train_sents = [
    [('张', 'NR', 'B-PER'), ('三', 'NR', 'I-PER'),
     ('在', 'P', 'O'),
     ('北京', 'NS', 'B-LOC'),
     ('工作', 'VV', 'O')],
    [('李', 'NR', 'B-PER'), ('四', 'NR', 'I-PER'),
     ('加入', 'VV', 'O'),
     ('腾讯', 'NN', 'B-ORG'),
     ('公司', 'NN', 'I-ORG')],
    [('王', 'NR', 'B-PER'), ('五', 'NR', 'I-PER'),
     ('访问', 'VV', 'O'),
     ('上海', 'NS', 'B-LOC')],
    [('清华', 'NN', 'B-ORG'), ('大学', 'NN', 'I-ORG'),
     ('在', 'P', 'O'),
     ('中国', 'NR', 'B-LOC')],
]

# ========== 2. 特征提取 ==========
def word2features(sent, i):
    """提取位置i处的词特征"""
    word = sent[i][0]
    postag = sent[i][1]

    features = {
        'bias': 1.0,
        'word': word,
        'word.lower': word.lower(),
        'postag': postag,
    }
    # 前一个词的特征
    if i > 0:
        prev_word = sent[i-1][0]
        features.update({
            '-1:word': prev_word,
            '-1:postag': sent[i-1][1],
        })
    else:
        features['BOS'] = True   # 句首标记

    # 后一个词的特征
    if i < len(sent) - 1:
        next_word = sent[i+1][0]
        features.update({
            '+1:word': next_word,
            '+1:postag': sent[i+1][1],
        })
    else:
        features['EOS'] = True   # 句末标记

    return features

def sent2features(sent):
    return [word2features(sent, i) for i in range(len(sent))]

def sent2labels(sent):
    return [label for _, _, label in sent]

# 提取特征
X_train = [sent2features(s) for s in train_sents]
y_train = [sent2labels(s) for s in train_sents]

# ========== 3. 训练CRF模型 ==========
crf = sklearn_crfsuite.CRF(
    algorithm='lbfgs',
    c1=0.1,           # L1正则化系数
    c2=0.1,           # L2正则化系数
    max_iterations=100,
    all_possible_transitions=True,   # 允许所有可能的转移
)
crf.fit(X_train, y_train)

# ========== 4. 预测 ==========
test_sent = [('赵', 'NR'), ('六', 'NR'), ('去', 'VV'),
             ('深圳', 'NS'), ('出差', 'VV')]
X_test = [sent2features([(w, p, 'O') for w, p in test_sent])]
y_pred = crf.predict(X_test)[0]

print("NER结果:")
for (word, _), tag in zip(test_sent, y_pred):
    print(f"  {word} → {tag}")

# ========== 5. 评估 ==========
# labels = list(crf.classes_)
# metrics.flat_f1_score(y_test, y_pred, average='weighted', labels=labels)

# ========== 6. 查看学到的转移权重 ==========
# 从CRF模型中提取标签转移矩阵
from collections import Counter
transitions = Counter()
for (from_tag, to_tag), weight in crf.transition_features_.items():
    transitions[(from_tag, to_tag)] = weight

print("\n标签转移权重（Top 10）:")
for (f, t), w in transitions.most_common(10):
    print(f"  {f} → {t}: {w:.4f}")
```

## 六、CRF的优缺点

### 6.1 优点

- **全局最优**：整个序列联合解码，不会产生局部最优问题
- **特征灵活**：可使用任意数量、重叠的特征（词、词性、前后缀、上下文等）
- **标签约束**：通过转移特征可以强制约束合法的标签序列
- **效果优秀**：在有标注数据的序列标注任务上，CRF长期是SOTA
- **可与深度学习结合**：BiLSTM-CRF、Transformer-CRF 是经典架构

### 6.2 缺点

- **训练速度慢**：需要计算配分函数 $Z(X)$，涉及前向传播，复杂度高
- **特征工程依赖**：效果高度依赖人工设计的特征模板
- **推理速度中等**：维特比解码复杂度 $O(n \cdot |S|^2)$
- **无法端到端**：不像神经网络可以自动学习特征
- **长距离依赖弱**：线性链结构只建模相邻标签间的依赖

### 6.3 CRF的演进路线

```
HMM（生成式）
    │
    │ 解决: 特征灵活性问题
    ▼
CRF（判别式，全局归一化）
    │
    │ 解决: 人工特征工程的瓶颈
    ▼
BiLSTM-CRF（神经网络+CRF解码层）
    │
    │ 解决: 更强特征表示 + 并行效率
    ▼
Transformer-CRF（注意力机制 + CRF）
```

## 七、小结

| 要点 | 内容 |
|------|------|
| 核心思想 | 判别式模型，直接建模 $P(Y\|X)$，全局归一化 |
| vs HMM | CRF是判别式，HMM是生成式；CRF能用重叠特征，HMM不能 |
| vs Softmax | Softmax逐位置分类，CRF全局联合解码，能约束标签合法性 |
| 特征函数 | 转移特征 $f(y_{t-1}, y_t)$ + 状态特征 $f(y_t, X, t)$ |
| 配分函数 | $Z(X)$ 对所有可能序列归一化，是训练的主要计算开销 |
| 解码算法 | 维特比算法（与HMM相同） |
| 典型应用 | 命名实体识别（NER）、词性标注、中文分词 |
| 深度学习结合 | BiLSTM-CRF / Transformer-CRF 仍是NER的主流架构 |
