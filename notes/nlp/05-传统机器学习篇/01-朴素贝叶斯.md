# 朴素贝叶斯

朴素贝叶斯（Naive Bayes）是一族基于贝叶斯定理的概率分类器，在NLP领域有着广泛而深远的应用。尽管"朴素"二字暗示了它对现实世界的某种简化假设，但正是这种简化让它在文本分类任务中表现出"简单、高效、出奇好用"的特点。从垃圾邮件过滤到情感分析，朴素贝叶斯往往是工程师们首选的**基线模型（baseline）**。

---

## 一、贝叶斯定理回顾

贝叶斯定理是整个朴素贝叶斯分类器的数学基石，它描述了在观察到新证据后，如何更新我们对某个假设的信念。

### 1.1 公式

$$P(A|B) = \frac{P(B|A) \cdot P(A)}{P(B)}$$

各部分含义：

| 符号 | 名称 | 含义 |
|------|------|------|
| P(A) | 先验概率 | 在观察到B之前，A发生的概率 |
| P(A\|B) | 后验概率 | 在观察到B之后，A发生的概率 |
| P(B\|A) | 似然 | 在A成立时，观察到B的概率 |
| P(B) | 证据因子 | B出现的边际概率（归一化常数） |

### 1.2 一个直观的例子

> 假设某疾病发病率为1%（先验）。检测仪器对该病的检出率为99%（似然），但对健康人的误报率为5%。现有一人检测结果为阳性，他真正患病的概率是多少？

```
P(病|阳) = P(阳|病) × P(病) / P(阳)
         = 0.99 × 0.01 / (0.99×0.01 + 0.05×0.99)
         = 0.0099 / 0.0594
         ≈ 16.7%
```

尽管检测看起来很"准"，但由于先验概率很低，阳性结果并不意味着大概率患病——这就是贝叶斯思维的力量。

## 二、朴素贝叶斯在文本分类中的应用

### 2.1 核心思想

对于一篇待分类文档 $d$，我们需要计算它属于每个类别 $c$ 的概率，然后选择概率最大的类别：

$$c^* = \arg\max_{c} P(c|d) = \arg\max_{c} \frac{P(d|c) \cdot P(c)}{P(d)}$$

由于 $P(d)$ 对所有类别相同，可以省略：

$$c^* = \arg\max_{c} P(c) \cdot P(d|c)$$

### 2.2 "朴素"假设

文档 $d$ 由词 $w_1, w_2, ..., w_n$ 组成。计算 $P(d|c)$ 需要计算 $P(w_1, w_2, ..., w_n | c)$。**朴素贝叶斯假设所有特征（词）在给定类别下条件独立**：

$$P(w_1, w_2, ..., w_n | c) = \prod_{i=1}^{n} P(w_i | c)$$

这个假设在NLP中显然不成立（"机器"和"学习"经常共现），但实践证明它不影响分类效果——这正是"朴素"的精髓。

### 2.3 ASCII 图解分类流程

```
 ┌──────────────────┐
 │  训练文档集合      │
 │  ┌──┐ ┌──┐ ┌──┐  │
 │  │d1│ │d2│ │d3│  │     训练阶段
 │  └──┘ └──┘ └──┘  │
 └────────┬─────────┘
          │ 统计词频
          ▼
 ┌──────────────────┐     ┌───────────────────┐
 │  先验 P(c)        │     │  似然 P(w|c)       │
 │  P(体育) P(科技)  │     │  P("进球"|体育)    │
 └────────┬─────────┘     └─────────┬─────────┘
          │                          │
          └──────────┬───────────────┘
                     ▼
          ┌────────────────────┐
          │  新文档 d → 分类     │     预测阶段
          │  c* = argmax P(c|d) │
          └────────────────────┘
```

## 三、多项式NB vs 伯努利NB

朴素贝叶斯有多种变体，在文本分类中最常用的是**多项式朴素贝叶斯**和**伯努利朴素贝叶斯**。

### 3.1 多项式朴素贝叶斯（Multinomial NB）

- **特征表示**：词频（一个词出现几次就算几次）
- **适用场景**：文档长度较长、词频信息有意义的文本分类
- **公式**：$P(w_i|c) = \frac{count(w_i, c)}{\sum_{w} count(w, c)}$

### 3.2 伯努利朴素贝叶斯（Bernoulli NB）

- **特征表示**：二值（词出现=1，不出现=0）
- **适用场景**：短文本，或"词是否出现"比"出现几次"更重要的场景
- **特点**：会显式考虑词不出现的概率

### 3.3 对比表

| 特性 | 多项式NB (MultinomialNB) | 伯努利NB (BernoulliNB) |
|------|--------------------------|------------------------|
| 特征类型 | 计数（词频） | 二值（出现/不出现） |
| 是否考虑词频 | 是 | 否 |
| 不出现的词 | 不参与计算 | 参与计算（贡献负对数） |
| 典型场景 | 长文档分类 | 短文本分类 |
| sklearn 类 | MultinomialNB | BernoulliNB |

## 四、拉普拉斯平滑

### 4.1 零概率问题

如果测试文档中出现了一个训练集里从未见过的词（或在某类别下从未出现过的词），会导致：

$$P(w_{new}|c) = 0 \Rightarrow P(d|c) = \prod P(w_i|c) = 0$$

一个零概率会让整个文档的概率归零——这显然不合理。

### 4.2 解决方案：拉普拉斯平滑（加一平滑）

$$P(w_i|c) = \frac{count(w_i, c) + \alpha}{\sum_{w} count(w, c) + \alpha \cdot |V|}$$

其中 $\alpha$ 是平滑参数（通常取1），$|V|$ 是词表大小。

```
不平滑:  P("区块链"|科技) = 0 / 100 = 0        ← 灾难！

拉普拉斯: P("区块链"|科技) = (0+1) / (100+1×5000)
                                   = 1 / 5100   ← 合理的小值
```

## 五、完整代码：朴素贝叶斯中文文本分类器

### 5.1 从零实现

```python
import numpy as np
from collections import defaultdict, Counter

class NaiveBayesClassifier:
    """从零实现的朴素贝叶斯文本分类器"""

    def __init__(self, alpha=1.0):
        self.alpha = alpha          # 拉普拉斯平滑参数
        self.word_probs = {}        # P(词|类别)
        self.class_probs = {}       # P(类别)
        self.vocab = set()

    def fit(self, X, y):
        """
        X: List[List[str]]  每个元素是分词后的文档
        y: List[str]        类别标签
        """
        # 统计每个类别的文档数
        class_counts = Counter(y)
        total_docs = len(y)

        # 计算先验 P(类别)
        self.class_probs = {c: count / total_docs for c, count in class_counts.items()}

        # 统计每个类别下每个词的出现次数
        word_counts = defaultdict(lambda: defaultdict(int))  # {类别: {词: 次数}}
        for words, label in zip(X, y):
            for w in words:
                word_counts[label][w] += 1
                self.vocab.add(w)

        vocab_size = len(self.vocab)

        # 计算似然 P(词|类别)，带拉普拉斯平滑
        for c in class_counts:
            total_words_in_class = sum(word_counts[c].values())
            self.word_probs[c] = {
                w: (word_counts[c].get(w, 0) + self.alpha) / (total_words_in_class + self.alpha * vocab_size)
                for w in self.vocab
            }

    def predict(self, words):
        """预测单个文档的类别"""
        best_class, best_log_prob = None, float('-inf')
        for c, prior in self.class_probs.items():
            log_prob = np.log(prior)
            for w in words:
                if w in self.word_probs[c]:
                    log_prob += np.log(self.word_probs[c][w])
            if log_prob > best_log_prob:
                best_log_prob = log_prob
                best_class = c
        return best_class


# ========== 使用示例 ==========
# 训练数据（已分词）
train_docs = [
    "苹果 发布 了 新款 手机".split(),
    "华为 推出 旗舰 手机".split(),
    "微软 发布 操作系统".split(),
    "谷歌 推出 搜索引擎".split(),
    "皇马 赢得 比赛 冠军".split(),
    "巴萨 晋级 决赛".split(),
]
train_labels = ["科技", "科技", "科技", "科技", "体育", "体育"]

# 训练
nb = NaiveBayesClassifier(alpha=1.0)
nb.fit(train_docs, train_labels)

# 预测
test_doc = "苹果 将 发布 新款 操作系统".split()
print(f"预测类别: {nb.predict(test_doc)}")  # 输出: 科技
```

### 5.2 使用 sklearn 的 MultinomialNB

```python
from sklearn.feature_extraction.text import CountVectorizer, TfidfTransformer
from sklearn.naive_bayes import MultinomialNB, BernoulliNB
from sklearn.pipeline import Pipeline
from sklearn.metrics import classification_report

# 模拟数据
texts = [
    "这款 手机 拍照 效果 很好 电池 续航 长",
    "手机 屏幕 清晰 运行 流畅 性价比 高",
    "发货 速度快 包装 精美 质量 好",
    "商家 服务 态度 差 退货 麻烦",
    "质量 差 用 了 几天 就 坏 了",
    "太 差 了 完全 是 假货",
]
labels = ["好评", "好评", "好评", "差评", "差评", "差评"]

# 构建Pipeline：向量化 + TF-IDF + 朴素贝叶斯
model = Pipeline([
    ('vect', CountVectorizer()),
    ('tfidf', TfidfTransformer()),
    ('clf', MultinomialNB(alpha=1.0)),   # alpha即拉普拉斯平滑参数
])

model.fit(texts, labels)

# 预测新评论
test_texts = ["手机 质量 好 推荐", "假货 差评"]
preds = model.predict(test_texts)
for text, pred in zip(test_texts, preds):
    print(f"[{pred}] {text}")
```

## 六、优缺点分析

### 6.1 优点

- **训练和预测速度极快**：仅需统计词频，时间复杂度 $O(n \cdot d)$
- **对高维数据友好**：词表通常上万维，NB依然高效
- **需要训练数据少**：即便每类只有几十条样本也能工作
- **可解释性好**：每个词对每个类别的贡献都可查

### 6.2 缺点

- **"朴素"假设过强**：无法捕捉词与词之间的依赖关系（如"不 好"这样的否定）
- **对特征相关性敏感**：当特征高度相关时，概率估计会失真
- **零概率问题**：必须依赖平滑，参数选择影响效果
- **词序信息丢失**：词袋模型天然无法建模词序

### 6.3 适用场景与不适用场景

| 场景 | 是否适合朴素贝叶斯 | 原因 |
|------|-------------------|------|
| 垃圾邮件过滤 | ★★★★★ | 特征独立假设在此场景出奇有效 |
| 新闻分类 | ★★★★☆ | 类别界限清晰，主题词区分度高 |
| 情感分析 | ★★★☆☆ | 否定、反讽等依赖上下文，NB表现一般 |
| 实体识别 | ★★☆☆☆ | 序列标注任务需要建模上下文依赖 |

## 七、小结

| 要点 | 内容 |
|------|------|
| 核心思想 | 基于贝叶斯定理，利用条件独立假设进行分类 |
| 关键公式 | $P(c\|d) \propto P(c) \cdot \prod P(w_i\|c)$ |
| 多项式NB | 使用词频，适合长文本，sklearn中为 `MultinomialNB` |
| 伯努利NB | 使用二值特征，适合短文本，sklearn中为 `BernoulliNB` |
| 拉普拉斯平滑 | $P(w\|c) = \frac{count(w,c)+\alpha}{\sum count + \alpha\|V\|}$，解决零概率 |
| 优点 | 速度快、高维友好、所需数据少 |
| 缺点 | 独立假设过强，无法建模词序和词间依赖 |
| 典型应用 | 垃圾邮件过滤、新闻分类、情感分析基线 |
