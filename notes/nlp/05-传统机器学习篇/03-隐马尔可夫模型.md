# 隐马尔可夫模型

隐马尔可夫模型（Hidden Markov Model, HMM）是统计建模中处理**序列数据**的经典工具。在NLP领域，HMM曾是词性标注、中文分词、命名实体识别等序列标注任务的**主流方法**。尽管如今深度学习已占据主导，但理解HMM对于掌握序列建模思想至关重要——它是理解CRF、BiLSTM-CRF乃至Transformer的基石。

---

## 一、HMM五要素

HMM用一个双层结构来描述序列：观测到的序列背后隐藏着一个状态序列。

### 1.1 五要素详解

| 要素 | 符号 | 说明 | NLP示例（词性标注） |
|------|------|------|---------------------|
| 状态集 | $S = \{s_1, ..., s_N\}$ | 所有可能的隐状态 | 词性集合{名词, 动词, 形容词, ...} |
| 观测集 | $V = \{v_1, ..., v_M\}$ | 所有可能的观测值 | 词汇表{我, 爱, 编程, ...} |
| 转移概率 | $A = [a_{ij}]$ | $a_{ij} = P(s_j \| s_i)$ 从状态i到j的概率 | P(动词→名词)：动词后接名词的概率 |
| 发射概率 | $B = [b_j(k)]$ | $b_j(k) = P(v_k \| s_j)$ 状态j产生观测k的概率 | P("编程"\|动词) |
| 初始概率 | $\pi = [\pi_i]$ | $\pi_i = P(q_1 = s_i)$ 序列首状态概率 | 句子首词是名词的概率 |

### 1.2 ASCII图解HMM结构

```
  时间:    t=1      t=2      t=3      t=4
           │        │        │        │
  隐状态:  ┌───┐   ┌───┐   ┌───┐   ┌───┐
           │q₁ │──→│q₂ │──→│q₃ │──→│q₄ │     （隐状态序列）
           └─┬─┘   └─┬─┘   └─┬─┘   └─┬─┘
             │       │       │       │         发射概率 B
             ▼       ▼       ▼       ▼
  观测值:    o₁      o₂      o₃      o₄       （观测序列）

           │←─a₁₂─→│←─a₂₃─→│←─a₃₄─→│         转移概率 A

  示例（中文分词）:
  隐状态:  [B] ──→ [E] ──→ [S] ──→ [B]       B=词首 M=词中 E=词尾 S=单字
           │        │        │        │
           ▼        ▼        ▼        ▼
  观测值:  我       爱       他       的       （可见的汉字序列）

  分词结果: 我 / 爱 / 他 / ... （根据BEMS标注切词）
```

## 二、HMM的三个基本问题

HMM的应用围绕三个核心问题展开：

### 2.1 问题一览

| 问题 | 描述 | 算法 | NLP应用 |
|------|------|------|---------|
| 评估 | 给定模型λ，求观测序列O的概率P(O\|λ) | **前向算法** | 识别哪个模型更匹配序列 |
| 解码 | 给定模型λ和观测O，求最可能的隐状态序列Q | **维特比算法** | 词性标注、分词、NER |
| 学习 | 仅给观测O，如何估计模型参数λ | **Baum-Welch** | 无监督训练HMM |

### 2.2 前向算法（评估问题）

计算观测序列 $O = o_1 o_2 ... o_T$ 的概率。定义前向变量 $\alpha_t(i) = P(o_1 o_2 ... o_t, q_t = s_i | \lambda)$。

```
前向算法流程（动态规划）:

步骤1（初始化）:  α₁(i) = πᵢ · bᵢ(o₁)          i = 1..N

步骤2（递推）:    αₜ(j) = [ Σᵢ αₜ₋₁(i) · aᵢⱼ ] · bⱼ(oₜ)   t=2..T

步骤3（终止）:    P(O|λ) = Σᵢ αT(i)


图解（T=3, N=2 的计算过程）:

    α₁(1) ──→ α₂(1) ──→ α₃(1)
      │  ╲      │  ╲      │
      │   ╲      │   ╲     │
    α₁(2) ──→ α₂(2) ──→ α₃(2)

    每个 αₜ(j) = 上面所有指向它的 αₜ₋₁(i) 加权求和 × bⱼ(oₜ)
```

### 2.3 维特比算法（解码问题）

寻找最可能的隐状态序列 $Q^* = \arg\max_Q P(Q|O, \lambda)$。定义 $\delta_t(i) = \max_{q_1...q_{t-1}} P(q_1...q_{t-1}, q_t=s_i, o_1...o_t | \lambda)$。

```
维特比算法流程:

步骤1（初始化）:  δ₁(i) = πᵢ · bᵢ(o₁)          ψ₁(i) = 0

步骤2（递推）:    δₜ(j) = maxᵢ [ δₜ₋₁(i) · aᵢⱼ ] · bⱼ(oₜ)
                 ψₜ(j) = argmaxᵢ [ δₜ₋₁(i) · aᵢⱼ ]     ← 记录回溯指针

步骤3（终止）:    P* = maxᵢ δT(i)
                 q*T = argmaxᵢ δT(i)

步骤4（回溯）:    q*t = ψₜ₊₁(q*t₊₁)    t = T-1, ..., 1


图解（寻找最优路径）:

    δ₁(1) ══════→ δ₂(1) ══════→ δ₃(1) ══→ 最优
      ║ ╲          ║ ╲          ║
      ║  ╲         ║  ╲         ║
    δ₁(2) ────→ δ₂(2) ────→ δ₃(2)

    ═══ = 最优路径（保留每步最大值，回溯得到）
    ─── = 被淘汰的路径
```

### 2.4 Baum-Welch 算法（学习问题）

 Baum-Welch 是 EM 算法在 HMM 上的特例，用于无监督学习模型参数。由于实现较复杂且NLP中通常有标注数据，实际应用较少，这里仅做了解。

## 三、HMM用于词性标注的完整Python实现

```python
import numpy as np
from collections import defaultdict

class HMMPosTagger:
    """基于HMM的词性标注器"""

    def __init__(self):
        self.tags = []           # 词性列表
        self.vocab = []          # 词汇表
        self.tag2idx = {}
        self.word2idx = {}
        self.A = None            # 转移矩阵 [N, N]
        self.B = None            # 发射矩阵 [N, M]
        self.pi = None           # 初始概率 [N]

    def train(self, sentences):
        """
        sentences: List[List[(word, tag)]]
          每个sentence是 (词, 词性) 元组列表
        """
        # 构建词表和词性表
        tag_set, vocab_set = set(), set()
        for sent in sentences:
            for word, tag in sent:
                tag_set.add(tag)
                vocab_set.add(word)

        self.tags = sorted(tag_set)
        self.vocab = sorted(vocab_set)
        self.tag2idx = {t: i for i, t in enumerate(self.tags)}
        self.word2idx = {w: i for i, w in enumerate(self.vocab)}
        N, M = len(self.tags), len(self.vocab)

        # 统计转移、发射、初始计数
        trans_count = np.zeros((N, N))    # A
        emit_count = np.zeros((N, M))     # B
        init_count = np.zeros(N)          # pi

        for sent in sentences:
            for i, (word, tag) in enumerate(sent):
                t_idx = self.tag2idx[tag]
                w_idx = self.word2idx[word]
                emit_count[t_idx][w_idx] += 1
                if i == 0:
                    init_count[t_idx] += 1
                else:
                    prev_tag = sent[i-1][1]
                    trans_count[self.tag2idx[prev_tag]][t_idx] += 1

        # 加一平滑后归一化
        self.A = (trans_count + 1) / (trans_count.sum(axis=1, keepdims=True) + N)
        self.B = (emit_count + 1) / (emit_count.sum(axis=1, keepdims=True) + M)
        self.pi = (init_count + 1) / (init_count.sum() + N)

    def viterbi(self, words):
        """维特比算法解码最优词性序列"""
        T = len(words)
        N = len(self.tags)
        delta = np.zeros((T, N))     # 最优概率
        psi = np.zeros((T, N), dtype=int)  # 回溯指针

        # 初始化
        for j in range(N):
            w_idx = self.word2idx.get(words[0], -1)
            emit = self.B[j][w_idx] if w_idx >= 0 else 1.0 / len(self.vocab)
            delta[0][j] = self.pi[j] * emit

        # 递推
        for t in range(1, T):
            for j in range(N):
                probs = delta[t-1] * self.A[:, j]
                psi[t][j] = np.argmax(probs)
                delta[t][j] = probs[psi[t][j]]
                w_idx = self.word2idx.get(words[t], -1)
                emit = self.B[j][w_idx] if w_idx >= 0 else 1.0 / len(self.vocab)
                delta[t][j] *= emit

        # 回溯
        path = [np.argmax(delta[T-1])]
        for t in range(T-2, -1, -1):
            path.append(psi[t+1][path[-1]])
        path.reverse()

        return [(words[i], self.tags[path[i]]) for i in range(T)]


# ========= 使用示例：词性标注 =========
train_data = [
    [("我", "r"), ("爱", "v"), ("自然", "n"), ("语言", "n"), ("处理", "v")],
    [("他", "r"), ("正在", "d"), ("学习", "v"), ("算法", "n")],
    [("这", "r"), ("是", "v"), ("一本", "m"), ("好", "a"), ("书", "n")],
    [("自然", "n"), ("语言", "n"), ("处理", "v"), ("很有趣", "a")],
    [("我", "r"), ("正在", "d"), ("读", "v"), ("书", "n")],
]

hmm = HMMPosTagger()
hmm.train(train_data)

# 预测
test = ["我", "爱", "算法"]
result = hmm.viterbi(test)
print("词性标注结果:")
for word, tag in result:
    print(f"  {word} / {tag}")
```

## 四、HMM用于中文分词

### 4.1 BEMS标注体系

将分词转化为序列标注问题，每个字标记为四种状态之一：

```
状态含义:
  B (Begin)   - 词的开头
  M (Middle)  - 词的中间
  E (End)     - 词的结尾
  S (Single)  - 单字成词

示例:
  句子:    自  然  语  言  处  理  很  有  趣
  标注:    B   E   B   E   B   E   S   S   S
  分词:   |自然| |语言| |处理| |很| |有| |趣|

ASCII图解:
  字:     自    然    语    言    处    理    很    有    趣
          ↓     ↓     ↓     ↓     ↓     ↓     ↓     ↓     ↓
  状态:   B ─→ E     B ─→ E     B ─→ E     S     S     S
          └──词1──┘  └──词2──┘  └──词3──┘   ↑     ↑     ↑
                                              词4  词5  词6
```

### 4.2 分词实现思路

```python
def hmm_segment(text, hmm_model):
    """
    用HMM进行中文分词
    text: 待分词的中文字符串
    """
    chars = list(text)
    # 维特比解码得到BEMS序列
    tagged = hmm_model.viterbi(chars)
    tags = [tag for _, tag in tagged]

    # 根据BEMS标签切词
    words = []
    word = ""
    for char, tag in zip(chars, tags):
        if tag == 'S':
            words.append(char)
        elif tag == 'B':
            word = char
        elif tag == 'M':
            word += char
        elif tag == 'E':
            word += char
            words.append(word)
            word = ""
    return words

# 注：需要先在BEMS标注的语料上训练HMM
# 常用语料：人民日报标注语料库、MSR等
```

## 五、HMM的局限性

尽管HMM在序列标注任务中曾大放异彩，但它存在几个根本性局限：

### 5.1 两个独立假设过强

HMM做了两个假设：

1. **马尔可夫假设**：当前状态只依赖前一个状态（一阶）
2. **观测独立假设**：当前观测只依赖当前状态

```
HMM假设的局限性:

  HMM认为:     qₜ 只看 qₜ₋₁
              oₜ 只看 qₜ

  实际情况:     qₜ 受 qₜ₋₁ qₜ₋₂ ... 影响
              oₜ 与 oₜ₋₁ oₜ₊₁ 相关
              （比如"打"球 vs "打"电话，词义依赖上下文）
```

### 5.2 生成式模型的局限

HMM是**生成式模型**，需要建模 $P(\text{观测}, \text{状态})$。这导致：

- 需要建模观测的分布（词的生成），而这在NLP中很困难
- 无法灵活引入**丰富的重叠特征**（如前后缀、大小写、上下文词等）
- 标注序列和观测序列的依赖关系建模不够灵活

### 5.3 与后续模型的对比

| 特性 | HMM | CRF | BiLSTM-CRF |
|------|-----|-----|------------|
| 模型类型 | 生成式 | 判别式 | 判别式+神经网络 |
| 特征利用 | 仅发射+转移 | 任意特征函数 | 自动学习特征 |
| 上下文 | 一阶马尔可夫 | 全局（线性链） | 双向长距离 |
| 标注精度 | 良好 | 优秀 | 最优 |
| 训练速度 | 快 | 中等 | 慢 |
| 可解释性 | 高 | 中 | 低 |

> 这正是为什么CRF后来取代了HMM，而BiLSTM-CRF又进一步取代了纯CRF。

## 六、小结

| 要点 | 内容 |
|------|------|
| 核心思想 | 用隐状态序列建模观测序列，状态间满足一阶马尔可夫性 |
| 五要素 | 状态集S、观测集V、转移概率A、发射概率B、初始概率π |
| 评估问题 | 前向算法计算 P(O\|λ) |
| 解码问题 | **维特比算法**求最可能状态序列（NLP中最常用） |
| 学习问题 | Baum-Welch（EM算法）无监督学习参数 |
| 词性标注 | 隐状态=词性，观测=词，用维特比解码 |
| 中文分词 | BEMS标注体系，将分词转化为序列标注 |
| 主要局限 | 假设过强、生成式模型、无法利用丰富重叠特征 |
| 后继者 | CRF解决了特征限制问题，BiLSTM-CRF进一步提升了效果 |
