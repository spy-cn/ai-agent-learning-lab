面对琳琅满目的模型和技术方案，如何根据实际需求选择最合适的路线？本章提供系统化的决策框架，帮助你在不同场景下做出最优选择。

## 1. 按任务类型的推荐方案

### 1.1 文本分类

```
任务难度
  │
  │ 大模型时代
  │ +----------------------------------+
  │ |     Few-shot / GPT-4 API         |  < 50条样本
  │ +----------------------------------+
  │ |     SetFit / 小样本学习           |  50-500条样本
  │ +----------------------------------+
  │ |     BERT微调 (推荐)               |  500-10000条样本
  │ +----------------------------------+
  │ |     LLM全量微调 (LoRA)           |  > 10000条样本
  │ +----------------------------------+
  └──────────────────────────────────────> 数据量
```

| 方案 | 数据量 | 训练成本 | 推理成本 | 准确率 | 推荐场景 |
|------|--------|----------|----------|--------|----------|
| Zero-shot (GPT-4) | 0 | 0 | 高 | ~85% | 快速验证 |
| Few-shot (GPT-4) | 10-50 | 0 | 高 | ~88% | 无标注数据 |
| SetFit | 50-500 | 低 | 低 | ~90% | 小样本场景 |
| BERT微调 | 500+ | 中 | 低 | ~93% | **最佳性价比** |
| LLM微调(LoRA) | 10000+ | 高 | 中 | ~95% | 追求极致 |

## 3. 成本vs效果权衡

### 3.1 API调用 vs 本地部署

| 维度 | API调用 (GPT-4/GPT-4o) | 本地部署 (Llama3/Qwen2) |
|------|------------------------|------------------------|
| **前期成本** | $0 (按量付费) | GPU采购(~$2万+) |
| **单次推理成本** | $0.01-0.06/千token | ~$0.0001/千token |
| **延迟** | 0.5-3秒 | 0.1-1秒 (取决于硬件) |
| **数据隐私** | 数据上传第三方 | **完全本地** |
| **可控性** | 低 (受API限制) | 高 (可定制) |
| **维护成本** | 0 | 需要运维能力 |
| **模型更新** | 自动获取最新 | 需要手动升级 |

### 3.2 盈亏平衡分析

```python
def calculate_breakeven(
    api_cost_per_1k: float = 0.01,      # API每千token成本
    local_cost_per_1k: float = 0.0001,  # 本地每千token成本
    fixed_local_cost: float = 20000,    # 本地GPU一次性成本
    tokens_per_month: float = 10_000_000 # 每月token使用量
):
    """计算使用本地部署和API调用的盈亏平衡点"""
    
    monthly_api_cost = (tokens_per_month / 1000) * api_cost_per_1k
    monthly_local_cost = (tokens_per_month / 1000) * local_cost_per_1k
    
    print(f"API月成本: ${monthly_api_cost:.2f}")
    print(f"本地月运营成本: ${monthly_local_cost:.2f}")
    
    # 计算回本时间（月）
    monthly_savings = monthly_api_cost - monthly_local_cost
    breakeven_months = fixed_local_cost / monthly_savings
    
    print(f"每月节省: ${monthly_savings:.2f}")
    print(f"回本周期: {breakeven_months:.1f}个月")
    
    return breakeven_months

# 示例：每月1000万token
# calculate_breakeven(tokens_per_month=10_000_000)
# API月成本: $100.00
# 本地月运营成本: $1.00
# 每月节省: $99.00
# 回本周期: 202.0个月 (~17年)
# 
# 每月1亿token:
# calculate_breakeven(tokens_per_month=100_000_000)
# 回本周期: ~20个月
```

## 5. 特殊场景建议

### 5.1 小样本场景（< 500条标注数据）

```
策略选择:
  ├── 零样本: 使用大模型API (GPT-4) 直接判断
  ├── 少样本: Prompt中给3-5个示例
  ├── SetFit: 句子Transformer + 对比学习 (效果最好)
  └── 数据增强: EDA/回译/同义词替换
```

```python
# SetFit 小样本分类示例
from setfit import SetFitModel, Trainer, TrainingArguments

model = SetFitModel.from_pretrained("BAAI/bge-small-zh-v1.5")

args = TrainingArguments(
    batch_size=16,
    num_epochs=1,
    num_iterations=20,  # 对比学习迭代
)

trainer = Trainer(
    model=model,
    args=args,
    train_dataset=train_dataset,  # 仅需数百条
)
trainer.train()
```

### 5.2 长文本处理

| 策略 | 实现 | 优点 | 缺点 |
|------|------|------|------|
| 截断 | 取前N个token | 简单 | 丢失信息 |
| 滑动窗口 | 分段独立处理 | 覆盖全文 | 片段间无关联 |
| 层次化 | 先分块，再聚合 | 保留全文信息 | 复杂度高 |
| 长上下文模型 | Longformer/BigBird | 原生支持 | 模型选择少 |
| 现代LLM | Llama3.1(128K) | 最灵活 | 计算量大 |

## 小结

本章提供了系统化的NLP任务模型选择决策框架。

| 要点 | 内容 |
|------|------|
| **文本分类** | 数据量500+ → BERT微调是最佳性价比 |

| **决策三要素** | 数据量 + 精度要求 + 资源约束 |
| **成本权衡** | 月token > 1亿时本地部署开始有优势 |
| **小样本** | SetFit效果最好，GPT API最方便 |
| **推理优化** | 量化 + ONNX + 批量推理三件套 |
