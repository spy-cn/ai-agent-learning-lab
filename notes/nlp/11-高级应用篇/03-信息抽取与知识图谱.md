# 信息抽取与知识图谱

信息抽取（Information Extraction, IE）是NLP的核心任务之一，目标是从非结构化文本中自动提取结构化信息。知识图谱（Knowledge Graph, KG）则将这些结构化信息组织成图网络，支持高效的查询和推理。本章将从IE任务体系出发，深入关系抽取、事件抽取，并介绍知识图谱的构建、表示学习与应用。

| 任务 | 英文 | 粒度 | 输入 | 输出 | 典型数据集 |
|------|------|------|------|------|-----------|
| 命名实体识别 | NER | Token级 | 句子 | 实体提及+类别 | CoNLL-03, OntoNotes |

## 关系抽取方法

关系抽取（Relation Extraction）识别文本中已标注实体之间的语义关系，输出结构化的（头实体, 关系, 尾实体）三元组。

### 方法一：流水线方法（Pipeline）

先NER识别所有实体，然后对每个实体对进行关系分类。

```
┌──────────── 流水线关系抽取 ────────────┐
│                                        │
│  文本: "乔布斯创立了苹果公司"            │
│      │                                 │
│      ▼                                 │
│  NER: [PER:乔布斯] [ORG:苹果公司]       │
│      │                                 │
│      ▼                                 │
│  实体对: (乔布斯, 苹果公司)              │
│      │                                 │
│      ▼                                 │
│  关系分类器 → 创始人-是                   │
│      │                                 │
│      ��                                 │
│  三元组: (乔布斯, 创始人, 苹果公司)       │
└────────────────────────────────────────┘
```

```python
import torch
import torch.nn as nn

class PipelineRelationClassifier(nn.Module):
    """流水线关系分类模型
    输入：句子 + 实体位置标记 → 分类关系类型
    """
    def __init__(self, bert_model, num_relations, hidden_dim=768):
        super().__init__()
        self.bert = bert_model
        self.dropout = nn.Dropout(0.1)
        
        # 关系分类头：拼接[CLS] + 两个实体的平均表示
        self.classifier = nn.Sequential(
            nn.Linear(hidden_dim * 3, hidden_dim),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(hidden_dim, num_relations)
        )
    
    def forward(self, input_ids, attention_mask, e1_start, e1_end, e2_start, e2_end):
        """
        Args:
            input_ids: (B, L)
            attention_mask: (B, L)
            e1_start, e1_end: 头实体的起止位置
            e2_start, e2_end: 尾实体的起止位置
        Returns:
            relation_logits: (B, num_relations)
        """
        outputs = self.bert(input_ids=input_ids, attention_mask=attention_mask)
        hidden = outputs.last_hidden_state  # (B, L, D)
        
        # [CLS] token
        cls_rep = hidden[:, 0, :]  # (B, D)
        
        # 实体表示：平均池化
        e1_rep = self._entity_pooling(hidden, e1_start, e1_end)
        e2_rep = self._entity_pooling(hidden, e2_start, e2_end)
        
        # 拼接三个表示
        combined = torch.cat([cls_rep, e1_rep, e2_rep], dim=-1)  # (B, 3D)
        
        relation_logits = self.classifier(self.dropout(combined))
        
        return relation_logits
    
    def _entity_pooling(self, hidden, start_idx, end_idx):
        """对实体包含的token做平均池化"""
        batch_size = hidden.size(0)
        pooled = torch.zeros(batch_size, hidden.size(-1), device=hidden.device)
        
        for b in range(batch_size):
            s, e = start_idx[b], end_idx[b]
            if s <= e:
                pooled[b] = hidden[b, s:e+1].mean(dim=0)
        
        return pooled
```

### 方法二：联合抽取（Joint Extraction）

同时识别实体和关系，避免流水线方案的错误传播。CASREL模型将关系建模为"头实体 → 关系 → 尾实体"的级联二分类。

```
┌───────────────── CASREL 联合抽取 ─────────────────┐
│                                                    │
│  步骤1: 头实体识别                                   │
│  输入句子 → 对每个token做头实体start/end二分类        │
│                                                    │
│  步骤2: 关系和尾实体识别                              │
│  对每个已识别的头实体 + 每种关系类型:                   │
│    → 对每个token做尾实体start/end二分类               │
│                                                    │
│  输出: (head, relation, tail) 三元组列表              │
└────────────────────────────────────────────────────┘
```

```python
class CASRELJointExtractor(nn.Module):
    """CASREL联合实体关系抽取模型（简化版）"""
    def __init__(self, bert_dim=768, num_relations=24):
        super().__init__()
        
        # 头实体识别器
        self.head_start = nn.Linear(bert_dim, 1)  # 每个token是否是头实体起点
        self.head_end = nn.Linear(bert_dim, 1)    # 每个token是否是头实体终点
        
        # 尾实体识别器（每种关系独立的二分类器）
        self.tail_start = nn.ModuleList([
            nn.Linear(bert_dim, 1) for _ in range(num_relations)
        ])
        self.tail_end = nn.ModuleList([
            nn.Linear(bert_dim, 1) for _ in range(num_relations)
        ])
        
        self.num_relations = num_relations
    
    def extract_triplets(self, hidden_states, attention_mask, threshold=0.5):
        """
        从编码表示中抽取三元组
        
        Args:
            hidden_states: (B, L, D) BERT编码
            attention_mask: (B, L)
            threshold: 解码阈值
        Returns:
            triplets: list of (head_span, relation_idx, tail_span)
        """
        B, L, _ = hidden_states.shape
        all_triplets = []
        
        for b in range(B):
            # 步骤1: 识别头实体
            h_start_probs = torch.sigmoid(self.head_start(hidden_states[b])).squeeze()  # (L,)
            h_end_probs = torch.sigmoid(self.head_end(hidden_states[b])).squeeze()      # (L,)
            
            # 解码头实体span
            head_entities = self._decode_spans(h_start_probs, h_end_probs, threshold)
            
            if not head_entities:
                continue
            
            # 步骤2: 对每种关系，识别尾实体
            for h_start, h_end in head_entities:
                # 头实体信息：平均池化
                head_rep = hidden_states[b, h_start:h_end+1].mean(dim=0)
                
                for r in range(self.num_relations):
                    # 融入头实体信息的尾实体识别
                    # 简化：直接对token做二分类
                    t_start_probs = torch.sigmoid(
                        self.tail_start[r](hidden_states[b])
                    ).squeeze()
                    t_end_probs = torch.sigmoid(
                        self.tail_end[r](hidden_states[b])
                    ).squeeze()
                    
                    tail_entities = self._decode_spans(
                        t_start_probs, t_end_probs, threshold
                    )
                    
                    for t_start, t_end in tail_entities:
                        all_triplets.append({
                            "head": (h_start.item(), h_end.item()),
                            "relation": r,
                            "tail": (t_start.item(), t_end.item())
                        })
        
        return all_triplets
    
    def _decode_spans(self, start_probs, end_probs, threshold):
        """解码实体span：匹配start>threshold和最近的end>threshold的位置"""
        spans = []
        start_candidates = (start_probs > threshold).nonzero(as_tuple=True)[0]
        end_candidates = (end_probs > threshold).nonzero(as_tuple=True)[0]
        
        for s in start_candidates:
            # 找最近的合法end位置
            valid_ends = end_candidates[end_candidates >= s]
            if len(valid_ends) > 0:
                e = valid_ends[0]
                if e - s < 20:  # 限制最大span长度
                    spans.append((s, e))
        
        return spans

print("CASREL联合实体关系抽取模型已定义")
```

## 事件抽取

事件抽取（Event Extraction）比关系抽取更复杂，需要识别触发事件的词（触发词）以及事件参与者的角色（论元）。

### 事件抽取模型架构

```python
class EventExtractionModel(nn.Module):
    """事件抽取模型：触发词识别 + 论元角色分类"""
    def __init__(self, bert_model, num_event_types, num_argument_roles, hidden_dim=768):
        super().__init__()
        self.bert = bert_model
        
        # 触发词识别：对每个token分类事件类型 + BIO标注
        self.trigger_classifier = nn.Linear(hidden_dim, num_event_types + 1)  # +1 for "No trigger"
        
        # 论元角色分类：对每个实体pair分类论元角色
        self.argument_classifier = nn.Linear(hidden_dim * 2, num_argument_roles + 1)
        
        self.num_event_types = num_event_types
    
    def forward(self, input_ids, attention_mask):
        """阶段1: 触发词识别"""
        outputs = self.bert(input_ids=input_ids, attention_mask=attention_mask)
        hidden = outputs.last_hidden_state  # (B, L, D)
        
        trigger_logits = self.trigger_classifier(hidden)  # (B, L, num_types+1)
        
        return trigger_logits
    
    def classify_argument(self, hidden, trigger_idx, entity_idx):
        """
        阶段2: 论元角色分类
        Args:
            hidden: (B, L, D) BERT输出
            trigger_idx: 触发词位置
            entity_idx: 候选论元实体位置
        """
        trigger_rep = hidden[:, trigger_idx, :]   # (B, D)
        entity_rep = hidden[:, entity_idx, :]     # (B, D)
        
        combined = torch.cat([trigger_rep, entity_rep], dim=-1)  # (B, 2D)
        role_logits = self.argument_classifier(combined)  # (B, num_roles+1)
        
        return role_logits

print("事件抽取模型架构已定义")
```

## 知识图谱构建

知识图谱将信息抽取的结果组织成图结构，节点表示实体，边表示关系。

### 知识图谱构建Pipeline

```
┌─────────────────────────────────────────────────────────────────────┐
│                 知识图谱构建完整 Pipeline                              │
│                                                                      │
│  ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌─────┐│
│  │ 非结构化  │──▶│  实体识别 │──▶│  实体消歧 │──▶│  关系抽取 │──▶│知识 ││
│  │ 文本     │   │  (NER)   │   │ (Entity  │   │  (RE)    │   │融合 ││
│  └──────────┘   └──────────┘   │ Link)   │   └──────────┘   └──┬──┘│
│                                └──────────┘                    │    │
│                                                                ▼    │
│  ┌─────────────────────────────────────────────────────────┐       │
│  │                    5. 图谱存储与查询                       │       │
│  │  ┌───────────────────────────────────────────────────┐  │       │
│  │  │              图数据库 (Neo4j / NebulaGraph)        │  │       │
│  │  │  节点(Node): 实体                                    │  │       │
│  │  │  边(Edge):   关系                                    │  │       │
│  │  │  属性(Property): 附加信息                             │  │       │
│  │  └───────────────────────────────────────────────────┘  │       │
│  └─────────────────────────────────────────────────────────┘       │
└─────────────────────────────────────────────────────────────────────┘
```

### 知识图谱构建代码示例

```python
from dataclasses import dataclass
from typing import List, Dict, Tuple
import json

@dataclass
class Entity:
    """知识图谱中的实体"""
    id: str
    name: str
    type: str  # PER, ORG, LOC, PRODUCT等
    properties: Dict[str, str] = None
    
    def __post_init__(self):
        if self.properties is None:
            self.properties = {}

@dataclass
class Relation:
    """知识图谱中的关系"""
    head: Entity
    relation: str
    tail: Entity
    confidence: float = 1.0

class KnowledgeGraph:
    """简易知识图谱实现"""
    def __init__(self):
        self.entities: Dict[str, Entity] = {}  # id -> Entity
        self.relations: List[Relation] = []
        self.adjacency: Dict[str, List[Tuple[str, str]]] = {}  # head_id -> [(relation, tail_id)]
    
    def add_entity(self, entity: Entity):
        """添加实体"""
        if entity.id not in self.entities:
            self.entities[entity.id] = entity
            self.adjacency[entity.id] = []
    
    def add_relation(self, relation: Relation):
        """添加关系"""
        self.add_entity(relation.head)
        self.add_entity(relation.tail)
        
        self.relations.append(relation)
        self.adjacency[relation.head.id].append(
            (relation.relation, relation.tail.id)
        )
    
    def query_neighbors(self, entity_id: str, relation: str = None):
        """查询邻居节点"""
        if entity_id not in self.adjacency:
            return []
        
        neighbors = self.adjacency[entity_id]
        if relation:
            neighbors = [(r, t) for r, t in neighbors if r == relation]
        
        return [(rel, self.entities[tid]) for rel, tid in neighbors]
    
    def get_triplets(self) -> List[Tuple[str, str, str]]:
        """获取所有三元组"""
        return [(r.head.name, r.relation, r.tail.name) for r in self.relations]
    
    def to_neo4j_cypher(self) -> str:
        """生成Neo4j Cypher导入语句"""
        statements = []
        
        for entity in self.entities.values():
            statements.append(
                f"CREATE (n:{entity.type} {{id:'{entity.id}', name:'{entity.name}'}})"
            )
        
        for rel in self.relations:
            statements.append(
                f"MATCH (a {{id:'{rel.head.id}'}}), (b {{id:'{rel.tail.id}'}}) "
                f"CREATE (a)-[:{rel.relation}]->(b)"
            )
        
        return "\n".join(statements)
    
    def summary(self):
        """图谱摘要"""
        return {
            "num_entities": len(self.entities),
            "num_relations": len(self.relations),
            "entity_types": set(e.type for e in self.entities.values()),
            "relation_types": set(r.relation for r in self.relations),
            "triplets": self.get_triplets()
        }


# 构建示例知识图谱
kg = KnowledgeGraph()

# 添加科技公司知识
apple = Entity("apple", "苹果公司", "ORG", {"成立": "1976", "总部": "库比蒂诺"})
tim = Entity("tim_cook", "蒂姆·库克", "PER", {"职位": "CEO"})
steve = Entity("steve_jobs", "史蒂夫·乔布斯", "PER", {"职位": "联合创始人"})
iphone = Entity("iphone", "iPhone", "PRODUCT", {"首次发布": "2007"})

kg.add_entity(apple)
kg.add_entity(tim)
kg.add_entity(steve)
kg.add_entity(iphone)

kg.add_relation(Relation(steve, "创始人", apple))
kg.add_relation(Relation(tim, "CEO", apple))
kg.add_relation(Relation(apple, "发布", iphone))
kg.add_relation(Relation(tim, "位于", Entity("california", "加州", "LOC")))

print("=== 知识图谱摘要 ===")
for k, v in kg.summary().items():
    print(f"{k}: {v}")

print("\n=== 查询苹果公司的邻居 ===")
neighbors = kg.query_neighbors("apple")
for rel, entity in neighbors:
    print(f"  -{rel}-> {entity.name} ({entity.type})")
```

## 知识图谱表示学习

知识图谱表示学习（KG Embedding）将实体和关系映射到低维向量空间，使得图谱中的结构信息可以通过向量运算表示。

### TransE：最经典的KG嵌入模型

```
┌─────────────────── TransE 原理 ───────────────────┐
│                                                     │
│  核心假设: h + r ≈ t                                │
│                                                     │
│  ┌─────┐     ┌──────┐     ┌─────┐                  │
│  │  h   │ + │  r    │ ≈ │  t   │                   │
│  │ 实体 │     │ 关系 │     │ 实体 │                   │
│  │ 头   │     │ 向量 │     │ 尾   │                   │
│  └─────┘     └──────┘     └─────┘                  │
│                                                     │
│  评分函数: f(h, r, t) = -||h + r - t||               │
│  距离越小 → 三元组成立的可能性越大                     │
│                                                     │
│  示例:                                               │
│  苹果公司 + 创始人是 ≈ 乔布斯                          │
│  北京 + 首都是 ≈ 中国 (?) ← 错误！TransE不佳            │
└─────────────────────────────────────────────────────┘
```

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class TransE(nn.Module):
    """TransE知识图谱嵌入模型"""
    def __init__(self, num_entities, num_relations, embedding_dim=100, margin=1.0):
        super().__init__()
        # 实体和关系的嵌入矩阵
        self.entity_embeddings = nn.Embedding(num_entities, embedding_dim)
        self.relation_embeddings = nn.Embedding(num_relations, embedding_dim)
        
        # 初始化嵌入
        nn.init.xavier_uniform_(self.entity_embeddings.weight)
        nn.init.xavier_uniform_(self.relation_embeddings.weight)
        
        # L2归一化（可选）
        self.entity_embeddings.weight.data = F.normalize(
            self.entity_embeddings.weight.data, p=2, dim=1
        )
        self.relation_embeddings.weight.data = F.normalize(
            self.relation_embeddings.weight.data, p=2, dim=1
        )
        
        self.margin = margin
        self.embedding_dim = embedding_dim
    
    def forward(self, heads, relations, tails):
        """
        Args:
            heads: (B,) 头实体索引
            relations: (B,) 关系索引
            tails: (B,) 尾实体索引
        Returns:
            scores: (B,) TransE评分 ||h + r - t||
        """
        h = self.entity_embeddings(heads)       # (B, D)
        r = self.relation_embeddings(relations) # (B, D)
        t = self.entity_embeddings(tails)       # (B, D)
        
        # L2距离: ||h + r - t||
        scores = torch.norm(h + r - t, p=2, dim=-1)
        
        return scores
    
    def negative_sampling_loss(self, pos_heads, pos_relations, pos_tails, 
                                neg_heads, neg_relations, neg_tails):
        """基于负采样的Margin Ranking Loss"""
        # 正样本得分
        pos_scores = self.forward(pos_heads, pos_relations, pos_tails)
        
        # 负样本得分（破坏头或尾实体）
        neg_scores_head = self.forward(neg_heads, pos_relations, pos_tails)
        neg_scores_tail = self.forward(pos_heads, pos_relations, neg_tails)
        
        # Margin Ranking Loss
        loss_head = F.relu(self.margin + pos_scores - neg_scores_head).mean()
        loss_tail = F.relu(self.margin + pos_scores - neg_scores_tail).mean()
        
        loss = loss_head + loss_tail
        
        return loss
    
    def predict_tail(self, head_idx, relation_idx, tail_candidates=None):
        """预测尾实体：找使 ||h + r - t|| 最小的 t"""
        with torch.no_grad():
            h = self.entity_embeddings(torch.tensor([head_idx]))  # (1, D)
            r = self.relation_embeddings(torch.tensor([relation_idx]))  # (1, D)
            h_plus_r = h + r  # (1, D)
            
            if tail_candidates is not None:
                all_tails = self.entity_embeddings(tail_candidates)  # (N, D)
            else:
                all_tails = self.entity_embeddings.weight  # (E, D)
            
            # 计算所有尾实体的分数
            scores = -torch.norm(h_plus_r - all_tails, p=2, dim=-1)  # (N,)
            best_idx = scores.argmax().item()
            
            return best_idx, scores


# 模拟训练
num_entities, num_relations = 100, 10
model = TransE(num_entities, num_relations, embedding_dim=100)

# 模拟一批训练数据
B = 32
pos_heads = torch.randint(0, num_entities, (B,))
pos_relations = torch.randint(0, num_relations, (B,))
pos_tails = torch.randint(0, num_entities, (B,))
neg_heads = torch.randint(0, num_entities, (B,))
neg_relations = torch.randint(0, num_relations, (B,))
neg_tails = torch.randint(0, num_entities, (B,))

loss = model.negative_sampling_loss(
    pos_heads, pos_relations, pos_tails,
    neg_heads, neg_relations, neg_tails
)
print(f"TransE训练损失: {loss.item():.4f}")
```

### RotatE：复数空间旋转

```
┌─────────────────── RotatE 原理 ───────────────────┐
│                                                     │
│  核心: 将实体嵌入复数空间，关系建模为旋转              │
│                                                     │
│  h, t ∈ C^k (复数向量)                               │
│  r ∈ C^k, |r_i| = 1 (单位复数，即纯旋转)              │
│                                                     │
│  评分函数: f(h, r, t) = -||h ∘ r - t||               │
│  (Hadamard积：逐元素复数乘法)                         │
│                                                     │
│  优势: 可以建模对称关系                                │
│    对称关系 r: 存在 h,t 使得 h∘r=t 且 t∘r=h          │
│    即 r*r = 1 → r 是 ±1 或虚数                       │
│                                                     │
│  可建模的关系模式:                                     │
│  - 对称:   r_i ∈ {±1}                                │
│  - 反对称: r_i*r_i ≠ 1                               │
│  - 反转:   r1 = conj(r2)                             │
│  - 组合:   r3 = r1 ∘ r2                              │
└─────────────────────────────────────────────────────┘
```

### KG嵌入模型对比

| 模型 | 得分函数 | 空间 | 对称 | 反对称 | 反转 | 组合 |
|------|---------|------|------|--------|------|------|
| TransE | -‖h+r-t‖ | R | ✗ | ✓ | ✓ | ✗ |
| DistMult | <h,r,t> | R | ✓ | ✗ | ✗ | ✗ |
| ComplEx | Re(<h,r,conj(t)>) | C | ✓ | ✓ | ✓ | ✗ |
| RotatE | -‖h∘r-t‖ | C | ✓ | ✓ | ✓ | ✓ |

```python
"""
完整信息抽取流程：NER + 关系抽取
运行前安装: pip install spacy
           python -m spacy download en_core_web_sm
"""

import spacy
import re
from typing import List, Dict, Tuple

class InformationExtractor:
    """基于spaCy的信息抽取器"""
    
    def __init__(self):
        # 加载英文模型
        self.nlp = spacy.load("en_core_web_sm")
        
        # 定义关系抽取规则
        self.relation_patterns = {
            "founder": [
                r"(\w+) founded (\w+)",
                r"(\w+) was founded by (\w+)",
                r"(\w+) established (\w+)"
            ],
            "ceo": [
                r"(\w+) is the CEO of (\w+)",
                r"(\w+) serves as CEO of (\w+)",
                r"(\w+) leads (\w+)"
            ],
            "acquired": [
                r"(\w+) acquired (\w+)",
                r"(\w+) purchased (\w+)",
                r"(\w+) bought (\w+)"
            ],
            "located_in": [
                r"(\w+) is based in (\w+)",
                r"(\w+) is located in (\w+)",
                r"(\w+) headquartered in (\w+)"
            ]
        }
    
    def extract_entities(self, text: str) -> List[Dict]:
        """提取命名实体"""
        doc = self.nlp(text)
        entities = []
        
        for ent in doc.ents:
            entities.append({
                "text": ent.text,
                "label": ent.label_,
                "start": ent.start_char,
                "end": ent.end_char,
                "description": spacy.explain(ent.label_)
            })
        
        return entities
    
    def extract_relations_rule(self, text: str) -> List[Tuple[str, str, str]]:
        """基于规则的关系抽取"""
        relations = []
        doc = self.nlp(text)
        
        for token in doc:
            # 基于依存句法分析的关系抽取
            if token.dep_ == "nsubj" and token.head.pos_ == "VERB":
                subject = token.text
                verb = token.head.text
                
                # 找宾语
                for child in token.head.children:
                    if child.dep_ in ("dobj", "pobj", "attr"):
                        obj = child.text
                        relations.append((subject, verb, obj))
        
        return relations
    
    def extract_all(self, text: str) -> Dict:
        """完整的信息抽取"""
        return {
            "text": text,
            "entities": self.extract_entities(text),
            "relations": self.extract_relations_rule(text),
            "document_length": len(text)
        }


# 演示代码
def demo_ie():
    """信息抽取演示"""
    extractor = InformationExtractor()
    
    texts = [
        "Steve Jobs founded Apple Inc. in Cupertino, California in 1976.",
        "Tim Cook is the CEO of Apple and leads the company from its headquarters.",
        "Microsoft acquired Activision Blizzard for $68.7 billion in January 2023.",
        "Elon Musk co-founded Tesla and is based in Austin, Texas."
    ]
    
    for text in texts:
        print(f"\n{'='*60}")
        print(f"文本: {text}")
        
        result = extractor.extract_all(text)
        
        print(f"\n实体:")
        for ent in result["entities"]:
            print(f"  [{ent['label']}] {ent['text']} ({ent['description']})")
        
        print(f"\n关系 (基于依存句法):")
        for subj, verb, obj in result["relations"]:
            print(f"  ({subj}, {verb}, {obj})")

if __name__ == "__main__":
    demo_ie()
```

## 小结

| 要点 | 内容 |
|------|------|
| IE三大子任务 | NER（实体识别）→ RE（关系抽取）→ EE（事件抽取），粒度递进 |
| NER方案 | 序列标注(BIO/BIOES)，常用spaCy/HanLP/基于BERT的token分类 |
| 关系抽取方法 | 流水线（先NER再分类）/ 联���抽取（CASREL级联）/ LLM prompt引导 |
| CASREL核心 | 头实体识别 → 对每种关系做尾实体识别，避免流水线错误传播 |
| 事件抽取 | 识别触发词 + 论元角色分类，比RE更复杂，需理解事件结构 |
| KG构建Pipeline | 实体识别→实体消歧→关系抽取→知识融合→图谱存储 |
| TransE | 向量空间h+r≈t，简单高效但无法建模对称/1-N关系 |
| RotatE | 复数空间旋转，可建模对称/反对称/反转/组合四种关系模式 |
| KG存储 | Neo4j图数据库，Cypher查询语言，支持图遍历和子图匹配 |
| KBQA | 自然语言问题 → 实体链接+关系识别 → 图查询 → 答案生成 |
