# 对话系统

对话系统（Dialogue System）是NLP中最贴近实际应用的方向之一。从Siri到ChatGPT，从客服机器人到智能音箱，对话系统已经深入日常生活的方方面面。本章将系统介绍对话系统的分类、传统Pipeline架构、基于LLM的端到端方案，以及对话管理与评估的核心技术。

## 对话系统分类

```
┌─────────────────────────────────────────────────────────────────────┐
│                       对话系统三大分类                               │
├──────────┬──────────────────────────────┬───────────────────────────┤
│  类型     │  目标                         │  典型应用                 │
├──────────┼──────────────────────────────┼───────────────────────────┤
│ 任务型    │  完成特定任务                   │  订票、客服、预约、       │
│ (Task)   │  (有明确的目标和槽位)            │  天气查询、设备控制        │
├──────────┼──────────────────────────────┼───────────────────────────┤
│ 闲聊型    │  进行开放式自然对话              │  情感陪伴、社交、         │
│ (Chitchat)│  (无明确目标，关注体验)          │  闲聊机器人               │
├──────────┼──────────────────────────────┼───────────────────────────┤
│ 问答型    │  回答事实性问题                 │  知识问答、FAQ��统、       │
│ (QA)     │  (基于知识库或文档)              │  技术支持                 │
└──────────┴──────────────────────────────┴───────────────────────────┘
```

现代对话系统越来越多地融合这三种类型，如ChatGPT既能完成任务型对话（通过工具调用），也能进行闲聊和问答。

## 传统任务型对话系统架构（Pipeline）

传统任务型对话系统采用模块化Pipeline架构，每个模块各司其职：

### Pipeline各模块详解

| 模块 | 英文全称 | 输入 | 输出 | 关键技术 |
|------|----------|------|------|----------|
| NLU | Natural Language Understanding | 用户原始文本 | 意图标签 + 槽位值 | 文本分类 + 序列标注 |
| DST | Dialogue State Tracking | 当前轮NLU + 历史状态 | 更新后的对话状态 | 状态跟踪模型 |
| POL/DPL | Policy Learning | 对话状态 | 下一步系统动作 | 强化学习/规则 |
| NLG | Natural Language Generation | 系统动作 + 状态 | 自然语言回复 | 模板/Seq2Seq/LLM |

### 意图识别和槽位填充的Joint模型

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class JointIntentSlotModel(nn.Module):
    """联合意图识别和槽位填充模型
    共享BERT编码器，两个分类头分别输出意图和槽位序列
    """
    def __init__(self, bert_model, num_intents, num_slots, hidden_dim=768):
        super().__init__()
        self.bert = bert_model
        self.dropout = nn.Dropout(0.1)
        
        # 意图分类头（取[CLS] token表示）
        self.intent_classifier = nn.Linear(hidden_dim, num_intents)
        
        # 槽位分类头（对每个token做BIO标注）
        self.slot_classifier = nn.Linear(hidden_dim, num_slots)
    
    def forward(self, input_ids, attention_mask):
        """
        Args:
            input_ids: (B, L)
            attention_mask: (B, L)
        Returns:
            intent_logits: (B, num_intents)
            slot_logits: (B, L, num_slots)
        """
        # BERT编码
        outputs = self.bert(input_ids=input_ids, attention_mask=attention_mask)
        hidden_states = outputs.last_hidden_state  # (B, L, 768)
        pooled_output = outputs.pooler_output     # (B, 768) — [CLS]表示
        
        # 意图分类
        intent_logits = self.intent_classifier(self.dropout(pooled_output))
        
        # 槽位标签分类（每个token）
        slot_logits = self.slot_classifier(self.dropout(hidden_states))
        
        return intent_logits, slot_logits

class ATISDataExample:
    """ATIS数据集示例：航空订票领域"""
    def __init__(self):
        self.text = "show me flights from Boston to San Francisco on Monday"
        self.tokens = ["show", "me", "flights", "from", "Boston", 
                       "to", "San", "Francisco", "on", "Monday"]

                      "O", "B-toloc", "I-toloc", "O", "B-depart_date"]

# 损失计算演示
def compute_joint_loss(intent_logits, slot_logits, intent_labels, slot_labels, 
                       attention_mask, slot_weight=0.5):
    """
    联合损失 = 意图分类CE + 槽位填充CE
    
    Args:
        intent_logits: (B, num_intents)
        slot_logits: (B, L, num_slots)
        intent_labels: (B,)
        slot_labels: (B, L)
        attention_mask: (B, L)
        slot_weight: 槽位损失的权重系数
    """
    # 意图分类损失
    intent_loss = F.cross_entropy(intent_logits, intent_labels)
    
    # 槽位填充损失（仅计算有效token位置）
    slot_logits_flat = slot_logits.view(-1, slot_logits.size(-1))
    slot_labels_flat = slot_labels.view(-1)
    
    slot_loss = F.cross_entropy(
        slot_logits_flat, slot_labels_flat, ignore_index=-100, reduction='none'
    )
    
    # 按有效token平均
    active_mask = attention_mask.view(-1).bool()
    slot_loss = slot_loss[active_mask].mean()
    
    # 联合损失
    total_loss = intent_loss + slot_weight * slot_loss
    
    return total_loss, intent_loss, slot_loss

print("联合意图识别和槽位填充模型已定义")
```

### 对话状态跟踪（DST）实现

```python
class DialogueStateTracker:
    """对话状态跟踪器"""
    def __init__(self, slots_definition):
        """
        Args:
            slots_definition: dict, 定义需要跟踪的槽位及其类型
                例如: {"destination": "city", "date": "date", "departure": "city"}
        """
        self.slots_def = slots_definition
        self.state = {slot: None for slot in slots_definition}
        self.turn_count = 0
    
    def update(self, intent, extracted_slots):
        """
        基于新轮次的NLU结果更新对话状态
        
        Args:
            intent: str, 当前轮识别的意图
            extracted_slots: dict, 当前轮提取的槽位值
        """
        self.turn_count += 1
        
        # 更新意图（支持意图变更）
        self.state["current_intent"] = intent
        
        # 增量更新槽位值（保留之前已填充的槽位）
        for slot, value in extracted_slots.items():
            if slot in self.slots_def and value is not None:
                self.state[slot] = value
    
    def get_missing_slots(self):
        """获取尚未填充的槽位"""
        return [s for s in self.slots_def if self.state[s] is None]
    
    def is_complete(self):
        """检查所有必要槽位是否已填充"""
        return len(self.get_missing_slots()) == 0
    
    def get_state_summary(self):
        """获取状态摘要"""
        return {
            "turn": self.turn_count,
            "intent": self.state.get("current_intent"),
            "filled": {k: v for k, v in self.state.items() if v and k != "current_intent"},
            "missing": self.get_missing_slots(),
            "complete": self.is_complete()
        }

# 模拟对话状态跟踪

})
```

```python
class RuleBasedPolicy:
    """基于规则的对话策略"""
    def __init__(self):
        self.action_templates = {
            "ask_slot": {
                "destination": "请问您要去哪个城市？",
                "departure_date": "请问您计划哪天出发？",
                "departure_location": "请问您从哪个城市出发？",
                "passengers": "请问几位乘客？",
            },
            "confirm": "请确认您的订单：{date}从{depart}飞往{dest}，{pax}位乘客。",
            "search_and_present": "正在为您查询{date}从{depart}飞往{dest}的航班...",
            "greeting": "您好！我是航班助手，有什么可以帮您？",
            "goodbye": "感谢使用，祝您旅途愉快！",
        }
    
    def select_action(self, state):
        """
        根据当前对话状态选择下一步动作
        
        Args:
            state: DialogueStateTracker实例
        Returns:
            action: (action_type, params)
        """
        # 首次对话
        if state.turn_count == 0:
            return ("greeting", None)
        
        # 用户结束对话
        if state.state.get("current_intent") == "goodbye":
            return ("goodbye", None)
        
        # 检查缺失槽位 → 逐一询问
        missing = state.get_missing_slots()
        if missing:
            slot_to_ask = missing[0]
            return ("ask_slot", slot_to_ask)
        
        # 所有槽位已填充 → 确认并执行
        return ("confirm_and_execute", state.state)
    
    def generate_response(self, action, state):
        """生成自然语言回复"""
        action_type, params = action
        
        if action_type == "greeting":
            return self.action_templates["greeting"]
        
        elif action_type == "ask_slot":
            return self.action_templates["ask_slot"][params]
        
        elif action_type == "confirm_and_execute":
            s = state.state
            return self.action_templates["confirm"].format(
                date=s.get("departure_date", "未知"),
                depart=s.get("departure_location", "未知"),
                dest=s.get("destination", "未知"),
                pax=s.get("passengers", "未知")
            )
        
        elif action_type == "goodbye":
            return self.action_templates["goodbye"]
        
        return "抱歉，我不太理解您的问题。"
    
    def generate_response(self, action, state):
```

## 基于LLM的端到端对话系统

LLM的出现重构了对话系统范式——不再需要复杂的Pipeline，而是通过精巧的Prompt设计直接驱动对话。

```
┌─────────────────────────────────────────────────────────────────────┐
│           Pipeline方案 vs LLM端到端方案                                │
├─────────────┬─────────────────────────┬─────────────────────────────┤
│  维度        │  传统Pipeline            │  LLM端到端                   │
├─────────────┼─────────────────────────┼─────────────────────────────┤
│  模块化       │  NLU→DST→POL→NLG       │  单一大模型完成所有            │
│  开发成本     │  各模块独立开发/标注      │  需高质量提示词工程           │
│  可控性       │  高，每步可干预           │  中，通过prompt/system控制    │
│  灵活性       │  低，新增领域需重开发      │  高，直接适配                │
│  维护成本     │  高，多模块串联维护        │  低，主要维护prompt          │
│  推理效率     │  高（小模型组合）          │  较低（大模型推理）           │
│  适用场景     │  高频/确定领域             │  灵活多领域/长尾场景          │
└─────────────┴─────────────────────────┴─────────────────────────────┘
```

```python
        hotels = {
            "北京": [{"name": "北京国际饭店", "rating": 4.5, "price": 688},
                    {"name": "如家精选", "rating": 4.0, "price": 328}],
            "上海": [{"name": "上海外滩酒店", "rating": 4.8, "price": 888},
                    {"name": "汉庭酒店", "rating": 3.8, "price": 258}]
        }
        city_hotels = hotels.get(city, [{"name": f"{city}商务酒店", "rating": 4.0, "price": 400}])
        
        return {
            "hotels": city_hotels,
            "check_in": check_in,
            "check_out": check_out,
            "guests": guests,
            "total_nights": 1,  # 简化计算
            "status": "confirmed"
        }
    
    def _get_weather(self, city, date=None):
        """模拟天气查询API"""
        weather_data = {
            "北京": {"temp": 28, "condition": "晴", "humidity": "45%"},
            "上海": {"temp": 32, "condition": "多云", "humidity": "70%"},
            "广州": {"temp": 35, "condition": "雷阵雨", "humidity": "85%"}
        }
        weather = weather_data.get(city, {"temp": 25, "condition": "晴", "humidity": "50%"})
        weather["city"] = city
        weather["date"] = date or datetime.now().strftime("%Y-%m-%d")
        return weather
    
    def process_turn(self, user_input: str) -> str:
        """处理单轮对话"""
        # 添加用户消息到历史
        self.conversation_history.append({
            "role": "user", "content": user_input
        })
        
        # 调用LLM
        response = self._call_llm_with_functions()
        
        # 处理函数调用
        final_response = self._handle_response(response)
        
        return final_response
    
    def _call_llm_with_functions(self):
        """调用LLM并传入函数定义"""
        response = self.client.chat.completions.create(
            model=self.model,
            messages=self.conversation_history,
            functions=self.functions,
            function_call="auto",
            temperature=0.7
        )
        return response.choices[0].message
    
    def _handle_response(self, message) -> str:
        """处理LLM响应，递归处理函数调用"""
        if message.function_call:
            # LLM请求调用函数
            func_name = message.function_call.name
            func_args = json.loads(message.function_call.arguments)
            
            # 执行实际函数
            if func_name in self.function_registry:
                result = self.function_registry[func_name](**func_args)
            else:
                result = {"error": f"未知函数: {func_name}"}
            
            # 将函数结果添加到对话历史
            self.conversation_history.append({
                "role": "assistant",
                "content": None,
                "function_call": {
                    "name": func_name,
                    "arguments": message.function_call.arguments
                }
            })
            
            self.conversation_history.append({
                "role": "function",
                "name": func_name,
                "content": json.dumps(result, ensure_ascii=False)
            })
            
            # 再次调用LLM生成最终回复
            final_message = self._call_llm_with_functions()
            response_text = final_message.content or ""
        else:
            response_text = message.content or ""
        
        # 添加助手回复到历史
        self.conversation_history.append({
            "role": "assistant", "content": response_text
        })
        
        return response_text
    
    def get_conversation_summary(self) -> str:
        """获取对话摘要（用于长期记忆）"""
        summary_prompt = [
            {"role": "system", "content": "请用一段话总结以下对话的要点。"},
            {"role": "user", "content": json.dumps(
                [m for m in self.conversation_history if m["role"] != "system"],
                ensure_ascii=False
            )}
        ]
        
        response = self.client.chat.completions.create(
            model=self.model,
            messages=summary_prompt,
            max_tokens=200
        )
        
        return response.choices[0].message.content
    
    def reset(self):
        """重置对话"""
        self.conversation_history = []

```
```
┌─────────────────────────────────────────────────────────────────────┐
```

```
│                                                                      │
│  ┌─────────────────────────────────────────────────────┐            │

│  │  - 当前对话的最近N轮历史                               │            │
│  │  - 存储方式：直接拼接在prompt中                        │            │
```

```python
import numpy as np
from typing import List, Tuple

class SimpleMemoryManager:
    """简化版对话记忆管理器"""
    
    def __init__(self, max_short_term_turns: int = 10, 
                 long_term_threshold: int = 5):
        self.short_term = []  # 短期记忆：最近对话
        self.long_term = []   # 长期记忆：关键信息摘要
        self.max_turns = max_short_term_turns
        self.threshold = long_term_threshold
    
    def add_turn(self, user: str, assistant: str):
        """添加一轮对话到短期记忆"""
        self.short_term.append({"user": user, "assistant": assistant})
        
        # 超过阈值时，压缩早期对话到长期记忆
        if len(self.short_term) > self.max_turns:
            old_turns = self.short_term[:self.threshold]
            self.short_term = self.short_term[self.threshold:]
            
            # 生成摘要存入长期记忆
            summary = self._summarize(old_turns)
            self.long_term.append(summary)
    
    def _summarize(self, turns: List[dict]) -> str:
        """简单的摘要生成（实际应用中用LLM生成）"""
        topics = [f"{t['user']} → {t['assistant'][:50]}" for t in turns]
        return " | ".join(topics)
    
    def get_context(self) -> str:
        """获取完整的上下文信息"""
        context_parts = []
        
        if self.long_term:
            context_parts.append("【历史对话摘要】\n" + "\n".join(self.long_term))
        
        if self.short_term:
            recent = "\n".join(
                f"用户: {t['user']}\n助手: {t['assistant']}" 
                for t in self.short_term
            )
            context_parts.append(f"��最近对话】\n{recent}")
        
        return "\n\n".join(context_parts)
    
    def search_long_term(self, query: str) -> List[str]:
        """检索长期记忆中与查询相关的内容"""
        # 简化：返回包含关键词的记忆
        matches = [m for m in self.long_term if query in m]
        return matches if matches else self.long_term[-3:]


# 演示

    ("北京", "北京今天晴天，28°C。"),
    ("谢谢", "不客气！还有什么需要吗？"),
    ("帮我订机票", "好的，请问从哪出发，去哪里？"),
]

    memory.add_turn(user, assistant)
```

## 对话评估

对话系统的评估远比传统NLP任务复杂，因为一个好的回复不仅需要"正确"，还需要"合适"、"流畅"、"有信息量"。

```
┌─────────────────────────────────────────────────────────────────────┐
│                    对话系统评估体系                                   │
├──────────┬────────────────────────────┬─────────────────────────────┤
│  评估维度  │  方法                        │  局限性                      │
├──────────┼────────────────────────────┼─────────────────────────────┤
│  流畅度    │  困惑度(Perplexity)          │  不反映回复质量               │
│  多样性    │  Distinct-N (去重n-gram比例) │  只衡量词级别                 │
│  相关性    │  嵌入相似度                   │  需要参考回复                │
│  一致性    │  自一致/事实验证              │  难以自动化                   │
│  任务完成  │  槽位填充率/任务成功率          │  需要标注                    │
│  人工评估  │  Likert量表/成对比较          │  成本高，不可规模化            │
└──────────┴────────────────────────────┴─────────────────────────────┘
```

```python
import math
from collections import Counter

def distinct_n(text: str, n: int = 2) -> float:
    """
    Distinct-N: 衡量回复多样性
    值越高，回复中的重复模式越少（多样性越好）
    """
    tokens = text.split()
    if len(tokens) < n:
        return 0.0
    
    ngrams = [tuple(tokens[i:i+n]) for i in range(len(tokens) - n + 1)]
    unique_ngrams = len(set(ngrams))
    total_ngrams = len(ngrams)
    
    return unique_ngrams / total_ngrams if total_ngrams > 0 else 0.0

def response_length_diversity(responses: list) -> dict:
    """评估一组回复的多样性指标"""
    all_text = " ".join(responses)
    
    return {
        "avg_length": sum(len(r.split()) for r in responses) / len(responses),
        "distinct_1": distinct_n(all_text, 1),
        "distinct_2": distinct_n(all_text, 2),
        "distinct_3": distinct_n(all_text, 3),
        "num_responses": len(responses)
    }
```

## 小结

| 要点 | 内容 |
|------|------|
| 对话系统三分类 | 任务型（完成目标）/ 闲聊型（开放式）/ 问答型（事实性）|
| 传统Pipeline | NLU（意图+槽位）→ DST（状态更新）→ POL（动作选择）→ NLG（回复生成）|
| NLU核心技术 | 意图分类 + 槽位填充（BIO序列标注），可用Joint模型联合学习 |
| DST核心职责 | 增量维护对话状态，追踪已填充/缺失槽位，处理指代消解 |
| POL核心决策 | 缺失槽位则追问 → 全部填充则确认执行 → 任务完成则结束 |
| LLM端到端优势 | 无需复杂Pipeline，通过Prompt直接驱动，灵活适应新领域 |
| Function Calling | LLM决定何时调用外部API → Runtime执行 → 结果回传LLM生成回复 |
| 记忆机制 | 短期记忆（对话历史引用）/ 长期记忆（向量数据库语义检索）|
| 对话评估难点 | BLEU/ROUGE不适用，需结合自动指标（Perplexity/Distinct）和人工评估 |
| 评估核心指标 | 流畅度/多样性/相关性/一致性/任务完成率 |
