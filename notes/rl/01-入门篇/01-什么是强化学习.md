# 什么是强化学习

## 核心定义

**强化学习（Reinforcement Learning, RL）** 是机器学习的三大分支之一，研究**智能体（Agent）**如何通过与**环境（Environment）**的交互，通过试错来学习最优的**策略（Policy）**，以最大化长期累积**奖励（Reward）**。

```
┌─────────────────────────────────────────────────┐
│              强化学习交互循环                      │
│                                                 │
│         ┌─────────────┐                         │
│         │   智能体     │                         │
│         │   Agent     │                         │
│         └──────┬──────┘                         │
│                │ 动作 Action (aₜ)               │
│                ▼                                │
│         ┌─────────────┐                         │
│         │    环境      │                         │
│         │ Environment │                         │
│         └──────┬──────┘                         │
│                │ 状态 State (sₜ₊₁)              │
│                │ 奖励 Reward (rₜ₊₁)             │
│                ▼                                │
│         ┌─────────────┐                         │
│         │   智能体     │                         │
│         │   Agent     │                         │
│         └─────────────┘                         │
│                                                 │
└─────────────────────────────────────────────────┘
```

## 核心概念

### 1. 智能体（Agent）
学习和做决策的主体。例如：下棋的AI、自动驾驶系统、游戏角色。

### 2. 环境（Environment）
智能体交互的外部世界。例如：棋盘、道路交通系统、游戏世界。

### 3. 状态（State, s）
对环境的描述，包含智能体做决策所需的所有信息。

```python
# 示例：CartPole中的状态
# [小车位置, 小车速度, 杆子角度, 杆子角速度]
state = [0.05, 0.02, -0.03, 0.01]
```

### 4. 动作（Action, a）
智能体可以执行的操作。

```python
# 离散动作空间：CartPole（左推/右推）
actions = [0, 1]  # 0=向左推, 1=向右推

# 连续动作空间：机械臂控制
actions = [joint1_torque, joint2_torque, joint3_torque]  # 连续值
```

### 5. 奖励（Reward, r）
环境对智能体动作的即时反馈信号。

```python
# CartPole的奖励设计
reward = 1.0  # 每保持杆子直立一步，得1分
# 游戏结束（杆子倒了）时回合终止
```

### 6. 策略（Policy, π）
智能体根据状态选择动作的规则，是RL的核心。

```python
# 确定性策略：π(s) = a
action = policy(state)  # 直接输出一个动作

# 随机性策略：π(a|s) = P(a|s)
action_probabilities = policy(state)  # 输出动作的概率分布
action = np.random.choice(actions, p=action_probabilities)
```

### 7. 回报（Return, G）
从时刻t开始，未来累积折扣奖励：

$$G_t = r_{t+1} + \gamma r_{t+2} + \gamma^2 r_{t+3} + ... = \sum_{k=0}^{\infty} \gamma^k r_{t+k+1}$$

其中 γ ∈ [0, 1] 是**折扣因子**：

| γ值 | 含义 | 效果 |
|-----|------|------|
| γ = 0 | 只看当前奖励 | 短视策略 |
| γ = 1 | 未来奖励不衰减 | 长远规划（可能不收敛） |
| γ = 0.99 | 常用值 | 兼顾当前与未来 |
| γ = 0.9 | 较短视 | 快速收敛 |

## 与监督学习的区别

| 维度 | 监督学习 | 强化学习 |
|------|----------|----------|
| **数据** | 有标注的静态数据集 | 通过交互动态生成 |
| **反馈** | 即时、明确的标签 | 延迟的奖励信号 |
| **目标** | 最小化预测误差 | 最大化累积奖励 |
| **探索** | 不需要探索 | 必须主动探索未知 |
| **数据分布** | 固定分布 | 受策略影响而变化 |
| **示例** | 图像分类、房价预测 | 下棋、玩游戏、机器人控制 |

## RL问题分类

```
强化学习
├── 按环境是否已知
│   ├── 基于模型 (Model-Based)：已知环境动力学
│   └── 无模型 (Model-Free)：未知环境，通过交互学习
│
├── 按学习方法
│   ├── 基于价值 (Value-Based)：学习Q值，如DQN
│   ├── 基于策略 (Policy-Based)：直接优化策略，如REINFORCE
│   └── Actor-Critic：结合两者，如PPO、SAC
│
├── 按动作空间
│   ├── 离散动作：游戏AI、棋类
│   └── 连续动作：机器人控制、自动驾驶
│
└── 按智能体数量
    ├── 单智能体 (Single-Agent)
    └── 多智能体 (Multi-Agent)
```

| 领域 | 示例 | 使用的算法 |
|------|------|------------|
| **游戏** | AlphaGo（围棋）、OpenAI Five（Dota2） | MCTS + 深度RL |

## RL的核心挑战

### 1. 探索-利用困境（Exploration-Exploitation Tradeoff）
- **利用**：选择当前已知的最佳动作（可能错过更好的）
- **探索**：尝试新动作（可能短期内获得较少奖励）

```python
# ε-greedy策略
def epsilon_greedy(q_values, epsilon=0.1):
    if random.random() < epsilon:
        return random.choice(actions)  # 探索
    else:
        return argmax(q_values)        # 利用
```

### 2. 延迟奖励（Credit Assignment Problem）
动作的效果可能在很久之后才体现：
- 下棋：某一步的好坏直到终局才知道
- 俄罗斯方块：堆积的错误在游戏结束时才爆发

### 3. 样本效率
RL通常需要大量交互数据，这在现实世界中代价很高。

### 4. 稳定性与收敛性
深度RL训练不稳定，对超参数敏感。

## 第一个直觉：多臂老虎机

最简单的RL问题——多臂老虎机（Multi-Armed Bandit），没有状态，只有动作和奖励：

```python
import numpy as np

class MultiArmedBandit:
    """多臂老虎机：最简单的RL问题"""
    
    def __init__(self, k=10):
        # k个摇臂，每个有真实期望奖励
        self.k = k
        self.q_true = np.random.normal(0, 1, k)  # 真实值
    
    def pull(self, action):
        """拉杆，返回奖励（有噪声）"""
        reward = np.random.normal(self.q_true[action], 1)
        return reward

# ε-greedy策略
def solve_bandit(bandit, steps=1000, epsilon=0.1):
    q_est = np.zeros(bandit.k)  # 估计值
    n = np.zeros(bandit.k)      # 每个臂被拉次数
    
    for step in range(steps):
        if np.random.random() < epsilon:
            action = np.random.randint(bandit.k)  # 探索
        else:
            action = np.argmax(q_est)              # 利用
        
        reward = bandit.pull(action)
        n[action] += 1
        q_est[action] += (reward - q_est[action]) / n[action]  # 增量更新
    
    return q_est

# 运行
bandit = MultiArmedBandit(k=10)
estimates = solve_bandit(bandit)
print(f"估计值: {estimates}")
print(f"真实值: {bandit.q_true}")
print(f"最优臂: {np.argmax(bandit.q_true)} (真实) vs {np.argmax(estimates)} (估计)")
```
