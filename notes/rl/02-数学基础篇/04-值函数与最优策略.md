# 值函数与最优策略

## 两种值函数

### 状态值函数 V(s)

在策略π下，状态s的长期价值：

$$V^\pi(s) = E_\pi\left[\sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \bigg| S_t = s\right]$$

### 动作值函数 Q(s,a)

在状态s执行动作a，之后遵循策略π的长期价值：

$$Q^\pi(s,a) = E_\pi\left[\sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \bigg| S_t = s, A_t = a\right]$$

### V与Q的关系

```
V^π(s) = Σ_a π(a|s) · Q^π(s,a)           # 对动作求期望

Q^π(s,a) = Σ_s' P(s'|s,a) [R + γ·V^π(s')] # 展开下一状态

最优关系:
V*(s) = max_a Q*(s,a)
```

## 最优策略

**最优策略** $\pi^*$ 是使值函数最大化的策略：

$$\pi^*(s) = \arg\max_a Q^*(s, a)$$

```python
import numpy as np

# 从Q表提取最优策略
def extract_optimal_policy(Q):
    """从Q*提取贪心策略"""
    policy = np.argmax(Q, axis=1)  # 每个状态选Q值最大的动作
    return policy

# 示例
Q_star = np.array([
    [1.0, 5.0, 2.0, 0.5],  # s0: 最优动作是a1（Q=5.0）
    [3.0, 1.0, 4.0, 2.0],  # s1: 最优动作是a2（Q=4.0）
    [0.5, 0.3, 0.1, 0.9],  # s2: 最优动作是a3（Q=0.9）
])

policy = extract_optimal_policy(Q_star)
print(f"最优策略: {policy}")  # [1, 2, 3]
```

## 策略改进定理

如果对于所有状态s：

$$Q^\pi(s, \pi'(s)) \geq V^\pi(s)$$

那么 $\pi'$ 至少和π一样好：$V^{\pi'}(s) \geq V^\pi(s)$。

**推论**：贪心策略 $\pi'(s) = \arg\max_a Q^\pi(s,a)$ 保证不劣于原策略。

```python
def policy_improvement(env, V, gamma=0.99):
    """策略改进：基于V^π计算更好的策略"""
    n_states = env.n_states
    n_actions = env.n_actions
    new_policy = np.zeros((n_states, n_actions))
    
    for s in range(n_states):
        # 计算每个动作的Q值
        q_values = np.zeros(n_actions)
        for a in range(n_actions):
            for prob, next_s, reward, done in env.get_transitions(s, a):
                v_next = 0 if done else V[next_s]
                q_values[a] += prob * (reward + gamma * v_next)
        
        # 贪心选择最优动作
        best_action = np.argmax(q_values)
        new_policy[s, best_action] = 1.0  # 确定性贪心策略
    
    return new_policy
```

## 策略迭代算法

交替进行策略评估和策略改进，直到收敛：

```
初始化随机策略 π₀
重复：
  1. 策略评估：计算 V^π
  2. 策略改进：π_new = greedy(V^π)
  3. 如果 π_new == π，停止
```

```python
def policy_iteration(env, gamma=0.99, theta=1e-8, max_iter=1000):
    """完整的策略迭代算法"""
    n_states = env.n_states
    n_actions = env.n_actions
    
    # 初始化随机策略
    policy = np.ones((n_states, n_actions)) / n_actions
    
    for iteration in range(max_iter):
        # 1. 策略评估
        V = policy_evaluation(env, policy, gamma, theta)
        
        # 2. 策略改进
        new_policy = policy_improvement(env, V, gamma)
        
        # 3. 检查是否收敛
        if np.array_equal(new_policy, policy):
            print(f"策略迭代在第{iteration+1}次迭代后收敛")
            break
        
        policy = new_policy
    
    return policy, V
```

## 优势函数（Advantage Function）

衡量动作a相对于平均水平的优势：

$$A^\pi(s, a) = Q^\pi(s, a) - V^\pi(s)$$

```python
# 优势函数的意义
"""
A > 0：动作a比平均水平好 → 应该更多选择
A < 0：动作a比平均水平差 → 应该更少选择
A = 0：动作a和平均水平一样

策略梯度中使用优势函数可以降低方差：
∇J(π) = E[ A^π(s,a) · ∇log π(a|s) ]
"""
```

## 完整示例：GridWorld策略迭代

```python
import numpy as np

class GridWorld:
    def __init__(self, size=4, gamma=0.99):
        self.size = size
        self.gamma = gamma
        self.n_states = size * size
        self.n_actions = 4
        self.terminal = [(0, 0), (size-1, size-1)]
        self.actions = [(-1,0), (0,1), (1,0), (0,-1)]
    
    def to_coord(self, s):
        return (s // self.size, s % self.size)
    
    def to_state(self, coord):
        return coord[0] * self.size + coord[1]
    
    def get_transitions(self, s, a):
        coord = self.to_coord(s)
        dr, dc = self.actions[a]
        new_coord = (coord[0] + dr, coord[1] + dc)
        if not (0 <= new_coord[0] < self.size and 0 <= new_coord[1] < self.size):
            new_coord = coord
        next_s = self.to_state(new_coord)
        done = new_coord in self.terminal
        reward = 0.0 if done else -1.0
        return [(1.0, next_s, reward, done)]


def policy_evaluation(env, policy, gamma=0.99, theta=1e-8):
    V = np.zeros(env.n_states)
    while True:
        delta = 0
        for s in range(env.n_states):
            if env.to_coord(s) in env.terminal:
                continue
            v = 0
            for a in range(env.n_actions):
                for prob, next_s, r, done in env.get_transitions(s, a):
                    v_next = 0 if done else V[next_s]
                    v += policy[s, a] * prob * (r + gamma * v_next)
            delta = max(delta, abs(V[s] - v))
            V[s] = v
        if delta < theta:
            break
    return V


def policy_improvement(env, V, gamma=0.99):
    policy = np.zeros((env.n_states, env.n_actions))
    for s in range(env.n_states):
        if env.to_coord(s) in env.terminal:
            continue
        q = np.zeros(env.n_actions)
        for a in range(env.n_actions):
            for prob, next_s, r, done in env.get_transitions(s, a):
                v_next = 0 if done else V[next_s]
                q[a] += prob * (r + gamma * v_next)
        best = np.argmax(q)
        policy[s, best] = 1.0
    return policy


def policy_iteration(env, gamma=0.99):
    policy = np.ones((env.n_states, env.n_actions)) / env.n_actions
    for i in range(1000):
        V = policy_evaluation(env, policy, gamma)
        new_policy = policy_improvement(env, V, gamma)
        if np.array_equal(new_policy, policy):
            print(f"收敛于第{i+1}次迭代")
            break
        policy = new_policy
    return policy, V


# 运行
env = GridWorld(size=4, gamma=0.99)
policy, V = policy_iteration(env)

print("\n最优状态值函数 V*:")
for r in range(4):
    print(" ".join(f"{V[r*4+c]:6.2f}" for c in range(4)))

print("\n最优策略:")
action_symbols = ['↑', '→', '↓', '←']
for r in range(4):
    row = []
    for c in range(4):
        s = r * 4 + c
        if (r, c) in env.terminal:
            row.append('  G')
        else:
            a = np.argmax(policy[s])
            row.append(f"  {action_symbols[a]}")
    print("".join(row))

# 输出:
# 最优策略:
#   G  ←  ←  ←
#   ↑  ↑  ↑  ↓
#   ↑  ↑  ↓  ↓
#   ↑  →  →  G
```

## 值函数估计误差

### Bellman误差

$$\text{Bellman Error} = |V(s) - (R + \gamma V(s'))|$$

### TD误差

$$\delta_t = R_{t+1} + \gamma V(S_{t+1}) - V(S_t)$$

```python
# TD误差在RL中非常重要
# 它是值函数更新的基础
# V(s) ← V(s) + α · δ_t

# 如果δ_t > 0：实际比预期好，应该增加V(s)
# 如果δ_t < 0：实际比预期差，应该减少V(s)
```

## 收敛性分析

### 策略迭代

| 性质 | 说明 |
|------|------|
| **收敛保证** | 有限MDP必然在有限步内收敛 |
| **迭代次数** | 通常远少于状态数 |
| **每次迭代** | 策略评估(O(\|S\|³)) + 改进(O(\|S\|²\|A\|)) |

### 值迭代

| 性质 | 说明 |
|------|------|
| **收敛保证** | 贝尔曼算子是γ-压缩 |
| **收敛速度** | 误差每步缩小γ倍 |
| **复杂度** | 每次迭代O(\|S\|²\|A\|) |

## 小结

| 概念 | 公式 | 用途 |
|------|------|------|
| **V^π(s)** | $E[G_t \| S_t=s]$ | 状态价值评估 |
| **Q^π(s,a)** | $E[G_t \| S_t=s, A_t=a]$ | 动作价值评估 |
| **V\*(s)** | $\max_a Q^*(s,a)$ | 最优状态值 |
| **π\*(s)** | $\arg\max_a Q^*(s,a)$ | 最优策略 |
| **A^π(s,a)** | $Q^π(s,a) - V^π(s)$ | 动作优势 |
| **策略迭代** | 评估→改进→重复 | 求解最优策略 |
| **值迭代** | 直接求V* | 更简洁的求解 |
