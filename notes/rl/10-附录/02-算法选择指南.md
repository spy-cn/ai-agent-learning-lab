# RL 算法选择指南

面对一个 RL 问题，如何选择最合适的算法？本指南提供系统化的决策流程。

## 决策树

```
                        开始
                         │
                    ┌────┴────┐
                    │ 动作空间？│
                    └────┬────┘
                    ┌────┴────────┐
                 离散              连续
                    │                │
              ┌─────┴─────┐    ┌────┴────┐
            小规模       大规模    │ 样本效率？│
            (表格)      (神经网络)  └────┬────┘
              │           │        ┌────┴────┐
         Q-Learning    DQN系列   高优先    低优先
         SARSA       ┌──┴──┐       │        │
                     │样本效率？│   SAC     PPO
                     └──┬──┘  TD3(备选) A2C(快速)
                   ┌────┴────┐
                 高优先      低优先
                   │          │
              DQN+PER       PPO
              Rainbow      A2C
```

## 按场景选择

### 场景一：经典控制（CartPole, MountainCar）

```
推荐: PPO 或 DQN

理由:
  - 状态维度低 (≤10)
  - 动作离散且少 (2~3个)
  - 需要快速验证
  - SB3一行代码搞定
```

```python
from stable_baselines3 import PPO
model = PPO("MlpPolicy", env, verbose=1)
model.learn(total_timesteps=50000)
```

### 场景二：Atari 游戏

```
推荐: DQN (含变体) 或 PPO+CNN

DQN系列选择:
  - 基础DQN: 快速验证
  - Double DQN: 减少Q值过估计
  - Rainbow: 追求最优性能
  - R2D2: 部分可观测环境

PPO选择:
  - IMPALA-CNN 架构
  - 多环境并行 (8~16个)
  - 更稳定但样本效率略低
```

```python
# DQN for Atari
from stable_baselines3 import DQN
from stable_baselines3.common.atari_wrappers import AtariWrapper

env = AtariWrapper(gym.make('BreakoutNoFrameskip-v4'))
model = DQN("CnnPolicy", env, learning_rate=1e-4, 
            buffer_size=100000, verbose=1)
```

### 场景三：机器人连续控制（MuJoCo）

```
推荐: SAC (首选) 或 PPO

SAC 优势:
  - 样本效率最高
  - 自动温度调节
  - 内置探索
  - 稳定性好

PPO 优势:
  - 简单易调
  - 多环境并行快
  - 适合大规模分布式训练

不推荐:
  - DDPG: 不稳定
  - DQN: 仅支持离散动作
```

### 场景四：多智能体系统

```
推荐流程:
  1. 先尝试 IQL (最简单的基线)
  2. 如果有效但不够好 → MAPPO
  3. 如果需要集中训练 → MADDPG
  4. 竞争环境 → Self-Play

协作环境:  MAPPO > QMIX > MADDPG > IQL
竞争环境:  Self-Play + PPO/SAC
混合环境:  MADDPG (集中Critic)
```

### 场景五：仅有离线数据

```
推荐: TD3+BC (简单) 或 IQL (鲁棒)

关键约束:
  - 不能与环境交互
  - 需要避免外推误差
  - 数据质量至关重要

选择依据:
  - 数据质量高 → TD3+BC
  - 数据质量一般 → IQL
  - 需要保守估计 → CQL
  - 需要动作约束 → BCQ
```

## 关键决策因素

### 1. 动作空间类型

| 动作空间 | 推荐算法 | 不适用 |
|----------|----------|--------|
| 离散（少量） | Q-Learning, DQN, PPO | DDPG, TD3, SAC |
| 离散（大量） | PPO, Autoregressive | 标准DQN |
| 连续（低维） | SAC, TD3, PPO | Q-Learning |
| 连续（高维） | PPO, SAC | DDPG |
| 混合 | PPO, SAC(离散版) | — |

### 2. 样本效率需求

```
样本效率排名 (从高到低):
1. Model-Based (Dyna, MuZero, Dreamer)  ← 最少样本
2. Off-Policy (SAC, TD3, DQN)           ← 可复用旧数据
3. On-Policy (PPO, A2C, REINFORCE)      ← 只用新数据

如果环境交互成本高（如真实机器人）:
  → 必须用 Model-Based 或 Off-Policy

如果模拟器很快（如游戏）:
  → PPO 即可，简单稳定
```

### 3. 稳定性需求

```
稳定性排名 (从高到低):
1. PPO     ← 最稳定，调参容易
2. SAC     ← 很稳定，自动温度
3. TD3     ← 较稳定，需调超参
4. DQN     ← 中等，需调ε策略
5. DDPG    ← 不稳定，敏感
6. REINFORCE ← 高方差
```

### 4. 并行化需求

```
并行能力排名:
1. PPO/A2C  ← 天然支持多环境
2. A3C      ← 异步多线程
3. IMPALA   ← 分布式架构
4. SAC/TD3  ← 单环境即可（off-policy）
```

### 5. 状态空间维度

| 状态类型 | 维度 | 推荐特征提取 |
|----------|------|-------------|
| 向量（低维） | <20 | MLP [64, 64] 或 [128, 128] |
| 向量（高维） | 20~100 | MLP [256, 256] |
| 图像 | 84×84×4 | NatureCNN 或 IMPALA-CNN |
| 点云/3D | — | PointNet 或 Transformer |
| 图结构 | — | GNN |
| 文本/语言 | — | Transformer / BERT |

## 算法调参优先级

```
调参重要性 (从高到低):

1. 学习率 (learning_rate)
   → 影响最大，先调这个
   → 范围: 1e-5 ~ 1e-3 (对数搜索)

2. 奖励折扣 (gamma)
   → 影响长期规划能力
   → 范围: 0.9 ~ 0.999

3. 网络架构 (hidden_dims)
   → 影响表达能力
   → 常用: [64,64], [128,128], [256,256]

4. 探索参数
   → ε (DQN): 衰减率
   → α (SAC): 自动或 [0.1, 0.2]
   → clip_range (PPO): 0.1~0.3

5. Batch Size
   → 影响梯度估计质量
   → 常用: 32, 64, 128, 256

6. Replay Buffer Size
   → 影响数据多样性
   → 常用: 1e4~1e6

7. 其他算法特有参数
   → GAE lambda, tau, ent_coef 等
```

## SB3 一行代码选择

```python
# === 终极选择指南 ===

# 离散动作 + 简单任务
model = DQN("MlpPolicy", env)

# 离散动作 + 图像输入
model = DQN("CnnPolicy", env)

# 连续动作 + 样本效率优先
model = SAC("MlpPolicy", env)

# 连续动作 + 稳定性优先
model = PPO("MlpPolicy", env)

# 连续动作 + 精确控制
model = TD3("MlpPolicy", env)

# 多环境并行 + 快速训练
model = PPO("MlpPolicy", vec_env)  # vec_env含多个并行环境

# 离散动作 + 快速基线
model = A2C("MlpPolicy", env)
```

## 避坑指南

| 常见错误 | 正确做法 |
|----------|----------|
| 从复杂算法开始 | 先跑通 DQN/PPO 基线 |
| 不做观测归一化 | 用 VecNormalize |
| 不设随机种子 | set_seed(42) |
| 只跑一个种子 | 至少 3~5 个种子 |
| 不保存检查点 | 用 CheckpointCallback |
| 不监控训练 | 用 TensorBoard/W&B |
| 学习率太大 | 从 3e-4 开始 |
| 网络太大 | 从 [64,64] 开始 |
| 忽视奖励设计 | 仔细设计奖励函数 |
| 不评估确定性策略 | evaluate(deterministic=True) |
