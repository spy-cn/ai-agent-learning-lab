# 分层强化学习（Hierarchical RL）

分层RL通过**分层抽象**处理长horizon任务，将复杂问题分解为多个层次的子问题。

## 为什么需要分层RL？

```
问题: 长horizon任务的奖励极度稀疏

例: "去厨房拿可乐"
  状态: 整个房子的位置
  动作: 前后左右移动
  奖励: 只有拿到可乐才有

传统RL: 在上千万步的探索后可能偶然学会
分层RL: 分解为高层"去厨房" → 低层"导航到目标"
```

```
┌──────────────────────────────────────────────┐
│            分层RL架构                         │
│                                              │
│  高层策略 (Manager):                          │
│    输入: 当前状态 + 目标                       │
│    输出: 子目标 (如"去厨房")                   │
│    时间尺度: 每100步决策一次                  │
│         │                                    │
│         ▼ 子目标                              │
│  低层策略 (Worker):                           │
│    输入: 当前状态 + 子目标                     │
│    输出: 原子动作 (如"向前走")                 │
│    时间尺度: 每步决策                         │
│         │                                    │
│         ▼ 原子动作                            │
│  环境                                         │
│                                              │
└──────────────────────────────────────────────┘
```

## 经典框架

### 1. Options框架

Options = 子策略 + 终止条件

```
Option = (策略π, 终止函数β)

π: S × A → [0,1]    子策略
β: S → [0,1]         终止概率

高层在option之间选择，低层执行option直到终止
```

```python
import torch
import torch.nn as nn
import numpy as np

class Option:
    """一个Option = 子策略 + 终止函数"""
    
    def __init__(self, option_id, policy_net, termination_net, 
                 initiation_set=None):
        self.id = option_id
        self.policy = policy_net      # 低层策略
        self.termination = termination_net  # 终止函数
        self.initiation_set = initiation_set  # 可启动此option的状态集
    
    def act(self, state):
        """执行option"""
        action = self.policy(state)
        return action
    
    def should_terminate(self, state):
        """是否终止此option"""
        prob = self.termination(state)
        return np.random.random() < prob


class HierarchicalAgent:
    """分层RL智能体"""
    
    def __init__(self, n_options, state_dim, action_dim):
        # 高层策略: 选择option
        self.meta_policy = nn.Sequential(
            nn.Linear(state_dim, 128),
            nn.ReLU(),
            nn.Linear(128, n_options)
        )
        
        # 多个option
        self.options = [
            Option(
                i,
                policy_net=LowLevelPolicy(state_dim, action_dim),
                termination_net=TerminationNet(state_dim)
            )
            for i in range(n_options)
        ]
        
        self.current_option = None
        self.option_duration = 0
    
    def act(self, state):
        """分层决策"""
        # 检查是否需要选新option
        if self.current_option is None or \
           self.current_option.should_terminate(state) or \
           self.option_duration >= 50:  # 最大持续时间
            
            # 高层选择option
            logits = self.meta_policy(state)
            probs = torch.softmax(logits, dim=-1)
            option_id = torch.multinomial(probs, 1).item()
            self.current_option = self.options[option_id]
            self.option_duration = 0
        
        # 低层执行
        action = self.current_option.act(state)
        self.option_duration += 1
        
        return action, self.current_option.id
```

### 2. FeUdal Networks (FeUdal)

Manager输出**隐式目标**（隐空间中的方向），Worker执行：

```python
class FeUdalNetworks(nn.Module):
    """FeUdal Networks: Manager-Worker架构"""
    
    def __init__(self, state_dim, action_dim, 
                 goal_dim=64, hidden_dim=128, k=10):
        super().__init__()
        self.goal_dim = goal_dim
        self.k = k  # Manager每k步决策一次
        
        # 状态编码器（共享）
        self.percept = nn.Sequential(
            nn.Linear(state_dim, hidden_dim),
            nn.ReLU(),
        )
        
        # Manager: 输出目标方向
        self.manager = nn.LSTMCell(hidden_dim, goal_dim)
        
        # Worker: 翻译目标为动作
        self.worker = nn.ModuleDict({
            'goal_embed': nn.Linear(goal_dim, action_dim * k),
            'rnn': nn.LSTMCell(hidden_dim + action_dim, action_dim * k),
            'action': nn.Linear(action_dim * k, action_dim)
        })
        
        self.manager_hidden = None
        self.worker_hidden = None
        self.current_goal = None
        self.step_count = 0
    
    def forward(self, state):
        features = self.percept(state)
        
        # Manager每k步更新一次目标
        if self.step_count % self.k == 0:
            self.manager_hidden = self.manager(features, self.manager_hidden or 
                (torch.zeros(1, self.goal_dim), torch.zeros(1, self.goal_dim)))
            self.current_goal = self.manager_hidden[0]  # h方向作为目标
        
        # Worker: 目标 → 动作分布
        goal_embed = self.worker['goal_embed'](self.current_goal)
        worker_input = torch.cat([features, goal_embed], dim=-1)
        self.worker_hidden = self.worker['rnn'](worker_input, self.worker_hidden or
            (torch.zeros(1, self.goal_dim), torch.zeros(1, self.goal_dim)))
        logits = self.worker['action'](self.worker_hidden[0])
        
        self.step_count += 1
        return logits
    
    def intrinsic_reward_worker(self, goals_history, states_history):
        """Worker的内在奖励: cos(goal, state变化)"""
        if len(states_history) < 2:
            return 0
        
        state_diff = states_history[-1] - states_history[-2]
        goal = goals_history[-1]
        
        # 余弦相似度作为奖励
        cos_sim = torch.cosine_similarity(goal, state_diff, dim=-1)
        return cos_sim.item() / self.k
```

```python
            # Manager设定新目标
            self.current_goal = self.manager.act(state)

        worker_input = torch.cat([state, self.current_goal])
        action = self.worker.select_action(worker_input)
        
        return action
    
    def step(self, state, action, reward, next_state, done):
        """存储经验"""
        self.transition_buffer.append((state, self.current_goal, next_state))
        
        # Worker经验: 内在奖励 = -到目标的距离
        worker_input = torch.cat([state, self.current_goal])
        worker_next_input = torch.cat([next_state, self.current_goal])
        
        # Worker奖励: 状态是否接近目标
        state_diff = next_state - state
        intrinsic_reward = -torch.norm(state_diff - self.current_goal)
        
        self.worker_buffer.add(worker_input, action, intrinsic_reward, 
                              worker_next_input, done)
        
        # 每c步更新Manager
        if len(self.transition_buffer) >= self.c:
            self.update_manager(state, done)
    
    def update_manager(self, current_state, done):
        """更新Manager（带目标重标记）"""
```

## 子目标发现

自动发现有用的子目标：

```python
class SubgoalDiscovery:
    """子目标发现"""
    
    def __init__(self, state_dim):
        self.state_dim = state_dim
    
    def discover_via_clustering(self, visited_states):
        """用聚类发现常见的状态簇"""
        from sklearn.cluster import KMeans
        
        states = np.array(visited_states)
        kmeans = KMeans(n_clusters=10)
        kmeans.fit(states)
        
        # 簇中心作为子目标
        subgoals = kmeans.cluster_centers_
        return subgoals
    
    def discover_via_bottleneck(self, visited_states, transitions):
        """发现瓶颈状态（连接不同区域的关节点）"""
        # 用图论方法找割点
        from collections import defaultdict
        
        # 构建状态转移图
        graph = defaultdict(set)
        for s, s_next in transitions:
            graph[tuple(s)].add(tuple(s_next))
        
        # 找瓶颈（图中割点）
        # ... 简化版: 找度数低但连接重要区域的状态
        
        bottlenecks = self.find_articulation_points(graph)
        return bottlenecks
```

## 分层RL的优势

| 优势 | 说明 |
|------|------|
| **处理长horizon** | 时间抽象减少决策频率 |
| **探索高效** | 高层指导探索方向 |
| **迁移** | 低层技能可跨任务复用 |
| **样本效率** | 分层结构加速学习 |
| **可解释性** | 层次化决策更易理解 |

## 应用场景

```
1. 视频游戏（Minecraft, StarCraft）
   高层: "采集木材" → "建造房屋" → "制作工具"
   低层: 移动、交互

2. 机器人操控
   高层: "到达位置A" → "抓取物体" → "移动到B"
   低层: 关节控制

3. 自动驾驶
   高层: 路线规划 → 变道决策
   低层: 方向盘、油门控制

4. 自然语言指令跟随
   高层: 解析"先开灯再关门"
   低层: 执行"开灯"/"关门"动作
```

## 小结

| 方法 | 分层机制 | 特点 |
|------|----------|------|
| **Options** | 子策略+终止条件 | 理论清晰 |
| **FeUral** | Manager-Worker | 隐式目标 |
| **HIRO** | Off-policy分层 | 样本高效 |
| **MAXQ** | 值函数分解 | 递归分解 |
| **h-DQN** | 双层DQN | 目标驱动 |

分层RL是解决长horizon、稀疏奖励问题的关键技术。
