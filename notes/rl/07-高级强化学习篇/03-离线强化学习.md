# 离线强化学习（Offline RL）

离线RL从**完全固定的数据集**中学习策略，不与环境交互。解决RL在真实世界中难以在线探索的问题。

## 为什么需要离线RL？

```
标准RL（在线）:
  环境 → 交互 → 学习 → 更好的策略 → 更多交互
  问题: 真实世界试错代价高（医疗、金融、机器人）

离线RL:
  固定数据集 → 学习 → 策略
  数据来源: 历史记录、日志、传感器数据
  不需要任何额外交互
```

```
┌──────────────────────────────────────────────────┐
│              RL数据来源对比                       │
├──────────────────┬───────────────────────────────┤
│     在线RL       │          离线RL               │
├──────────────────┼───────────────────────────────┤
│ 实时与环境交互    │ 从固定数据集学习              │
│ 探索未知行为      │ 不能探索新行为               │
│ 适合仿真环境     │ 适合真实世界应用              │
│ 例: PPO, SAC     │ 例: BCQ, CQL, IQL            │
└──────────────────┴───────────────────────────────┘
```

## 核心挑战：外推误差

```
问题: 数据集只包含行为策略采样的动作
      策略想评估数据集外的动作的Q值

  数据集D: {(s, a_behavior, r, s')}
  要评估: Q(s, a_new)  其中 a_new ≠ a_behavior

  Q网络对未见过的(s, a_new)组合可能给出任意值
  → 可能被错误高估 → 策略学到坏行为

  这就是 "Extrapolation Error" 外推误差
```

```
外推误差的来源:
┌─────────────────────────────────────────────┐
│  数据集覆盖:                                │
│  只有虚线区域的状态-动作被数据覆盖           │
│                                             │
│     action                                  │
│       │  ╱──────╲                          │
│       │ ╱ 数据  ╲╱                         │
│       │╱  覆盖  ╱│╲                        │
│       │──────╱──│──╲← 外推区域(未知)       │
│       │     ╱   │   ╲                      │
│       └──────→ state                      │
│                                             │
│  Q网络在外推区域可能给出任意大的值           │
│  max操作会选这些错误的值 → 过估计           │
└─────────────────────────────────────────────┘
```

## BCQ（Batch-Constrained Q-Learning）

BCQ限制策略只选择数据集中见过的动作：

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import numpy as np

class BCQ:
    """Batch-Constrained Q-Learning"""
    
    def __init__(self, state_dim, action_dim, max_action,
                 phi=0.05, lr=1e-3, gamma=0.99, tau=0.005):
        self.gamma = gamma
        self.tau = tau
        self.phi = phi  # 动作过滤阈值
        self.max_action = max_action
        
        # VAE生成数据集相似的动作
        self.vae = VAE(state_dim, action_dim, latent_dim=action_dim*2)
        
        # 扰动网络（微调动作）
        self.perturbation = PerturbationNetwork(state_dim, action_dim)
        
        # Twin Q网络
        self.q1 = QNetwork(state_dim, action_dim)
        self.q2 = QNetwork(state_dim, action_dim)
        self.q1_target = QNetwork(state_dim, action_dim)
        self.q2_target = QNetwork(state_dim, action_dim)
        self.q1_target.load_state_dict(self.q1.state_dict())
        self.q2_target.load_state_dict(self.q2.state_dict())
        
        self.q_optimizer = optim.Adam(
            list(self.q1.parameters()) + list(self.q2.parameters()), lr=lr
        )
    
    def select_action(self, state):
        """受约束的动作选择"""
        state = torch.FloatTensor(state).unsqueeze(0)
        
        # 1. 用VAE生成数据集相似的动作
        with torch.no_grad():
            generated_actions = self.vae.decode(
                self.vae.encode(state).sample(n=100)  # 生成100个候选
            )
        
        # 2. 用扰动网络微调
        perturbed = generated_actions + self.perturbation(state, generated_actions)
        perturbed = perturbed.clamp(-self.max_action, self.max_action)
        
        # 3. 选Q值最高的
        with torch.no_grad():
            q_values = self.q1(state, perturbed)
            best_idx = q_values.argmax()
            action = perturbed[best_idx]
        
        return action.numpy()
    
    def train(self, replay_buffer, batch_size=256, n_iterations=1000):
        for _ in range(n_iterations):
            states, actions, rewards, next_states, dones = replay_buffer.sample(batch_size)
            
            # === Q网络训练 ===
            with torch.no_grad():
                # 生成下一状态的动作
                next_actions = self.select_action_batch(next_states)
                q1_next = self.q1_target(next_states, next_actions)
                q2_next = self.q2_target(next_states, next_actions)
                min_q = torch.min(q1_next, q2_next)
                target_q = rewards + self.gamma * (1 - dones) * min_q
            
            current_q1 = self.q1(states, actions)
            current_q2 = self.q2(states, actions)
            
            q_loss = F.mse_loss(current_q1, target_q) + F.mse_loss(current_q2, target_q)
            
            self.q_optimizer.zero_grad()
            q_loss.backward()
            self.q_optimizer.step()
            
            # 软更新
            self.soft_update(self.q1, self.q1_target)
            self.soft_update(self.q2, self.q2_target)
```

## CQL（Conservative Q-Learning）

CQL通过**正则化**使数据集外动作的Q值降低：

$$L_{CQL} = \alpha \left[E_{s \sim D, a \sim \mu}\left[Q(s,a)\right] - E_{(s,a) \sim D}\left[Q(s,a)\right]\right] + L_{TD}$$

```python
class CQL:
    """Conservative Q-Learning"""
    
    def __init__(self, state_dim, action_dim, alpha=1.0, lr=1e-3):
        self.alpha = alpha  # 保守程度
        
        self.q1 = QNetwork(state_dim, action_dim)
        self.q2 = QNetwork(state_dim, action_dim)
        
        # Actor（用SAC式的策略）
        self.actor = GaussianPolicy(state_dim, action_dim)
        
        self.q_optimizer = optim.Adam(
            list(self.q1.parameters()) + list(self.q2.parameters()), lr=lr
        )
    
    def conservative_loss(self, states, actions, next_states):
        """CQL保守损失"""
        # 从策略采样随机动作
        with torch.no_grad():
            random_actions = torch.randn_like(actions).clamp(-1, 1)
            policy_actions, _, _ = self.actor.sample(states)
            next_policy_actions, _, _ = self.actor.sample(next_states)
        
        # Q值对数据集内外的差异
        q1_data = self.q1(states, actions)
        q1_random = self.q1(states, random_actions)
        q1_policy = self.q1(states, policy_actions)
        q1_next = self.q1(next_states, next_policy_actions)
        
        # 保守正则化: 降低数据集外动作的Q值
        logsumexp = torch.logsumexp(
            torch.cat([q1_random, q1_policy, q1_next], dim=0), dim=0
        ).mean()
        
        cql_loss = self.alpha * (logsumexp - q1_data.mean())
        
        return cql_loss
    
    def train_step(self, batch):
        states, actions, rewards, next_states, dones = batch
        
        # 标准TD损失
        with torch.no_grad():
            next_actions, log_probs, _ = self.actor.sample(next_states)
            q1_next = self.q1_target(next_states, next_actions)
            q2_next = self.q2_target(next_states, next_actions)
            min_q = torch.min(q1_next, q2_next) - log_probs
            target_q = rewards + self.gamma * (1 - dones) * min_q
        
        td_loss = F.mse_loss(self.q1(states, actions), target_q)
        
        # CQL保守损失
        cql_loss = self.conservative_loss(states, actions, next_states)
        
        total_loss = td_loss + cql_loss
        
        self.q_optimizer.zero_grad()
        total_loss.backward()
        self.q_optimizer.step()
```

## IQL（Implicit Q-Learning）

IQL通过**期望回归**避免查询数据集外的动作：

```python
class IQL:
    """Implicit Q-Learning"""
    
    def __init__(self, state_dim, action_dim, expectile=0.7, 
                 temperature=3.0, lr=3e-4):
        self.expectile = expectile  # 期望回归参数
        self.temperature = temperature
        
        # V网络（不需要目标网络）
        self.v_net = QNetwork(state_dim, 1)
        
        # Twin Q网络
        self.q1 = QNetwork(state_dim, action_dim)
        self.q2 = QNetwork(state_dim, action_dim)
        self.q1_target = QNetwork(state_dim, action_dim)
        self.q2_target = QNetwork(state_dim, action_dim)
        self.q1_target.load_state_dict(self.q1.state_dict())
        self.q2_target.load_state_dict(self.q2.state_dict())
        
        # Advantage-Weighted Regression策略
        self.actor = GaussianPolicy(state_dim, action_dim)
        
        self.v_optimizer = optim.Adam(self.v_net.parameters(), lr=lr)
        self.q_optimizer = optim.Adam(
            list(self.q1.parameters()) + list(self.q2.parameters()), lr=lr
        )
        self.actor_optimizer = optim.Adam(self.actor.parameters(), lr=lr)
    
    def expectile_loss(self, pred, target, expectile=0.7):
        """期望回归损失"""
        diff = target - pred
        loss = torch.where(
            diff > 0,
            expectile * diff**2,
            (1 - expectile) * diff**2
        )
        return loss.mean()
    
    def train_step(self, batch):
        states, actions, rewards, next_states, dones = batch
        
        # === Step 1: 训练V网络 ===
        with torch.no_grad():
            q1 = self.q1_target(states, actions)
            q2 = self.q2_target(states, actions)
            min_q = torch.min(q1, q2)
        
        v_pred = self.v_net(states)
        v_loss = self.expectile_loss(v_pred, min_q, self.expectile)
        
        self.v_optimizer.zero_grad()
        v_loss.backward()
        self.v_optimizer.step()
        
        # === Step 2: 训练Q网络（用V代替max操作）===
        with torch.no_grad():
            target_v = self.v_net(next_states)
            target_q = rewards + self.gamma * (1 - dones) * target_v
        
        q1_pred = self.q1(states, actions)
        q2_pred = self.q2(states, actions)
        q_loss = F.mse_loss(q1_pred, target_q) + F.mse_loss(q2_pred, target_q)
        
        self.q_optimizer.zero_grad()
        q_loss.backward()
        self.q_optimizer.step()
        
        # 软更新目标Q
        self.soft_update(self.q1, self.q1_target)
        self.soft_update(self.q2, self.q2_target)
        
        # === Step 3: 用AWR训练Actor ===
        with torch.no_grad():
            q = torch.min(self.q1(states, actions), self.q2(states, actions))
            v = self.v_net(states)
            advantage = q - v
            # 指数加权（只学好的动作）
            weight = (advantage * self.temperature).exp().clamp(max=100)
        
        log_probs, _ = self.actor.evaluate_actions(states, actions)
        actor_loss = -(weight * log_probs).mean()
        
        self.actor_optimizer.zero_grad()
        actor_loss.backward()
        self.actor_optimizer.step()
```

### IQL的关键优势

1. **完全不需要评估数据集外的动作**
2. V网络避免了max操作的过估计
3. AWR只用好的动作训练Actor
4. 实现简单，稳定性好

## TD3+BC

最简单的离线RL方法——TD3加上行为克隆正则化：

$$L_{actor} = -E[Q(s, \pi(s))] + \lambda \cdot E[(\pi(s) - a_{expert})^2]$$

```python
class TD3BC:
    """TD3 + Behavior Cloning"""
    
    def __init__(self, state_dim, action_dim, alpha=2.5, lr=3e-4):
        self.alpha = alpha  # BC权重
        self.td3 = TD3(state_dim, action_dim)
    
    def actor_loss(self, states, expert_actions):
        """TD3损失 + α × BC损失"""
        # TD3部分: 最大化Q值
        pred_actions = self.td3.actor(states)
        q1, _ = self.td3.critic(states, pred_actions)
        td3_loss = -q1.mean()
        
        # BC部分: 接近专家动作
        bc_loss = F.mse_loss(pred_actions, expert_actions)
        
        # 加权组合（需要归一化）
        lam = self.alpha / q1.abs().mean().detach()
        total_loss = td3_loss + lam * bc_loss
        
        return total_loss
```

## 算法对比

| 算法 | 核心思想 | 优点 | 缺点 |
|------|----------|------|------|
| **BCQ** | 限制动作在数据集范围 | 直接约束 | 需要VAE |
| **CQL** | 降低数据集外Q值 | 理论保证 | 超参敏感 |
| **IQL** | 期望回归避免外推 | 简单稳定 | 表达力有限 |
| **TD3+BC** | RL + BC正则化 | 最简单 | 性能一般 |

## 数据集质量的重要性

```python
# 离线RL高度依赖数据集质量

# 数据集类型:
# 1. 专家数据: 只有好的轨迹
#    → 难学出鲁棒策略（不知道什么不好）
#
# 2. 随机数据: 均匀随机探索
#    → 覆盖广但好行为少
#
# 3. 混合数据: 专家+随机+中等
#    → 最佳: 既有好的样本，又覆盖状态空间
#
# 4. 噪声专家: 专家策略+噪声
#    → 实用: 真实日志通常如此

# 数据集覆盖率是离线RL的关键
def dataset_coverage_analysis(dataset):
    """分析数据集的覆盖率"""
    states = dataset['states']
    
    # 计算状态空间覆盖
    from sklearn.neighbors import KernelDensity
    kde = KernelDensity(kernel='gaussian', bandwidth=0.1)
    kde.fit(states)
    
    # 在均匀网格上评估密度
    grid = np.meshgrid(*[np.linspace(-1, 1, 20) for _ in range(states.shape[1])])
    grid_points = np.column_stack([g.ravel() for g in grid])
    
    log_densities = kde.score_samples(grid_points)
    
    coverage = np.exp(log_densities).mean()
    return coverage
```

## 小结

| 概念 | 说明 |
|------|------|
| **外推误差** | 数据集外Q值不可靠 |
| **BCQ** | 约束动作在数据集范围 |
| **CQL** | 降低数据集外Q值 |
| **IQL** | 避免查询数据集外动作 |
| **TD3+BC** | RL + BC正则化 |
| **数据质量** | 决定离线RL上限 |

离线RL使RL可以应用于真实世界的安全关键场景。
