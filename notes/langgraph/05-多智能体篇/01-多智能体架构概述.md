# 多智能体架构概述

多智能体系统（Multi-Agent System）是 LangGraph 的高阶应用——多个专业化的 Agent 协同工作，解决复杂任务。

---

## 为什么需要多智能体

### 单 Agent 的局限

```
单 Agent 处理所有任务:
┌──────────────────────────────────┐
│           全能 Agent               │
│  • 搜索信息                        │
│  • 写代码                          │
│  • 数据分析                        │
│  • 文件处理                        │
│  • 翻译                           │
│  • 数学计算                        │
│  • 图表生成                        │
│  • ...（什么都做，什么都不精）       │
└──────────────────────────────────┘

问题:
- 🔴 Prompt 过长（工具太多）
- 🔴 上下文窗口浪费
- 🔴 不同任务互相干扰
- 🔴 难以调试
- 🔴 无法并行
```

### 多 Agent 的优势

```
专业分工:
┌──────────┐  ┌──────────┐  ┌──────────┐
│ 研究 Agent│  │ 编码 Agent│  │ 分析 Agent│
│ • 搜索    │  │ • 写代码  │  │ • SQL    │
│ • 阅读    │  │ • 测试    │  │ • 统计    │
│ • 总结    │  │ • 修复    │  │ • 可视化  │
└──────��───┘  └──────────┘  └──────────┘
       ↑              ↑              ↑
       └──────────────┼──────────────┘
                      │
                ┌──────────┐
                │ 协调者    │
                │ (路由分配) │
                └──────────┘

优势:
- ✅ 每个 Agent Prompt 精简专注
- ✅ 可以并行工作
- ✅ 独立调试和优化
- ✅ 可按需加载（动态激活）
```

## 多智能体架构模式

### 模式一：Supervisor（监督者）

```
              ┌─────────────┐
              │  Supervisor │
              │  (中心调度)  │
              └──────┬──────┘
                     │
        ┌────────────┼────────────┐
        │            │            │
        ▼            ▼            ▼
   ┌─────────┐ ┌─────────┐ ┌─────────┐
   │ Agent A │ │ Agent B │ │ Agent C │
   │ (研究)   │ │ (写作)   │ │ (审核)   │
   └─────────┘ └─────────┘ └─────────┘
```

**特点**：
- 中心化控制，Supervisor 决策一切
- 适合有明确流程的任务
- 最容易实现和调试

### 模式二：Hierarchical（层级）

```
              ┌──────────────┐
              │  Top Manager │
              └──────┬───────┘
                     │
         ┌───────────┼───────────┐
         │           │           │
         ▼           ▼           ▼
    ┌─────────┐ ┌─────────┐ ┌─────────┐
    │Team Lead│ │Team Lead│ │Team Lead│
    │ (技术)   │ │ (内容)   │ │ (数据)   │
    └────┬────┘ └────┬────┘ └────┬────┘
         │           │           │
    ┌────┴────┐┌────┴────┐┌────┴────┐
    │Worker A ││Worker C ││Worker E │
    │Worker B ││Worker D ││Worker F │
    └─────────┘└─────────┘└─────────┘
```

**特点**：
- 多级管理结构
- 适合大型复杂项目
- 模拟真实组织结构

### 模式三：Swarm（群智协作）

```
    ┌─────────┐
    │ Agent A │ ──────┐
    └─────────┘       │
         │            ▼
         ▼       ┌─────────┐
    ┌─────────┐  │ Agent C │
    │ Agent B │◄─┤ (接力)  │
    └─────────┘  └────┬────┘
                      │
                      ▼
                 ┌─────────┐
                 │ Agent D │
                 └─────────┘
```

**特点**：
- 去中心化
- Agent 之间直接传递任务
- 灵活但难以预测

### 模式四：Network（网络）

```
         ┌─────────┐
    ┌───►│ Agent A ├────┐
    │    └─────────┘    │
    │         ▲         ▼
┌─────────┐   │   ┌─────────┐
│ Agent D │◄──┴──►│ Agent B │
└─────────┘       └─────────┘
    │                  ▲
    │    ┌─────────┐    │
    └───►│ Agent C ├────┘
         └─────────┘
```

**特点**：
- 任意 Agent 可与任意 Agent 通信
- 最灵活但最复杂
- 需要防止无限循环

## 通信机制

### 共享状态通信

```python
class TeamState(TypedDict):
    """团队共享状态"""
    messages: Annotated[list, add_messages]
    current_task: str
    assigned_agent: str
    research_results: Annotated[list, operator.add]  # 共享研究结果
    code_output: str
    review_feedback: str

# 所有 Agent 读写同一个 State
def research_agent(state: TeamState) -> dict:
    results = do_research(state["current_task"])
    return {"research_results": results}  # 写入共享状态

def writing_agent(state: TeamState) -> dict:
    # 读取研究结果
    context = state["research_results"]
    draft = write_based_on(context)
    return {"code_output": draft}
```

### 消息传递通信

```python
from langchain_core.messages import HumanMessage, AIMessage

def agent_a(state: State) -> dict:
    """Agent A 发消息给 Agent B"""
    msg = AIMessage(
        content="这是研究结果...",
        additional_kwargs={"to": "agent_b"}  # 指定接收者
    )
    return {"messages": [msg]}

def router(state: State) -> str:
    """根据消息路由"""
    last_msg = state["messages"][-1]
    recipient = last_msg.additional_kwargs.get("to")
    if recipient:
        return recipient
    return END
```

### Handoff 机制

```python
def handoff(state: State) -> dict:
    """Agent 交接任务"""
    return {
        "messages": [AIMessage(
            content=f"任务完成，交接给下一个Agent",
            additional_kwargs={
                "handoff": True,
                "from": "research_agent",
                "to": "writing_agent",
                "summary": "已完成资料收集",
                "context": "收集了5篇相关论文"
            }
        )]
    }
```

## 最简多 Agent 示例

```python
"""
最简单的双 Agent 系统：研究 + 写作
"""
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
from langchain_openai import ChatOpenAI
import operator

llm = ChatOpenAI(model="gpt-4o")

class TeamState(TypedDict):
    messages: Annotated[list, add_messages]
    task: str
    research: str
    draft: str
    final: str

def researcher(state: TeamState) -> dict:
    """研究 Agent"""
    response = llm.invoke(
        f"你是研究专家。针对以下任务，收集关键信息:\n{state['task']}"
    )
    return {"research": response.content}

def writer(state: TeamState) -> dict:
    """写作 Agent"""
    response = llm.invoke(
        f"你是写作专家。基于以下研究结果写一篇文章:\n{state['research']}"
    )
    return {"draft": response.content}

def reviewer(state: TeamState) -> dict:
    """审核 Agent"""
    response = llm.invoke(
        f"你是审核专家。审核以下文章并提出修改意见:\n{state['draft']}"
    )
    return {"final": response.content, "messages": [response]}

# 构建团队
builder = StateGraph(TeamState)
builder.add_node("researcher", researcher)
builder.add_node("writer", writer)
builder.add_node("reviewer", reviewer)

builder.add_edge(START, "researcher")
builder.add_edge("researcher", "writer")
builder.add_edge("writer", "reviewer")
builder.add_edge("reviewer", END)

team = builder.compile()

# 运行
result = team.invoke({
    "task": "写一篇关于AI发展趋势的文章",
    "messages": [],
    "research": "",
    "draft": "",
    "final": ""
})
```

## 架构选择指南

| 模式 | 适合场景 | 控制力 | 复杂度 | 并行性 |
|------|----------|--------|--------|--------|
| Supervisor | 明确流程 | ⭐⭐⭐⭐⭐ | 低 | 中 |
| Hierarchical | 大型项目 | ⭐⭐⭐⭐ | 高 | 高 |
| Swarm | 创意探索 | ⭐⭐ | 中 | 中 |
| Network | 复杂协作 | ⭐⭐ | 很高 | 高 |

---

## 小结

| 要点 | 说明 |
|------|------|
| 多 Agent | 多个专业 Agent 分工协作 |
| Supervisor | 中心调度，最常用 |
| Hierarchical | 多级管理，适合大型项目 |
| Swarm | 去中心化，灵活 |
| 共享状态 | 最简单的通信方式 |
| Handoff | 任务交接机制 |
