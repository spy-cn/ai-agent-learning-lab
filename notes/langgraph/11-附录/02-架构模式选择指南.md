# 架构模式选择指南

帮你决定何时用 Chain、何时用 Graph、何时用多 Agent。

---

## 决策树

```
你的任务是什么？
│
├── 单次 LLM 调用
│   └── 直接用 OpenAI SDK / LangChain LLM
│
├── 固定步骤流水线（A→B→C）
│   └── LangChain Chain (LCEL)
│
├── 需要条件分支
│   ├── 简单（2-3个分支）
│   │   └── LangGraph（单图）
│   └── 复杂（多个专业模块）
│       └── LangGraph + 子图
│
├── 需要循环
│   ├── Agent工具调用循环
│   │   └── create_react_agent
│   └── 质量迭代/自我修正
│       └── LangGraph 自定义循环
│
├── 需要状态管理
│   ├── 简单对话记忆
│   │   └── LangGraph + MemorySaver
│   └── 跨会话持久化
│       └── LangGraph + PostgresSaver
│
├── 需要人工介入
│   └── LangGraph + interrupt
│
├── 多个 Agent 协作
│   ├── 有明确调度者
│   │   └── Supervisor 模式
│   ├── 层级管理
│   │   └── Hierarchical 模式
│   └── 自由接力
│       └── Swarm 模式
│
└── 需要 RAG
    ├── 简单检索+生成
    │   └── 基础 RAG 图
    ├── 自适应策略
    │   └── Adaptive RAG
    └── 带自我评估
        └── Self-RAG / CRAG
```

---

## 复杂度 vs 能力

```
简单 ←────────────────────────────────────→ 复杂

[直接API] → [Chain] → [简单Graph] → [复杂Graph] → [多Agent]
   │           │          │             │              │
   │           │          │             │              │
 单次调用    线性流程   条件+循环    子图+HITL    Supervisor/Swarm
```

---

## 多 Agent 何时用

| 场景 | 推荐 |
|------|------|
| 单一领域任务 | 单 Agent |
| 2-3个不同领域 | Supervisor |
| 5+个专业 Agent | Hierarchical |
| 创意探索类任务 | Swarm |
| 固定工作流 | 单图 + 子图 |

## 模型选择

| 任务 | 推荐模型 |
|------|----------|
| 复杂推理 | GPT-4o / Claude 3.5 Sonnet |
| 路由/分类 | GPT-4o-mini / Claude Haiku |
| 代码生成 | GPT-4o / Claude 3.5 Sonnet |
| 简单问答 | GPT-4o-mini |
| 工具调用 | GPT-4o (最佳工具调用) |

## 成本优化清单

```
□ 用小模型做简单任务（路由、分类）
□ 合并 LLM 调用（一次做多件事）
□ 使用缓存
□ 批量处理
□ 控制循环次数
□ 并行化串行操作
□ 精简 Prompt
□ 流式输出减少用户感知延迟
```
