# 01 - 数字图像基础

## 一、什么是数字图像？

数字图像是由**像素（Pixel）**组成的二维矩阵，每个像素包含颜色或亮度信息。

```
数字图像 = 采样(Sampling) + 量化(Quantization)
```

### 图像的数学表示

```
单通道（灰度）图像：
         width (列)
        ┌──────────────┐
   h  │ p(0,0) p(0,1) │
   e  │ p(1,0) p(1,1) │    p(y, x) ∈ [0, 255]
   i  │ p(2,0) p(2,1) │
   g  │      ...      │
   h  │ p(h-1, w-1)   │
   t  └──────────────┘
        (行)

三通道（彩色）图像：
    I(y, x) = [B(y,x), G(y,x), R(y,x)]
    
    即每个像素位置存储 3 个值：蓝、绿、红
```

## 二、图像类型

| 类型 | 通道数 | 每像素位数 | OpenCV 标志 |
|------|--------|-----------|-------------|
| 二值图像 | 1 | 1 bit | - |
| 灰度图像 | 1 | 8 bit | `cv2.IMREAD_GRAYSCALE` (0) |
| 彩色图像 | 3 | 24 bit | `cv2.IMREAD_COLOR` (1) |
| 带透明度 | 4 | 32 bit | `cv2.IMREAD_UNCHANGED` (-1) |

## 三、色彩空间

### RGB vs BGR

```
重要：OpenCV 默认使用 BGR 顺序，而非 RGB！

RGB: (R, G, B) → (255, 0, 0) = 红色
BGR: (B, G, R) → (0, 0, 255) = 红色（OpenCV默认）
```

### 常见颜色对照

| 颜色 | RGB | BGR (OpenCV) | Hex |
|------|-----|-------------|-----|
| 红 | (255, 0, 0) | (0, 0, 255) | #FF0000 |
| 绿 | (0, 255, 0) | (0, 255, 0) | #00FF00 |
| 蓝 | (0, 0, 255) | (255, 0, 0) | #0000FF |
| 白 | (255, 255, 255) | (255, 255, 255) | #FFFFFF |
| 黑 | (0, 0, 0) | (0, 0, 0) | #000000 |
| 黄 | (255, 255, 0) | (0, 255, 255) | #FFFF00 |

## 四、坐标系统

```
OpenCV 图像坐标系：

    (0,0) ────────────→ X (width/columns)
      │
      │
      │
      │
      ↓
      Y (height/rows)

注意：
- 原点 (0,0) 在左上角
- Y 轴向下为正
- 像素访问：img[y, x] = img[行, 列]
```

```python
import cv2
import numpy as np

# 创建 400x600 的黑色图像
img = np.zeros((400, 600, 3), dtype=np.uint8)

# 画坐标轴参考线
cv2.line(img, (0, 0), (600, 0), (255, 255, 255), 1)  # X轴
cv2.line(img, (0, 0), (0, 400), (255, 255, 255), 1)  # Y轴

# 标记点
cv2.circle(img, (300, 200), 5, (0, 0, 255), -1)  # 红色点在 (x=300, y=200)
cv2.putText(img, "(300, 200)", (310, 195),
            cv2.FONT_HERSHEY_SIMPLEX, 0.5, (0, 255, 0), 1)

# 访问该像素
print(f"像素值: {img[200, 300]}")  # 注意是 img[y, x]

cv2.imshow("Coordinate System", img)
cv2.waitKey(0)
```

## 五、图像的数据类型

OpenCV 中常见的图像数据类型：

| 数据类型 | NumPy 类型 | 值域 | 用途 |
|----------|-----------|------|------|
| CV_8U | np.uint8 | 0 - 255 | 最常用 |
| CV_16U | np.uint16 | 0 - 65535 | HDR、深度图 |
| CV_32F | np.float32 | 0.0 - 1.0 | 计算中间结果 |
| CV_64F | np.float64 | 任意精度 | 精确计算 |

```python
# 类型转换
img_float = img.astype(np.float32) / 255.0  # 归一化到 [0, 1]
img_uint8 = (img_float * 255).astype(np.uint8)  # 转回 uint8

# OpenCV 类型转换函数
img_float32 = cv2.convertScaleAbs(img, alpha=1/255.0)  # 不推荐
```

## 六、图像分辨率与质量

### 分辨率

```
分辨率 = 宽 × 高（像素数）

常见分辨率：
┌──────────────┬─────────────┐
│  分辨率名称   │  像素尺寸    │
├──────────────┼─────────────┤
│  QVGA        │  320 × 240  │
│  VGA         │  640 × 480  │
│  HD 720      │  1280 × 720 │
│  Full HD     │  1920 × 1080│
│  4K UHD      │  3840 × 2160│
│  8K UHD      │  7680 × 4320│
└──────────────┴─────────────┘
```

### DPI vs 分辨率

```
DPI (Dots Per Inch) = 每英寸像素数
物理尺寸 = 像素数 / DPI

例：1920 像素 / 300 DPI = 6.4 英寸
```

## 七、图像的频域概念

```
图像可以分解为不同频率的组成部分：

┌──────────────────┬──────────────────┐
│   低频 (LF)       │   高频 (HF)       │
├──────────────────┼──────────────────┤
│  平滑区域         │  边缘和细节        │
│  整体亮度         │  噪声             │
│  大面积颜色       │  纹理             │
└──────────────────┴──────────────────┘

应用：
- 低通滤波 → 降噪、模糊
- 高通滤波 → 锐化、边缘检测
```

## 小结

| 概念 | 要点 |
|------|------|
| 数字图像 | 像素矩阵 |
| 色彩顺序 | OpenCV 用 BGR |
| 坐标原点 | 左上角 (0,0) |
| 像素访问 | `img[y, x]`（先行后列） |
| 数据类型 | uint8 最常用 |
| 分辨率 | 宽 × 高 |
